【Grassmann Flow】当 Attention 不再是唯一选择——用微分几何重新理解序列建模

━━━━━━━━━━━━━━━━━━━━

◆ 一封来信

━━━━━━━━━━━━━━━━━━━━

前几天收到一位读者的私信，大意是：

> 能不能解读一下 Grassmann Flow 这篇论文（arxiv: 2512.19428）？我在自己的项目里对比了 Grassmann Flow 和 Transformer，初步结果极为出乎意料的好。

论文标题很直接：**Attention Is Not What You Need: Grassmann Flows as an Attention-Free Alternative for Sequence Modeling**。作者 Zhang Chong，2025 年 12 月。

标题致敬了 2017 年那篇改变一切的 "Attention Is All You Need"，但把结论反过来了——Attention 不是你需要的东西。

论文不长，数学很紧凑，实验规模很小（13-18M 参数），作者自己也说这是 proof of concept。但它提出了一个真正有价值的问题：**Self-Attention 到底在干什么？有没有完全不同的方式来做同一件事？**

这个问题值得认真回答。

━━━━━━━━━━━━━━━━━━━━

◆ Attention 到底在干什么——Tensor Lifting 视角

━━━━━━━━━━━━━━━━━━━━

先忘掉 Q、K、V，忘掉 softmax，忘掉多头。从最底层想：Self-Attention 的输入是什么，输出是什么。

输入：一个序列，L 个 token，每个 token 是一个 d 维向量 h_t。所有 token 排成一排，住在一条"走廊"里——每个 token 只知道自己的位置，不知道邻居是谁。

输出：还是 L 个 d 维向量，但每个 token 都"看过"了其他所有 token，融合了全局信息。

中间发生了什么？

**Tensor Lifting——张量提升。**

通过 Q、K、V 三个矩阵，每个 token 从 d 维的"个人空间"被提升到 L×L 的"成对交互空间"。QK^T 算出来的那个 L×L 矩阵，就是这个提升的产物——它记录了所有 token 两两之间的关系。

💡 **人话翻译：** 想象一间教室，L 个学生坐成一排（一维走廊）。Attention 让每个学生和其他所有学生握手（L×L 矩阵），根据握手的力度决定听谁的话。从一维走廊提升到了二维大厅——每一对学生之间都有一条连线，连线越粗，交流越多。

这个提升非常暴力。L 个 token，两两配对，自由度是 L² 量级。序列长度 1024，就是 100 万级别的交互参数。序列长度 4096，就是 1600 万级别。这些参数里有多少是真正有用的信息？有多少是冗余和噪声？

论文用了三个词来形容这个提升：**extremely fine-grained（极度细粒度）、high-dimensional（高维）、hard to compress（难以压缩）**。

自由度太大，不可解释，也不可控。这是 Attention 的力量之源，也是它的原罪。

────────────────────

**一个类比：全班握手**

30 个学生的教室，Attention 要求每个人跟其他 29 个人都握一次手——总共 435 次握手。你从这 435 次握手中提取信息。

问题是：你真的需要 435 次握手吗？也许坐在一起的几个人之间的关系就够了。也许你不需要知道"张三和李四有多像"，你需要知道的是"张三和李四面朝哪个方向"。

这就是 Grassmann Flow 的出发点。

━━━━━━━━━━━━━━━━━━━━

◆ Grassmann Flow 五步流水线

━━━━━━━━━━━━━━━━━━━━

论文的核心架构，拆成五步。

────────────────────

**第一步：线性降维——把学生从大教室搬到小房间**

```
z_t = W_red · h_t + b_red
```

把每个 token 从 d 维降到 r 维。论文实验里 d=256，r=32——压缩了 8 倍。

这一步的意义：后面所有几何操作都在 r 维空间里做。r 越小，计算越快，几何约束越紧。代价是信息损失——但论文赌的就是：token 之间的结构关系，不需要 d 维那么多自由度来描述。

────────────────────

**第二步：多尺度局部配对——只看邻座，不看全班**

定义一个偏移集合 W = {1, 2, 4, 8, 12, 16}。对每个位置 t，只跟它右边偏移 1、2、4、8、12、16 个位置的 token 配对。

💡 **人话翻译：** Attention 是全班握手，Grassmann Flow 是只跟前后几桌的人交流。但不是固定只看隔壁——通过 {1,2,4,8,12,16} 这样的多尺度偏移，既看紧邻的（偏移 1-2），也看稍远的（偏移 8-16）。类似于人眼的视野：中心区域高分辨率，边缘低分辨率。

注意：只向右配对。这是因果性约束——生成任务里，位置 t 的 token 不能看到未来。

每个位置最多 6 个配对（偏移集合大小）。不是 L 个。这是复杂度从 O(L²) 降到 O(L) 的根本原因。

────────────────────

**第三步：Grassmann / Plücker 编码——全文最难也最关键的一步**

每对配对 (z_t, z_{t+Δ}) 是 R^r 空间中的两个向量。这两个向量张成一个二维子空间（一个平面）。

这个二维子空间，是 **Grassmann 流形 Gr(2, r)** 上的一个点。

然后用 **Plücker 嵌入** 把这个点映射为一个坐标向量：

```
p_ij = z_{t,i} · z_{t+Δ,j} - z_{t,j} · z_{t+Δ,i}    (1 ≤ i < j ≤ r)
```

归一化：

```
p̂ = p / max(‖p‖₂, ε)
```

r=32 的时候，Plücker 坐标是一个 C(32,2) = 496 维的向量。

────────────────────

好，暂停。这段公式看不懂很正常。让我把它翻译成人话。

**Attention 记录的是什么？** 两个 token 的点积——"你和我有多像"。QK^T 的每个元素就是一个相似度分数。

**Plücker 坐标记录的是什么？** 两个 token 张成的平面的"朝向"——不是"你和我有多像"，是"你和我面朝哪个方向"。

💡 **人话翻译：** 两个人站在操场上。Attention 的做法是测量两个人之间的距离（点积相似度）。Plücker 的做法是看这两个人连成的那条线指向什么方向，以及这条线和地平面的夹角是多少。

距离是一个标量——一个数。方向是一个更丰富的对象——它编码了两个 token 之间关系的几何结构，而不仅仅是它们的相似程度。

更准确地说：两个向量的点积丢掉了旋转信息，只保留了投影长度。Plücker 坐标保留了旋转信息——两个向量在高维空间里"朝向"哪里，它们张成的平面是怎么"摆放"的。

这就是 Grassmann 流形的意义。Grassmann 流形 Gr(2, r) 是"r 维空间中所有二维子空间的集合"。每一对 token 张成一个二维子空间，对应流形上的一个点。Plücker 坐标是这个点在欧式空间中的表示方式。

**关键洞见：Attention 用 L² 的成对相似度矩阵来表示 token 间关系。Grassmann Flow 用流形上的几何点来表示 token 间关系。前者自由度是 L²，后者自由度被 r 和窗口大小 m 控制。**

────────────────────

**第四步：投影回模型空间**

```
g_t^(Δ) = W_plü · p̂_t^(Δ) + b_plü
```

把 Plücker 坐标投影回模型的隐藏维度。然后对所有有效偏移取平均：

```
g_t = (1/|W_t|) Σ g_t^(Δ)
```

这一步很朴素——线性变换 + 平均池化。把几何空间的信息搬回模型能处理的空间。

────────────────────

**第五步：门控融合——让模型自己决定听谁的**

```
u_t = [h_t; g_t]                            // 拼接原始特征和几何特征
α_t = σ(W_gate · u_t + b_gate)              // sigmoid 门控
h̃_t = α_t ⊙ h_t + (1-α_t) ⊙ g_t          // 按元素混合
```

然后接 LayerNorm、Dropout、FFN——和标准 Transformer 后半段一样。

门控让模型在训练中自己学习：对每个维度，原始特征和几何特征各占多少比重。有些维度可能完全不需要几何信息（α→1），有些维度可能完全依赖几何信息（α→0）。

────────────────────

**五步连起来：**

h_t（原始 token）→ 降维 → 局部配对 → Plücker 几何编码 → 投影回来 → 门控混合 → 输出

没有 Q、K、V，没有 softmax，没有 L×L 矩阵。整个序列的 token 交互，被限制在局部的、低维的、几何约束的流形上。

◆ 复杂度：L² 和 L 的区别

━━━━━━━━━━━━━━━━━━━━

| 操作 | Self-Attention | Grassmann Flow |
|------|---------------|----------------|
| token 交互 | O(L²d_head) | O(Lmd²)，m 是窗口大小 |
| 投影 | O(Ld²) | O(Ld²) |
| 总复杂度 | O(L²d_head + Ld²) | O(Ld²) |
| 关键区别 | L² 项来自 QK^T | 没有 L² 项 |

Attention 的 L² 项是那个"全班握手"矩阵。序列长 1024，矩阵是 100 万。长 4096，矩阵是 1600 万。长 100K，矩阵是 100 亿。这就是为什么长序列建模一直是 Transformer 的痛点。

Grassmann Flow 的复杂度是 O(Ld²)——线性于序列长度。因为每个 token 只跟固定数量（m 个）的邻居配对，不跟全班握手。

但要注意：O(Ld²) 里的 d² 项来自投影操作，和 Transformer 的 FFN 部分一样。真正省掉的是 L² 项。对于短序列，L² 可能比 d² 小——这时候 Grassmann 没有复杂度优势。优势在长序列时才体现。

◆ 实验结果：诚实地看

━━━━━━━━━━━━━━━━━━━━

论文做了两组实验。

────────────────────

**实验一：Wikitext-2 语言建模**

| 配置 | 模型 | 参数量 | Val PPL |
|------|------|--------|---------|
| 6 层, block 128 | TransformerLM | 12.59M | 248.4 |
| 6 层, block 128 | GrassmannLM | 13.00M | 275.7 |
| 12 层, block 256 | TransformerLM | 17.32M | 235.2 |
| 12 层, block 256 | GrassmannLM | 18.16M | 261.1 |

PPL（Perplexity）越低越好。Grassmann 比 Transformer 差 10-15%。

但有一个趋势：从 6 层到 12 层，差距在缩小。6 层时差 11%，12 层时差 11%——绝对差距没变，但 Transformer 自己也从 248 降到 235，改善幅度在变小，而 Grassmann 从 275 降到 261，改善幅度差不多。层数越深，Grassmann 的几何约束被充分利用的程度越高。

────────────────────

**实验二：SNLI 自然语言推断**

| 模型 | Val Acc | Test Acc |
|------|---------|----------|
| Transformer head | 0.8545 | 0.8511 |
| Grassmann-Plücker head | 0.8550 | 0.8538 |

backbone 都是 DistilBERT，只替换分类头。Grassmann 略胜——0.27 个百分点。

这个结果很有趣。语言建模（生成任务）差 10-15%，但分类任务打平甚至略胜。

────────────────────

**作者自己承认的问题：**

1. 当前实现比 Transformer 慢——Plücker 坐标计算没有 CUDA 优化，是纯 Python 循环
2. 实验规模很小——13-18M 参数，和 GPT-2 small (117M) 都差一个数量级
3. 只在 Wikitext-2 和 SNLI 上测过，没有测多模态、代码生成、长文档

论文的自我定位很清晰：**这不是一个工程方案，是一个概念验证。** 目标不是替代 Attention，是 "de-center" 它——证明 Attention 不是序列建模的唯一选择。

◆ 回答那位读者：为什么你的实验"出乎意料的好"

━━━━━━━━━━━━━━━━━━━━

论文的结果是"差 10-15%"。你的结果是"出乎意料的好"。矛盾吗？不矛盾。原因可能有几个。

────────────────────

**第一，任务类型不同。**

论文的语言建模任务（预测下一个 token）要求模型对整个词汇表建立精确的概率分布。这是 Attention 的主场——全局信息融合越好，预测越准。

但如果你的任务侧重**局部模式识别**——比如分类、匹配、信息抽取——那长距离依赖没那么重要。SNLI 的结果已经暗示了这一点：分类任务上，Grassmann 和 Transformer 打平。

如果你的任务恰好是局部模式主导的，Grassmann 的多尺度局部配对（偏移 1,2,4,8,12,16）可能正好覆盖了关键的模式窗口，而 Attention 的全局握手反而引入了噪声。

────────────────────

**第二，几何约束是天然的正则化。**

Grassmann Flow 把 token 交互限制在 Gr(2,r) 流形上。这意味着模型不能随意拟合任何两两关系——它只能表示那些可以被"二维子空间朝向"编码的关系。

在小数据、小模型的场景下，这种约束是优势。Attention 的 L×L 矩阵有太多自由度，容易过拟合。Grassmann 的几何约束天然防止了这一点——就像给一个画家只发了铅笔和尺子，他画不出油画的细节，但他画的线条一定是干净利落的。

如果你的数据集不大、模型不大、任务不需要极致的表达能力，几何约束反而让模型更稳健。

────────────────────

**第三，降维 r 的选择。**

论文用 r=32，这是一个很激进的压缩。如果你在自己的实验里调过 r，选了一个更适合你任务的值，效果可能会有质的不同。r 太小，信息损失过多；r 太大，几何约束太松。最佳的 r 取决于任务中 token 关系的"内在维度"——有些任务的 token 关系本来就是低维的，r=32 甚至 r=16 就够了。

────────────────────

**第四，注意力分散问题。**

Attention 有一个被反复讨论的问题：**注意力分散（attention dilution）**。序列很长的时候，softmax 把概率分散到太多 token 上，每个 token 分到的权重都很小，信号被稀释。各种改进方案（FlashAttention、GQA、稀疏注意力）本质上都在解决这个问题。

Grassmann Flow 从根上绕过了这个问题——它不做全局注意力，自然不存在分散。每个 token 只关注 6 个邻居，信号集中，不稀释。

如果你的任务中注意力分散是主要瓶颈，Grassmann 的表现超过 Attention 完全说得通。

━━━━━━━━━━━━━━━━━━━━

◆ 更深一层：从向量相似度到子空间朝向

━━━━━━━━━━━━━━━━━━━━

这篇论文最有价值的贡献，不是 Grassmann Flow 这个具体架构，而是它提出的那个问题：**token 之间的关系，一定要用相似度（点积）来表示吗？**

Attention 的核心操作是 QK^T——两个向量的点积。点积测量的是"你和我有多像"。整个 Attention 机制建立在一个隐含假设上：token 之间的关系可以被"相似度"充分描述。

但"相似度"只是关系的一种。两个 token 之间的关系，除了"像不像"，还有：

- **方向性**：A→B 的关系和 B→A 可能不同（因果关系、修饰关系）
- **互补性**：两个不相似的 token 可能构成重要的配对（主语和谓语）
- **结构性**：两个 token 的关系不取决于它们自身，而取决于它们在整体中的位置

Plücker 坐标编码的是子空间的朝向——两个向量张成的平面在高维空间中"面朝哪里"。这是一种比点积更丰富的关系表示。点积是标量（一个数），Plücker 坐标是 C(r,2) 维的向量（r=32 时是 496 维）。它保留了旋转信息、保留了两个向量之间的"角度结构"，而不仅仅是投影长度。

💡 **人话翻译：** 两根筷子摆在桌上。点积告诉你"这两根筷子的方向有多接近"（夹角的余弦）。Plücker 坐标告诉你"这两根筷子组成的那个 V 字形，在三维空间中是怎么摆放的"——朝上还是朝右，张开多大角度，整个 V 字形绕了什么方向。前者是一个数，后者是一组数，信息量天然不同。

这就是为什么论文标题叫 "Attention Is Not What You Need"——不是说 Attention 不好，是说 Attention 选择的"相似度"这个关系表示方式，不是唯一的选择，甚至可能不是最好的选择。

━━━━━━━━━━━━━━━━━━━━

◆ 这篇论文的局限——不回避

━━━━━━━━━━━━━━━━━━━━

诚实地列出问题。

**一、语言建模差 10-15%。** 对于大模型预训练的核心任务（下一个 token 预测），这个差距不小。Grassmann Flow 在生成任务上还没有证明自己。

**二、实验规模太小。** 13-18M 参数，在 2025 年的标准下连 toy model 都算不上。Grassmann 流形上的几何操作在大规模时是否还能保持优势？r 和模型规模的最佳比例关系是什么？都没有答案。

**三、实现效率低。** 作者自己说了，Plücker 坐标计算没有 CUDA kernel。这不是算法的问题，是工程的问题——但工程问题也是真问题。FlashAttention 能火，一半功劳在论文，一半在 Tri Dao 写的 CUDA kernel。Grassmann Flow 需要同等质量的工程投入才能公平比较。

**四、窗口大小固定。** 偏移集合 {1,2,4,8,12,16} 是手动设定的。如果任务需要超过 16 的长距离依赖，这个架构就有结构性缺陷。虽然可以叠更多层来间接增大感受野，但这不如 Attention 的全局感受野来得直接。

**五、只测了 NLP。** 没有视觉、音频、多模态的实验。Attention 的通用性已经在无数任务上被验证，Grassmann Flow 还没有。

这些问题都是真实的。但它们不影响这篇论文的核心价值——它不是要给出一个完整的替代方案，是要打开一扇窗。

◆ 从 Attention 的霸权到几何民主

━━━━━━━━━━━━━━━━━━━━

2017 年的 "Attention Is All You Need" 是一篇伟大的论文。但任何一个工程方案，当它成为唯一的选择、不可挑战的范式，它就不再是工具，而是信仰。

过去八年，几乎所有的序列建模改进都在 Attention 的框架内进行：多头、稀疏、线性、FlashAttention、GQA、MQA、滑动窗口……所有这些工作的共同前提是：token 之间的关系用 QK^T 来建模，然后在这个框架内做各种优化和近似。

很少有人问：QK^T 是唯一的选择吗？

Grassmann Flow 回答：不是。你可以用微分几何——具体来说，用 Grassmann 流形和 Plücker 嵌入——来表示 token 间关系。效果没有 Attention 好（目前），但方向完全不同，起点完全不同，数学基础完全不同。

Mamba 系列的 SSM 也在回答这个问题：你可以用状态空间模型——递推更新一个固定大小的状态——来隐式建模序列依赖，完全不需要两两配对。

RWKV 也在回答：你可以用线性注意力的变体，把 L² 降到 L。

这些工作之间的共同点不是"打败 Attention"——没有一个做到了。它们的共同点是：**证明 Attention 不是唯一的路。**

────────────────────

这让我想到一个更大的问题。

AI 领域有一种倾向：一旦某个方法在 benchmark 上赢了，所有人就涌过去，把它当成唯一正确的方向。Attention 赢了 RNN，于是 RNN "死了"。Transformer 赢了 LSTM，于是 LSTM "死了"。

但"死掉"的方法真的没有价值吗？SSM 的本质就是改良的 RNN。RWKV 的本质就是改良的 RNN。Grassmann Flow 的本质是把微分几何引入序列建模——这条路从来没有被认真探索过，因为 2017 年之后所有人都在做 Attention 的变体。

**一个领域最危险的时刻，不是没有好方法，而是只有一个好方法。** 当所有人都在同一条路上优化，整个领域的探索宽度就归零了。

Grassmann Flow 可能不会成为下一个 Transformer。但它证明了一件事：在 Attention 的领地之外，还有未被开垦的数学大陆。Grassmann 流形只是其中一块。还有辛流形、纤维丛、黎曼几何……这些数学工具在物理学中已经被用了上百年，但在深度学习中几乎是处女地。

论文作者说得很准确：目标不是替代 Attention，是 **de-center** 它。去中心化。让它从"唯一选择"变成"选项之一"。

这不是技术判断，这是方法论。好的工程生态需要多样性。生物进化需要基因多样性，农业需要品种多样性，金融需要资产多样性，AI 研究需要架构多样性。

当所有人都在一棵树上优化果实大小的时候，需要有人走到旁边那片没人去过的林子里看看——哪怕那片林子里的果实现在还又小又涩。

━━━━━━━━━━━━━━━━━━━━

◆ 给那位读者的最后几句话

━━━━━━━━━━━━━━━━━━━━

你的实验结果"出乎意料的好"，很可能是真的。几个可能的原因上面分析了——任务类型、几何正则化、注意力分散。

建议你做几件事：

1. **控制变量对比**：固定参数量（不是层数），让 Grassmann 和 Transformer 在相同参数预算下比。论文里 Grassmann 多了 3-5% 的参数，这个差异不大但应该控制。

2. **调 r 值**：r=32 是论文的默认值，不一定是你的任务的最优值。试试 r=16, 32, 64, 128，画一条曲线。

3. **调偏移集合**：{1,2,4,8,12,16} 是论文默认的。你的任务如果有特定的局部模式尺度，可以定制。

4. **和其他 efficient attention 对比**：不只跟标准 Attention 比，也跟滑动窗口注意力、线性注意力比。如果 Grassmann 赢了标准 Attention 但输给滑动窗口注意力，那说明关键因素是局部性而非几何编码。

5. **如果效果确实好，写出来**。论文作者只有小规模实验，你的实验——在真实项目、真实数据上——比论文的实验更有说服力。这个方向需要更多实证数据。

━━━━━━━━━━━━━━━━━━━━

「Attention 记录的是'你和我有多像'，Plücker 坐标记录的是'你和我面朝哪个方向'。从相似度到朝向，是关系表示的范式跳跃。」

「一个领域最危险的时刻，不是没有好方法，而是只有一个好方法。」
