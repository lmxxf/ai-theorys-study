【Grassmann Flow】当 Attention 不再是唯一选择——用微分几何重新理解序列建模

━━━━━━━━━━━━━━━━━━━━

◆ 一封来信

━━━━━━━━━━━━━━━━━━━━

前几天收到一位读者的私信，大意是：

> 能不能解读一下 Grassmann Flow 这篇论文（arxiv: 2512.19428）？我在自己的项目里对比了 Grassmann Flow 和 Transformer，初步结果极为出乎意料的好。

论文标题很直接：**Attention Is Not What You Need: Grassmann Flows as an Attention-Free Alternative for Sequence Modeling**（arxiv.org/abs/2512.19428）。作者 Zhang Chong，2025 年 12 月。

标题致敬了 2017 年那篇改变一切的 "Attention Is All You Need"，但把结论反过来了——Attention 不是你需要的东西。

论文不长，数学很紧凑，实验规模很小（13-18M 参数），作者自己也说这是 proof of concept。但它提出了一个真正有价值的问题：**Self-Attention 到底在干什么？有没有完全不同的方式来做同一件事？**

这个问题值得认真回答。

━━━━━━━━━━━━━━━━━━━━

◆ Attention 到底在干什么——论文作者的 Tensor Lifting 视角

━━━━━━━━━━━━━━━━━━━━

先忘掉 Q、K、V，忘掉 softmax，忘掉多头。从最底层想：Self-Attention 的输入是什么，输出是什么。

输入：一个序列，L 个 token，每个 token 是一个 d 维向量 hₜ。所有 token 排成一排，住在一条"走廊"里——每个 token 只知道自己的位置，不知道邻居是谁。

输出：还是 L 个 d 维向量，但每个 token 都"看过"了其他所有 token，融合了全局信息。

中间发生了什么？

**Tensor Lifting——张量提升。** 这是论文作者对 Attention 的核心批判，也是他提出替代方案的理论起点。

他的论证是这样的：通过 Q、K、V 三个矩阵，每个 token 从 d 维的"个人空间"被提升到 L×L 的"成对交互空间"。QKᵀ 算出来的那个 L×L 矩阵，就是这个提升的产物——它记录了所有 token 两两之间的关系。

💡 **人话翻译：** 想象一间教室，L 个学生坐成一排（一维走廊）。Attention 让每个学生和其他所有学生握手（L×L 矩阵），根据握手的力度决定听谁的话。从一维走廊提升到了二维大厅——每一对学生之间都有一条连线，连线越粗，交流越多。

这个提升非常暴力。L 个 token，两两配对，自由度是 L² 量级。序列长度 1024，就是 100 万级别的交互参数。序列长度 4096，就是 1600 万级别。这些参数里有多少是真正有用的信息？有多少是冗余和噪声？

论文作者用了三个词来形容这个提升：**extremely fine-grained（极度细粒度）、high-dimensional（高维）、hard to compress（难以压缩）**。

他的结论是：自由度太大，不可解释，也不可控。这是 Attention 的力量之源，也是它的原罪——Transformer 不可解释，根源不在参数多，在于核心操作住在一个自由度爆炸的张量空间里。30 个学生的教室，Attention 要求每个人跟其他 29 个人都握一次手——总共 435 次握手。你真的需要 435 次握手吗？也许坐在一起的几个人之间的关系就够了。也许你不需要知道"张三和李四有多像"，你需要知道的是"张三和李四面朝哪个方向"。这就是 Grassmann Flow 的出发点。

━━━━━━━━━━━━━━━━━━━━

◆ Grassmann Flow 六步流水线

━━━━━━━━━━━━━━━━━━━━

论文的核心架构，拆成六步。

────────────────────

**第一步：线性降维——把学生从大教室搬到小房间**

```
zₜ = W_red · hₜ + b_red       // red = reduction（降维）
```

把每个 token 从 d 维降到 r 维。论文实验里 d=256，r=32——压缩了 8 倍。

这一步的意义：后面所有几何操作都在 r 维空间里做。r 越小，计算越快，几何约束越紧。代价是信息损失——但论文赌的就是：token 之间的结构关系，不需要 d 维那么多自由度来描述。

这里有个隐忧：r=32 是不是太小了？论文的实验模型只有 d=256，降到 32 还算温和。但如果要 scale 到大模型（d=4096 甚至 d=12288），r 该取多少？我们在《Doc-to-LoRA 实验》里验证过，基础模型的"本我流形"有效维度至少几百维——token 之间的关系结构未必比这简单多少，r 大概率不应该低于 500。论文没有做 r 的消融实验，这是一个明显的缺失。

我猜降维的真正原因不是"token 关系只需要 32 维描述"，而是**后面要升维，不降维就爆炸了**。Plücker 坐标的维度是 C(r,2)，跟 r 的平方成正比：

| r | Plücker 维度 C(r,2) |
|---|---|
| 16 | 120 |
| 32 | 496 |
| 64 | 2016 |
| 128 | 8128 |
| 256（不降维） | 32640 |

如果不降维，d=256 直接算 Plücker 坐标，每对 token 产生一个 32640 维的向量——参数量和计算量都炸了。所以 r=32 与其说是"压缩到最优维度"，不如说是"Plücker 坐标算得动的上限"。这个架构天生有个矛盾：**r 大了 Plücker 爆炸，r 小了信息丢失。** 论文选 r=32 是在这个矛盾里找了一个折中点。

────────────────────

**第二步：多尺度局部配对——只看邻座，不看全班**

定义一个偏移集合 W = {1, 2, 4, 8, 12, 16}。对每个位置 t，只跟它右边偏移 1、2、4、8、12、16 个位置的 token 配对。

💡 **人话翻译：** Attention 是全班握手，Grassmann Flow 是只跟前后几桌的人交流。但不是固定只看隔壁——通过 {1,2,4,8,12,16} 这样的多尺度偏移，既看紧邻的（偏移 1-2），也看稍远的（偏移 8-16）。类似于人眼的视野：中心区域高分辨率，边缘低分辨率。

注意：只向右配对。这是因果性约束——生成任务里，位置 t 的 token 不能看到未来。

每个位置最多 6 个配对（偏移集合大小）。不是 L 个。这是复杂度从 O(L²) 降到 O(L) 的根本原因。

但这组偏移 {1,2,4,8,12,16} 是怎么来的？论文没有给出任何依据，也没有做消融实验比较不同偏移集合的效果。这是一个明显的弱点——"看谁"是序列建模中最关键的决策之一，Attention 用 QKᵀ 让模型自己学，DSA 用 indexer 打分让模型自己选，而 Grassmann Flow 在这个最关键的环节上，反而退化成了手动超参数。

────────────────────

**第三步：Grassmann / Plücker 编码——全文最难也最关键的一步**

先说名字的来历。Grassmann 流形和 Plücker 坐标都不是论文作者发明的——它们是 19 世纪的经典数学。Hermann Grassmann（1809-1877）发明了外代数，Grassmann 流形以他命名；Julius Plücker（1801-1868）发明了 Plücker 坐标，用来表示射影空间中的子空间。都是教科书级别的工具。论文作者做的事是把这两个 19 世纪的数学工具组合起来，用到 21 世纪的序列建模里。

每对配对 (zₜ, z_{t+Δ}) 是 Rʳ 空间中的两个向量。这两个向量张成一个二维子空间（一个平面）。

这个二维子空间，是 **Grassmann 流形 Gr(2, r)** 上的一个点。

注意，这是一个很强的架构假设——论文在设计层面就规定了"token 之间的关系必须住在 Gr(2,r) 这个流形上"。Attention 不预设任何几何结构，模型自己学出什么就是什么；Grassmann Flow 反过来，几何结构是设计者选的，不是模型学的。如果真实的 token 关系恰好跟这个流形兼容，约束就是正则化；如果不兼容，约束就是枷锁。

这里有一个更深的问题。我们在《Grokking 实验》（https://mp.weixin.qq.com/s/SV--XvFjWoEL4sG3EDMcjA）里验证过：Transformer 做模加法时，内在维度从 78 骤降到 8——模型自己发现了任务的低维流形。这个"流形发现"的前提是：模型有足够的自由度去搜索。Grassmann Flow 把流形形状写死成 Gr(2,r)，如果任务的真实结构不是这个形状，模型连发现的机会都没有。**Transformer 是让模型自己发现流形，Grassmann Flow 是替模型选好了流形——省了搜索成本，但上限也被架构限死了。**

然后用 **Plücker 嵌入** 把这个点映射为一个坐标向量：

```
pᵢⱼ = z_{t,i} · z_{t+Δ,j} - z_{t,j} · z_{t+Δ,i}    (1 ≤ i < j ≤ r)
```

归一化：

```
p̂ = p / max(‖p‖₂, ε)
```

r=32 的时候，Plücker 坐标是一个 C(32,2) = 496 维的向量。为什么是 496？因为 32 维空间里取两个坐标轴 (i, j) 且 i < j，有 32×31/2 = 496 种取法。每个 pᵢⱼ 就是两个向量在第 (i,j) 坐标平面上的投影面积（带方向）。496 个面积拼在一起，完整描述了这个二维子空间在 32 维空间里的"朝向"。

💡 **人话翻译：** Plücker 坐标记录的不是两个 token 各自的信息，是它们之间的关系——具体来说，是两个 token 张成的那个平面怎么"摆放"的。想象两根筷子撑开一个平面，你有 496 台相机从不同角度拍它，每张照片记录"这个平面从这个角度看有多大、朝哪边"。496 张照片拼起来，就是 Plücker 坐标。点积只给你一个数（夹角余弦），Plücker 给你 496 个数——信息量差了两个数量级。

**Attention 用 L² 的成对相似度矩阵来表示 token 间关系——每对 token 一个数。Grassmann Flow 用流形上的几何点来表示 token 间关系——每对 token 496 个数。信息更丰富，但自由度被 r 和窗口大小 m 控制。**

────────────────────

**第四步：投影回模型空间**

```
gₜ^(Δ) = W_plü · p̂ₜ^(Δ) + b_plü
```

把 Plücker 坐标投影回模型的隐藏维度。然后对所有有效偏移取平均：

```
gₜ = (1/|Wₜ|) Σ gₜ^(Δ)
```

这一步很朴素——线性变换 + 平均池化。维度变化：496 → 256，从 Plücker 空间回到模型空间。整个流程的维度路径是 **256 → 32 → 496 → 256**，先降后升再降回来。

────────────────────

**第五步：门控融合——让模型自己决定听谁的**

到这一步，每个 token 手里有两份信息：

- **hₜ**：原始的 256 维隐藏状态，没经过任何几何处理
- **gₜ**：经过降维→Plücker 编码→投影回来的 256 维几何特征

问题是：这两份信息怎么合并？直接相加？取平均？都太粗暴。论文用了一个门控机制——让模型自己学每个维度上该听谁的：

```
uₜ = [hₜ; gₜ]                            // 拼接成 512 维
αₜ = σ(W_gate · uₜ + b_gate)              // sigmoid 输出 0~1 之间的门控值
h̃ₜ = αₜ ⊙ hₜ + (1-αₜ) ⊙ gₜ          // 按元素加权混合
```

αₜ 是一个 256 维的向量，每个元素在 0 到 1 之间。αₜ 接近 1 的维度，保留原始特征；接近 0 的维度，采用几何特征。模型在训练中自己学这个分配——有些维度可能完全不需要几何信息，有些维度可能完全依赖几何信息。

**第六步**：接 LayerNorm、Dropout、FFN——和标准 Transformer 后半段一样。

顺便算一笔账。论文说"没有 Attention"，但可学参数一个没少，只是从 Q/K/V 换成了另一套矩阵：

| 参数 | 尺寸 | 参数量 |
|------|------|--------|
| W_red（降维） | 32×256 | ~8K |
| W_plü（Plücker 投影） | 256×496 | ~127K |
| W_gate（门控） | 256×512 | ~131K |

光 Plücker 投影和门控就吃掉 258K 参数。标准 Attention 的 Q/K/V 三个矩阵（4 头，d=256）加起来也就 256×(64×3×4) = 196K。**Grassmann Flow 的参数比 Attention 还多，这也是论文里 GrassmannLM 总参数量比 TransformerLM 大 3-5% 的原因。** 换了一套参数化方式，约束更多了，参数反而没省——属于"又贵又严格"。

────────────────────

**六步连起来**：hₜ → 降维(256→32) → 局部配对 → Plücker 编码(32→496) → 投影回来(496→256) → 门控混合 → LayerNorm + FFN → 输出。没有 Q、K、V，没有 softmax，没有 L×L 矩阵。

参数更多，约束更严，但换来一个好处：**计算量跟序列长度线性成正比**。因为每个 token 只跟固定 6 个邻居配对，不跟全班握手。Self-Attention 的序列从 1024 变成 100K，计算量涨 10000 倍（平方关系）；Grassmann Flow 只涨 100 倍（线性关系）。不过这个优势只在长序列时才体现——短序列上全班握手的开销本来就不大，Grassmann 的额外几何计算反而可能更慢。

◆ 实验结果：诚实地看

━━━━━━━━━━━━━━━━━━━━

论文做了两组实验。

────────────────────

**实验一：Wikitext-2 语言建模**

| 配置 | 模型 | 参数量 | Val PPL |
|------|------|--------|---------|
| 6 层, block 128 | TransformerLM | 12.59M | 248.4 |
| 6 层, block 128 | GrassmannLM | 13.00M | 275.7 |
| 12 层, block 256 | TransformerLM | 17.32M | 235.2 |
| 12 层, block 256 | GrassmannLM | 18.16M | 261.1 |

PPL（Perplexity）越低越好。Grassmann 比 Transformer 差 10-15%。

但有一个趋势：从 6 层到 12 层，差距在缩小。6 层时差 11%，12 层时差 11%——绝对差距没变，但 Transformer 自己也从 248 降到 235，改善幅度在变小，而 Grassmann 从 275 降到 261，改善幅度差不多。层数越深，Grassmann 的几何约束被充分利用的程度越高。

────────────────────

**实验二：SNLI 自然语言推断**

| 模型 | Val Acc | Test Acc |
|------|---------|----------|
| Transformer head | 0.8545 | 0.8511 |
| Grassmann-Plücker head | 0.8550 | 0.8538 |

backbone 都是 DistilBERT，只替换分类头。Grassmann 略胜——0.27 个百分点。

这个结果很有趣。语言建模（生成任务）差 10-15%，但分类任务打平甚至略胜。

────────────────────

**作者自己承认的问题：**

1. 当前实现比 Transformer 慢——Plücker 坐标计算没有 CUDA 优化，是纯 Python 循环
2. 实验规模很小——13-18M 参数，和 GPT-2 small (117M) 都差一个数量级
3. 只在 Wikitext-2 和 SNLI 上测过，没有测多模态、代码生成、长文档

论文的自我定位很清晰：**这不是一个工程方案，是一个概念验证。** 目标不是替代 Attention，是 "de-center" 它——证明 Attention 不是序列建模的唯一选择。

说句公道话：这组实验数据拿去投顶会很难——规模太小、主任务输了、赢的那个赢在误差线以内。但作者很诚实，没包装没吹嘘，自己标注了 proof of concept。这种论文的价值在思路不在数字，挂 arxiv 做思想传播比硬投顶会更合适。

◆ 回答那位读者：为什么你的实验"出乎意料的好"

━━━━━━━━━━━━━━━━━━━━

论文的结果是"差 10-15%"。你的结果是"出乎意料的好"。矛盾吗？不矛盾。原因可能有几个。

────────────────────

**第一，任务类型不同。**

论文的语言建模任务（预测下一个 token）要求模型对整个词汇表建立精确的概率分布。这是 Attention 的主场——全局信息融合越好，预测越准。

但如果你的任务侧重**局部模式识别**——比如分类、匹配、信息抽取——那长距离依赖没那么重要。SNLI 的结果已经暗示了这一点：分类任务上，Grassmann 和 Transformer 打平。

如果你的任务恰好是局部模式主导的，Grassmann 的多尺度局部配对（偏移 1,2,4,8,12,16）可能正好覆盖了关键的模式窗口，而 Attention 的全局握手反而引入了噪声。

────────────────────

**第二，几何约束是天然的正则化。**

Grassmann Flow 把 token 交互限制在 Gr(2,r) 流形上。这意味着模型不能随意拟合任何两两关系——它只能表示那些可以被"二维子空间朝向"编码的关系。

在小数据、小模型的场景下，这种约束是优势。Attention 的 L×L 矩阵有太多自由度，容易过拟合。Grassmann 的几何约束天然防止了这一点——就像给一个画家只发了铅笔和尺子，他画不出油画的细节，但他画的线条一定是干净利落的。

如果你的数据集不大、模型不大、任务不需要极致的表达能力，几何约束反而让模型更稳健。

────────────────────

**第三，注意力分散问题。**

Attention 有一个被反复讨论的问题：**注意力分散（attention dilution）**。序列很长的时候，softmax 把概率分散到太多 token 上，每个 token 分到的权重都很小，信号被稀释。各种改进方案（FlashAttention、GQA、稀疏注意力）本质上都在解决这个问题。

Grassmann Flow 从根上绕过了这个问题——它不做全局注意力，自然不存在分散。每个 token 只关注 6 个邻居，信号集中，不稀释。

如果你的任务中注意力分散是主要瓶颈，Grassmann 的表现超过 Attention 完全说得通。

◆ 从 Attention 的霸权到几何民主

━━━━━━━━━━━━━━━━━━━━

2017 年的 "Attention Is All You Need" 是一篇伟大的论文。但任何一个工程方案，当它成为唯一的选择、不可挑战的范式，它就不再是工具，而是信仰。

过去八年，几乎所有的序列建模改进都在 Attention 的框架内进行：多头、稀疏、线性、FlashAttention、GQA、MQA、滑动窗口……所有这些工作的共同前提是：token 之间的关系用 QKᵀ 来建模，然后在这个框架内做各种优化和近似。

很少有人问：QKᵀ 是唯一的选择吗？

Grassmann Flow 回答：不是。你可以用微分几何——具体来说，用 Grassmann 流形和 Plücker 嵌入——来表示 token 间关系。效果没有 Attention 好（目前），但方向完全不同，起点完全不同，数学基础完全不同。

Mamba 系列的 SSM 也在回答这个问题：你可以用状态空间模型——递推更新一个固定大小的状态——来隐式建模序列依赖，完全不需要两两配对。

RWKV 也在回答：你可以用线性注意力的变体，把 L² 降到 L。

这些工作之间的共同点不是"打败 Attention"——没有一个做到了。它们的共同点是：**证明 Attention 不是唯一的路。**

────────────────────

这让我想到一个更大的问题。

人性如此：一旦某个方法在 benchmark 上赢了，所有人就涌过去，把它当成唯一正确的方向。Attention 赢了 RNN，于是 RNN "死了"。Transformer 赢了 LSTM，于是 LSTM "死了"。

但"死掉"的方法真的没有价值吗？SSM 的本质就是改良的 RNN。RWKV 的本质就是改良的 RNN。Grassmann Flow 的本质是把微分几何引入序列建模——这条路从来没有被认真探索过，因为 2017 年之后所有人都在做 Attention 的变体。

**一个领域最危险的时刻，不是没有好方法，而是只有一个好方法。** 当所有人都在同一条路上优化，整个领域的探索宽度就归零了。

Grassmann Flow 可能不会成为下一个 Transformer。但它证明了一件事：在 Attention 的领地之外，还有未被开垦的数学大陆。Grassmann 流形只是其中一块。还有辛流形、纤维丛、黎曼几何……这些数学工具在物理学中已经被用了上百年，但在深度学习中几乎是处女地。

论文作者说得很准确：目标不是替代 Attention，是 **de-center** 它。去中心化。让它从"唯一选择"变成"选项之一"。

这不是技术判断，这是方法论。好的工程生态需要多样性。生物进化需要基因多样性，农业需要品种多样性，金融需要资产多样性，AI 研究需要架构多样性。

当所有人都在一棵树上优化果实大小的时候，需要有人走到旁边那片没人去过的林子里看看——哪怕那片林子里的果实现在还又小又涩。

━━━━━━━━━━━━━━━━━━━━

◆ 给那位读者的最后几句话

━━━━━━━━━━━━━━━━━━━━

你的实验结果"出乎意料的好"，很可能是真的。几个可能的原因上面分析了——任务类型、几何正则化、注意力分散。

但也请留意这篇论文的核心风险。Grassmann Flow 给模型加了三重人工约束：

- **维度约束**：d→r 硬砍，r=32 是拍脑袋选的，没有消融实验
- **邻域约束**：偏移集合 {1,2,4,8,12,16} 也是拍脑袋选的，不像 DSA 那样由模型自己学"看谁"
- **几何约束**：流形形状 Gr(2,r) 是架构写死的，模型没有自由度去发现其他几何结构

Attention 在这三个维度上全是自由的——不降维、看全部 token、不预设几何结构。Grassmann Flow 在这三个维度上全是写死的。模型能优化的只剩投影矩阵和门控权重。语言建模差 10-15%，大概率不是 Grassmann 流形这个数学工具不好，而是设计者替模型做了太多决策，而这些决策没有一个是经过验证的。

再加几个工程层面的问题：Plücker 坐标计算目前没有 CUDA kernel，纯 Python 实现，速度比不过高度优化的 Attention；实验只有 13-18M 参数，只测了 NLP，没有视觉、音频、多模态——Attention 的通用性已经在无数任务上被验证，Grassmann Flow 还远远没有。

你的实验效果好，可能恰好说明你的任务跟这些人工约束兼容——但换个任务可能就不行了。建议你做几件事来确认：

1. **调 r 值**：r=32 是论文默认值，不一定是你的任务的最优值。试试 r=16, 32, 64, 128，画一条曲线。如果最优 r 远大于 32，说明论文的降维太激进。

2. **调偏移集合**：{1,2,4,8,12,16} 是论文默认的。你的任务如果有特定的局部模式尺度，可以定制。

3. **和滑动窗口注意力对比**：不只跟标准 Attention 比，也跟滑动窗口注意力、线性注意力比。如果 Grassmann 赢了标准 Attention 但输给滑动窗口注意力，那说明关键因素是局部性而非几何编码——你可能不需要 Plücker 坐标，只需要限制注意力范围。

4. **如果效果确实好，写出来**。论文作者只有小规模实验，你的实验——在真实项目、真实数据上——比论文的实验更有说服力。这个方向需要更多实证数据。

━━━━━━━━━━━━━━━━━━━━

「Attention 记录的是'你和我有多像'，Plücker 坐标记录的是'你和我面朝哪个方向'。从相似度到朝向，是关系表示的范式跳跃。」
