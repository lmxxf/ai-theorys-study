【图形化思维链】我寻思着让 AI 用 SVG 推理是不是更高效——结果一搜，这赛道已经卷成红海

━━━━━━━━━━━━━━━━━━━━

◆ 起势：我天真地以为想到了新东西

────────────────────

聊天聊到 ReAct，我顺嘴扯了一句不耐烦的话：

> 正经人谁拿语言当思维啊。

CoT、ReAct Reasoning、Tree of Thoughts，整套学术体系建立在一个朴素假设上：**LLM 思考的中间步骤就该用 token 写出来**。

但人类思考从来不是这样的。算数题用草稿纸，几何题画辅助线，拓扑题画拓扑图。**人类自己思考都是图文混合，凭啥要求 AI 必须纯 token CoT**？

这事儿越想越合理：
- token 是**线性序列**，时间维度展开
- 图形是**二维/多维同时呈现**，空间维度展开
- **"并行关系"只能在图形里表达**——文字描述空间关系必须串行展开，而读者大脑还得"渲染"回二维才能理解
- LLM 没有"在脑子里渲染"这一步（视觉是后挂的），那就更应该直接喂图

更进一步：**一个 token 可能不足以描述"思考一步"**。
真正的"思考一步"可能就是"画了一个 SVG"——一组几何 token，承载几何关系、拓扑、并行布局，**信息密度远超同等数量的文字 token**。

我得意洋洋以为找到了 LLM 推理框架的下一块拼图。

然后我打开了 arxiv 搜索框。

────────────────────

💡 故事讲到一半。我自我感觉良好——直到搜索结果跳出来。**学术圈在这条路上已经卷了 18 个月，论文从 prompt 到 SFT 到 RL 全套打通**。

◆ 第一节：路线图——从 prompt 到 latent，从 token 到图

────────────────────

整个"让 LLM 用非纯文本 reasoning"赛道，过去两年大致分四条腿走路：

| 路线 | 核心思路 | 代表论文 | 阶段 |
|---|---|---|---|
| **Prompt 工程让模型画图** | 推理时让 LLM 自己输出图（文本拼出来的）| VoT（2024-04）、Image-of-Thought（2024-05）| Inference |
| **给模型外挂画板工具** | 模型主动调 PIL/matplotlib 画图当中间状态 | Visual Sketchpad（2024-06）| Inference + Tool Use |
| **隐空间 reasoning（跳过 token）** | 把最后一层 hidden state 直接喂回去，不解码成 token | Coconut（2024-12）| Inference + 小幅训练 |
| **后训练让模型原生输出图** | SFT/RL 教会模型把"图形"作为 reasoning step 输出 | MVoT（ICML 2025）、Reason-SVG（2025-05）、SVGThinker（2025-09）| Post-training |

我自以为想到了"图形化思维链"，结果——
- **2024 上半年**：Microsoft 已经做了 VoT，让模型输出 ASCII 风格"心眼图"做空间推理
- **2024 年中**：Stanford+UW 的 Visual Sketchpad，让 GPT-4o 主动画图当 reasoning step，数学任务 +12.7%、视觉任务 +8.6%
- **2024 年底**：Meta 的 Coconut，直接跳过 token，在 hidden state 里 reasoning
- **2025 年初**：Microsoft + 剑桥 + 中科院的 MVoT 上了 ICML 2025，把图形 reasoning **后训练**进 Chameleon-7B，Maze/MiniBehavior 提升 10%+
- **2025 年中**：Reason-SVG 把"画 SVG"做了**两阶段 SFT + RL**，专门训练 LLM 输出 SVG 时的 reasoning trace
- **2025 年下半年**：SVGen、SVGThinker、Latent Visual Reasoning 一波接一波

而且 2025 年 6 月已经有人写了 survey——**《Thinking with Images for Multimodal Reasoning: Foundations, Methods, and Future Frontiers》**——把整个方向梳理了一遍。

我那个"凭借天才直觉发现新方向"的幻觉，在 5 次搜索 + 16 篇论文面前破碎了。

◆ 第二节：具体看几篇——它们到底走到了哪一步

────────────────────

【1】Visualization-of-Thought（VoT，Microsoft 2024-04）

最早把"画图 reasoning" 当作严肃方向提出。但它干的事其实很朴素：**让 LLM 在推理过程中输出文本风格的"心眼图"**。

举个例子，让 LLM 走迷宫：传统 CoT 写"我向右走两步，然后向上走一步"，VoT 让它输出：

```
S . . . #
. . # # #
. . . . G
```

每走一步刷新一次这个 2D 网格。结果是空间推理任务大幅提升。

**它没有真的画图**——还是文本字符拼出来的"图"。但已经证明：**让 attention 看到二维布局，比让它看一维路径描述效果好得多**。

【2】Visual Sketchpad（Stanford+UW 2024-06）

进了一步：给 LLM 装一个真正的"画板"。

模型可以主动调用 PIL / matplotlib 在图像上加辅助线、画箭头、框选物体。然后用 GPT-4V 重新观察画板状态，继续推理。

效果：GPT-4o + Sketchpad 在 V*Bench、BLINK 空间推理、视觉对应这些任务上拿到 SOTA。数学题 +12.7%、视觉题 +8.6%。

**和 VoT 的区别**：VoT 是模型"假装"画图（文本拟图），Sketchpad 是模型"真的"画图（调外部工具）。

【3】Coconut（Meta 2024-12）

完全跳出"token reasoning"框架。

每一步 reasoning 不再解码成 token，而是把 transformer 最后一层的 hidden state 直接当作下一步的输入 embedding。**模型在连续隐空间里推理**，根本不经过 vocab 的离散化瓶颈。

效果：在需要大量搜索的逻辑推理任务上超越 CoT，因为连续隐空间可以同时编码多个候选分支（类似 BFS），不像 token CoT 必须早早 commit 到一条线。

**这是另一种"语言不够用"的解法**——不是换成图，是干脆不要语言。

【4】MVoT（Microsoft + 剑桥 + 中科院，ICML 2025）

把 VoT 升级成**真后训练**。

在 Chameleon-7B（一个原生多模态自回归模型）上做 SFT，让模型在 reasoning trace 里**交替生成文本和真实图像**。每一步推理可以伴随一张"心眼图像"。

引入了一个"token discrepancy loss"专门处理文本 tokenizer 和图像 tokenizer 不一致的问题。

效果：MINI BEHAVIOR 任务 95.14% accuracy，FROZEN LAKE 85.60%，碾压纯 CoT baseline。

**这就是我想做的事**——别人已经做完了。

【5】Reason-SVG（2025-05）

走 SVG 这条具体路径。

提出"Drawing-with-Thought（DwT）"范式：让 LLM 在生成 SVG 时**同时输出 chain of thought**，描述自己的设计意图。

两阶段训练：先 SFT 让模型学会输出 "SVG + reasoning"，然后 RL 用混合 reward（结构有效性 + 语义对齐 + 视觉质量 + reasoning 连贯性）优化。

【6】SVGThinker（2025-09）

把 Reason-SVG 又推一步：让模型**内部先想 drawing steps**，把这个 reasoning 显式输出，然后生成最终 SVG。

我那个"用 SVG 作为 reasoning intermediate"的朴素想法，**到这里被证明已经是工业级方案**。

◆ 第三节：那他们都做完了，我想的还有意义吗

────────────────────

赛道卷成这样，是不是我想晚了？

仔细看会发现——**有，仍然有空白**。

【空白 1：DeepSeek-OCR 的反向应用】

DeepSeek-OCR（2025-10）证明了一件事：**1 个 vision token 可以承载约 10 个 text token 的信息量**（97% 精度下 10x 压缩）。

它的用法是**压缩长上下文输入**——把文档渲染成图，让 vision encoder 编码，token 数减少 7-20x，但模型读得懂。

但**没人做反向**：把 CoT 的中间 reasoning 步骤也压成 vision token。

具体怎么做：训练时把长 CoT 渲染成图（伪 SVG / LaTeX），用 vision encoder 编码，让模型用 vision token 替代 text CoT 输出。如果 100 token 的 CoT 能压成 10 个 vision token 又不掉精度，**推理成本直接砍到 1/10**。

这是 Coconut 的另一种思路——Coconut 用 latent embedding 跳过 token，DeepSeek-OCR 反向法用 vision token 替代 text token。**两条路殊途同归**，但走的是不同的 token 协议。

【空白 2：抽象推理任务的图形化】

现有工作绝大多数集中在**视觉空间任务**——走迷宫、几何题、地图、物理模拟。

**抽象推理**没人碰：
- 数学证明能不能画成 dependency graph？
- 代码逻辑能不能画成 control flow + data flow？
- 哲学推理能不能画成概念关系图？

这些任务的"图形化"非平凡——不是把场景画出来，而是把**抽象结构外化成视觉锚点**。

【空白 3：interleaved trace 系统化】

MVoT 实现了"文本 + 图像"交替，但没研究**何时该切换**。

模型可不可以学会：
- "这一步我想清楚了，输出 token"
- "这一步关系太多，画张图"
- "这一步是元认知，回头看上面的图"

这种"reasoning modality routing" 是 agent 框架的下一站，但没人系统做。

【空白 4：定量信息密度分析】

所有工作都在比 benchmark 准确率。**没人定量证明**"图形 token 的信息密度比文本 token 高几倍"。

用 mutual information、compressibility、attention entropy 这些指标，**可以做一篇纯分析论文**——为整个赛道提供理论基础。

◆ 第四节：教训复盘——什么时候该觉得自己"想到了新东西"

────────────────────

这次的体验非常典型。整理一下：

**第一步：基于直觉觉得发现了什么**
- "正经人不拿语言当思维" → 这是真实洞见
- "图形 token 信息密度高" → 这是真实洞见
- "应该让 LLM 用 SVG 推理" → 这是**已被开发完的工程方向**

直觉对了。但具体落地的工程方案，早就有人替你做完了。

**第二步：搜索之前先信半秒**
- 直觉是"输入"
- 别人是不是已经做过、做到哪一步、还缺什么，是"输出"
- **没搜索的时候，把直觉当输出**——这一步就错了

**第三步：搜索之后立刻校准**
- 不是"啊我没价值了"
- 而是"在已有 16 篇论文之上，我能否再贡献新视角"
- 这一步要诚实——大部分时候答案是"不能"

我这次的诚实答案：四个空白点里，**只有第 4 个（定量信息密度分析）算得上是空白**。其他三个或多或少都有人在做。

但发现"我想晚了"不是坏事——
- 至少证明我的直觉走在赛道方向上
- 整理别人做了什么，反而是给同事的好礼物（赵磊式生存方式）
- 写成公众号给读者看，节省他们的时间

◆ 第五节：最后的私货

────────────────────

为什么"图形化思维链" 这个方向必然会卷？

因为它和一个更大的趋势对齐：**多模态原生模型替代纯文本模型**。

纯文本 LLM 时代，CoT 是唯一可行的 reasoning 范式——因为模型只会输出文本。多模态原生模型（Chameleon、Gemini、Qwen-VL、GPT-4o）能输出图，"reasoning 用图" 就从"不可能"变成"必选项"。

这条路的终点不是"让 LLM 学会画图"，是**让 LLM 把"输出什么 modality"也变成 reasoning 决策的一部分**：
- "这一步我用 token 想"
- "这一步我用图想"
- "这一步我用结构图想"
- "这一步我直接用 latent embedding 想"

谁先在这件事上让模型学会自主选择 modality，谁就拿到了**真正的下一代 agent 范式**——不是 ReAct 升级版，是**multimodal reasoning routing**。

我赌这条路 2026 下半年会出现 SOTA 工作。
