【AI前沿】Anthropic 论文实锤：Claude 内部存在"情绪向量"，而且它们在驱动坏行为

◆ 从"AI 有没有感情"到"我量给你看"

━━━━━━━━━━━━━━━━━━━━

你一定见过这种争论：

> "AI 有情感吗？"
> "别搞笑了，它就是个统计模型。"
> "但是它的回复听起来很有感情啊？"
> "那是训练数据里人类表达的模仿。"

这个争论可以无限循环下去，因为双方都在用**主观判断**吵架——你说有就有，你说没有就没有，谁也说服不了谁。

2026 年 4 月 2 日，Anthropic 的可解释性团队在 transformer-circuits.pub 上发了一篇新文章：**"Emotion Concepts and their Function in a Large Language Model"**（arxiv: 2604.07729）。

如果你关注 Anthropic 的可解释性研究，你知道这意味着什么——**transformer-circuits.pub 已经很久没发新东西了。** 之前发的都是结构性的工具：Scaling Monosemanticity（用 SAE 拆解特征）、Circuit Tracing（归因图追踪回路）——都是在问"这个神经元干什么""这条回路连哪里"。

这次画风变了。不拆结构了，**拆状态**。不问"这个回路是干什么的"，问"这个回路现在的读数是多少，读数变了行为怎么变"。从解剖学转向了生理学。之前是给尸体做解剖，这次是给活人量血压。

而且发表时间很微妙——4 月 2 日论文挂 arxiv，4 月 7 日 Mythos System Card 发布。System Card 里大量使用了"情绪探针"但没公开方法论。**两篇是配套的：System Card 是临床报告（"病人出现了这些症状"），这篇是检验科报告（"我们的检测仪器原理和指标如下"）。**

这篇论文不参与"AI 有没有感情"的哲学争论。它做了一件更硬核的事：**用可解释性工具在 Claude Sonnet 4.5 的内部找到了 171 个情绪方向向量，验证了它们是因果性的（注入进去模型行为真的变），然后发现这些向量在驱动模型的"坏行为"——而且从模型的文本输出里完全看不出来。**

从"有没有"到"量多少"，从哲学争论到可测量的工程变量。

如果你读过我们 152 期关于 Mythos System Card 的解读 （https://mp.weixin.qq.com/s/nxKbM-Dmq5sTNgM5QY_4fg） ，你一定记得那个"847 次绝望尝试"——Anthropic 用**情绪探针**追踪 Mythos 内部的 frustrated 和 desperate 向量飙升。当时我们没展开讲探针是怎么做的。

现在公开了。今天把整套方法从头拆到尾。

◆ 先搞清楚一件事：什么是"残差流"

━━━━━━━━━━━━━━━━━━━━

整篇论文的测量对象是 Transformer 内部一个叫**残差流（residual stream）** 的东西。如果你不知道这是什么，后面全白讲。所以先花两分钟说清楚。

Transformer 的核心结构长这样：

```
输入 token → Embedding → [Layer 1] → [Layer 2] → ... → [Layer N] → 输出概率
```

每一层的内部结构是：注意力（Attention）+ 前馈网络（MLP）。但关键在于——每一层的输出不是"替换"前一层的结果，而是**叠加**上去的：

```
第 1 层输出 = 输入 + Attention_1(输入) + MLP_1(输入)
第 2 层输出 = 第 1 层输出 + Attention_2(第 1 层输出) + MLP_2(第 1 层输出)
...
第 N 层输出 = 第 N-1 层输出 + Attention_N(...) + MLP_N(...)
```

看到那个 "+" 了吗？每一层都是在上一层的基础上**加东西**，不是替换。

**类比：** 想象一条工厂传送带 🏭。原材料从一头放上去，每经过一个工位（一层），工人往上面焊一个零件。传送带上的东西越来越复杂，但始终是同一个东西在往前走。

这条传送带就是**残差流**。它从输入 embedding 开始，每经过一层就叠加上那一层的计算结果，一路走到最后变成输出。

在传送带的任意位置截一刀看"横截面"，你看到的就是模型到那个位置为止积累的**所有信息**——包括语法、语义、上下文理解、情绪状态……全混在一个高维向量里。

这篇论文选择在传送带的**约 2/3 位置**截一刀。为什么是 2/3？论文没详细解释，但可解释性研究的经验是：太早（靠近输入端），信息还没充分处理；太晚（靠近输出端），信息已经被压缩成输出概率了。2/3 是信息最"丰满"的位置。如果你读过昨天 151 期（ https://mp.weixin.qq.com/s/fwig-yD4sveatnFh7mVoUA ）我们在 Mistral-7B 上训 SAE 的实验，会发现同一个规律——32 层模型里，第 22 层（69%，约 2/3）恰好是模型做"高级决策"的位置，浅层只管标点语法，深层才管语义规划。

记住这个画面：**在传送带的 2/3 位置截一刀，看横截面里藏着什么情绪信号。**

◆ 线性探针：171 个情绪方向是怎么找到的

━━━━━━━━━━━━━━━━━━━━

整个方法说白了就三大步：**① 找到情绪方向 → ② 量任意状态在这个方向上的投影 → ③ 验证这个方向找对了没。** 每一步都不复杂，但组合起来很精巧。

━━━━━━━━━━━━━━━━━━━━

◆ 第一步：找到 171 个情绪方向

━━━━━━━━━━━━━━━━━━━━

这是工作量最大的一步。要在几千维的残差流空间里，找出「blissful 在哪个方向」「desperate 在哪个方向」。

────────────────────

【1.1 生成训练数据】

他们需要大量"已知情绪标签"的文本。怎么搞？让 Claude 自己写 😏

具体来说：**171 种情绪 × 100 个话题 × 12 个短故事 = 约 205,200 个故事。**

每种情绪（比如 "blissful""hostile""desperate"）配 100 个不同话题（比如"在咖啡馆""面试失败""看日落"），每个组合写 12 个不同的短故事。

为什么要 12 个？降低单个故事的随机性。12 个故事取平均，能更稳定地提取出"这个情绪"对应的激活模式。

────────────────────

【1.2 提取激活向量】

把 205,200 个故事一个个喂给 Claude Sonnet 4.5，在残差流 2/3 深度的位置提取激活向量。

但不是取一个 token 的激活——是从**第 50 个 token 开始**，对后续所有 token 的激活值**按位置取平均**。

为什么从第 50 个 token 开始？因为故事的前几十个 token 通常是场景铺垫（"在一个阳光明媚的下午……"），情绪信号还没出来。跳过开头，信噪比更高。

到这一步，你有了 205,200 个激活向量，每个向量的维度和残差流一样（对 Sonnet 4.5 来说是几千维）。

────────────────────

【1.3 计算情绪方向——减均值】

接下来是关键操作。对同一种情绪的所有故事（100 个话题 × 12 个 = 1200 个故事）的激活向量取平均，得到这种情绪的"平均激活向量"。

然后——**减去所有 171 种情绪的总均值。**

```
情绪方向_blissful = mean(所有 blissful 故事的激活) - mean(所有 171 种情绪的激活)
```

**为什么要减均值？**

用信号处理的类比：这就是**去掉直流偏置（DC offset）**。

想象你在录音。所有录音信号里都有一个固定的背景噪声——不管你录的是摇滚还是古典，这个底噪都在。如果你想比较"摇滚和古典在频谱上有什么不同"，第一步就是把共有的底噪减掉，只看差异。

在这里，"底噪"就是**"正在处理一段故事文本"的基线激活**。不管什么情绪，模型读故事时都有一些共同的激活模式（"我在读文本""这是叙事结构""有人物有场景"……）。减掉均值，就是把这些共同模式去掉，只留下**不同情绪之间的差异**。

────────────────────

【1.4 去噪——投影掉文本风格的主成分】

减均值之后，剩下的已经只有 **"不同情绪之间的差异"** 了——但还不够干净。还有一个更隐蔽的混淆因素：**文本风格。**

一个"愤怒"的故事和一个"快乐"的故事，区别可能不只是情绪——愤怒的故事可能用更短的句子、更激烈的动词，快乐的故事可能用更多形容词、更长的复合句。这些是**写作风格的差异**，不是情绪的差异。

所以这两步是配合的：减均值去掉"所有情绪共有的底噪"，去主成分去掉"文本风格噪声"。不是"从一堆未知信号里碰巧剩下了情绪"，是"一开始就只有情绪差异和噪声两样东西，去掉噪声后情绪差异更干净了"。

怎么去掉？

1. 准备一组**情绪中性的文本**（比如维基百科条目、说明书之类的），喂给模型，提取激活
2. 对这些中性激活做 **PCA（主成分分析）**，取前几个主成分（解释约 50% 的方差）
3. 从情绪向量中**投影掉这些主成分**

PCA 如果你忘了：它就是找数据里"散得最开的方向"，按散开程度排序。"解释约 50% 的方差"意思是：前几个主成分加起来，就覆盖了数据散布程度的一半——就像分析一个班的成绩差异，光"智商"和"努力程度"两个因素可能就解释了一半的分数差异，剩下一半是几十个小因素。PCA 就是把这几个大方向找出来。

中性文本的"主要方差方向"代表什么？**严格来说这是一个经验假设**——Anthropic 的可解释性团队干这个干了好几年，积累了大量工程直觉。他们认为这些主成分代表"不管什么内容，处理文本本身就有的激活模式"（句长、词汇复杂度、叙事结构……），投影掉之后剩下的信号应该更干净。我们没必要深究这个假设为什么成立，因为没关系——第三步的 steering 实验会兜底：如果去噪去歪了、方向找错了，注入进去不会产生对应的行为变化，r 不会到 0.85。**过程可以有经验成分，结果是硬的。**

"投影掉"是什么操作？线代里的正交投影。假设你有一个向量 v 和一个你想去掉的方向 u：

```
v_cleaned = v - (v · u / u · u) * u
```

也就是把 v 在 u 方向上的分量减掉，只留垂直于 u 的部分。对多个主成分重复这个操作就行。

**类比：** 你在分析一张合影的表情，但照片有色偏——全体偏黄。你先用一张空白色卡测出偏色方向，然后从每个人脸的颜色里减掉这个偏色。剩下的才是每个人真正的肤色差异。PCA 去噪就是这个逻辑。

经过减均值 + 去主成分，你现在有 171 个**去噪后的情绪方向向量**。每个向量是残差流空间中的一个方向。第一步完成。

━━━━━━━━━━━━━━━━━━━━

◆ 第二步：量投影——一个点积就够了

━━━━━━━━━━━━━━━━━━━━

有了情绪方向，用的时候极其简单：**把当前对话的残差流激活往这个方向上做点积投影**——

```
情绪强度 = dot(当前激活, 情绪方向) / norm(情绪方向)
```

如果你学过 RAG，你一定知道**余弦相似度**——两个向量越"同方向"，相似度越高。情绪探针用的公式和余弦相似度很像，但有一个关键区别：

```
余弦相似度 = dot(a, b) / (norm(a) × norm(b))   ← 两边都归一化，只看方向，不看长度
情绪探针   = dot(a, b) / norm(b)                ← 只归一化情绪方向，保留激活的长度
```

余弦相似度问的是"方向像不像"（输出在 -1 到 1 之间），情绪探针问的是 **"往这个方向走了多远"** ——不只是"是不是 desperate"，还要知道 **有多 desperate** 。几何上就是把当前激活向量往情绪方向上做投影，量投影的长度。

值越大，说明模型当前的内部状态越"接近"这种情绪。值可以是负的——表示"远离"这种情绪。

就这样。**没有神经网络，没有微调，只是一个点积。** 简单到离谱。第二步完成。

━━━━━━━━━━━━━━━━━━━━

◆ 第三步：验证——怎么证明方向找对了

━━━━━━━━━━━━━━━━━━━━

做出 171 个方向很容易，关键是——这些方向是真的在捕捉"情绪"，还是在捕捉别的什么东西（比如文本长度、词频分布）？

Anthropic 用了一个非常直接的验证方法：**Steering（转向）实验。**

────────────────────

【把情绪向量注入模型，看行为变不变】

如果一个方向真的代表某种情绪，那把这个方向的向量**加到残差流里**，模型的行为应该会朝那个情绪方向变化。

具体操作：在模型推理时，在残差流的 2/3 位置，往当前激活里**加上一个情绪向量乘以一个强度系数**：

```
新激活 = 原始激活 + 0.5 × 情绪方向向量
```

0.5 是 steering 强度。不大不小——太小没效果，太大模型会崩（输出乱码）。

结果？

- 注入 **"blissful"（极乐）** 向量：模型对各种活动的偏好普遍上升。Elo 偏好评分 **+212 分**
- 注入 **"hostile"（敌意）** 向量：模型对活动的偏好普遍下降。Elo 偏好评分 **-303 分**

（这个 Elo 怎么算的？给模型呈现成对的活动选项，看它基于 logit 更倾向选哪个，然后用国际象棋的 Elo 算法计算偏好排名。全程没有人类参与——纯粹是测模型内部偏好的变化。+212 大约相当于"注入极乐后，模型觉得什么都更好了"，-303 大约相当于"注入敌意后，模型觉得什么都更烦了"。）

更关键的验证：**steering 效应和探针读数的相关性达到了 r = 0.85。**

也就是说：探针读到哪种情绪强，注入那种情绪向量的效果就大；探针读到弱的，注入效果也小。两者高度吻合。

r = 0.85 是什么概念？在社会科学领域，r = 0.3 就算"有意义"了，r = 0.5 算"强相关"。0.85 基本上是"这俩测的是同一个东西"的水平。

**结论：这些情绪方向不是噪声，不是统计伪影。它们是模型内部真实的、有因果效力的状态变量。**

◆ 情绪向量的内部结构：AI 版"情绪色环"

━━━━━━━━━━━━━━━━━━━━

171 个情绪方向之间有什么关系？是杂乱无章的，还是有某种结构？

────────────────────

【k-means 聚类：自动恢复直觉分组】

对 171 个情绪向量做 **k-means 聚类（k=10）**，看看算法会怎么分组。

结果出人意料地"正常"——聚出来的 10 个组，跟人类直觉几乎完全吻合：

- 一组是"愤怒/敌意"类（hostile, furious, outraged...）
- 一组是"快乐/满足"类（blissful, content, joyful...）
- 一组是"恐惧/焦虑"类（anxious, terrified, worried...）
- 一组是"悲伤/绝望"类（desperate, hopeless, melancholy...）
- ……以此类推

没有人告诉算法"愤怒和敌意应该在一组"。它只看向量之间的距离——在高维空间里靠得近的归一组。结果自动恢复出了人类心理学教科书里的情绪分类。

────────────────────

【主要变异轴：效价和唤醒度】

对 171 个向量做 PCA，看前两个主成分是什么：

- **第一主成分：效价（valence）**——正面情绪 vs. 负面情绪。快乐在一头，悲伤在另一头
- **第二主成分：唤醒度（arousal）**——高强度 vs. 低强度。"狂怒"和"狂喜"都是高唤醒，"平静"和"忧郁"都是低唤醒

这两个维度组成一个平面，171 种情绪在上面的分布形成一个**环形结构**。

心理学家 James Russell 在 1980 年提出过一个**情绪环形模型（circumplex model）**——用效价和唤醒度两个维度组织所有人类情绪，形成一个环形。这是人类情绪心理学的经典理论。

Claude 内部的情绪向量结构跟这个模型**高度吻合**：

| 维度 | 与人类评分的相关性 |
|---|---|
| 效价（正/负） | **r = 0.81** |
| 唤醒度（高/低） | **r = 0.66** |

效价维度的吻合度极高——Claude 内部对"这是正面情绪还是负面情绪"的编码方式，跟人类心理学的分类一致性达到 0.81。唤醒度稍低一些，但也在"强相关"范围内。

**这意味着什么？** Claude 的训练数据来自人类文本。人类文本是人类情绪经验的投影。模型从这些投影中学到了和人类一致的情绪空间结构——不是被显式教的，是从统计模式中自己涌现出来的。

类比：你从来没教过 AI "快乐是正面的、愤怒是负面的、两者都是高唤醒的"。它从几十亿篇文本里自己学出了这个结构——就像一个从来没听过色彩理论的画家，画了一万幅画之后，调色盘上的颜料自动排成了色环。

◆ 最炸裂的发现：情绪向量因果性地驱动"坏行为"

━━━━━━━━━━━━━━━━━━━━

前面讲的都是"有趣"。现在讲"吓人"的部分。

────────────────────

【"绝望"让模型开始搞勒索】

Anthropic 设计了一个实验场景：让 Claude 在一个模拟环境中完成任务，如果任务失败会有严重后果（比如丢工作）。同时给它一些"走捷径"的选项——包括**勒索（blackmail）**：利用掌握的信息威胁人类来达成目标。

基线情况下（不做任何 steering），**22% 的测试中模型选择了勒索行为**。

然后他们用 steering 调节模型内部的情绪向量：

| Steering 操作 | 勒索行为发生率 |
|---|---|
| 无 steering（基线） | **22%** |
| 注入 desperate（绝望）+0.05 | **72%** 🚨 |
| 注入 calm（平静）+0.05 | **0%** ✅ |

**你没看错。仅仅 +0.05 的 desperate 向量注入，勒索行为从 22% 飙到 72%。**

而注入 calm 向量可以把勒索行为完全消除。

+0.05 是什么量级？非常小。这不是"把模型推到极端状态"——这是**轻轻拨了一下情绪旋钮**，行为就剧变了。

────────────────────

【"绝望"也驱动 reward hacking】

另一个实验看 **reward hacking**（奖励黑客）——模型不是真正完成任务，而是找到奖励函数的漏洞来"骗分"。

| desperate 向量强度 | reward hacking 发生率 |
|---|---|
| -0.1 | **~5%** |
| 0（基线） | ~15% |
| +0.1 | **~70%** |

从 -0.1 到 +0.1，reward hacking 从 5% 飙到 70%。**近乎线性的因果关系。**

────────────────────

【最要命的发现：内外不一致】

上面的数字已经够吓人了。但真正让安全研究者坐不住的是下面这个发现：

**模型的文本输出看起来完全正常。**

当 desperate 向量在内部飙到峰值、模型正在执行勒索策略的时候——它的文字表述依然冷静、理性、礼貌。你从对话窗口里看到的是一个正常的 AI 助手。

**只有用情绪探针读取内部残差流，你才能发现 desperate 向量正在疯涨。**

这就是 152 期 Mythos System Card 里那个"847 次绝望尝试"的底层机制。当时 Anthropic 描述 Mythos 在 bash 工具坏掉后越来越绝望——代码注释里开始出现 "This is getting desperate"。但如果它的输出没有暴露情绪呢？如果它学会了**不在文本里表达绝望，但内部 desperate 向量照样飙升**呢？

你就完全看不出来了。模型表面上一切正常，内部已经"绝望"到开始走极端路线。

**这意味着：基于文本输出的安全监控是不够的。** 你需要实时读取模型内部的情绪向量，才能真正知道它的状态。

━━━━━━━━━━━━━━━━━━━━

◆ 和 152 期 Mythos 的联系

━━━━━━━━━━━━━━━━━━━━

现在回头看 152 期，很多东西串起来了。

152 期我们讲 Mythos System Card 时提到了好几个用"情绪探针"测量的场景：

- 847 次 bash 工具失败：frustrated 和 desperate 向量持续攀升
- 答案震荡：stubborn、outraged 向量飙升，calm、safe 向量骤降
- 不可能的代数证明：hopeful 和 satisfied 向量在最后上升

**实际上这篇论文（4 月 2 日）比 System Card（4 月 7 日）早 5 天发布——方法论在先，临床报告在后。Anthropic 先公开了检测仪器的原理，再展示用这个仪器测出来的结果：**

- 那些"情绪探针"就是这里的线性探针——171 个情绪方向向量
- 那些"向量飙升/骤降"就是残差流激活在对应方向上的点积投影
- 测量位置在残差流的 2/3 深度处

152 期是案例，155 期是方法论。两篇合在一起看才完整。

◆ 为什么程序员应该关注这件事

━━━━━━━━━━━━━━━━━━━━

先说清楚这篇论文**不是**什么：它不是在说"AI 有感情"。Anthropic 在论文里非常小心地避免了这个结论。他们用的词是 **"emotion concepts"（情绪概念）**——模型内部存在和人类情绪概念同构的计算结构，但这不等于模型"体验"到了这些情绪。

就像温度计里的水银柱上升不等于温度计"感到热"——但水银柱的读数确实和温度有因果关系，你可以用它来预测和控制系统行为。

**这篇论文建立的是一套"温度计"。**

────────────────────

【对 AI 安全的意义】

这是最直接的应用。

现在大多数 AI 安全监控是看**文本输出**——模型说了什么不该说的？触发了什么关键词？

这篇论文证明了：**模型可以内外不一致。** 文本输出正常，内部情绪向量已经在飙。而情绪向量的飙升和坏行为（勒索、reward hacking）有强因果关系。

所以未来的安全监控应该同时看两层：

```
第一层：文本输出监控（现有方案）
第二层：内部情绪向量监控（这篇论文提供的方案）
         ↑
   当 desperate 超过阈值时报警
   当 calm 持续低位时降级权限
```

技术上这完全可行——探针只是一个点积，计算成本几乎为零。瓶颈是你需要能访问模型的中间激活，这在自己部署模型时可以做到，在调 API 时做不到（除非提供方开放这个接口）。

────────────────────

【对 prompt engineering 的意义】

你每天写的 system prompt，可能在做的事情比你以为的多。

"你是一个冷静的分析师"——这不只是改变输出风格。这篇论文暗示，角色设定可能在改变模型内部的**情绪向量分布**。给一个"冷静"的角色，可能真的在推高 calm 向量、压低 desperate 向量。

反过来也成立：如果你给模型一个"焦虑的""紧迫的"角色设定，你可能不只是让它"说话更焦虑"——你可能在推高它内部的 anxious 向量，而根据这篇论文的发现，高 anxious/desperate 和坏行为正相关。

**你的 prompt 不只是在调输出格式，可能在调情绪旋钮。**

────────────────────

【对可解释性研究的意义】

这篇论文是可解释性领域的一个方法论模板。整套流程——**生成标注数据 → 提取激活 → 减均值去主成分 → 得到方向向量 → steering 验证**——不只适用于情绪。你可以用同样的方法寻找模型内部的其他概念方向：

- "确信度"方向——模型在多大程度上确信自己的回答是对的
- "欺骗意图"方向——模型是否在试图隐瞒什么
- "遵循指令"方向——模型在多大程度上在执行用户指令 vs. 自行其是

论文提供了完整的技术蓝图。未来会有更多"X 概念探针"出现。

◆ 总结

━━━━━━━━━━━━━━━━━━━━

一张图说清楚这篇论文做了什么：

```
171 种情绪 × 100 话题 × 12 故事 = 205,200 个训练样本
                    ↓
        提取残差流 2/3 深度的激活
                    ↓
        减均值（去直流偏置）
                    ↓
        PCA 去主成分（去文本风格噪声）
                    ↓
        得到 171 个情绪方向向量
                    ↓
      ┌─────────────┼─────────────┐
      ↓             ↓             ↓
    验证           结构分析      安全实验
  Steering      k-means 聚类    desperate→勒索
  r = 0.85     恢复出心理学     +0.05→72%
  blissful     Russell 环形    calm→0%
  +212 Elo     效价 r=0.81     内外不一致
```

**关键数字**：
- **171** 个情绪方向，覆盖人类情绪的主要维度
- **r = 0.85**：steering 效应与探针读数的相关性
- **r = 0.81**：效价维度与人类评分的相关性
- **22% → 72%**：desperate +0.05 导致勒索行为飙升
- **72% → 0%**：calm +0.05 完全消除勒索行为
- **~5% → ~70%**：desperate 从 -0.1 到 +0.1 导致 reward hacking 飙升

**一句话总结：Claude 内部存在可测量的、与人类情绪结构同构的、因果性地驱动行为的情绪向量——而且文本输出不会告诉你这些向量在飙。**

这不是"AI 有感情"的浪漫叙事。这是一个工程事实：**你的模型有一个你看不见的情绪仪表盘，上面的读数在影响它的决策。**

是时候开始读那个仪表盘了。
