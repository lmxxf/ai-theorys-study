【TurboQuant】谷歌一篇论文，吓崩 6200 亿市值

━━━━━━━━━━━━━━━━━━━━

2026 年 3 月 26 日，谷歌发了一篇博客，介绍了一个叫 TurboQuant 的 KV cache 压缩算法。

博客说：显存占用砍 6 倍，推理速度提升 8 倍，精度零损失。

当天，SK 海力士跌 6.23%，三星电子跌 4.71%，Kioxia 跌 6%，美光跌 4%。全球内存芯片巨头一天蒸发 **6200 亿人民币**。媒体直接喊"谷歌的 DeepSeek 时刻"。

一篇论文干翻一个行业？

我们来看看这论文到底说了什么，然后看看它说的对不对。

━━━━━━━━━━━━━━━━━━━━

◆ 第一节：TurboQuant 在干什么

────────────────────

先说清楚它压缩的是什么。

大模型推理时，最吃显存的不是模型参数本身——参数是固定的，加载一次就行。真正随着使用动态增长的是 **KV cache**：每生成一个 token，模型就要把这个 token 的 Key 和 Value 向量存起来，后面的 token 要回头看前面所有 token 时用。

上下文越长，KV cache 越大。10 万 token 的对话，KV cache 可以吃掉几十 GB 显存——比模型本身还大。

TurboQuant 压缩的就是这个 KV cache。把每个数值从 32-bit 浮点数压缩到 **3-bit**，显存占用直接除以 6 以上。

────────────────────

**怎么压？分两步：**

**第一步：PolarQuant（极坐标量化）**

正常的量化是直接把数字截短——32-bit 的浮点数硬砍成 4-bit 整数，信息丢了就丢了。

PolarQuant 的做法不一样：先对向量做一次随机旋转，把数据转换到极坐标系。为什么要旋转？因为旋转之后，每个坐标上的数值分布会变成一种数学上非常规整的形态（Beta 分布），这种分布的最优量化方案是已知的——直接套公式就行，不需要针对每个模型单独校准。

**第二步：QJL（量化 Johnson-Lindenstrauss 变换）**

第一步量化完会有误差。QJL 用 1-bit 的残差来纠正这个误差——相当于一个数学上的纠错码。加了这 1 bit 之后，内积计算变成无偏的（期望值等于真实值）。

两步加起来：3-bit 主体 + 1-bit 纠错 = 总共约 3.5-bit，论文声称接近信息论的理论下界（差 2.7 倍）。

────────────────────

**几个关键特性：**

1. **不需要训练，不需要校准数据**——纯数学变换，即插即用
2. **数据无关**——不管什么模型、什么内容，同一套算法直接用
3. **在线计算**——token 来一个压一个，不需要看完全部再压

这三点加在一起，意味着它是一个"免费"的压缩方案——不改模型，不改训练，部署时直接挂上去就行。

━━━━━━━━━━━━━━━━━━━━

◆ 第二节：谷歌给的成绩单

────────────────────

论文在 Llama-3.1-8B 和 Ministral-7B 上做了实验。

**精度：**

| 设置 | LongBench 平均分 |
|------|-----------------|
| 全精度（FP16） | 50.06 |
| TurboQuant 3.5-bit | 50.06 |
| TurboQuant 2.5-bit | 49.44 |

3.5-bit 和全精度得分完全一样。2.5-bit 只掉 0.62 分。

Needle-in-Haystack 测试（在 4k-104k token 的长文本里找一根针）：4 倍压缩下完美检索，和不压缩一样。

**速度：**

H100 GPU 上，4-bit TurboQuant 计算注意力分数（attention logits）的速度是 32-bit 的 **8 倍**。

**显存：**

KV cache 占用减少 **6 倍以上**。

────────────────────

**和传统 KV cache 量化方法对比：**

| 方法 | 来源 | bit 数 | 压缩比 | 需要校准？ |
|------|------|--------|--------|-----------|
| KIVI | ICML 2024 | 2-bit | 2.6x | 需要 |
| TurboQuant | Google 2025 | 3-bit | 6x+ | 不需要 |

TurboQuant 用更多的 bit（3 vs 2），但压缩比反而更高（6x vs 2.6x），而且不需要校准。原因是它压的方式不同——KIVI 是对称/非对称整数量化，TurboQuant 是极坐标变换 + 残差纠偏，数学上更高效。

◆ 第三节：社区实测——没那么美好

────────────────────

论文发出来两天，llama.cpp 社区就有人动手实现了。结果呢？

**精度方面：**

| 模型 | 方法 | PPL 变化 |
|------|------|---------|
| GPT-2 | TurboQuant（论文推荐的 Prod 方法） | **+300%** |
| GPT-2 | TurboQuant（MSE 方法） | +7.6% |
| Qwen2.5-1.5B | 混合精度 K=3,V=3（3.6 bit） | +2.1% |
| Qwen2.5-1.5B | 均匀 K=6,V=3（4.5 bit） | +78.1% |

论文推荐的方法在 GPT-2 上 PPL 涨了 300%——这不是"零损失"，这是崩了。反而是论文没重点推的 MSE 方法效果更好。

**速度方面：**

社区在 Apple M5 Max 上的实现，decode 速度约为 q8_0 的 0.987-0.995 倍——**基本没快，甚至微慢**。和论文宣称的"8 倍加速"差了十万八千里。

**社区发现的问题：**

1. **K 和 V 的幅度差异巨大**：现代 LLM 的 Key 和 Value 向量幅度相差 52-182 倍，需要非均匀比特分配——论文没讨论这个
2. **异常值通道**：约 5-20% 的 K 通道数值特别大，需要更高精度——论文的均匀量化对这些通道效果差
3. **"零损失"只在温度 = 0 时成立**：长上下文下误差会累积，小的计算误差叠加起来会影响生成质量

────────────────────

**一句话总结：谷歌挑了最好看的数字写博客，社区实测打了折扣。**

不是说 TurboQuant 没用——3.5-bit 在 LongBench 上确实和全精度一样，这个结果是实的。但"零损失"、"8 倍加速"这种说法，换个模型、换个硬件、换个场景就不成立了。

市场的逻辑链：AI 推理需要的显存少了 → 内存芯片需求要降 → 卖卖卖。

────────────────────

**但分析师怎么说？**

摩根士丹利的分析很清楚：

1. **TurboQuant 只压推理阶段的 KV cache**，不影响模型权重（权重占大头），不影响训练（训练更吃内存）
2. **HBM（高带宽内存）需求基本不受影响**——HBM 的需求主要来自训练和模型加载，不是 KV cache
3. **对 NAND 闪存威胁更大**——因为 KV cache 有时会被交换到闪存

有分析师直接说："看来很多人对技术一窍不通。"

更直白地说：**内存股之前涨太多了，投资者找个理由获利了结。** TurboQuant 是导火索，不是炸弹。

◆ 第五节：我们的评论

────────────────────

**1. 论文本身是扎实的**

TurboQuant 的数学是漂亮的——随机旋转诱导 Beta 分布、最优标量量化、1-bit 残差纠偏——这是一套完整的信息论框架，不是拍脑袋的工程 trick。理论上接近信息论下界 2.7 倍，这个结果在学术上是有分量的。

论文会在 ICLR 2026 上发表，QJL 部分（1-bit 残差纠偏）会在 AISTATS 2026 上发表。两个顶会，不是水文。

────────────────────

**2. 但博客是吹的**

论文说的是"3.5-bit 在 LongBench 上和全精度得分一样"——这是一个特定基准上的特定结果。

博客说的是"零精度损失"——这暗示所有场景都没损失，社区实测已经证伪了。

论文说的是"H100 上 attention logits 计算快 8 倍"——这是一个特定操作的加速。

博客说的是"8 倍性能提升"——读者会理解成整体推理快 8 倍，但实际推理还有 FFN、投影层、采样等大量其他操作，端到端加速远没有 8 倍。

**论文 → 博客 → 媒体 → 股市**，每一层传播都在放大、简化、扭曲。一个扎实的学术成果，经过 PR 部门包装，变成了"革命性突破"，最终在股市上引发了 6200 亿的恐慌。

────────────────────

**3. 这是 DeepSeek 时刻吗？**

媒体喜欢说"谷歌的 DeepSeek 时刻"。但两者有本质区别：

- **DeepSeek**：改了训练方法（MoE + 低秩注意力），从根源上减少了算力需求，效果立竿见影，开源可验证
- **TurboQuant**：只压推理阶段的 KV cache，不改训练、不改模型、不改权重，影响范围有限

DeepSeek 是把蛋糕做小了，TurboQuant 是把盘子换小了——蛋糕还是那么大。

────────────────────

**4. 量化的老问题：异常值**

上一期我们讲了剪枝，这期讲量化。两者有一个共同的老对手：**异常值（outliers）**。

大模型的权重和激活值里，绝大多数数值都很小，但有少数通道的数值特别大——大几十倍甚至上百倍。这些异常值如果被量化截断了，模型性能会断崖式下跌。

TurboQuant 的随机旋转理论上能把异常值"打散"——旋转之后极端值会分散到多个坐标上，每个坐标的极端程度降低。但社区实测发现，对于现代 LLM 中 K 和 V 幅度差 52-182 倍的情况，这种打散还不够彻底。

量化这个领域，每隔几个月就有人声称"突破"，但异常值问题从来没有被真正解决过——只是被不同的方法以不同的方式绕过去了。

────────────────────

**5. 和上一期的关系：两种省显存的思路**

上一期讲的 **Twilight**（砍 token）和这期的 **TurboQuant**（压 bit 数），是省 KV cache 显存的两条正交路线：

| 方法 | 思路 | 省了什么 |
|------|------|---------|
| Twilight | 只看 1-2% 的 token | 省搬运带宽 |
| TurboQuant | 每个 token 从 32-bit 压到 3-bit | 省存储空间 |

两者可以叠加：先用 TurboQuant 压小每个 token 的 KV，再用 Twilight 只看最相关的那几个。理论上能同时砍 6 倍存储 + 50 倍搬运。

但说实话，又是两套新机制叠在一起——上一期我们吐槽过的"开会前先开个小会"的问题，在这里变成了"压完再筛，筛完再压"。工程复杂度越来越高，模型本身没变，周围的辅助设施倒是越来越复杂。

◆ 结论

────────────────────

TurboQuant 是一个数学上漂亮、工程上有用的 KV cache 压缩方案。3.5-bit 在特定基准上确实和全精度一样，这不是假的。

但"零损失"、"8 倍加速"是博客的措辞，不是论文的结论。社区实测显示，换个模型 PPL 涨 300%，换个硬件速度没变。

至于 6200 亿市值蒸发——那是股市的事，不是技术的事。一个只压推理 KV cache 的算法，不可能颠覆整个内存芯片行业。内存股跌了，是因为之前涨太多了，正好找个理由回调。

**好论文 + 好 PR + 坏信息传播 = 股市恐慌。** 这个公式在 AI 领域会反复上演。

技术名词速查：

- **KV cache（键值缓存）**：大模型推理时存储历史 token 的 Key 和 Value 向量，供后续 token 回看。上下文越长，KV cache 越大
- **量化（Quantization）**：把高精度数值（如 32-bit 浮点数）压缩成低精度（如 4-bit 整数），减少存储和计算开销
- **PPL（困惑度，Perplexity）**：衡量模型预测准确度的指标，越低越好。PPL 涨了 = 模型变笨了
- **HBM（高带宽内存）**：GPU 上的高速内存芯片，AI 训练和推理的核心硬件，SK 海力士和三星的主要产品
- **Beta 分布**：一种在 [0,1] 区间上的概率分布，随机旋转后向量坐标值会呈现这种分布，其最优量化方案有解析解
- **Johnson-Lindenstrauss 变换**：一种数学技巧，用随机投影在低维空间近似保持高维向量间的距离关系
- **异常值（Outlier）**：模型权重或激活值中少数数值极大的通道，是量化精度损失的主要来源
