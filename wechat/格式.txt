微信公众号文章格式规范

━━━━━━━━━━━━━━━━━━━━

◆ 文件格式

• 文章用 .txt 纯文本，不用 markdown
• 封面图用 .svg，转 .png 发布
• 命名：45.txt / cover45.svg / cover45.png（数字递增）

━━━━━━━━━━━━━━━━━━━━

◆ 排版符号

「强调」用直角引号（微信公众号不支持 **加粗** 语法，用直角引号代替）
【小节标题】用中文方括号（◆ 下面的子内容）
• 列表圆点
✓ 正确 / ✗ 错误
⚠️ 警告/重点
💡 名词解释/人话翻译（给小白看的解释用这个标志）
━━━ 粗分隔线（章节之间）
──── 细分隔线（小节之间）

◆ 标题层级

【关键词】文章主标题（文章开头第一行）
◆ 一级标题（大节）
【二级标题】（小节，在 ◆ 下面的子内容）
• 列表项

注意：▸ 不再使用，统一用 ◆ 作为大节标题

━━━━━━━━━━━━━━━━━━━━

◆ 表格格式

用 ASCII 字符画表格（纯文本 code 块里显示对齐）：

  +----------+-----------------+------------------------------+
  | 列1      | 列2             | 列3                          |
  +----------+-----------------+------------------------------+
  | 内容     | 内容            | 内容                         |
  | 内容     | 内容            | 内容                         |
  +----------+-----------------+------------------------------+

表格框线字符：
• 角和边：+ - |
• 不用 Unicode 框线（┌ ─ │ 这些），在不同字体里宽度不一致，会错位

◆ 风格要点

• 语言接地气，少装逼
• 技术名词要翻译成人话（用 💡 标记）
• 可以用表情但别滥用
• 段落短一点，别堆大段
• 重点用「直角引号」强调，不用 **加粗**（微信不支持）
• 数据和对比用表格呈现，比列表更清晰

━━━━━━━━━━━━━━━━━━━━

◆ 封面图要点

• 直白，一眼能看懂
• 别搞太抽象的
• 主标题要大、醒目
• 配色跟主题相关
• 别画复杂 logo，画不好就用简化图形或文字代替
• 动画效果微信不支持，做静态版本

【SVG 中文字体配置】

cairosvg 转 PNG 时需要指定本地中文字体，否则中文显示为方块。

在 SVG 开头加：

  <svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 800 600">
    <style>
      text { font-family: 'Noto Sans CJK SC', sans-serif; }
    </style>
    ...
  </svg>

系统可用的中文字体（fc-list :lang=zh family）：
• Noto Sans CJK SC（简体中文，推荐）
• Noto Serif CJK SC（简体中文，衬线）
• Noto Sans CJK TC（繁体中文）
• AR PL UKai TW MBE（台湾楷书）

转换命令：
  cairosvg cover77.svg -o cover77.png -d 150
