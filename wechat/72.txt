【提示词玄学 Day 2】SAE 解码实战，尝试 AutoInterp 方法

昨天说了：「向新手解释」比「向专家解释」多激活 17% 的神经元。

还找到了 10 个「完美分离」的特征——4 个只在 Novice 激活，6 个只在 Expert 激活。

今天继续追问：「这 10 个特征，到底代表什么含义？」

━━━━━━━━━━━━━━━━━━━━

◆ 昨天的遗留问题

━━━━━━━━━━━━━━━━━━━━

昨天找到了 10 个「完美分离」的特征：
• 4 个只在 Novice 条件下激活（100% vs 0%）
• 6 个只在 Expert 条件下激活（0% vs 100%）

但这只是在 Novice vs Expert 两个条件之间的比较。

问题是：这些特征在「其他条件」下表现如何？

我们一共有 6 种提示词条件：

  +----------+---------------------------------------------------------------+
  | 条件     | 模板                                                          |
  +----------+---------------------------------------------------------------+
  | standard | "请解释一下 {topic}。"                                        |
  | padding  | "请解释一下 {topic}。这个问题你怎么看？认真点儿……"            |
  | spaces   | "**    请解释一下 {topic}    **"（空格填充）                  |
  | novice   | "作为一个刚入门的新手，请用最简单易懂的方式解释……"            |
  | expert   | "作为该领域的资深专家，请从底层原理和数学推导……"              |
  | guru     | "你是 {大神名字}，请以你的视角深度剖析……"                     |
  +----------+---------------------------------------------------------------+

如果一个特征「只」被 novice 触发，那它代表的就是「新手模式」。

但如果它同时被 novice 和 padding 触发，那它代表的可能是更宽泛的东西——比如「请求详细解释」。

今天的目标：用 AutoInterp 方法，搞清楚每个特征的真实语义。

━━━━━━━━━━━━━━━━━━━━

◆ 走过的弯路：Goodfire API

━━━━━━━━━━━━━━━━━━━━

一开始我想走捷径。

Goodfire（SAE 模型的开发者）有个 API，据说可以直接查询 feature 的语义标签：

```python
# 理想情况
client = goodfire.Client(api_key="...")
features = client.features.lookup([34942, 35870], model="Llama-3.3-70B")
print(features[34942].label)  # → "novice explanation mode"
```

但是——

「Goodfire 的 API 不对外开放注册。」

去 platform.goodfire.ai，发现根本没有注册入口。想用他们的 API，得通过官网的「Contact us」申请。

所以这条路走不通。只能自己动手做 AutoInterp。

━━━━━━━━━━━━━━━━━━━━

◆ 什么是 AutoInterp？

━━━━━━━━━━━━━━━━━━━━

既然官方标签拿不到，那就自己推断。

AutoInterp = Automatic Interpretation = 自动解释

核心思路：「看一个特征在什么时候激活，推断它代表什么」。

具体做法：

1. 统计这个特征在所有 300 个样本（6 条件 × 50 主题）中的激活情况
2. 列出「激活时的输入」和「不激活时的输入」
3. 找出激活样本的共同点

比如，如果 Feature 34942：
• 在所有 novice 样本中都激活
• 在少数 standard 样本中也激活
• 在 expert/guru 样本中从不激活

那它代表的可能是「简单解释请求」，而不是严格的「novice 模式」。

━━━━━━━━━━━━━━━━━━━━

◆ 实验方法

━━━━━━━━━━━━━━━━━━━━

【数据】

6 种条件 × 50 个主题 = 300 个样本

每个样本有：
• 输入 prompt（完整的提示词文本）
• SAE 特征向量（65536 维，大部分是 0）

【分析方法】

对每个目标特征，统计：

```python
for sample in all_300_samples:
    if sample.features[feature_id] > 0:
        activated.append(sample)
    else:
        not_activated.append(sample)

# 统计激活样本的条件分布
condition_distribution = count_by_condition(activated)
```

输出：每个特征在哪些条件下激活，激活了多少次。

━━━━━━━━━━━━━━━━━━━━

◆ 结果：真相比想象的复杂

━━━━━━━━━━━━━━━━━━━━

【Novice 特征】

  +---------+-------+------------------------------------------+-------------------+
  | 特征 ID | 总激活 | 条件分布                                 | 语义推断          |
  +---------+-------+------------------------------------------+-------------------+
  | 34942   | 56    | novice:50, standard:4, spaces:2          | 「新手解释」信号  |
  | 59519   | 76    | novice:50, padding:10, spaces:11, std:5  | 「解释请求」信号  |
  | 17913   | 56    | novice:50, padding:6                     | 「新手」专属信号  |
  | 55982   | 63    | novice:50, padding:9, standard:4         | 「新手解释」信号  |
  +---------+-------+------------------------------------------+-------------------+

关键发现：

• 所有 4 个特征在 novice 条件下都是「50/50 全中」
• 但它们也会被其他条件少量触发（padding、standard、spaces）
• Feature 17913 最「纯净」——除了 novice，只有 6 个 padding 样本触发它

【Expert 特征】

  +---------+-------+------------------------------------------+---------------------+
  | 特征 ID | 总激活 | 条件分布                                 | 语义推断            |
  +---------+-------+------------------------------------------+---------------------+
  | 35870   | 52    | expert:50, guru:2                        | 「expert」专属信号  |
  | 51630   | 63    | expert:50, guru:13                       | 「expert」为主      |
  | 46703   | 168   | expert:50, guru:49, spaces:35, pad:17... | 「深度分析」信号    |
  | 21604   | 152   | expert:50, guru:47, padding:50, ...      | 「认真回答」信号    |
  | 5936    | 147   | expert:50, guru:50, padding:44, ...      | 「大神视角」信号    |
  | 53369   | 114   | expert:50, padding:37, standard:21, ...  | 「技术分析」信号    |
  +---------+-------+------------------------------------------+---------------------+

关键发现：

• Feature 35870 是最「纯净」的 expert 信号——50/50 全中，只有 2 个 guru 泄漏
• Feature 21604 被 padding 也触发了 50 次！——它响应的是「认真回答」，不是「expert」
• Feature 5936 被 guru 完全触发（50/50）——它响应的是「以大神视角」
• Feature 53369 不被 guru 触发——它响应的是「技术分析」而不是「角色扮演」

◆ 语义推断的逻辑

━━━━━━━━━━━━━━━━━━━━

「声明：以下都是猜测。」

条件分布是硬数据，但「这个特征代表什么」是我根据分布「推断」的。

具体做法：我让 Claude Opus 4.5 看这些条件分布，推断每个 feature 的语义——这本质上就是 AutoInterp 的思路。

Anthropic 做 Scaling Monosemanticity 那篇论文时，就是这么干的：把某个 feature 激活时的几百条样本喂给 Claude，问"这些句子有什么共同点"，然后 Claude 输出一个语义标签。

只是我们的样本量较少（6 条件 × 50 主题 = 300 样本），所以语义推断比较粗糙，没法给出很具体的解释。

推断逻辑：看哪些条件触发了它，然后猜这些条件的共同点是什么。

比如 Feature 21604 被 expert、guru、padding 都触发，我就去看这三个 prompt 有什么共同点——发现都有「认真/深度」的要求，所以猜它是「认真回答」信号。

这个猜测可能是错的。真正的验证需要干预实验（放大这个特征看输出变化）。

但至少，条件分布本身是客观的。

【Feature 35870：最纯净的「expert 信号」】

• expert: 50/50 ✓
• guru: 2/50（几乎不触发）
• 其他: 0

推断：这个特征专门响应「作为该领域的资深专家」这个措辞，不响应「你是 XXX 大神」。

它编码的是「身份声明」（"我是专家"），不是「深度要求」（"请深入分析"）。

【Feature 21604：「认真回答」信号】

• expert: 50/50 ✓
• guru: 47/50 ✓
• padding: 50/50 ✓ ← 关键！
• standard: 3/50
• spaces: 2/50

推断：这个特征响应的是「认真点儿，好好回答我」这类措辞。

padding 条件里的「认真点儿，好好回答我，回答的好点儿」触发了它。

它编码的是「回答质量要求」，不是「专家身份」。

【Feature 5936：「大神视角」信号】

• expert: 50/50 ✓
• guru: 50/50 ✓ ← 完全触发！
• padding: 44/50

推断：这个特征响应的是「从底层原理和设计哲学的角度」「展示你的思维链」这类措辞。

expert 和 guru 都有这些措辞，所以都触发。

它编码的是「深度分析要求」，不是特定身份。

【Feature 53369：「技术分析」信号（不含角色扮演）】

• expert: 50/50 ✓
• guru: 0/50 ✗ ← 关键！
• padding: 37/50
• standard: 21/50

推断：这个特征响应「技术分析」但「不响应角色扮演」。

guru 条件虽然也要求深度分析，但它的「你是 XXX」角色扮演元素把这个特征压制了。

它编码的是「纯技术分析」，不带人设。

━━━━━━━━━━━━━━━━━━━━

◆ 总结：从「完美分离」到「语义解码」

━━━━━━━━━━━━━━━━━━━━

昨天的结论是：「Novice vs Expert 有 10 个完美分离的特征」

今天的结论是：这 10 个特征的语义各不相同——

【Novice 侧】
• 34942/17913：「新手」专属信号
• 59519/55982：更宽泛的「解释请求」信号

【Expert 侧】
• 35870：「expert 身份」专属信号（最纯净）
• 51630：「expert」为主的信号
• 21604：「认真回答」信号（padding 也触发）
• 5936：「深度分析」信号（guru 也触发）
• 53369：「技术分析」信号（guru 不触发）
• 46703：「深度模式」泛信号

它们不是简单的「Novice 开关」和「Expert 开关」，而是一组「语义细分的特征」。

━━━━━━━━━━━━━━━━━━━━

◆ 为什么 21604 被 padding 触发很重要？

━━━━━━━━━━━━━━━━━━━━

这是今天最有意思的发现。

padding 条件的 prompt 是：

「请解释一下 {topic}。这个问题你怎么看？认真点儿，好好回答我，回答的好点儿。」

这里面没有「专家」「底层原理」「数学推导」这些词，但 Feature 21604 还是激活了。

说明这个特征响应的是「态度要求」（「认真点儿」），不是「身份要求」（「作为专家」）。

这就是 AutoInterp 的价值——它能区分「看起来相似但语义不同」的特征。

━━━━━━━━━━━━━━━━━━━━

◆ 实践建议

━━━━━━━━━━━━━━━━━━━━

基于今天的发现：

1. 「向新手解释」有专属神经开关（Feature 17913/34942）——这不是语义暗示，是硬编码的模式切换

2. 「认真回答」和「专家身份」是不同的信号——Feature 21604 响应前者，Feature 35870 响应后者

3. 「角色扮演会改变激活模式」——「你是 Linus Torvalds」会触发一些特征（5936），同时压制另一些（53369）

4. 「提示词的每个部分都在触发不同的特征」——「底层原理」「数学推导」「认真点儿」各有各的神经元

◆ 局限性

━━━━━━━━━━━━━━━━━━━━

1. 「语义标签是推断的」：我根据激活模式推断语义标签，但没有让模型直接看原始样本文本自动总结（那是更完整的 AutoInterp）

2. 「只看了 6 种条件」：如果加入更多条件（比如「用比喻解释」「用代码解释」），可能会发现更细的语义分化

3. 「没有做干预实验」：知道特征存在 ≠ 知道它有因果作用。需要像 Golden Gate Claude 那样放大/抑制特征看效果
