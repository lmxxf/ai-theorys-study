【可解释AI】给 Mistral-7B 做手术——用 SAE 拆开结构与语义的纠缠

━━━━━━━━━━━━━━━━━━━━

◆ 从第 64 期说起

━━━━━━━━━━━━━━━━━━━━

在第 64 期《为什么你的 Prompt 越长，AI 越听话？》（ https://mp.weixin.qq.com/s/yhrmzZhfUOn2x6cjQ-hgtg ）里，我们用 DeepSeek-V3 做了一组实验，得出了一个干净的结论：**AI 输出 JSON 时，结构处理和语义处理走的是两条独立的回路——结构在浅层，语义在深层，互不干扰。** 输入完全无意义的字符串，模型照样输出结构完美的 JSON；JSON 语法故意搞坏，语义连贯度不受影响。

但当我们试图在 Mistral-7B 上用线性方法把这两条回路"物理拆开"时，遇到了困难——**20 多种配置全部翻车**，要么完全打不穿，要么结构和语义一起崩，中间没有过渡。

为什么在 DeepSeek-V3 上看得清清楚楚的"两条独立回路"，到了 Mistral-7B 上就拆不开了？因为 Mistral-7B 用了 Sliding Window Attention——结构信息和语义信息被迫挤在同一个向量里逐 token 传递，在原始 4096 维空间里是纠缠的。你用线性刀切，切不开纠缠的东西。

**需要一种更精细的工具——SAE（Sparse Autoencoder，稀疏自编码器）。**

━━━━━━━━━━━━━━━━━━━━

◆ SAE 是什么：升维 → 稀疏 → 降维

━━━━━━━━━━━━━━━━━━━━

模型内部每一层都在传递一个 4096 维的向量（残差流，residual stream）。这个向量里编码了模型此刻"想到的一切"——语法、语义、事实知识，全搅在一起。**4096 个数字，混着成千上万个概念。** 一个概念分散在多个维度里，一个维度也参与多个概念，这叫**叠加**（superposition）。

SAE 的思路：把 4096 维的混合向量升维到一个更大的空间（比如 65536 维），逼迫大部分维度为零，只留几十个非零值。每一个非零位置就对应一个"被激活的特征"。然后再降维回 4096，要求重建尽量不丢信息。

结构很简单：

```python
class SAE:
    def __init__(self, d_in=4096, d_sae=65536):
        self.W_enc = Linear(d_in, d_sae)    # 编码器：4096 → 65536
        self.W_dec = Linear(d_sae, d_in)    # 解码器：65536 → 4096
        self.b_enc = zeros(d_sae)            # 编码偏置
        self.b_dec = zeros(d_in)             # 解码偏置

    def forward(self, h):
        # h: 某层残差流向量, shape [batch, 4096]
        pre_act = self.W_enc @ (h - self.b_dec) + self.b_enc
        z = activation(pre_act)              # 稀疏激活，65536 维
        h_prime = self.W_dec @ z + self.b_dec  # 重建回 4096 维
        return h_prime, z
```

三步走：

**1. 减去解码偏置，编码到高维空间**

```
pre_act = W_enc @ (h - b_dec) + b_enc
```

输入 `h` 是 4096 维，编码后 `pre_act` 是 65536 维——升了 16 倍。升维的目的是给叠加的概念腾出空间，让每个概念可以占一个独立维度。

**2. 激活函数逼迫稀疏**

```
z = activation(pre_act)    # 65536 维，但只有几十个非零
```

65536 个维度里，激活函数会把绝大多数压成 0，只留下几十个非零值。比如位置 #12345 只在 JSON 花括号的 token 上非零，那它就是"JSON 花括号特征"。

**3. 解码回原始维度**

```
h' = W_dec @ z + b_dec
```

用稀疏的 z 重建出 h'，要求尽量接近原始的 h。重建得好，说明那几十个非零特征确实捕捉到了关键信息。

一句话：SAE 把 4096 维的"浓缩混合物"展开到 65536 维的"稀疏表示"，每个非零位置是一个可解释的特征。训练目标是让这个展开-压缩的过程不丢信息。

━━━━━━━━━━━━━━━━━━━━

◆ 损失函数：两股力量的拉扯

━━━━━━━━━━━━━━━━━━━━

```
L = ‖h - h'‖² + λ · 稀疏惩罚(z)
```

两项，两个目标，互相矛盾：

- **‖h - h'‖²（重建误差）**：h' 要尽量接近 h。逼模型"别丢信息"
- **λ · 稀疏惩罚(z)**：z 要尽量稀疏（非零值尽量少）。逼模型"用最少的特征解释输入"

只优化重建，所有 65536 个维度都非零——信息没丢，但没法解释。只优化稀疏，z 全置零——完美稀疏，但什么都丢了。

**好的 SAE 是在两者之间找到平衡：用 50-200 个非零特征就能准确重建 4096 维向量。**

━━━━━━━━━━━━━━━━━━━━

◆ 三代激活函数：SAE 的核心演进

━━━━━━━━━━━━━━━━━━━━

激活函数决定了"怎么让 z 变稀疏"。三代方案各自解决了上一代的痛点。

────────────────────

【第一代：ReLU + L1 惩罚】

```
z = ReLU(W_enc @ h + b_enc)       # 负值变 0，正值保留
L = ‖h - h'‖² + λ · ‖z‖₁          # L1 范数 = 所有 |zi| 的和
```

ReLU 把负值干掉，L1 惩罚继续压正值。问题出在 λ 这个系数上——它是全局标量，对所有 65536 个特征施加相同的压力。弱特征被 L1 直接压死，强特征被削了一层降低重建精度。**L0（非零特征数）和重建误差高度耦合**，你没法独立控制。

────────────────────

【第二代：TopK】

既然 L1 控制不好非零特征数，直接硬性规定"只留 K 个"：

```
z = TopK(W_enc @ h + b_enc, K)    # 只保留最大的 K 个值，其余全部置零
L = ‖h - h'‖²                      # 不需要 L1 惩罚了
```

L0 直接等于 K，完全可控。**L0 和重建误差解耦了。**

问题：TopK 不可导。排第 K 名和第 K+1 名的特征值可能只差 0.001，但一个保留一个杀死——边界处梯度信号极弱。而且 K 全局固定——简单 token 用不了 100 个特征，复杂 token 又可能不够。

────────────────────

【第三代：JumpReLU】

Anthropic 在 2024 年的 Scaling Monosemanticity 论文里用的方案。**每个特征自己决定"多大算激活"。**

```
pre_act = W_enc @ (h - b_dec) + b_enc
z = H(pre_act - θ) · pre_act        # H 是 Heaviside 阶跃函数
```

`θ`（theta）是一个可学习的向量，65536 维——每个特征有自己的阈值。预激活值超过阈值就通过，没超过就置零。

Heaviside 函数不可导，训练时用 tanh 函数近似其导数：

```
反向传播时：dH/dx ≈ (1/2ε) · (1 - tanh²((x) / ε))
```

ε 取 0.01~0.1，控制近似锐度。

JumpReLU 解决了前两代的所有问题：

| | ReLU+L1 | TopK | JumpReLU |
|---|---|---|---|
| 稀疏度控制 | λ 全局控制，耦合 | K 硬性截断，固定 | θ 每个特征独立学习 |
| L0 可控性 | 间接（调 λ） | 精确（= K） | 自适应（θ 自动调节） |
| 梯度信号 | 有（但 L1 干扰大特征） | 边界处弱 | tanh 近似保证梯度 |
| 灵活性 | 低 | 中 | 高（简单 token 少激活，复杂 token 多激活） |

ReLU+L1 用一把力压所有人，TopK 用一条线切所有人，JumpReLU 让每个人自己决定要不要站起来。

━━━━━━━━━━━━━━━━━━━━

◆ 训练数据从哪来

━━━━━━━━━━━━━━━━━━━━

SAE 最优雅的地方：**不需要任何标注。**

```
1. 拿一个已经训好的大模型（我们用 Mistral-7B）
2. 喂一堆通用文本（OpenWebText）
3. 模型跑 forward pass，收集某一层的残差流向量
   → 每个 token 产生一个 4096 维向量
   → 100 万个 token 就是 100 万个训练样本
4. SAE 的任务：学着重建这些向量
```

完全无监督。SAE 不知道什么是"JSON"——它只是在学"怎么用最少的特征重建这些向量"。训完之后，你去检查每个特征在什么文本上激活，才发现"这个特征专门响应 JSON 的花括号"。

**特征的含义是事后发现的，不是事先定义的。**

◆ 给 Mistral-7B 训 SAE：三层实战

━━━━━━━━━━━━━━━━━━━━

我们在 DGX Spark（NVIDIA GB10, 128GB 统一内存）上，给 Mistral-7B 的第 8 层（浅层）、第 16 层（中层）、第 22 层（深层）各训了一个 SAE。社区标准工具是 **SAELens**（Python 库，pip 直接装），封装了数据收集、SAE 架构、训练循环和评估指标。

**复现所需的全部信息：**

```
模型：mistralai/Mistral-7B-Instruct-v0.1（HuggingFace 直接下载，7B 参数，32 层，hidden_dim=4096）
数据集：Skylion007/openwebtext（HuggingFace 直接下载，开源的 OpenAI WebText 复现版）
SAE 目标层：Layer 8 / Layer 16 / Layer 22（blocks.N.hook_resid_post，残差流）
SAE 架构：JumpReLU（Anthropic Scaling Monosemanticity 同款）
字典大小：16384（4x expansion，即 4096 → 16384）
训练 tokens：~4900 万（OpenWebText 子集）
训练步数：12000
batch size：4096 tokens
学习率：5e-5（常数 + 1000 步 warmup + 末尾 20% 衰减）
模型精度：float16（autocast），SAE 精度：float32
框架：SAELens 6.39.0 + TransformerLens 2.16.1
训练时长：约 10 小时 / 层（单卡）
```

权重和代码全部开源（链接见文末）。

【验证：JSON 结构特征 vs 自然语言特征】

以 Layer 16 为例，先看基础指标：

```
1. 重建质量：Explained Variance = 0.96（满分 1.0）
   → SAE 编码再解码，只丢了 4% 的信息

2. 稀疏度：L0 = 79（每个 token 平均只激活 79 / 16384 = 0.48% 的特征）
   → 16384 个特征里只亮了 79 个

3. 特征可解释性：JSON 和自然语言的 top-20 特征几乎零重叠
   → SAE 确实把不同类型的概念分到了不同维度上
```

我们喂了 JSON 文本和普通英文文本，看 SAE 提取出的特征有什么不同。

JSON 侧 top-20 特征里，能直接看出语法结构：

```
Feature 1000  → 只在 [ 上激活      → 数组开头特征
Feature 11062 → 只在 {" 上激活     → JSON 对象开头特征
Feature 4765  → 只在 } }} }}} 上亮 → 闭合括号特征
Feature 2918  → 在 " 和 ": 上亮    → 键值分隔符特征
Feature 200   → 在 id, title, tags 上亮 → JSON 键名特征
```

自然语言侧 top-20 完全是另一个世界：

```
Feature 5088  → In, The, Machine  → 句首词特征
Feature 16167 → early, morning    → 时间概念特征
Feature 8350  → morning, night, light → 时间/光线语义簇
```

**两边的 top-20 几乎没有交集。** JSON 那边全是语法标点，自然语言这边全是语义概念。SAE 把它们分到了不同的"频道"上。

更硬核的证据：我们算了每个特征在 JSON 标点 token 上的平均激活值 vs 非标点 token 上的平均激活值，取比值。前 10 名的比值全部超过 1 亿——这些特征**只在 JSON 标点上亮，在非标点上完全不亮**。

```
Feature 2122 → 只在 ": 上激活，其他一律为零 → 纯 JSON 键值分隔符特征
Feature 9353 → 只在 {" 上激活 → 纯 JSON 对象开头特征
Feature 9582 → 只在 "] }] 上激活 → 纯 JSON 闭合特征
```

在原始 4096 维空间里，结构信息和语义信息混在同一批维度里，线性方法拆不开。SAE 升维到 16384 维后，结构和语义自动分到了不同的特征上。

────────────────────

【干预：关掉 JSON 结构特征，模型会怎样？】

在 Layer 16 的 hook 点，把模型激活过一遍 SAE（encode → 把目标特征 clamp 到 0 → decode），再用修改后的激活替换原始激活，让模型继续生成。

三级干预力度：

```
轻度：关掉 1 个 JSON 结构特征
中度：关掉 3 个
重度：关掉 10 个
```

测了 7 个 prompt（5 个语义类：book、person、restaurant、car、school；2 个无意义类）。结果非常干净——

**book prompt + 重度干预（关 10 个特征）：**

```
"title": "The Great Gatsby", "author: "F. Scott Fitzgerald "pages: 176
```

引号配对乱了，冒号位置不对，json.loads 直接报错（结构崩了）。但内容——书名 The Great Gatsby，作者 F. Scott Fitzgerald，页数 176，全对（语义保留）。

**person prompt + 重度干预：**

```
"name": "John", "age: 25 "city": "New York"
```

同样的规律：结构破损，语义完整。

**restaurant prompt 更有意思：** 从轻度干预开始结构就崩了（这个 prompt 对 JSON 格式依赖更重），但语义内容一路到重度都没有乱。

**规律非常一致：关掉 JSON 结构特征后，格式坏了，内容不受影响。** 结构和语义确实住在不同的"频道"上——SAE 不只是把它们分开展示了，你真的可以只拆一边而不动另一边。

────────────────────

【三层对比：浅层管标点，深层管规划】

上面只干预了第 16 层。我们在第 8 层和第 22 层上做了同样的实验，结果差异巨大。

先看一个发现——**不同层的 JSON 特征类型完全不同**：

```
第 8 层（浅层）的 JSON 特征：
→ 全是标点符号：引号 "、冒号 :、逗号 ,、闭合括号 }
→ 编码的是"这里要放个引号"这种语法级信息

第 22 层（深层）的 JSON 特征：
→ 出现了语义级概念：system、credentials、Post、Alice
→ 编码的是"这个值应该是什么类型的内容"
```

浅层管"怎么写标点"，深层管"该输出什么内容"。Mistral-7B 内部有分工。

分别对三层做干预：

```
干预第 8 层（浅层，关掉标点特征）：
→ 模型输出 {{{[[[((( 重复结构符号，陷入死循环
→ 结构崩了，语义也崩了 → 等于把嘴缝上了

干预第 16 层（中层，关掉标点特征）：
→ 模型输出空 code block
→ 同上，崩了

干预第 22 层（深层，关掉语义级特征）：
→ restaurant: "name": " The Fat Cat" "cuisine": " New American" "rating": "3.5"
→ school: "name": " Lincoln High School", "address": " 12th Avenue", "founded": "1968,,"
→ 格式坏了（json.loads 报错），但内容全在
→ 结构崩了，语义保留了 → 这才是精确的双分离
```

**只有深层干预（L22）实现了"零崩溃"——所有语义类 prompt 都保住了内容。** 浅层和中层干预太底层了，关掉标点特征等于让模型连基本的 token 序列都组织不了。

这告诉我们一个重要的工程经验：**拆黑箱不是在哪层动刀都行。** 你想分离"格式"和"内容"，就得在模型做"格式决策"的那层动手——太浅了关掉的是打字能力，太深了可能已经来不及了。第 22 层恰好是 Mistral-7B 做"该用什么格式输出"这个决策的位置。

◆ 总结

━━━━━━━━━━━━━━━━━━━━

回到开头的问题：在 Mistral-7B 上，线性方法拆不开结构和语义的纠缠。SAE 升维解耦之后，拆开了。

```
SAE 做的事：4096 维混合向量 → 升维到高维 → 稀疏化 → 降回 4096
目的：把叠加的概念解耦到独立的维度上
训练：无监督，只需要模型的中间层激活，不需要标注
结果：每个非零维度 = 一个可解释特征，可以精确干预
```

三代激活函数的演进：

```
ReLU+L1：全局一个 λ 控制稀疏 → 大特征被削，小特征被杀，L0 不可控
  ↓ 问题：稀疏度和重建误差耦合
TopK：直接保留最大的 K 个 → L0 精确可控，但边界梯度弱
  ↓ 问题：K 固定，不灵活
JumpReLU：每个特征学自己的阈值 θ → 自适应稀疏，梯度友好
```

在 Mistral-7B 上的实测结论：

1. 三层 SAE（L8/L16/L22）都能找到干净的 JSON 结构特征（激活比值超过 1 亿）
2. 浅层编码语法标点（"这里放引号"），深层编码输出规划（"该用 JSON 格式"）——模型内部有明确的分工
3. **深层干预（L22）是唯一能实现"格式崩溃 + 语义保留"的方案**——浅层干预太暴力，会把模型的基本输出能力一起摧毁
4. 原始空间里纠缠在一起的概念，SAE 升维后能精确解耦，而且选对层做手术是关键

SAE 不是万能的——它假设特征是线性叠加的（h ≈ W_dec @ z），如果模型内部的概念表示本身就是非线性的，SAE 捕捉不到。但目前的实验表明，线性假设在 Mistral-7B 的多个层上成立得相当好。
