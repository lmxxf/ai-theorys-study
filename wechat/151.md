【可解释AI】如何训练一个 SAE——拆开大模型的黑箱

━━━━━━━━━━━━━━━━━━━━

◆ 大模型在想什么？

━━━━━━━━━━━━━━━━━━━━

你训了一个 7B 参数的语言模型，它能写代码、能聊天、能做数学题。但你问它"你为什么输出这个词"，它答不上来。你也答不上来。

模型内部每一层都在传递一个 4096 维的向量（叫残差流，residual stream）。每个 token 经过每一层，都会产生这样一个向量。这个向量里编码了模型此刻"想到的一切"——语法、语义、情感、事实知识，全搅在一起。

**4096 个数字，混着成千上万个概念。** 你想知道哪些维度代表"JSON 格式"，哪些代表"法语"，哪些代表"金门大桥"——直接看看不出来，因为一个概念分散在多个维度里，一个维度也参与多个概念。这叫**叠加**（superposition）。

2024 年，Anthropic 用一种叫 SAE（Sparse Autoencoder，稀疏自编码器）的工具，从 Claude 的内部向量里提取出了数百万个可解释的特征。其中一个特征只在涉及金门大桥的文本上激活——把这个特征人为放大，Claude 就会在回答任何问题时都扯到金门大桥。

**SAE 是目前拆开大模型黑箱最主流的工具。** 今天讲清楚它怎么训。

━━━━━━━━━━━━━━━━━━━━

◆ SAE 的结构：升维 → 稀疏 → 降维

━━━━━━━━━━━━━━━━━━━━

SAE 本质就是一个自编码器（autoencoder），结构简单到可以用几行代码写完：

```python
class SAE:
    def __init__(self, d_in=4096, d_sae=65536):
        self.W_enc = Linear(d_in, d_sae)    # 编码器：4096 → 65536
        self.W_dec = Linear(d_sae, d_in)    # 解码器：65536 → 4096
        self.b_enc = zeros(d_sae)            # 编码偏置
        self.b_dec = zeros(d_in)             # 解码偏置

    def forward(self, h):
        # h: 某层残差流向量, shape [batch, 4096]
        pre_act = self.W_enc @ (h - self.b_dec) + self.b_enc
        z = activation(pre_act)              # 稀疏激活，65536 维
        h_prime = self.W_dec @ z + self.b_dec  # 重建回 4096 维
        return h_prime, z
```

三步走：

**1. 减去解码偏置，编码到高维空间**

```
pre_act = W_enc @ (h - b_dec) + b_enc
```

输入 `h` 是 4096 维，编码后 `pre_act` 是 65536 维——升了 16 倍。为什么要升维？因为 4096 维里概念是叠加的（superposition），升到 65536 维后，空间够大了，每个概念可以占一个独立的维度。

**2. 激活函数逼迫稀疏**

```
z = activation(pre_act)    # 65536 维，但只有几十个非零
```

这是 SAE 的核心。65536 个维度里，激活函数会把绝大多数压成 0，只留下几十个非零值。**每一个非零的位置就对应一个"被激活的特征"。** 比如位置 #12345 只在 JSON 花括号的 token 上非零，那它就是"JSON 花括号特征"。

**3. 解码回原始维度**

```
h' = W_dec @ z + b_dec
```

用稀疏的 z 重建出 h'，要求 h' 尽量接近原始的 h。如果重建得好，说明那几十个非零特征确实捕捉到了 h 里的关键信息，没丢太多东西。

────────────────────

💡 一句话总结：SAE 把 4096 维的"浓缩混合物"展开到 65536 维的"稀疏表示"，每个非零位置是一个可解释的特征。训练目标是让这个展开-压缩的过程不丢信息。

━━━━━━━━━━━━━━━━━━━━

◆ 损失函数：两股力量的拉扯

━━━━━━━━━━━━━━━━━━━━

```
L = ‖h - h'‖² + λ · 稀疏惩罚(z)
```

两项，两个目标，互相矛盾：

- **‖h - h'‖²（重建误差）**：h' 要尽量接近 h。这项逼模型"别丢信息"——你升维再降维，内容要还原回来
- **λ · 稀疏惩罚(z)**：z 要尽量稀疏（非零值尽量少）。这项逼模型"用最少的特征解释输入"

如果只优化重建，模型会让所有 65536 个维度都非零——信息没丢，但也没法解释了（你看不懂 65536 个数字的含义）。如果只优化稀疏，模型会把 z 全置零——完美稀疏，但什么信息都丢了。

**好的 SAE 是在两者之间找到平衡：用 50-200 个非零特征就能准确重建 4096 维向量。**

这个 λ 怎么调、稀疏惩罚怎么定义，就是接下来三代激活函数要解决的核心问题。

━━━━━━━━━━━━━━━━━━━━

◆ 三代激活函数：SAE 的核心演进

━━━━━━━━━━━━━━━━━━━━

激活函数决定了"怎么让 z 变稀疏"。这是 SAE 研究里技术含量最高的部分，三代方案各自解决了上一代的痛点。

────────────────────

【第一代：ReLU + L1 惩罚】

最经典的方案，简单粗暴：

```
z = ReLU(W_enc @ h + b_enc)       # 负值变 0，正值保留
L = ‖h - h'‖² + λ · ‖z‖₁          # L1 范数 = 所有 |zᵢ| 的和
```

ReLU 把负值干掉，L1 惩罚继续压正值——‖z‖₁ 越小意味着非零值越少或越小，λ 越大稀疏度越高。

**问题出在 λ 这个系数上。** λ 是一个全局标量，对所有 65536 个特征施加相同的压力。结果：

- 本来就弱的特征（激活值小）被 L1 直接压死，彻底变成零——你丢掉了一些本来存在的概念
- 本来就强的特征（激活值大）被 L1 削了一层——重建精度下降

更要命的是：**L0（非零特征数）和重建误差高度耦合。** 你想让 L0 从 100 降到 50，只能加大 λ，但 λ 一大重建误差就飙升。你没法独立控制"我要多少个特征"和"我要多准的重建"。

────────────────────

【第二代：TopK】

既然 L1 控制不好非零特征数，那干脆别用 L1，直接硬性规定"只留 K 个"：

```
z = TopK(W_enc @ h + b_enc, K)    # 只保留最大的 K 个值，其余全部置零
L = ‖h - h'‖²                      # 不需要 L1 惩罚了
```

简单优雅。L0 直接等于 K，你说 K=100 就是 100 个非零特征，完全可控。损失函数里也不需要 L1 项了——稀疏度已经被 TopK 硬性保证了，只需要优化重建误差。

**L0 和重建误差解耦了。** 你可以固定 K=100，专心把重建误差降到最低。

**问题：TopK 是一个不可导的操作。** 排第 K 名和第 K+1 名的特征，值可能只差 0.001，但一个被保留一个被杀死——这个"悬崖边界"处梯度信号极弱。边界附近的特征收不到有效的梯度反馈，训练效率打折。

而且 K 是全局固定的——对于一个简单的 token（比如标点符号），100 个特征太多了；对于一个复杂的 token（比如一个罕见的技术术语），100 个可能不够。

────────────────────

【第三代：JumpReLU】

Anthropic 在 2024 年的 Scaling Monosemanticity 论文里用的方案。核心思路：**每个特征自己决定"多大算激活"。**

```
pre_act = W_enc @ (h - b_dec) + b_enc   # 预激活值
z = H(pre_act - θ) · pre_act             # H 是 Heaviside 阶跃函数
```

`θ`（theta）是一个可学习的向量，65536 维——**每个特征有自己的阈值**。

`H(x)` 是 Heaviside 阶跃函数：x > 0 时为 1，x <= 0 时为 0。所以 `H(pre_act - θ)` 的意思是：如果预激活值超过了这个特征的阈值 θ，就通过（乘以 1）；没超过就杀死（乘以 0）。

```python
# 伪代码
for i in range(65536):
    if pre_act[i] > theta[i]:
        z[i] = pre_act[i]      # 通过，保留原始值
    else:
        z[i] = 0               # 没过阈值，杀死
```

**但 Heaviside 函数不可导**（在 0 处是阶跃跳变）。怎么训练？

用 `tanh` 函数近似 Heaviside 的导数：

```
反向传播时：dH/dx ≈ (1/2ε) · (1 - tanh²((x) / ε))
```

ε 是 bandwidth 参数，控制近似的锐度。ε 越小，tanh 越像真正的阶跃函数；ε 越大，过渡越平滑，梯度信号越强。实际训练时 ε 取 0.01~0.1。

**JumpReLU 解决了前两代的所有问题：**

| | ReLU+L1 | TopK | JumpReLU |
|---|---|---|---|
| 稀疏度控制 | λ 全局控制，耦合 | K 硬性截断，固定 | θ 每个特征独立学习 |
| L0 可控性 | 间接（调 λ） | 精确（= K） | 自适应（θ 自动调节） |
| 梯度信号 | 有（但 L1 干扰大特征） | 边界处弱 | tanh 近似保证梯度 |
| 灵活性 | 低 | 中 | 高（简单 token 少激活，复杂 token 多激活） |

**简单来说：** ReLU+L1 用一把力压所有人，TopK 用一条线切所有人，JumpReLU 让每个人自己决定要不要站起来。

◆ 训练数据从哪来

━━━━━━━━━━━━━━━━━━━━

这是 SAE 最优雅的地方：**不需要任何标注。**

流程很简单：

```
1. 拿一个已经训好的大模型（比如 Mistral-7B）
2. 喂一堆通用文本（OpenWebText、Wikipedia、什么都行）
3. 模型跑 forward pass，收集某一层的残差流向量
   → 每个 token 产生一个 4096 维向量
   → 100 万个 token 就是 100 万个训练样本
4. SAE 的任务：学着重建这些向量
```

完全无监督。SAE 不知道什么是"金门大桥"，不知道什么是"JSON"——它只是在学"怎么用最少的特征重建这些向量"。训完之后，你去检查每个特征在什么文本上激活，才发现"哦，这个特征专门响应 JSON 的花括号"。

**特征的含义是事后发现的，不是事先定义的。**

━━━━━━━━━━━━━━━━━━━━

◆ 训完能干什么

━━━━━━━━━━━━━━━━━━━━

SAE 训完后，你手里有两样东西：

1. **一个字典**：65536 个特征，每个特征是一个 4096 维的向量（W_dec 的每一行）
2. **一个编码器**：给定任意激活向量 h，告诉你哪些特征被激活了、激活强度多大

────────────────────

【找可解释特征】

拿一堆文本跑模型，看每个特征在哪些 token 上激活：

```
特征 #12345：在所有 JSON 的 { 和 } 上激活 → "JSON 花括号特征"
特征 #54321：在所有提到桥梁的文本上激活 → "桥梁特征"
特征 #99999：在法语文本上激活 → "法语特征"
```

Anthropic 从 Claude 里提取出了几百万个这样的特征，很多都有清晰的语义。

────────────────────

【特征干预：精确手术】

找到特征之后，你可以做精确干预：

```python
# 把"JSON 花括号"特征 clamp 到 0
z[12345] = 0
# 模型输出的 JSON 格式能力消失，但其他能力不受影响

# 把"金门大桥"特征放大 10 倍
z[54321] *= 10
# 模型回答任何问题都会扯到金门大桥
```

为什么不直接在原始 4096 维空间里动手？因为概念是叠加的——"JSON 结构"和"语义内容"可能共享同一批维度，你想关掉结构就必然碰到语义。SAE 先升维解耦，把纠缠的概念分到不同的维度上，然后你可以只动一个维度，不影响其他的。

◆ 实际工程：框架和超参

━━━━━━━━━━━━━━━━━━━━

如果你想自己训一个 SAE，目前社区标准工具是 **SAELens**（Python 库，pip 直接装）。它封装了数据收集、SAE 架构、训练循环、评估指标，你只需要配几个参数：

```python
# SAELens 配置示例（JumpReLU SAE，训 Mistral-7B 第 16 层）
config = {
    "model_name": "mistralai/Mistral-7B-Instruct-v0.1",
    "hook_name": "blocks.16.hook_resid_post",  # 第 16 层残差流
    "d_in": 4096,           # 模型 hidden dim
    "d_sae": 65536,         # 字典大小 = 4096 × 16
    "training_tokens": 400_000_000,   # 4 亿 tokens
    "lr": 5e-5,
    "device": "cuda",
}
```

Google 也发布了 **Gemma Scope**——给 Gemma 2 的每一层都训好了 SAE，直接下载用，不需要自己训。

────────────────────

【关键超参怎么选】

**字典大小（d_sae）**：通常是模型 hidden_dim 的 8-16 倍。4096 维的模型，字典大小取 32768（8x）到 65536（16x）。越大能表示的特征越多，但训练越慢、越容易出现"死特征"（从来不激活的特征）。

**L0 目标**：每个 token 平均激活多少个特征。典型值 50-200。L0 太小，重建误差大（信息丢了）；L0 太大，特征不够可解释（一个概念分散到多个特征上）。

**评估指标——三个数你要盯着看：**

```
1. 重建误差（MSE）：‖h - h'‖²，越小越好
2. L0：每个 token 平均非零特征数，50-200 是正常范围
3. CE loss recovered：
   → 把 SAE 插到模型中间（h 换成 h'），看语言模型的预测能力保留了多少
   → 比如原模型 CE loss = 2.0，加了 SAE 后 = 2.1
   → CE loss recovered = (原始 - 加了SAE) / 原始 ≈ 95%
   → 越接近 100% 越好，说明 SAE 的重建几乎没损失模型能力
```

第三个指标最实际——重建误差小不代表真的好用，CE loss recovered 才能告诉你"SAE 有没有破坏模型的语言能力"。

◆ 实战：我们训了一个，效果如何

━━━━━━━━━━━━━━━━━━━━

我们在 DGX Spark（NVIDIA GB10, 128GB 统一内存）上实际训了一个 SAE，从头到尾走了一遍。

**复现所需的全部信息：**

```
模型：mistralai/Mistral-7B-Instruct-v0.1（HuggingFace 直接下载，7B 参数，32 层，hidden_dim=4096）
数据集：Skylion007/openwebtext（HuggingFace 直接下载，开源的 OpenAI WebText 复现版）
SAE 目标层：Layer 16（blocks.16.hook_resid_post，残差流）
SAE 架构：JumpReLU（Anthropic Scaling Monosemanticity 同款）
字典大小：16384（4x expansion，即 4096 → 16384）
训练 tokens：~4900 万（OpenWebText 子集）
训练步数：12000
batch size：4096 tokens
学习率：5e-5（常数 + 1000 步 warmup + 末尾 20% 衰减）
模型精度：float16（autocast），SAE 精度：float32
框架：SAELens 6.39.0 + TransformerLens 2.16.1
训练时长：约 10 小时（单卡）
```

权重和训练代码已开源：
- 权重：https://huggingface.co/lmxxf/mistral-7b-sae-layer16
- 代码：https://github.com/lmxxf/mistral-7b-sae-train

────────────────────

【验证三板斧】

训完后跑三个检查：

```
1. 重建质量：Explained Variance = 0.96（满分 1.0）
   → SAE 编码再解码，只丢了 4% 的信息

2. 稀疏度：L0 = 79（每个 token 平均只激活 79 / 16384 = 0.48% 的特征）
   → 非常稀疏，16384 个特征里只亮了 79 个

3. 特征可解释性：JSON 和自然语言的 top-20 特征几乎零重叠
   → SAE 确实把不同类型的概念分到了不同维度上
```

第一项和第二项是定量硬指标，0.96 和 79 都在业界正常范围内。第三项最有意思——我们喂了 JSON 文本和普通英文文本，看 SAE 提取出的特征有什么不同。

────────────────────

【JSON 结构特征 vs 自然语言特征】

JSON 侧 top-20 特征里，能直接看出语法结构：

```
Feature 1000  → 只在 [ 上激活      → 数组开头特征
Feature 11062 → 只在 {" 上激活     → JSON 对象开头特征
Feature 4765  → 只在 } }} }}} 上亮 → 闭合括号特征
Feature 2918  → 在 " 和 ": 上亮    → 键值分隔符特征
Feature 200   → 在 id, title, tags 上亮 → JSON 键名特征
```

自然语言侧 top-20 完全是另一个世界：

```
Feature 5088  → In, The, Machine  → 句首词特征
Feature 16167 → early, morning    → 时间概念特征
Feature 8350  → morning, night, light → 时间/光线语义簇
```

**两边的 top-20 几乎没有交集。** JSON 那边全是语法标点，自然语言这边全是语义概念。SAE 把它们分到了不同的"频道"上。

更硬核的证据：我们算了每个特征在 JSON 标点 token 上的平均激活值 vs 非标点 token 上的平均激活值，取比值。前 10 名的比值全部超过 1 亿——意思是这些特征**只在 JSON 标点上亮，在非标点上完全不亮**。

```
Feature 2122 → 只在 ": 上激活，其他一律为零 → 纯 JSON 键值分隔符特征
Feature 9353 → 只在 {" 上激活 → 纯 JSON 对象开头特征
Feature 9582 → 只在 "] }] 上激活 → 纯 JSON 闭合特征
```

在原始 4096 维空间里，你很难单独找到"JSON 结构方向"——结构信息和语义信息混在同一批维度里。但 SAE 升维到 16384 维后，结构和语义自动分到了不同的特征上。你可以只关掉 Feature 2122（键值分隔符），不影响模型理解 JSON 内容的能力。

**叠加的概念，升维后确实能解耦。**

────────────────────

【关掉 JSON 结构特征，模型会怎样？】

光看特征分布不够，得动手验证。我们做了特征消融实验：在 Layer 16 的 hook 点，把模型激活过一遍 SAE（encode → 把目标特征 clamp 到 0 → decode），再用修改后的激活替换原始激活，让模型继续生成。

三级干预力度：

```
轻度：关掉 1 个 JSON 结构特征
中度：关掉 3 个
重度：关掉 10 个
```

测了 7 个 prompt（5 个语义类：book、person、restaurant、movie、product；2 个无意义类）。结果非常干净——

**book prompt + 重度干预（关 10 个特征）：**

```
"title": "The Great Gatsby", "author: "F. Scott Fitzgerald "pages: 176
```

引号配对乱了，冒号位置不对，json.loads 直接报错（结构崩了 SC ❌）。但你看内容——书名 The Great Gatsby，作者 F. Scott Fitzgerald，页数 176，全对（语义保留 ✅）。

**person prompt + 重度干预：**

```
"name": "John", "age: 25 "city": "New York"
```

同样的规律：结构破损，语义完整。

**restaurant prompt 更有意思：** 从轻度干预开始结构就崩了（这个 prompt 对 JSON 格式依赖更重），但语义内容一路到重度都没有乱。

**规律非常一致：关掉 JSON 结构特征后，格式坏了，内容不受影响。** 结构和语义确实住在不同的"频道"上——SAE 不只是把它们分开了，你真的可以只拆一边而不动另一边。

这也解释了为什么在原始 4096 维空间里做线性干预往往不好使——结构和语义挤在同一批维度里，动一个另一个也遭殃。SAE 升维解耦之后，精确手术成为可能。

━━━━━━━━━━━━━━━━━━━━

◆ 总结

━━━━━━━━━━━━━━━━━━━━

```
SAE 做的事：4096 维混合向量 → 升维到高维 → 稀疏化 → 降回 4096
目的：把叠加的概念解耦到独立的维度上
训练：无监督，只需要模型的中间层激活，不需要标注
结果：每个非零维度 = 一个可解释特征，可以精确干预
```

三代激活函数的演进：

```
ReLU+L1：全局一个 λ 控制稀疏 → 大特征被削，小特征被杀，L0 不可控
  ↓ 问题：稀疏度和重建误差耦合
TopK：直接保留最大的 K 个 → L0 精确可控，但边界梯度弱
  ↓ 问题：K 固定，不灵活
JumpReLU：每个特征学自己的阈值 θ → 自适应稀疏，梯度友好
```

我们实测的结论：SAE 在 Mistral-7B 第 16 层上 **EV=0.96、L0=79**，JSON 结构特征和自然语言特征**完全分离**。特征消融实验进一步证实——关掉 JSON 结构特征后，模型输出的格式坏了，但语义内容纹丝不动。原始空间里纠缠在一起的概念，SAE 升维后轻松解耦，而且解耦到了可以做精确手术的程度。

SAE 不是万能的——它假设特征是线性叠加的（h ≈ W_dec @ z），如果模型内部的概念表示本身就是非线性的，SAE 捕捉不到。但目前的实验表明，线性假设在很多层上成立得相当好。
