【Doc-to-LoRA 实验】AI 需要学会遗忘——我们用红楼梦亲手验证了

━━━━━━━━━━━━━━━━━━━━

◆ 先说背景：AI 现在怎么"记住"文档

━━━━━━━━━━━━━━━━━━━━

你问 Claude 或 GPT 一个问题，如果答案在某份文档里，你得把那份文档贴进对话框。下次再问，再贴一遍。模型每次都要重新"读"一遍那份文档，才能回答。它没有长期记忆，每次都是第一次见。

2026 年 2 月，Sakana AI 发了一篇论文 **Doc-to-LoRA**（arxiv: 2602.15902），提了一条不同的路：训练一个"超网络"（hypernetwork），输入文档，输出一组 LoRA 权重。把这组权重挂载到基础模型上，模型就"记住"了这份文档——以后查询不用带文档，知识已经在权重里了。

人话：超网络是一个专门"生成神经网络参数"的神经网络。你喂它一篇文档，它吐出一小块权重补丁（LoRA），贴到模型上，模型就像"读过这篇文档"一样。

这就是 AI 版的睡眠记忆巩固——白天读文档（context），超网络处理一遍（睡眠），知识搬进权重（长期记忆）。详细的分析在《Doc-to-LoRA 介绍》（https://mp.weixin.qq.com/s/zDhKbwklzuayMXUAqVThBQ）。

────────────────────

**问题在哪？**

文档长了怎么办？超网络一次只能处理 512 个 token（大约 400 个英文单词），文档超过这个长度就得切块。Doc-to-LoRA 的做法是：切成 N 块，每块独立生成一个 rank-8 的 LoRA，然后把 N 个 LoRA 沿 rank 维度拼接——1 块是 rank-8，10 块是 rank-80，100 块是 rank-800。

《Doc-to-LoRA 介绍》结尾我们留了一个疑问：

> 分块拼接的 rank 线性增长，是正确的方向吗？文档切 N 块，rank 就变成 8N。参数量线性增长，推理成本线性增长。文档越长，LoRA 越大。

我们还提了一个没人做过的消融实验：强制 N=1，只用 rank-8 装整篇文档——装不下的丢掉，留下的全是结构。和 N=K 的版本对比准确率，差多少就说明文档里有多少真正的冗余。

这个实验，我们自己跑了。

◆ 实验设计：红楼梦 vs 超网络

━━━━━━━━━━━━━━━━━━━━

我们需要一份足够长、细节足够密、不在任何训练集里的文档。

选了 **H. Bencraft Joly 1892 年翻译的红楼梦英译本**（Hung Lou Meng, or The Dream of the Red Chamber）前 8 章。这个版本足够冷门——1892 年的维多利亚时代英语，人名全是威妥玛拼音（Chia Pao-yue、Lin Tai-yue），不是现代翻译。Gemma-2-2b-it 的预训练数据里大概率没有这个版本的逐字内容。

────────────────────

**测试配置**

| 项目 | 值 |
|------|------|
| 文档 | 红楼梦英译本前 8 章，~85K tokens |
| 基础模型 | Gemma-2-2b-it |
| 超网络 | SakanaAI/doc-to-lora gemma_2b_d2l checkpoint-20000 |
| 分块参数 | max_ctx_chunk_len = 512 tokens/块 |
| 总块数 | 176 块 |
| 题目 | 24 道事实性问答，每章 3 题 |
| 硬件 | NVIDIA DGX Spark (GB10, 128GB 共享内存) |

────────────────────

**为什么不能直接"整本书 → 一个 LoRA"？**

你可能会问：既然分块拼接有问题，为什么不把整本书一次性喂给超网络，让它自己决定什么重要什么不重要，压成一个 rank-8 的 LoRA？

做不到。三个原因：

1. 超网络的编码器就是 Gemma-2B 本身，context window 最大 8192 tokens。前 8 章有 85K tokens，根本塞不进去。
2. 超网络训练时只见过 ≤512 tokens 的输入（`max_ctx_chunk_len=512`）。喂更长的文本完全超出训练分布。
3. 输出 rank 写死是 8——不管你喂多长的文本，出来都是 rank-8 的 LoRA，没有"根据内容重要性动态分配 rank"的能力。

所以 Doc-to-LoRA 目前的架构，**天然不支持"把一整本书压成一个紧凑的知识胶囊"这件事**。它只会切块、逐块压缩、拼接。拼接就意味着 rank 随块数线性增长，没有上限。

这就是我们要测的：这种"无上限堆 rank"的策略，到底能撑多远？

────────────────────

**实验逻辑**

超网络每处理一个 512-token 的块，生成一个 rank-8 的 LoRA。多块的 LoRA 沿 rank 维度拼接——1 块是 rank-16（8×(1+1)，包含一个 bias 项），5 块是 rank-48，176 块全文是 rank-1416。

我们要回答的问题很简单：**块越多，准确率越高吗？**

直觉上应该是。块越多，覆盖的文本越多，模型"记住"的内容越多，回答应该越准。对吧？

━━━━━━━━━━━━━━━━━━━━

◆ 结果：直觉错了

━━━━━━━━━━━━━━━━━━━━

| 条件 | 块数 | Rank | 文本覆盖 | 对应内容 | 准确率 |
|------|------|------|----------|----------|--------|
| Baseline | 0 | 0 | 0% | 无 | 25.0% |
| N=1 | 1 | 16 | 0.6% | 版权声明 | 16.7% |
| N=5 | 5 | 48 | 2.8% | 第 1 回开头 | 8.3% |
| N=20 | 20 | 168 | 11.4% | 约到第 1 回 | 4.2% |
| N=50 | 50 | 408 | 28.4% | 约到第 2-3 回 | 0.0% |
| N=100 | 100 | 808 | 56.9% | 约到第 4-5 回 | 0.0% |
| N=ALL(176) | 176 | 1416 | 100.0% | 前 8 章全部 | 0.0% |

**块越多，准确率越低。**

不是"边际递减"——是单调递减，一路降到零。

Baseline 什么都没挂，Gemma-2-2b-it 裸跑，24 道关于红楼梦的题，靠预训练里见过的红楼梦知识蒙对了 25%。

N=1 只喂了第一个 512-token 的块——注意，那 512 个 token 是什么？Project Gutenberg 的版权声明、译者信息、出版日期。**跟红楼梦的内容毫无关系。** 但这段垃圾信息生成的 LoRA 挂上去之后，模型回答红楼梦问题的准确率从 25% 降到了 16.7%。

垃圾信息不是"没用"，是"有毒"——它扰动了基础模型本来就知道的东西。

N≥50（rank-408）之后，模型的输出开始退化为重复 token。不是"答错了"，是**语言能力本身崩溃了**：

```
"îng îng îng îng îng îng îng îng..."
"первыхпервых первыхпервых..."
```

模型不再说人话了。

━━━━━━━━━━━━━━━━━━━━

◆ 为什么会这样——三层原因

━━━━━━━━━━━━━━━━━━━━

**第一层：OOD，超出训练分布**

超网络训练时见过多少块？

翻源码，超网络的配置里 `num_chunk_probs` 的最大 key 是 8。也就是说，训练时超网络最多见过 8 块拼接的情况。我们喂了 176 块——**22 倍于训练分布的上界**。

这就像一个只学过加减法的小学生，你让他算 176 次连乘。不是他数学差，是你出的题根本不在他的能力范围内。

但这不是全部。N=1 只有 1 块，完全在训练分布内，准确率照样比 baseline 低。说明 OOD 不是唯一的问题。

────────────────────

**第二层：噪声累积摧毁基础模型**

这是更本质的原因。

Gemma-2-2b-it 的 hidden dimension 是 2304。LoRA 只改 `down_proj`（MLP 层），权重矩阵是 [2304, 9216]。

算一笔账：

| 块数 N | 有效 rank | rank / hidden_dim | 扰动占比 |
|--------|----------|-------------------|----------|
| 1 | 16 | 0.7% | 微扰 |
| 5 | 48 | 2.1% | 轻微 |
| 20 | 168 | 7.3% | 显著 |
| 50 | 408 | 17.7% | 危险 |
| 100 | 808 | 35.1% | 毁灭性 |
| 176 | 1416 | 61.5% | 已超过原始维度的一半 |

rank-408 时，扰动已经占了原始维度的 18%。rank-1416 时，扰动维度超过了基础模型 hidden dimension 的 61%。

💡 **人话翻译：** LoRA 本来是给模型做微调的——像在一幅画上轻轻加几笔。rank-8 是加几笔，rank-1416 是在原画上盖了另一幅画。基础模型的"本我"被淹没了。

每个块独立生成 LoRA，块与块之间没有协调。拼接的结果是噪声的线性叠加——每个块各说各话，拼在一起不是合唱，是噪音。就像你让 176 个人同时对着你的耳朵说不同的事，你不会听得更清楚，你会聋。

────────────────────

**第三层：有效 rank 不能太大**

这是最深的一层，也是实验最有价值的发现。

LoRA 的本质是什么？是在基础模型的权重空间里加一个低秩扰动。"低秩"是关键词——它意味着扰动被限制在一个低维子空间里，不会破坏基础模型已经学好的高维结构。

基础模型经过预训练，它的知识和语言能力编码在一个高维的流形上——你可以把它想象成一个复杂的曲面。LoRA rank 小的时候，扰动只是在这个曲面上做微小的形变，曲面的整体拓扑结构不变，模型还是它自己。

但 rank 大到一定程度，扰动的维度开始接近流形本身的维度。这时候不是"在曲面上加一点变化"了，是**把曲面撕碎了**。

基础模型的"本我流形"就那么几百个有效维度。rank 大了，流形就坏了。流形坏了，语言能力就没了。

这就是为什么 N≥50 时模型输出变成 `"îng îng îng"` ——不是模型"答错了红楼梦的问题"，是模型**连说话的能力都丢了**。down_proj 层的 LoRA 扰动太大，MLP 的输出分布被彻底扭曲，解码器从正常的 token 概率分布退化成某几个 token 的无限重复。

────────────────────

**退一步想：LoRA 本来就不是干这个的**

如果不用 Doc-to-LoRA，直接用传统 LoRA 微调，拿红楼梦全文训几百步梯度下降，会怎样？最差也就是过拟合——模型说话带上红楼梦腔调，但至少它**记住了内容**，答题准确率不会比裸模型更低。

这恰好说明了问题的本质：**LoRA 是微调工具，不是记忆模块。** 传统 LoRA 微调能记住红楼梦，是因为梯度下降花了几百步把知识"学进去"了——改变的是模型的能力和风格，不是往里塞一份文档。Doc-to-LoRA 想让超网络一次前向传播就"生成"一个记忆胶囊，省掉梯度下降的过程。省是省了，但生成出来的权重没有经过优化，跟模型原有的知识结构不兼容——挂上去不是微调，是往一幅好好的画上泼墨。

LoRA 能微调，不能记忆。这是工具的边界。而 Doc-to-LoRA 偏要把它当记忆模块用，不断往里堆 rank——堆到 rank 不再 "Low" 的时候，讽刺的事情发生了：**LoRA（Low-Rank Adaptation）这个本来为防止灾难性遗忘而设计的工具，因为 rank 被堆得太高，自己触发了灾难性遗忘。**

◆ 回到《Doc-to-LoRA 介绍》的预言

━━━━━━━━━━━━━━━━━━━━

那篇文章里我们说：

> Doc-to-LoRA 现在的分块拼接是反过来的——文档越长，rank 越高。它在用维度扩张来保证信息不丢，本质上还是在做"完整复制"，只是换了个格式。

实验结果比我们预期的还要糟糕。不是"效果边际递减"，是**效果直接变成负的**。分块拼接不是"不够好"，是"会杀死模型"。

我们还说过：

> 真正复制人类记忆机制的设计应该是：重要的段落 → 高 rank 分配，冗余的段落 → 低 rank 甚至直接丢弃。最终 LoRA 的大小和文档长度解耦。

现在我们有数据支撑这个判断了。

────────────────────

**突触稳态假说的实证**

《Doc-to-LoRA 介绍》里我们讲了 SHY（突触稳态假说）：白天学习时突触全面增强，信噪比急剧下降；睡眠干的事不是"存入记忆"，而是选择性地削减突触强度——不重要的削多，重要的削少。**遗忘不是记忆的失败，是记忆的必要成本。**

这次的实验数据，是 SHY 在人工神经网络上的精确复现：

- 每多拼接一个块，就是在"全面增强突触"——不管有用没用，全部写进 LoRA
- rank 线性增长，信噪比急剧下降
- 最终模型崩溃——对应的就是"没有睡眠的大脑，学到的都是噪音"

Doc-to-LoRA 学会了"睡觉"——把文档从 context 搬进权重。但它的"睡觉"方式是**不做梦的睡觉**：来什么记什么，没有取舍，没有整合，没有遗忘。

真正的睡眠需要做梦——梦是大脑在主动做有损压缩：什么该留，什么该丢，什么该跟已有的知识整合。没有这一步，记忆就是垃圾堆积，最终压垮系统。

我们的实验证明了：**没有遗忘机制的记忆写入，比不写入更糟糕。**

◆ 从实验回到架构：什么才是真正的长期记忆

━━━━━━━━━━━━━━━━━━━━

Doc-to-LoRA 是事后打补丁——先训好模型，再单独训一个超网络去适配。超网络只认 Gemma-2-2b-it，换模型就得重新来。更根本的问题是：基础模型的内部表示结构不是为"被外部写入"设计的，LoRA 硬塞进去，小剂量还行，大剂量就毒发身亡。

真正的长期记忆需要什么？需要架构本身就为记忆更新留好接口。

────────────────────

《Mamba-3 介绍》（https://mp.weixin.qq.com/s/A8eJvexEDZwYMOsf1lSJAA）里我们讲了 Mamba-3。回忆一下 SSM 的状态更新公式：

```
state = A * state + B * new_token
```

A 矩阵每步衰减旧信息，B 矩阵选择性写入新信息。**遗忘是出厂设置。**

这个设计和 Doc-to-LoRA 的分块拼接有三个根本区别：

**一、state 大小固定，不随输入增长。**

Doc-to-LoRA：文档 N 块 → rank 8N → LoRA 尺寸线性膨胀。
SSM：不管处理了 100 个 token 还是 100 万个 token，state 就那么大。

这正好是《Doc-to-LoRA 介绍》里说的——**LoRA 大小和文档长度解耦**。SSM 天然做到了。

**二、遗忘是连续的，不是事后的。**

Doc-to-LoRA：先记住所有块，拼在一起，希望模型能从噪音里找到信号。
SSM：每一步都在遗忘——A 矩阵的衰减就是实时运行的遗忘曲线。旧信息不是被显式删除，是被新信息自然覆盖。

**三、写入和遗忘协同训练。**

Doc-to-LoRA：基础模型冻结，超网络单独训，两者没有协同。
SSM：A（遗忘）和 B（写入）是同一个训练过程学出来的，协同优化。模型从训练开始就知道"我的记忆是有限的"，所以它学会了在有限的 state 里放最重要的东西。

────────────────────

那篇文章结尾我们说：

> 真正的下一步，是把"记忆巩固"纳入模型的训练目标本身——让睡眠成为模型的出厂设置，而不是事后安装的插件。

Mamba 系列的 SSM 架构，就是这个"出厂设置"。

固定大小的 state，就是固定容量的长期记忆。A 矩阵的衰减，就是遗忘曲线。B 矩阵的选择性写入，就是海马体在决定什么值得记住。整个机制从第一个 training step 就在运行——不是事后打补丁，是架构本身的呼吸。

◆ 这个实验真正证明了什么

━━━━━━━━━━━━━━━━━━━━

不是"Doc-to-LoRA 不行"——它在论文场景（短文档、少量分块）下表现很好。

我们证明的是一个更底层的结论：

**任何没有遗忘机制的记忆写入方案，在输入规模增长时，都会从"有帮助"变成"有破坏"。**

这不是 Doc-to-LoRA 的 bug。这是所有"事后打补丁"式记忆方案的结构性上限。

LoRA rank 线性增长 → 扰动淹没本我流形 → 模型崩溃。这条因果链不依赖于具体的超网络实现，不依赖于文档内容，不依赖于基础模型的选择。它依赖的是一个更基本的事实：**低秩扰动之所以安全，恰恰因为它是低秩的。**

rank 一旦不低了，safety guarantee 就没了。

────────────────────

从生物学的角度看，这个结论其实不新鲜。赫布学习法则（Hebb's rule）说"一起激活的神经元会加强连接"——但如果只有加强没有削弱，突触会全部饱和，大脑变成一坨噪音。SHY 假说给出的解药就是睡眠中的全局突触下调。

我们的实验做的事，等价于**把一个大脑的突触全局上调了 61%，然后问它还能不能说话**。

答案是不能。

AI 需要学会遗忘，才能真正学会记忆。这不是哲学，是工程约束。

- 超网络的官方 API 只支持单块 internalize，多块消融需要拆开黑盒：逐块调用 `generate_weights()`，LoRA 存 CPU，最后搬回 GPU 用 `combine_lora()` 沿 rank 维度拼接，再 `apply_lora_to_layers()` 挂载
- LoRA 只修改 `down_proj`（MLP 层），每块 rank-8
- `combine_lora()` 内含 bias 项，所以 N 块的有效 rank = 8 × (N+1)，不是 8N
- 评估方式：关键词匹配（answer 中长度 >3 的词是否出现在 response 中）
