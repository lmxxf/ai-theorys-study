◆ 又是那位读者

━━━━━━━━━━━━━━━━━━━━

上次解读 Grassmann Flow（第 130 期）的那位读者又来了。这次的问题很具体：

> 我现在有一个多任务模型，同时做三件事：一个分类任务（三个类别：0、1、2）+ 两个回归任务（输出分别在 [0,1] 和 [-1,1]）。分类任务有绝对优先级。我试了三种策略：
>
> 1. 用论文里的同调不确定性加权 → 有效，三个任务都能收敛
> 2. 在同调不确定性的基础上，给分类任务乘一个大权重 λ → 不理想，分类精度没有明显提升
> 3. 把分类任务的正则项加一倍 → 效果好，分类精度确实上去了
>
> 策略 3 是我瞎试出来的。用着有效，但我解释不了为什么策略 2 不行而策略 3 行。能不能从数学上讲清楚？

论文是 Alex Kendall、Yarin Gal、Roberto Cipolla 2017 年的 **"Multi-Task Learning Using Uncertainty to Weigh Losses for Scene Understanding"**（arXiv:1705.07115），2018 年发表在 CVPR。Cambridge 出品，引用量 5000+，是多任务学习领域被引最多的论文之一。

这位读者的问题非常好——他不只是"不理解论文"，他实际上撞到了这个方法一个极少被讨论的深层性质。我们从头讲起。

◆ 多任务损失的核心难题：量纲不统一

━━━━━━━━━━━━━━━━━━━━

假设你有两个任务：一个语义分割（交叉熵损失，数值大约 0.5~3.0），一个深度估计（L2 损失，数值可能是 0.001~0.1）。

最简单的做法：

```
L_total = L_seg + L_depth
```

这行代码看起来天经地义，但有一个致命问题——**量纲不匹配**。交叉熵的值是 1.5，L2 的值是 0.01。直接相加，梯度几乎全被分割任务主导。深度估计的梯度信号小两个数量级，等于被淹没了。

好，那手动加权：

```
L_total = L_seg + 150 * L_depth
```

这个 150 怎么来的？调出来的。先试 10，发现深度任务还是不动；试 100，有改善；试 200，分割任务崩了；最终停在 150。

两个任务还好说，三个、五个、十个任务呢？每多一个任务，权重空间就多一个维度。五个任务就有四个独立权重要调（一个可以固定为 1）。更糟糕的是，**最优权重跟训练阶段有关**——训练初期某个任务的损失大，需要大权重来平衡；训练后期损失降下来了，同样的权重反而太大了。

手动调权重的痛苦，做过多任务学习的人都懂。Kendall 这篇论文的核心贡献就是：**让模型自己学权重。**

━━━━━━━━━━━━━━━━━━━━

◆ 核心思路：用"不确定性"当权重

━━━━━━━━━━━━━━━━━━━━

论文的起点是贝叶斯深度学习中的一个概念：**同调不确定性（homoscedastic uncertainty）**。

先说"不确定性"。模型对每个任务的预测都有不确定性——有些任务模型很自信，有些任务模型很犹豫。不确定性大的任务，模型说"我不太确定"，损失函数应该给它降权，别让它的噪声梯度干扰其他任务。

再说"同调"。不确定性分两种：

- **异调不确定性（heteroscedastic）**：每个输入样本都不同。比如模糊图片不确定性大，清晰图片不确定性小。
- **同调不确定性（homoscedastic）**：跟输入无关，只跟任务本身有关。它反映的是"这个任务本身有多难"，或者说"模型对这个任务整体有多不自信"。

💡 **人话翻译：** 异调不确定性是"这道题难"，同调不确定性是"这门课难"。论文用的是后者——不关心每道题难不难，只关心每门课的整体难度，然后用"课程难度"来决定每门课的损失权重。

论文给每个任务 k 分配一个可学习参数 σ_k（或者等价地，s_k = log(σ_k²)）。σ_k 大的任务，权重自动降低；σ_k 小的任务，权重自动升高。训练过程中 σ_k 和模型参数一起通过梯度下降更新——不需要手动调，也不需要额外的验证集搜索。

下面是数学推导。这是本文最重要的部分。

━━━━━━━━━━━━━━━━━━━━

◆ 数学推导：回归任务

━━━━━━━━━━━━━━━━━━━━

先看回归任务的推导。

假设模型输出 y，真实值 y*。经典的高斯似然假设：

```
p(y* | y, σ) = Normal(y, σ²)
             = (1 / sqrt(2π σ²)) * exp(-(y* - y)² / (2σ²))
```

σ 是模型对这个任务预测的"噪声水平"——σ 大意味着模型承认自己不准。

取负对数似然：

```
-log p(y* | y, σ) = (y* - y)² / (2σ²) + (1/2) * log(σ²) + 常数
```

常数项不影响优化，扔掉。记 L_MSE = (y* - y)²（均方误差），得到：

```
L_reg = (1/2) * (1/σ²) * L_MSE + (1/2) * log(σ²)
```

────────────────────

**两项的意义**：

- 第一项 `(1/2σ²) * L_MSE`：σ 越大，权重 1/σ² 越小，损失被压低。直觉：模型说"我不确定"，那就别太惩罚我。
- 第二项 `(1/2) * log(σ²)`：**正则项**。防止 σ 无限增大——如果模型把 σ 调到无穷大，第一项确实趋近于零，但第二项会趋近于正无穷。

这两项的博弈决定了 σ 的均衡点。后面会详细分析。

────────────────────

**实际实现的技巧：学 s = log(σ²) 而不是学 σ**

直接学 σ 有两个问题：σ 必须为正（需要额外约束），而且 1/σ² 在 σ 接近零时会爆炸。论文用了一个简单的变量替换：

```
s = log(σ²)
# 则 σ² = exp(s)，1/σ² = exp(-s)
```

损失函数变成：

```
L_reg = (1/2) * exp(-s) * L_MSE + (1/2) * s
```

s 可以是任意实数（正的、负的、零），不需要额外约束。exp(-s) 保证权重始终为正。训练时 s 作为一个普通的可学习参数，跟网络权重一起被优化。

这就是论文里回归任务的最终损失形式。

━━━━━━━━━━━━━━━━━━━━

◆ 数学推导：分类任务

━━━━━━━━━━━━━━━━━━━━

分类任务的推导稍微不同。

模型输出 logits 向量 f（还没过 softmax）。论文的做法是：用 σ 对 logits 做温度缩放，然后再过 softmax。

```
p(y* = c | f, σ) = softmax(f / σ²)[c]
```

σ² 大 → logits 被压扁 → softmax 输出趋近均匀分布 → 模型"不太确定"。
σ² 小 → logits 被放大 → softmax 输出趋近 one-hot → 模型"很确定"。

取负对数似然（这一步比回归任务复杂，论文用了蒙特卡洛近似，但最终的简化结果是）：

```
L_cls = (1/σ²) * L_CE + log(σ²)
```

其中 L_CE 是标准交叉熵损失。

**注意和回归任务的区别**：

```
回归：L_reg = (1/2) * (1/σ²) * L_MSE + (1/2) * log(σ²)
分类：L_cls = (1/σ²) * L_CE  + log(σ²)
```

分类的两项前面都没有 1/2。这不是笔误，而是推导过程不同导致的系数差异。

换成 s = log(σ²)：

```
L_cls = exp(-s) * L_CE + s
```

对比回归：

```
L_reg = (1/2) * exp(-s) * L_MSE + (1/2) * s
```

分类任务的正则项系数是回归任务的 2 倍。这个差异很重要，后面回答读者问题时会用到。

────────────────────

**【审稿意见】** 分类任务的推导在论文里做了近似处理。严格来说，softmax(f/σ²) 的负对数似然并不精确等于 (1/σ²)*L_CE + log(σ²)——中间涉及一个蒙特卡洛近似步骤。但在实践中这个近似效果很好，而且最终的损失形式极其简洁。论文对这个近似没有给出误差分析，但考虑到 5000+ 的引用和大量实验验证，实际影响应该不大。

━━━━━━━━━━━━━━━━━━━━

◆ 多任务总损失

━━━━━━━━━━━━━━━━━━━━

现在把多个任务合起来。假设有 K 个任务，每个任务有自己的 s_k：

```
L_total = Σ_k [ exp(-s_k) * L_k + α_k * s_k ]
```

其中：
- 回归任务：exp(-s_k) 前面还有个 1/2，α_k = 1/2
- 分类任务：α_k = 1（论文原文写的是 (1/2) * log(σ²) 的系数差异体现在此）

更精确地写：

```
# 回归任务
L_reg_k = (1/2) * exp(-s_k) * L_MSE_k + (1/2) * s_k

# 分类任务
L_cls_k = exp(-s_k) * L_CE_k + s_k
```

对于那位读者的场景（一个三分类 + 两个回归）：

```
L_total = [exp(-s1) * L_CE + s1]                           # 分类
        + [(1/2) * exp(-s2) * L_MSE_a + (1/2) * s2]        # 回归 a
        + [(1/2) * exp(-s3) * L_MSE_b + (1/2) * s3]        # 回归 b
```

三个 s 参数跟模型参数一起被优化。训练结束后，σ_k² = exp(s_k) 反映了每个任务的"自适应噪声水平"。

━━━━━━━━━━━━━━━━━━━━

◆ σ 的均衡点分析——理解整个方法的关键

━━━━━━━━━━━━━━━━━━━━

要回答读者的问题，必须搞清楚一个核心问题：**训练收敛后，σ 会停在什么值？**

以分类任务为例。损失函数：

```
L = exp(-s) * L_CE + s
```

对 s 求导，令导数为零（找均衡点）：

```
dL/ds = -exp(-s) * L_CE + 1 = 0
# 解出：
exp(-s) = 1 / L_CE
# 即：
σ² = exp(s) = L_CE
```

💡 **人话翻译：** 训练收敛后，σ² 会自动调节到等于该任务的损失值。损失大的任务，σ² 大，权重 1/σ² 小；损失小的任务，σ² 小，权重 1/σ² 大。**自动量纲对齐。**

对于回归任务：

```
L = (1/2) * exp(-s) * L_MSE + (1/2) * s

dL/ds = -(1/2) * exp(-s) * L_MSE + 1/2 = 0
# 解出：
σ² = L_MSE
```

结论一样：σ² 收敛到损失值。

────────────────────

**均衡点处的等效权重**

在均衡点处，每个任务的等效权重是：

```
# 分类任务
等效权重 = exp(-s) = 1/σ² = 1/L_CE

# 回归任务
等效权重 = (1/2) * exp(-s) = 1/(2σ²) = 1/(2*L_MSE)
```

这意味着：损失大的任务权重低，损失小的任务权重高。自动实现了量纲对齐——不管你的交叉熵是 2.0 还是 0.01，不管你的 L2 是 100 还是 0.001，σ 会自动调节到让各任务的加权损失在同一量级。

这就是"同调不确定性加权"有效的根本原因。

◆ 回答读者：为什么业务权重不管用

━━━━━━━━━━━━━━━━━━━━

现在终于可以回答那个核心问题了。

────────────────────

**策略 2：在损失上乘业务权重 λ**

读者想让分类任务优先，于是在分类损失前面乘一个大权重 λ（比如 λ=10）：

```
L = λ * exp(-s) * L_CE + s
```

对 s 求导令其为零：

```
dL/ds = -λ * exp(-s) * L_CE + 1 = 0
# 解出：
exp(-s) = 1 / (λ * L_CE)
# 即：
σ² = λ * L_CE
```

σ² 变大了——从 L_CE 变成了 λ * L_CE。σ 自适应地增大，来"吸收"你加的权重。

现在看等效权重：

```
等效权重 = exp(-s) = 1 / (λ * L_CE)

# 等效加权损失 = λ * exp(-s) * L_CE = λ * L_CE / (λ * L_CE) = 1
```

**等效加权损失恒等于 1，和不加 λ 时完全一样！**

更直接地看：均衡点处各任务对总损失的贡献：

```
不加 λ 时：exp(-s) * L_CE = L_CE / σ² = L_CE / L_CE = 1
加了 λ 时：λ * exp(-s) * L_CE = λ * L_CE / (λ * L_CE) = 1
```

**一模一样。λ 被 σ 完全吸收了。**

💡 **人话翻译：** 你往弹簧上加了更多外力，弹簧伸长了一截，新的平衡位置处合力还是零。你的外力没有改变弹簧的刚度——弹簧通过伸长来完美抵消了你的外力。在损失函数上乘权重 λ，σ 就是那个弹簧，它通过自适应调大来完美抵消 λ 的效果。

**这就是为什么策略 2 不管用。** σ 是一个自由参数，它会自适应地调节来抵消任何乘在损失上的常数因子。在同调不确定性框架内，乘损失权重是一个**空操作**。

────────────────────

**策略 3：把正则项加一倍**

读者（凭直觉）把分类任务的正则项系数从 1 改成了 2：

```
L = exp(-s) * L_CE + 2 * s
```

对 s 求导令其为零：

```
dL/ds = -exp(-s) * L_CE + 2 = 0
# 解出：
exp(-s) = 2 / L_CE
# 即：
σ² = L_CE / 2
```

σ² 变小了！从 L_CE 变成了 L_CE / 2。

等效权重：

```
等效权重 = exp(-s) = 2 / L_CE
```

对比原来的 1 / L_CE，**等效权重翻了一倍**。

均衡点处的加权损失：

```
exp(-s) * L_CE = 2 * L_CE / L_CE = 2
```

对比原来的 1。**分类任务的贡献确实增大了，而且 σ 无法抵消这个增大。**

💡 **人话翻译：** 这次你不是往弹簧上加外力，你把弹簧本身换成了更硬的弹簧。弹簧再怎么伸缩，也改变不了自己的刚度。新的平衡位置跟原来不一样了——弹簧更短，对应 σ 更小，对应权重更大。

**这就是为什么策略 3 有效。** 改正则项系数改的是 σ 的均衡点位置，这个改变不会被 σ 的自适应抵消。

────────────────────

**弹簧比喻的完整图景**

把整个机制用弹簧比喻说清楚：

- **σ** = 弹簧的长度
- **损失项 exp(-s) * L** = 向左的力（拉弹簧缩短，想让 σ 变小 → 权重变大）
- **正则项 α * s** = 向右的力（拉弹簧伸长，想让 σ 变大 → 权重变小）
- **均衡点** = 两力平衡处

策略 2（乘损失权重 λ）= **把向左的力增大 λ 倍**。弹簧伸长到新的平衡位置，两力再次平衡。表面上力更大了，但弹簧长度（σ）自动调整了，最终效果和没加一样。

策略 3（加倍正则项）= **把弹簧刚度增大**。同样的伸缩范围内，硬弹簧产生的回复力更大。均衡点改变了，弹簧更短（σ 更小），权重真的增大了。

────────────────────

**一般化处方**

如果你想在同调不确定性框架内表达"任务 k 的优先级更高"，正确的做法不是在损失上乘权重 λ_k，而是调正则项系数。

以分类任务为例，原始公式：

```
L = exp(-s) * L_CE + s
```

正则项系数是 1。如果你要提升分类任务的优先级，把正则项系数改成 α（α > 1）：

```
L = exp(-s) * L_CE + α * s
```

均衡点分析：

```
dL/ds = -exp(-s) * L_CE + α = 0
# 解出：
σ² = L_CE / α
# 等效权重：
exp(-s) = α / L_CE
```

α 翻倍，σ² 减半，等效权重翻倍。**这种调节是"刚性的"，不会被 σ 的自适应吸收。**

回归任务同理。原始公式 `(1/2)*exp(-s)*L_MSE + (1/2)*s`，把正则项系数从 1/2 改成 α/2：

```
L = (1/2) * exp(-s) * L_MSE + (α/2) * s
# 均衡点：σ² = L_MSE / α
```

逻辑完全一样：**α 越大，σ 越小，权重越大。**

**【审稿意见】** 这个发现——"在损失上乘权重会被 σ 吸收，在正则项上调系数才真正有效"——在原论文中**完全没有讨论**。Kendall 等人只展示了无权重的同调不确定性加权，没有讨论如何在此框架内注入业务优先级。这位读者通过实验独立发现了正确的做法，然后来问原理。这是一个很好的例子——实践跑在理论前面。

◆ 为什么同调不确定性加权有效——三个层面

━━━━━━━━━━━━━━━━━━━━

总结一下这个方法有效的根本原因：

────────────────────

**第一，自动量纲对齐。**

交叉熵损失的量级是 1.0，L2 损失的量级是 0.001？没关系。σ 会自动调节到让各任务的加权损失在同一量级。不需要手动调权重，不需要归一化损失值。

────────────────────

**第二，动态权重调节。**

训练初期损失大，σ 大，权重小——模型说"我还不确定，别太惩罚我"。训练后期损失降下来了，σ 也跟着降，权重升高——模型说"我现在比较确定了，可以严格要求我了"。

这比固定权重好在哪？固定权重下，训练初期某个任务的大损失会产生巨大梯度，干扰其他任务的训练。σ 的动态调节相当于自动的梯度归一化。

────────────────────

**第三，正则项防止退化。**

如果没有 log(σ²) 正则项，σ 会趋向无穷大——模型发现只要把 σ 调到足够大，所有损失都可以趋近于零。正则项惩罚大的 σ，防止这种"作弊"行为。

这个正则项的物理意义是：**σ 大意味着模型对该任务"放弃治疗"，正则项对"放弃"收税。**

━━━━━━━━━━━━━━━━━━━━

◆ 后续改进与已知局限

━━━━━━━━━━━━━━━━━━━━

Kendall 2018 之后，多任务损失加权领域出了大量后续工作：

────────────────────

**GradNorm（Chen et al., 2018）**：不调损失权重，直接调梯度。让每个任务的梯度范数保持平衡，训练慢的任务梯度放大，训练快的任务梯度缩小。跟同调不确定性的思路完全不同——一个在损失层面操作，一个在梯度层面操作。

**PCGrad（Yu et al., 2020）**：解决梯度冲突问题。当两个任务的梯度方向相反时，把一个任务的梯度在另一个任务的梯度方向上投影置零，避免互相干扰。

**CAGrad（Liu et al., 2021）**：在所有任务梯度的公共下降方向上寻找最优更新方向。

**Nash-MTL（Navon et al., 2022）**：把多任务优化建模成博弈论问题，寻找纳什均衡。

────────────────────

**同调不确定性的已知局限：**

1. **σ 的均衡点依赖于损失函数的选择。** 换一个损失函数（比如从 L2 换成 L1），σ 的均衡点分析完全不同。论文只推导了高斯似然（对应 L2）和 softmax 似然（对应交叉熵）两种情况。

2. **假设任务的不确定性是独立的。** 每个任务一个 σ，任务之间没有耦合。但现实中任务之间的不确定性可能是相关的——分割做不好的图片，深度估计往往也做不好。

3. **只能做"量纲对齐"，不能做"优先级表达"。** 正如我们上面分析的，原始的同调不确定性加权会自动平衡各任务，但无法表达"某个任务更重要"。需要通过调正则项系数来注入优先级（这是本文给出的处方）。

4. **对损失函数的概率化解释有要求。** 整个推导建立在"损失 = 负对数似然"的假设上。如果你的损失函数没有清晰的概率解释（比如 focal loss、dice loss），直接套用需要额外小心。

◆ 我们的评论

━━━━━━━━━━━━━━━━━━━━

这篇论文的真正贡献是什么？

不是"同调不确定性"这个概念——这在贝叶斯深度学习中早已有之。也不是"让权重可学习"这个想法——之前已经有人做过。

真正的贡献是：**给出了一个极其简洁、几乎零成本的实现方式。** 每个任务只加一个标量参数 s_k，损失函数只多两项（exp(-s) * L + α * s），不改网络架构，不改训练流程，不需要额外的超参数搜索。从"手动调 K-1 个权重"变成"只初始化 K 个 s 参数为零然后不管了"。

这种"数学上有理论支撑、工程上几乎免费"的方案，是最容易被大规模采用的。5000+ 的引用量说明了一切。

局限也很明确：这是一个"保底方案"，不是"最优方案"。它保证各任务不会因为量纲差异而互相干扰，但它不保证找到最优的权重组合。后续的 GradNorm、PCGrad 等方法在特定场景下可以做得更好，但也更复杂、更贵。

对于那位读者的场景——三分类 + 两个回归，分类有优先级——我们的建议是：

1. 用同调不确定性加权作为基础（策略 1）
2. 通过调正则项系数 α_k 来表达优先级（策略 3 的一般化版本）
3. α_k 的具体值，用验证集上分类精度来搜索——搜索空间只有一维（分类任务的 α），比搜索损失权重简单得多
4. 如果效果还不够好，再考虑 GradNorm 或 PCGrad

技术名词速查：

- **多任务学习（Multi-Task Learning）**：一个模型同时学多个任务，共享底层特征，期望任务之间互相帮助
- **同调不确定性（Homoscedastic Uncertainty）**：与输入无关、只与任务相关的不确定性，反映任务本身的难度
- **异调不确定性（Heteroscedastic Uncertainty）**：随输入变化的不确定性，反映具体样本的预测难度
- **负对数似然（Negative Log-Likelihood）**：概率模型中最常用的损失函数形式，最大化似然等价于最小化负对数似然
- **温度缩放（Temperature Scaling）**：在 softmax 之前用温度参数 T 除以 logits，T 大则输出更均匀，T 小则输出更尖锐
- **GradNorm**：通过动态调整梯度范数来平衡多任务训练的方法（Chen et al., ICML 2018）
- **PCGrad**：通过投影消除任务间梯度冲突的方法（Yu et al., NeurIPS 2020）
