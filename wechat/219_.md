【图像生成】AI 生图的目标函数演化史

218 期最后留了个尾巴：统一多模态模型里，理解走自回归，生成往往另挂一个扩散头——两个目标函数并存，缝合得很别扭。有读者会问：生成图像为什么非得扩散？预测下一个 token 不行吗？还有没有别的路？

有，而且不止一条。这篇把 AI 生图的几条技术路线按时间串一遍，但视角钉死在一个地方：**目标函数**。不看网络结构怎么改、参数怎么堆，只看一件事——你让模型"预测什么"。

先把结论放这：生图这十几年的演化，表面上是画质竞赛，底下其实是一场关于"图像到底是什么"的哲学辩论。每换一次目标函数，就是换了一套对图像的理解。

━━━━━━━━━━━━━━━━━━━━

◆ 第一代：GAN——图像是"骗过裁判的赝品"

━━━━━━━━━━━━━━━━━━━━

2014 年，Goodfellow 的 GAN 开局。它的目标函数和后面所有路线都不一样——不是"预测某个东西"，是**两个网络打架**：

min_G max_D  E[log D(x)] + E[log(1 − D(G(z)))]

【训练阶段】
生成器 G 吃一个随机向量，吐一张假图；判别器 D 同时看真图和假图，学着分辨真假。D 的 loss 是分类错误率，G 的 loss 是"被 D 识破的程度"。两边轮流更新：D 越来越会抓假，G 被逼着越造越真。

【生成阶段】
训练完了把 D 扔掉，只留 G——喂一个随机向量，**一次前向传播直接出图**。

这个"一步出图"是 GAN 至今无人能敌的优点，后面所有路线都要迭代几十上百步，速度全输给它。

但目标函数的毛病也刻在基因里。"骗过判别器"是个**对抗目标，不是重建目标**——G 只要找到几种能骗过 D 的图就完成任务了，没有任何东西逼它覆盖真实图像的全部多样性。于是 mode collapse（生成器只会画几种图，翻来覆去出同款）成了 GAN 的职业病。再加上两个网络拔河本来就难以平衡——一方碾压另一方，训练就崩——以及很难做精细的文本条件控制，GAN 在 2022 年前后被扩散赶下主桌，现在主要活在超分辨率、人脸生成这类专用场景里。

💡 mode collapse（模式坍缩）：真实图像分布有无数种"画法"，生成器却只学会其中几种就能骗过判别器，于是只生产那几种。对抗目标的结构性漏洞——它奖励"像真的"，不奖励"全都像"。

代表工作：GAN [arXiv:1406.2661]、StyleGAN 系列

━━━━━━━━━━━━━━━━━━━━

◆ 第二代：自回归离散 token——图像是"一篇按顺序写的作文"

━━━━━━━━━━━━━━━━━━━━

GPT 在文本上成功之后，一个自然的想法：能不能把同一套搬到图像上？

218 期讲过这条管线：先用 VQ 码本把图像量化成离散 token——一张 256×256 的图变成 32×32 = 1024 个整数，每个整数是码本里的条目编号。然后照搬语言模型，按从左到右、从上到下的**光栅顺序**把这 1024 个 token 排成一队，交叉熵预测下一个：

L = −Σ log P(xₜ | x₁...xₜ₋₁)

这条路线的吸引力是**架构复用**：和文本共用同一套 Transformer、同一个 softmax，词表里既有汉字也有图像块编号，模型根本不区分。天然适合统一多模态模型——Gemini 的原生图像生成走的就是这条路。

但它长期打不过扩散，三个原因，前两个 218 期点过，这里展开第三个：

一是**光栅顺序是装出来的因果**。文本的"下一个字"有真因果——语法、逻辑、指代，前文真的决定后文。但图像左上角的块并不"决定"它右边的块，像素之间是空间关系，不是时序关系。你逼着模型把空间结构当时间序列学，它就得浪费容量去拟合一个根本不存在的因果链。

二是**串行慢**。1024 个 token 串行吐 1024 步，每步一次前向传播。

三是**码本信息损失**。图像本来是连续的，硬量化成 8192 个码本条目的那一刻，高频细节已经丢了——这个伏笔后面 MAR 会回来收。

这条路线 2026 年还活得很好：OpenAI 的 4o 原生图像生成，官方公告明确自述是 "autoregressive model"，逐 token 出图（解码端可能还挂了扩散类渲染器，细节没公开）；Gemini 初代技术报告也写明能原生输出图像 token。

代表工作：DALL-E 初代 [arXiv:2102.12092]、Parti [arXiv:2206.10789]、GPT-4o 原生生图、Gemini 原生生图

━━━━━━━━━━━━━━━━━━━━

◆ 第三代：扩散——图像是"一张被雾遮住的照片"

━━━━━━━━━━━━━━━━━━━━

2020 年 DDPM 把问题整个换掉了：不预测"下一块是什么"，预测"这张噪声图里掺了什么噪声"。

【训练阶段】
拿一张真图 x₀，随机选一个噪声强度 t，按比例掺进高斯噪声 ε，得到一张半毁的图 x_t。让模型看着 x_t 和 t，把掺进去的噪声猜出来：

L = E‖ε − ε_θ(x_t, t)‖²

注意这是**均方误差——回归任务，不是分类**。模型输出的是连续值，不查任何字典。

【生成阶段】
从一张纯高斯噪声开始，反复"猜噪声→减掉一点"，几十步之后图像从雾里浮出来。

对着自回归的三宗罪，扩散正好逐条解了两条：

**没有人为顺序**——每一步去噪是全图所有像素同时更新，空间关系就是空间关系,不用假装是时序。构图、对称、远距离呼应这些全局结构,在每一步里都被整体看到。

**没有码本损失**——直接在连续空间里回归，细节不用先过一遍 8192 条目的字典。

代价同样明确：**采样慢**（几十步，每步一次完整前向传播——比 GAN 的一步慢两个数量级）；以及 218 期说过的，**和文本模型不兼容**——回归输出和语言模型的离散 softmax 是两套东西，统一模型只能缝合。

2021-2023 年，Stable Diffusion、DALL-E 2/3、Midjourney 把这条路线推成行业默认。

代表工作：DDPM [arXiv:2006.11239]、Stable Diffusion [arXiv:2112.10752]、DALL-E 3

━━━━━━━━━━━━━━━━━━━━

◆ 三代半：流匹配——扩散的数学整容版

━━━━━━━━━━━━━━━━━━━━

严格说不是新物种，是扩散的亲戚，但值得单列——因为 2026 年你看到的"扩散模型"，多数实际跑的是它。

扩散的去噪路径是随机过程推出来的，弯弯绕绕。流匹配换了个更干净的问法：在噪声分布和图像分布之间，**学一个速度场，把点平滑地从这头搬到那头**。

【训练阶段】
取一个噪声点 x₀ 和一张真图 x₁，在两点之间连条直线。在线上随机取个位置 x_t，让模型预测"该往哪个方向走、走多快"——目标速度近似就是直线方向 x₁ − x₀：

L = E‖v_θ(x_t, t) − (x₁ − x₀)‖²

（这是论文里"最优传输路径"那个特例的简化写法，原文还带一个很小的缩放项，不影响理解。）还是回归，还是均方误差，和扩散一个家族——论文自己就证明了扩散路径是流匹配的特例。区别在于搬运路径被拉直了——**路越直，采样步数越能压**。扩散要几十步走完的弯路，流匹配十几步甚至几步就到。

理论上两者等价（都是学概率分布之间的传输映射），工程上流匹配更好用。所以新一代主力模型悄悄完成了换代，以下都有官方技术报告背书：Stable Diffusion 3 的 rectified flow、Flux（官方自述 "rectified flow transformer"）、阿里 Qwen-Image（技术报告明确 flow matching 目标）、通义万相 Wan 和字节 Seedance 的视频生成。论文里说"扩散"，很多时候指的已经是它。

代表工作：Flow Matching [arXiv:2210.02747]、Stable Diffusion 3 [arXiv:2403.03206]、Flux

━━━━━━━━━━━━━━━━━━━━

◆ 旁支：掩码并行生成——把"顺序"还给模型自己挑

━━━━━━━━━━━━━━━━━━━━

在扩散如日中天的同时，离散 token 阵营内部也在自救。MaskGIT / Muse 一族的思路：自回归的问题不在离散 token，在**光栅顺序**——那把顺序扔了行不行？

还是 VQ 码本、还是交叉熵，但训练从"预测下一个"换成 BERT 式的掩码预测：随机蒙住一部分图像 token，让模型根据没蒙的部分猜蒙住的。

生成时玩法变了：先全部蒙住，一轮**并行**预测所有位置——但只保留置信度最高的一批，剩下的重新蒙上，下一轮再预测。论文的标准设置只要 **8 轮**，全图出来。

妙处有两个。一是**生成顺序由模型自己挑**：每轮留下的是它最有把握的部分——通常是大色块、背景、主体轮廓——难的细节留到后面几轮，等上下文更充分了再定。光栅顺序那个装出来的因果被扔掉了，顺序从人为约束变成了模型的自由变量。二是**快**：8 轮并行 vs 上千步串行，论文摘要里的数字是**最高加速 64 倍**。

代表工作：MaskGIT [arXiv:2202.04200]、Muse [arXiv:2301.00704]

━━━━━━━━━━━━━━━━━━━━

◆ 第四代：VAR——把假因果换成真因果

━━━━━━━━━━━━━━━━━━━━

2024 年，VAR（NeurIPS 2024 最佳论文）给自回归阵营翻了案。它保留"预测下一个"的形式，但把"下一个"的定义整个换掉：**不是下一个空间位置，是下一个分辨率尺度**。

先生成 1×1 的全图概览，再 2×2、4×4、8×8，一路细化到全分辨率。每个尺度内部所有 token **并行**出，尺度之间保持自回归——上一层的粗图作为条件，预测下一层的细图。

这一改最值得品的地方：**"粗轮廓决定细节"这个因果是真的。** 画家确实先打形再上调子，构图确实先于笔触，整体确实约束局部。光栅顺序的因果是装的，由粗到细的因果是真的。

同样是交叉熵、同样是"预测下一个"，把因果链从假的换成真的，质量和速度一起上来了。论文摘要的原话是："VAR, for the first time, makes GPT-style AR models surpass diffusion transformers in image generation"——首次让 GPT 式自回归在图像生成上超过扩散 Transformer（ImageNet 256 上 FID 1.73）。而且 256×256 的实现只用 10 个尺度，等于 10 步自回归就出图，比逐 token 自回归快约 20 倍。

代表工作：VAR [arXiv:2404.02905]

━━━━━━━━━━━━━━━━━━━━

◆ 第四代：MAR——回收"码本损失"那个伏笔

━━━━━━━━━━━━━━━━━━━━

同年，何恺明组从另一个方向动刀。VAR 修的是顺序，MAR 修的是**离散化**：图像本来是连续的，凭什么非要先过一遍 8192 条目的字典？

MAR 的做法：自回归照做，但每个位置不输出离散编号，**直接输出连续向量**。问题来了——交叉熵只能算在离散类别上，连续向量的概率分布怎么建模？答案是给每个位置挂一个**小扩散头**：以这个向量为条件，用扩散 loss 把它"渲染"成图像内容。

等于把两条路线杂交了：宏观顺序用自回归管（保住和文本架构的兼容性），微观每个位置的取值用扩散管（保住连续值的表达力）。VQ 码本被整个绕开,信息损失从源头消失。

代表工作：MAR [arXiv:2406.11838]

━━━━━━━━━━━━━━━━━━━━

◆ 收束：目标函数即世界观

━━━━━━━━━━━━━━━━━━━━

一张总表：

| 路线 | 预测什么 | loss | 顺序 | 代表 |
|----|----|----|----|----|
| GAN | 骗过判别器 | 对抗 | 一步出图 | StyleGAN |
| 自回归离散 | 下一个 token | 交叉熵 | 光栅（假因果） | Parti、Gemini |
| 扩散 | 掺进去的噪声 | 回归 | 无序，全图同步 | DDPM、DALL-E 3 |
| 流匹配 | 搬运速度场 | 回归 | 无序，全图同步 | SD3、Flux |
| 掩码并行 | 蒙住的 token | 交叉熵 | 模型自选 | MaskGIT、Muse |
| 下一尺度 | 下一个分辨率 | 交叉熵 | 由粗到细（真因果） | VAR |
| 连续自回归 | 下一个连续向量 | 自回归+扩散头 | 真因果+连续值 | MAR |

【暗线：一部给图像找因果的历史】

把表里"顺序"那一列单独抽出来看，演化的暗线就露出来了：

- **光栅顺序（假因果）**：把画画当写作文，假装左上角决定右下角
- **无序（取消因果）**：扩散全图同步去噪，干脆不回答"先画哪"
- **模型自选（因果当变量）**：掩码并行把顺序选择权还给模型
- **由粗到细（真因果）**：VAR 终于找到图像里真正的因果——构图先于笔触，整体约束局部

生图方法的进化史，就是一部"给图像找到正确因果结构"的历史。扩散赢了第一阶段，不是因为它找到了真因果，是因为它**取消了假因果**——在正确答案出现之前，不回答比答错强。

【接 218 期：谁能终结缝合？】

218 期说，现在的统一多模态模型多数是缝合的——理解走自回归交叉熵，生成另挂一个扩散头，两个目标函数并存。

看完这篇你就知道缝合为什么难受了：两套 loss 意味着两套梯度、两套表征偏好，梯度冲突那笔账（218 期第二笔税）在模型内部天天上演。

缝合不是纸上谈兵，有官方实证：腾讯混元 HunyuanImage 3.0 的技术报告（arXiv:2509.23951）写得明白——80B MoE 的统一自回归框架，图像部分在 VAE 特征上做扩散建模。**自回归骨架 + 扩散头，跟 MAR 的杂交思路同款**——工业界已经在用脚投票了。

而 VAR 和 MAR 的共同点值得注意：**都是自回归形式 + 真因果/连续值**。自回归形式保证和文本理解模块架构统一——同一个 Transformer、同一种"预测下一个"的训练范式；真因果和连续值保证生成质量不输扩散。要说明的是，VAR/MAR 目前还是学术热点，尚未见头部产品官方宣称采用——但混元 3.0 这种"AR 框架内嵌扩散"的混合路线，已经站在了同一个方向上。

【最后】

回到开头那句话。你让模型预测什么，就是在告诉它图像"是什么"：

- 预测下一个 token → 图像是一篇按顺序写的作文
- 预测掺进去的噪声 → 图像是一张被雾遮住的照片
- 预测下一个分辨率 → 图像是一幅由粗到细的素描

方法之争，本质是"图像的正确分解方式"之争。网络结构是执行层，目标函数才是世界观。
