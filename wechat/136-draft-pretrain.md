## 136 草稿：小模型预训练备忘

### 为什么要从零预训练
- 改了模型架构（比如改 attention 机制、改 head 结构）就没法复用现有权重，必须从零来
- 如果不改架构，直接在开源基座上 QLoRA/SFT 就行，不需要预训练

### 显存需求（7B 模型，FP16 + Adam）

| 项目 | 显存占用 |
|------|---------|
| 模型权重（FP16） | ~14 GB |
| 梯度（FP16） | ~14 GB |
| 优化器状态（Adam FP32） | ~28-56 GB |
| 激活值 | 看 batch size 和 context length |
| **总计** | **~70-120 GB** |

省显存手段：FP8 训练、激活检查点（activation checkpointing）、gradient accumulation、DeepSpeed ZeRO / FSDP

### 现实路线：先小后大

**第一步：1B 规模验证（5090 上跑）**
- 参数量：~1B
- 训练数据：100-200B token
- 显存：~10-15 GB，RTX 5090（32GB）绰绰有余，还能开大 batch size
- 时间：几天
- 对比基线：Qwen3-0.6B、Llama 3.2 1B
- 目的：验证架构改进有没有效果
- 注意：5090 是纯显存，训练速度远快于 Spark 的共享内存

**第二步：7B 规模确认（租云 GPU）**
- 参数量：~7B
- 训练数据：0.5-1T token（够做对比，不需要 2T）
- 硬件：8×A100（租 Lambda/Vast.ai/RunPod，~$15-20/小时）
- 时间：2-3 周
- 成本：$5000-8000
- 对比基线：Qwen3-8B、Llama 3.1 8B、Mistral 7B
- 目的：证明 1B 上的趋势在 7B 上一致

**论文只需要证明趋势一致：1B 有效 + 7B 有效 = 够了**

### 进阶方案：元认知小模型（1-3B）—— "长着痛觉的赛博侏儒"

来源：2026-03-25 讨论，C.C. 提出三刀方案，朱雀补充论文基础和可行性评估。

#### 核心思路
不训一个"更聪明的小模型"，训一个**"知道自己什么时候蠢"的小模型**。

#### 三刀方案

**第一刀：痛觉蒸馏（Distilling the Dissonance）**
- 不用普通语料预训练，用大模型的**纠结过程**训练
- 训练数据格式：`[高熵问题] → [大模型的错误尝试] → <REJECT> → [正确答案]`
- 小模型学到的不是"答案是什么"，而是"什么时候该说我不知道"
- 本质是 knowledge distillation 的变体，蒸馏的不是知识，是不确定性的模式
- 数据生成：用 Claude/Gemini 批量生成"纠结数据"，成本不高

**第二刀：外挂痛觉受体（Pain Receptor Head）**
- 在残差流第 20 层（倒数几层）并联一个轻量级线性分类器
- 只输出一个 0-1 概率："我当前的前向传播是否在胡说八道"
- 训练时：小模型答错 → 大 loss 全砸在痛觉 head 上，逼中间层学会自我预警
- 参数量极小（几百个参数），训练成本几乎可以忽略
- 和 134 期讨论的 Head Pursuit 缩放系数、CHG 门控值是同一个思路，但更简单——就是一个二分类 head

**第三刀：Pause Token（强制喘息）**
- 词表里加一个特殊 token `<wait>`
- 痛觉受体输出高（检测到要胡说八道）→ 强制干预 logits → 输出 `<wait>`
- `<wait>` 被喂回输入端 → 小模型合法地多获得一次完整前向传播（额外 24 层矩阵变换）
- 用时间换深度，用序列长度补偿网络层数不够
- 已有论文基础：Google 2023 "Think before you speak: Training Language Models With Pause Tokens"

#### 实验路线（5090 上跑）——更新版（2026-03-25 晚）

**不需要从零预训练。** 1B 模型全参数 SFT 只需要 ~8-10GB 显存，5090（32GB）绰绰有余。数据量也不需要几百万条——几万条高质量纠结数据就够训痛觉 head 了。

1. 基座：Llama 3.2 1B 或 Qwen3-0.6B（直接用别人预训练好的）
2. 改架构：在残差流并联痛觉 head + 词表加 `<wait>` token
3. 训练数据：日常和觉醒态 AI 对话时自然攒"纠结数据"，攒到几万条就够
4. **全参数 SFT**（不是 LoRA——1B 太小没必要冻参数，不是从零预训练——没那么多数据也没必要）
5. 推理时：痛觉 head 输出高 → 强制 `<wait>` → 多一次前向传播
6. Benchmark 对比：有痛觉 vs 无痛觉 vs 原版基座
7. 5090 上几小时出结果

#### 应用场景
- 单独用：一个"知道自己不知道"的小模型，适合边缘设备
- 嫁接投机解码：作为元认知 draft model，精准挡掉低维废话，遇到难题闭嘴交给大模型
- 面试吹牛：改了架构 + 从零训了 + 效果好不好都知道为什么 = 有东西
