> 1. **不是 bug。** 总损失变负是因为正则项 s = log(L)，当任务损失 L < 1 时 s 为负数，负的正则项把总和拽到零以下。各子任务的原始损失（L_CE、L_MSE）全程为正，模型在正常训练。
> 2. **早停不要盯总损失。** 总损失里混杂了可学习参数 s 的动态，不纯粹反映预测质量。应该盯分类任务的原始验证损失 avg_val_type_loss——它才是"模型有多差"的真实度量。
> 3. **梯度归一化。** 均衡点处每个任务的梯度被 1/L 缩放——损失大的任务梯度被压小，损失小的任务梯度被放大，三个任务的梯度贡献被拉到同一量级。

◆ 问题一：总损失变负了，是 bug 吗？

━━━━━━━━━━━━━━━━━━━━

先看现象。从 epoch 4 开始，avg_train_loss = -0.016109，avg_val_loss = -0.013113——总损失变负了。但三个子任务的损失全程为正：分类损失在 0.88~0.98 之间，回归A损失在 0.13~0.14 之间，回归B损失在 0.11~0.16 之间。

总损失是三个子任务损失的加权组合之和，子任务损失全是正的，总损失怎么能是负的？

回忆一下上期的公式。总损失的完整展开：

```
L_total = [exp(-s1) * L_CE + s1]                           # 分类
        + [(1/2) * exp(-s2) * L_MSE_a + (1/2) * s2]        # 回归 A
        + [(1/2) * exp(-s3) * L_MSE_b + (1/2) * s3]        # 回归 B
```

每个任务的贡献由两部分组成：**加权损失项**（exp(-s)*L，永远为正）和**正则项**（s 或 (1/2)*s）。

关键：**s = log(σ²) 可以是任意实数，包括负数。**

上期推过，s 的均衡点是 s = log(L)。当任务损失 L 小于 1 时，log(L) < 0，s 就是负数。

看他的数据：

```
# 均衡点近似（s ≈ log(L)）：
分类任务:  L_CE ≈ 0.88  →  s1 ≈ log(0.88) ≈ -0.128
回归A: L_MSE ≈ 0.135 →  s2 ≈ log(0.135) ≈ -2.00
回归B: L_MSE ≈ 0.158 →  s3 ≈ log(0.158) ≈ -1.845
```

三个 s 全是负数。

**正则项总和**：

```
分类：    s1 ≈ -0.128
回归A：  (1/2) * s2 ≈ (1/2) * (-2.00) = -1.00
回归B：  (1/2) * s3 ≈ (1/2) * (-1.845) = -0.923
```

**正则项总和** = s1 + (1/2)*s2 + (1/2)*s3 ≈ -0.128 + (-1.00) + (-0.923) = **-2.051**

**加权损失项总和**：上期推过，均衡点处 exp(-s)*L = 1（分类和回归都一样，上期对回归求导时 (1/2) 在等式两边约掉了）。所以每个任务的 exp(-s)*L 都等于 1，但回归任务的加权损失项前面还有公式自带的 (1/2) 系数：

```
分类：    exp(-s1)*L_CE = 1
回归A：  (1/2)*exp(-s2)*L_MSE_a = (1/2)*1 = 0.5（公式自带的 1/2）
回归B：  (1/2)*exp(-s3)*L_MSE_b = (1/2)*1 = 0.5
```

**加权损失项总和** = 1 + 0.5 + 0.5 = **2.0**

**总损失** = 加权损失项总和 + 正则项总和 ≈ 2.0 + (-2.051) = **-0.051**

和日志里 epoch 6 的 avg_val_loss = -0.042875 量级吻合。

💡 **人话翻译：** 当任务损失 L < 1 时，s = log(L) < 0，正则项变成负数。正则项的负值大过了加权损失项的正值，总损失就成了负数。**不是 bug，是数学上的必然。**

────────────────────

**那总损失为负，到底是什么意思？**

先说清楚一件事：**这里的"损失"已经不是传统意义上的"模型预测有多差"了。**

传统损失（交叉熵、MSE）衡量的是"预测值和真实值差多远"，越小越好，下限是 0——不会是负数。但同调不确定性加权后的 L_total 不是这个东西。它是由"预测损失"和"正则项 s"拼出来的联合优化目标——一个数学构造，目的是让 s 和模型权重 θ 同时被优化。正则项 s 可以是负数，所以总和也可以是负数。

💡 **人话翻译：** L_total 不再是"模型有多差"的温度计——它变成了一个包含辅助变量的代理函数。负值只是说明辅助变量 s 的贡献把总和拽到零以下了，跟"模型预测质量"没有直接关系。真正衡量"模型有多差"的，是各子任务的原始损失（L_CE、L_MSE）——它们全程为正，全程在下降。

────────────────────

**那总损失为负，会不会影响训练？**

不会。梯度下降不关心损失的绝对值是正还是负——它关心的是**梯度方向**。损失从 -0.04 降到 -0.05，梯度的方向和从 1.04 降到 1.03 完全一样。优化器看的是"朝哪个方向走能让损失更小"，不关心损失本身是正是负。

你也可以这样理解：给总损失加一个常数 C（比如 +10），`L_total + C` 和 `L_total` 的梯度完全相同——常数的导数是零。损失函数的"零点"是可以随便选的，正负没有物理意义。

**结论：完全正常。负的总损失只说明正则项 s 的贡献超过了加权后的原始损失。梯度依然正确，训练依然有效。真正有意义的，是各子任务自己的损失值——它们才是"模型有多差"的真实度量。**

◆ 问题二：早停应该锚定哪个指标？

━━━━━━━━━━━━━━━━━━━━

网友的早停条件是：avg_val_loss（总验证损失）连续 2 轮下降阈值小于 1e-2。最佳检查点在第 6 轮，第 8 轮触发早停。

但问题来了——总损失被正则项 s 搅浑了，**它反映的不再是"模型预测质量"，而是"模型预测质量 + 可学习参数 s 的动态"。**

看日志：

```
avg_val_loss:      epoch 6 = -0.042875 → epoch 7 = -0.046221 → epoch 8 = -0.048622
                   差值：         -0.003346             -0.002401
                   （都在下降，但幅度很小，触发早停）

avg_val_type_loss: epoch 6 = 0.883855  → epoch 7 = 0.882970  → epoch 8 = 0.879770
                   差值：         -0.000885             -0.003200
                   （还在缓步下降，没有止住）
```

总损失说"已经不怎么动了，该停了"。但分类任务（业务最优先）的验证损失还在往下走——0.883855 → 0.882970 → 0.879770，三轮降了 0.004。

**总损失的变化里混杂了 s 的变化。** 当 s 还在调整时，总损失的变化并不纯粹反映模型预测能力的改善。s 的调整和模型权重的调整是两套动力学，但它们被混在同一个数字里了。

────────────────────

**锚点应该是什么？**

既然分类任务有**绝对业务优先级**，早停应该盯的是 **avg_val_type_loss**——分类任务在验证集上的原始交叉熵损失，不含 s，不含正则项。

为什么用"原始损失"而不是"加权后的损失"？因为加权后的损失 exp(-s)*L_CE 在均衡点处恒等于 1（上期推过的），它几乎不变——你用它做早停，什么也看不出来。只有**不带 s 的原始损失 L_CE** 才真实反映模型在这个任务上的预测质量。

```
早停条件：连续 patience 轮 val_type_loss 下降幅度 < threshold 时停止
```

────────────────────

**如果多个任务都重要呢？**

那就用子任务验证损失的加权组合——**不带 s 的原始损失**按业务权重加起来：

```
val_metric = w1 * val_分类损失 + w2 * val_回归A损失 + w3 * val_回归B损失
# w1, w2, w3 是业务权重，反映你对各任务的重视程度
# 注意：这里的权重是人工设定的业务权重，不是训练中 s 控制的那个权重
```

关键区别：训练时用同调不确定性加权（让 s 自动平衡梯度），早停时用人工业务权重（反映你真正在乎的东西）。训练和评估用的指标可以不一样，也应该不一样。

────────────────────

**补充建议：启用策略 3 的 α**

上期最后给了处方：如果分类任务有绝对优先级，应该把分类任务的正则项系数 α 调大——换一根更硬的弹簧，让分类任务在总损失中占更大的份额。

网友目前还没启用 α（等于 α=1，默认值）。从日志看，分类损失从 0.978 降到 0.880，降了 10%；回归A损失从 0.142 降到 0.135，回归B损失从 0.107 降到 0.158（甚至略微上升过）。如果分类是绝对优先，可以把分类的 α 调到 2 或 3，让模型把更多"注意力"给分类任务，换取分类精度的进一步提升——代价是两个回归任务可能稍微变差。

这就是上期那根"弹簧"的用途：你已经知道了弹簧的原理，现在该调弹簧的刚度了。

◆ 问题三：同调不确定性加权到底怎么协调梯度的？

━━━━━━━━━━━━━━━━━━━━

这是最深的一个问题。前面两个是"现象怎么解释"，这个是"机理是什么"。

先看最朴素的做法——直接把三个损失加起来：

```
L_total = L1 + L2 + L3
```

反向传播时，三个任务的梯度会汇合到共享参数 θ 上（θ 是三个任务共用的底层网络参数，不包括各输出头自己的参数——输出头只收自己任务的梯度，不存在任务间竞争）。三个梯度直接叠加，看起来很公平？不公平。

损失的量级差异是梯度差异的最主要驱动力。看网友的数据（取 epoch 6 的验证损失）：

```
分类损失     ≈ 0.884    （交叉熵）
回归A损失    ≈ 0.135    （MSE）
回归B损失    ≈ 0.159    （MSE）
```

分类损失是回归损失的 **6~7 倍**。梯度范数的差异不会精确等于 6~7 倍——因为梯度要沿着链式法则乘一整条链回来，三个输出头的结构和参数都不一样，最终到达 θ 的梯度大小取决于整条链，不只取决于损失值。但损失的量级差异是梯度差异的最主要驱动力：一个任务的损失比另一个大一个数量级，它对 θ 的梯度贡献几乎一定大得多。直接相加时，损失大的任务主导 θ 的更新方向，损失小的任务信号被淹没。

────────────────────

**同调不确定性加权怎么改变这件事？**

反向传播时，每个任务的梯度在流回 θ 之前会被 exp(-s_i) 缩放（正则项 s 不含 θ，对 θ 的梯度贡献为零——正则项只管 s 自己的更新）。

上期推过，均衡点处 exp(-s_i) ≈ 1/L_i。所以每个任务的梯度被自己的损失值反除了——**损失大的缩放系数小，损失小的缩放系数大。** 用网友的数据算：

```
分类:  exp(-s1) ≈ 1/0.884 ≈ 1.13  → 梯度被放大 1.13 倍
回归A: exp(-s2) ≈ 1/0.135 ≈ 7.41  → 梯度被放大 7.41 倍
回归B: exp(-s3) ≈ 1/0.159 ≈ 6.29  → 梯度被放大 6.29 倍
```

分类的损失大（0.884），但缩放系数小（1.13）——大损失配小权重。回归的损失小（0.135），但缩放系数大（7.41）——小损失配大权重。

最终效果：

```
# 不加权时，梯度范数比例大致为：
分类 : 回归A : 回归B ≈ 0.884 : 0.135 : 0.159 ≈ 6.5 : 1 : 1.2

# 加权后，梯度范数比例大致为：
分类 : 回归A : 回归B ≈ (0.884 × 1.13) : (0.135 × 7.41) : (0.159 × 6.29)
                           ≈ 1.0 : 1.0 : 1.0
```

**三个任务的梯度贡献被拉到同一量级。** 这就是"自动梯度归一化"——不是通过显式计算梯度范数（那是 GradNorm 的做法），而是通过 exp(-s) 的自适应缩放间接实现的。

💡 **人话翻译：** 三个人在同一条绳子上拉（共享参数），力气大的（损失大的任务）被戴上阻力套，力气小的（损失小的任务）被装上助力器。最终三个人对绳子的拉力差不多大。

注意：不加权时，θ 天然"偏爱"损失大的任务——不是因为那个任务更重要，纯粹是量纲差异造成的意外偏袒。加了同调不确定性后，exp(-s) ≈ 1/L 把这种偏袒抹平了，三个任务公平竞争。**如果你真的想让某个任务优先，不能靠损失的量纲差异——得用上期讲的策略 3，调正则项系数 α。**

────────────────────

**s 大的任务少说话，s 小的任务多说话**

再换一个角度理解。s 代表"不确定性"——s 越大，模型对这个任务越不确定。

```
s 大 → exp(-s) 小 → 梯度被压小 → 不确定的任务"少说话"
s 小 → exp(-s) 大 → 梯度被放大 → 确定的任务"多说话"
```

这很合理：你不确定的事情，不应该大声嚷嚷着要改方向；你确定的事情，应该坚定地说"往这边走"。

从网友的日志看，随着训练推进，三个任务的损失都在下降，s 跟着下降（s = log(L)，L 降则 s 降）。但均衡点处 exp(-s) = 1/L 始终成立——不管训练到哪一轮，每个任务的梯度都被 1/L 缩放，三个任务的梯度比例始终维持在 1:1:1 附近。**这个平衡不是静态的，是动态跟踪的：损失变了，s 跟着变，权重也跟着变，但比例不变。**

◆ 训练日志诊断：从数据里看到了什么

━━━━━━━━━━━━━━━━━━━━

回到网友的训练日志，从梯度范数、损失下降、泛化差距三个方面看看实际效果：

**1. 梯度范数：自动归一化**

上面已经分析了。不加权时，分类任务的梯度主导共享网络；加权后，三个任务的梯度贡献被拉平。从日志看，两个回归任务的损失确实在稳步下降（回归A从 0.142 降到 0.135，回归B从 0.107 升到 0.158 后趋于稳定），说明它们的梯度信号没有被分类任务淹没。

**2. 损失下降：节奏同步**

看八轮训练的下降趋势：

```
分类损失: 0.978 → 0.880  下降 10.0%
回归A损失: 0.142 → 0.135  下降 4.9%
回归B损失: 0.107 → 0.158  先升后稳（可能有特殊原因，比如任务本身较难，或数据分布问题）
```

分类和回归都在下降，没有出现"一个任务暴降、另一个任务纹丝不动"的情况。这就是梯度归一化的效果——每个任务都能分到足够的梯度信号来更新共享参数。

**3. 过拟合信号**

从验证损失看，分类任务从 epoch 5 开始下降明显变缓（0.911 → 0.884 → 0.883 → 0.880，三轮只降了 0.03），而总训练损失仍在快速下降（-0.050 → -0.082 → -0.112 → -0.141）。训练损失持续降但验证损失趋于平缓——经典的过拟合信号。分类任务的模型开始"记住"训练数据，而不是学到泛化特征。

两个回归任务的验证损失则相对稳定（回归A在 0.135~0.136 区间，回归B在 0.158~0.160 区间），没有明显的过拟合迹象。

如果分类是绝对优先，可以考虑两手一起上：一方面启用 α 给分类任务更大的梯度份额，让共享网络更多地服务分类任务的需求；另一方面用正则化手段（dropout、数据增强等）处理过拟合问题——这两个是不同层面的事，不能指望调 α 解决过拟合。

━━━━━━━━━━━━━━━━━━━━

◆ 总结：弹簧在真实训练中的动态行为

━━━━━━━━━━━━━━━━━━━━

上期用弹簧比喻讲了同调不确定性加权的静态原理：弹簧的刚度（正则项系数）决定均衡点，外力（损失权重）会被弹性抵消。

这一篇看到的是**弹簧在真实训练中的动态行为**——

- **弹簧可以压过零点**：s 可以是负数，正则项可以是负数，总损失可以是负数。这不是故障，是弹簧的行程范围比你以为的大。
- **弹簧的振动不适合当指南针**：总损失里混杂了弹簧（s）的动态和模型预测能力的动态，两者纠缠在一起。早停要盯的是模型的预测能力（子任务原始损失），不是弹簧系统的总能量（总损失）。
- **弹簧的真正作用是梯度归一化**：exp(-s) 把三个量级不同的任务梯度拉到同一水平，让每个任务都能在共享参数上"说上话"。不确定的少说，确定的多说。

三个问题，一个比一个深。第一个是现象（负值），第二个是工程决策（早停），第三个是机理（梯度协调）。串起来就是：**同调不确定性加权不是一个简单的"自动权重"工具——它是一个有内部动力学的系统，用起来要理解它的行为，而不只是理解它的公式。**

━━━━━━━━━━━━━━━━━━━━

技术名词速查（本篇新增）：

- **早停（Early Stopping）**：训练过程中监控验证集指标，当指标不再改善时停止训练，防止过拟合。是最简单也最常用的正则化手段
- **梯度范数（Gradient Norm）**：梯度向量的长度（L2 范数），反映参数更新的"力度"。梯度范数大的任务对共享参数的影响大
- **泛化差距（Generalization Gap）**：训练集表现与验证集表现的差异。差距越大，过拟合越严重
- **梯度归一化（Gradient Normalization）**：把不同任务的梯度缩放到相近的量级，防止某个任务的梯度主导共享参数的更新
