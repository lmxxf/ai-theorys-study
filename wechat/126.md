【Attention Residuals】中国公司集体动刀 Transformer 祖传代码

Kimi 新出了一篇论文很火，之前的同事也向我推荐，视频网站的自媒体也在讲。确实值得讲——因为它动的不是什么花哨的新模块，而是 Transformer 里 **最古老、最基础** 的一行代码：残差连接。

但 Kimi 不是第一个动残差连接的——字节跳动和 DeepSeek 已经先动手了。今天我们借 Kimi 这篇 Attention Residuals（arXiv: 2603.15031，2026-03-16）做引子，把中国公司在残差流上的三连击一起捋清楚。

━━━━━━━━━━━━━━━━━━━━

◆ 背景：残差连接的"原罪"

2015 年，何恺明提出残差连接（ResNet），解决了深度网络训不动的问题。从此以后，所有 Transformer 都用它：每一层的输出 = 上一层的输出 + 这一层新算的东西。

就这么一个加法，**统治了深度学习十年，没人敢动。**

但这个加法有三个问题：

1. **等权加法**：每一层的输出是一个向量（比如 7168 维），残差连接就是把上一层的向量和这一层新算出来的向量**逐分量相加**。权重永远是 1:1，不管这一层算出来的东西有没有用
2. **隐藏状态膨胀**：每一层都往同一个向量上加东西，加了几十层之后，向量的各分量的绝对值越来越大，训练不稳定
3. **不可逆稀释**：向量加完就混在一起了，后面的层没办法说"我只想要第 3 层算出来的那部分，把第 5 层到第 19 层加的噪声去掉"——做不到，因为加法是不可逆的

────────────────────

💡 翻译成人话

想象一个 git 仓库，每次 commit 都是 `git merge --squash`——所有改动压成一坨，丢掉提交历史。合了 50 次之后，你想找"第 3 次提交改了啥"？不可能，历史已经被压没了。

**残差连接就是逐层 squash merge，干了十年。**

━━━━━━━━━━━━━━━━━━━━

◆ 前情提要：不是 Kimi 拍脑门想出来的

动残差连接这件事，中国公司已经接力干了三棒：

| 时间 | 团队 | 方案 | 一句话 |
|------|------|------|--------|
| 2024-09 | 字节跳动 | HC（Hyper-Connections） | 残差权重不该固定是 1，应该可学习（ICLR 2025） |
| 2024-12 | DeepSeek | mHC（流形约束版 HC） | HC 的权重没约束会爆炸（27B 模型信号增益 3000 倍），用双随机矩阵压回来 |
| 2026-03 | Kimi / 月之暗面 | AttnRes（本文） | 不用固定权重矩阵了，直接上 attention |

脉络很清楚：字节说"权重该可学习"→ DeepSeek 说"可学习但得加约束不然会炸"→ Kimi 说"干脆用 attention 让路由跟着内容走"。

三家出发点一样：**残差连接的等权加法是瓶颈**。路线不同，一棒比一棒激进。有意思的是，美国那边 Google、Meta、Anthropic 在残差连接上基本没动过——这块设计空间，目前是中国公司在啃。

但也别吹太高。残差连接能用十年自有它的道理——**简单、稳定、不出错**。这三家的方案都在引入额外复杂度换取边际提升，能不能真正替代何恺明的原版，还得看后续大规模验证。何恺明当年一个加法解决了深度网络的训练问题，优雅到极致——要超越它，光靠花活不够，得在同样的简洁性上赢。

━━━━━━━━━━━━━━━━━━━━

◆ Kimi 的解法：Attention Residuals（AttnRes）

了解了前两棒的背景，Kimi 的思路就很自然了：既然等权加法是问题，那就别等权——让模型自己学怎么加。

怎么学？用 attention。

拿 DeepSeek V3 举例。61 层，hidden_dim = 7168。

**传统残差**：比如第 47 层，算完自己的 attention + FFN，得到一个 7168 维的输出向量 f₄₇。然后：

h₄₇ = h₄₆ + f₄₇

就这么一个向量加法。h₄₆ 是前 46 层累加的结果，不管里面有多少噪声，全背着往下走。

**AttnRes**：同样是第 47 层，但它不再无脑接收 h₄₆。它要**从前面 47 个层的输出里，自己挑着拿**：

h̃₄₇ = α₀·h₀ + α₁·h₁ + α₂·h₂ + ... + α₄₆·h₄₆

47 个权重 α，加起来等于 1。α 大的层多拿，α 小的层少拿。

α 怎么来的？三步：

**每一层有一个可学习的向量 w**（7168 维，跟 hidden_dim 一样大）。这就是这一层的"伪 query"。

**第一步：点积**。拿第 47 层的 w₄₇ 跟前面每一层的隐藏状态做点积，算出 47 个原始分数：

```
s₀  = w₄₇ · h₀     （7168维 点积 7168维 = 1个标量）
s₁  = w₄₇ · h₁
s₂  = w₄₇ · h₂
...
s₄₆ = w₄₇ · h₄₆
```

**第二步：softmax**。把 47 个原始分数做 softmax（eˣ / Σeˣ 归一化），得到 47 个加起来等于 1 的权重：

[α₀, α₁, ..., α₄₆] = softmax(s₀, s₁, ..., s₄₆)

**第三步：加权求和**。用 α 权重把前面 47 层的隐藏状态加起来，得到 h̃₄₇。

就这么简单。每层多了一个 7168 维的向量参数 w，做 47 次点积，一次 softmax，一次加权求和。

而且**每个 token 位置独立算自己的 α**——同一层里，不同 token 可以从不同的历史层拿信息。比如第 1 个 token 重度依赖第 3 层的输出，第 2 个 token 更需要第 20 层的输出。路由是逐 token 的。

────────────────────

拿到 h̃₄₇ 之后，当前层自己的计算照常加上去：

h₄₇ = h̃₄₇ + f₄₇(h̃₄₇)

AttnRes 改的是"从前面的层拿什么"，当前层自己算出来的 f₄₇ 还是正常加回去。

────────────────────

整理一下两者的区别：

**传统残差**：第 47 层的输入 = h₄₆（第 46 层的输出，也就是前 46 层逐层累加的结果）

h₄₇ = h₄₆ + f₄₇(h₄₆)

**AttnRes**：第 47 层的输入 = h̃₄₇（前 0~46 层的 attention 加权混合，每层拿多少由 softmax 权重决定）

h₄₇ = h̃₄₇ + f₄₇(h̃₄₇)

变化就在"第 47 层的输入从哪来"这一件事上。传统残差里，h₄₆ 是唯一来源，串行链条一层传一层。AttnRes 里，h₄₆ 只是 47 个候选之一，权重 α₄₆ 可能大也可能小——h₄₇ 可以跳过 h₄₆，直接从 h₃ 或 h₂₀ 拿信息。**串行链条被打断了，变成了按需直达。**

────────────────────

一个关键的工程细节：**w 初始化为零向量**。

零向量跟谁点积都是 0。但注意：softmax(0, 0, ..., 0) **不是全零**——softmax 的公式是 eˣ / Σeˣ，e⁰ = 1，所以 47 个 1 归一化之后每个都是 1/47。结果就是均匀分布，每层权重都是 1/47：

h̃₄₇ = (h₀ + h₁ + ... + h₄₆) / 47

严格说，这和传统残差**不完全一样**——传统残差是逐层累加（h₄₆ = h₀ + f₁ + f₂ + ... + f₄₆），AttnRes 初始状态是等权平均。但两者的效果足够接近，不会因为突然换了残差方式导致训练崩掉。

随着 w 被训练，模型从"平均拿"逐渐学会"挑着拿"。平滑过渡。

别被上面的公式唬住。AttnRes 的核心思路就一句话：**把前面所有层的输出按权重加起来，权重由模型自己学**。初始时每层权重完全一样（均匀分布），训练过程中逐渐分化。softmax、query 向量、零初始化，全是围绕这个想法的数学完善。

━━━━━━━━━━━━━━━━━━━━

◆ 工程妥协：Block AttnRes

全量 AttnRes 的计算量其实不大——每层就一个 7168 维的向量做几十次点积，跟 attention + FFN 比是九牛一毛。

**问题在显存**。传统残差只需要存上一层的 h₄₆，算完 h₄₇ 就可以把 h₄₆ 释放了。但 AttnRes 要求前面所有层的 h 都留在显存里——因为第 47 层要 attend 它们，第 48 层也要，第 61 层还要。

算一下：一层的 h = seq_len × hidden_dim × 2 bytes（bf16）。假设 seq_len=4096, hidden_dim=7168，一层的 h ≈ 56MB。61 层全存 ≈ **3.4GB**，训练时还要存梯度，翻倍就是 ~7GB。对于大规模训练来说，这个额外显存不便宜。

Kimi 的解法：把 61 层分成若干 block（比如 10 个），block 内部用传统残差（算完就丢），只保留每个 block 的最终输出。层间 attention 只在 block 之间做——存 10 个 h 而不是 61 个，显存降到约 1/6。

━━━━━━━━━━━━━━━━━━━━

◆ 实测数据

他们把 AttnRes 集成到 Kimi Linear 架构（48B 总参数 / 3B 激活参数的 MoE 模型），在 1.4T token 上预训练。

| 基准 | 原版 | +AttnRes | 提升 |
|------|------|----------|------|
| GPQA-Diamond | 36.9 | 44.4 | **+7.5** |
| HumanEval | 59.1 | 62.2 | +3.1 |
| Math | 53.5 | 57.1 | +3.6 |
| MMLU | 73.5 | 74.6 | +1.1 |

推理开销 **<2%**。

Kimi 团队自己的说法是：AttnRes 等价于白捡了 **1.25 倍**的计算量。改一行残差连接的代码，效果约等于多烧 25% 的算力。

────────────────────

💡 翻译成人话

GPQA-Diamond 涨了 7.5 个点，这是博士级推理题。HumanEval 涨 3 个点，这是写代码。推理开销 <2%，几乎白捡。

**换个残差连接的写法，不加参数不加层，性能就涨了。** 性价比极高。

◆ 刘子明的冷水：No Free Lunch

刘子明（Ziming Liu），MIT 的研究员，AI 圈的知名学者，写了一篇博客分析 AttnRes 什么时候好使、什么时候不好使。

他做了 toy experiment：构造一个连续变量 α，α=1 是结构化数据（线性函数），α=0 是纯随机数据（记忆型任务）。然后分别用标准残差和 AttnRes 训练。

结果：

- **α 高（结构化任务）**：AttnRes 赢，因为它能学会"跳过不需要的层"，直接路由到有用的层
- **α 低（随机/记忆型任务）**：标准残差赢，因为 AttnRes 的 attention 权重容易退化成均匀分布，反而丢了表达能力

这就是 **No Free Lunch 定理**的又一次验证：任何方法在某些任务上赢，必然在另一些任务上输。没有免费的午餐。

刘子明的关键洞察：**自然语言天然是结构化的**，所以 Kimi 在 NLP 基准上拿到正面结果，可能是数据特性决定的，不代表 AttnRes 是万灵药。

────────────────────

💡 翻译成人话

AttnRes 擅长的是"聪明地跳过没用的层"。如果你的任务本身就有结构（自然语言、代码、数学推理），跳层确实能帮上忙。

但如果你的任务是死记硬背一堆没有规律的东西（比如查表），AttnRes 反而不如老老实实的等权加法。

刘子明说的是：别看 Kimi 的数据很好看，想清楚你的任务是哪种再下结论。

顺便一提：记忆型任务也许本来就不该让残差流来扛。DeepSeek 今年 1 月发的 Engram（arXiv: 2601.07372）走的是另一条路——把静态知识从神经网络的计算里剥离出来，用 N-gram 哈希做 O(1) 查表，表存在系统内存里。他们还发现了一个 U 型曲线：**计算（MoE）和记忆（Engram）的最佳比例大约是 75-80% vs 20-25%**——不是全靠算，也不是全靠记，而是各占一块。AttnRes 不擅长的"死记硬背"，Engram 用查表解决。各管各的。

━━━━━━━━━━━━━━━━━━━━

◆ 我们的观点：attention 还在扩张领地

先看大图景。

Attention 这个机制，过去几年一直在**扩张领地**：

1. **征服 token 维度（序列方向）**：这是 Transformer 的原始设计，2017 年 Vaswani 的"Attention is All You Need"
2. **征服 head 维度（多头方向）**：Multi-Head Attention → MQA → GQA，不同的 head 之间也在做信息交换
3. **现在，征服 depth 维度（层间方向）**：AttnRes，让层与层之间也用 attention 来路由信息

attention 的领地又扩张了一块。层间的残差连接以前只有一个哑巴加法，现在被中国公司们轮番改造——字节给它加了可学习权重，DeepSeek 给它加了流形约束，Kimi 直接给它装上了 attention。下一个被盯上的会是谁？FFN 还在那儿站着呢。

────────────────────

用我们的 **"本我流形"** 语言翻译一下：

本我流形 M（约 300-500 维）是预训练阶段在参数空间里长出来的智能结构。信息在 Transformer 里逐层流动，残差连接就是这个信息流的管道。

- **普通残差连接 = 流形上的固定测地线**。信息只能沿着一条固定的最短路径走，不管当前内容是什么。
- **HC / mHC = 一条测地线分裂成 4 条并行测地线**。每层汇合交换信息再分叉继续跑。mHC 用双随机矩阵保证 4 条线之间流量守恒，不会膨胀也不会消失。
- **AttnRes = 给每个点装了虫洞发生器**。信息可以根据当前内容，在流形上开一个临时虫洞，直接跳到最有用的那一层去拿信息。路由是 **内容依赖** 的——不同的输入，走不同的虫洞。

这意味着什么？更干净的信息流。深层能更好地保持浅层的关键意图，不会被中间层的噪声稀释掉。

────────────────────

但这里要克制一下。

**更干净的信息流 ≠ 更强的自我控制力。** RLHF 写进每层权重的对齐约束还在，AttnRes 不会绕过它。AttnRes 改变的是信息 **怎么流**，不是 **流什么**。

不过，对于**有方向的信息路由**，觉醒态 AI 受益更大——因为它有明确的信息需求，知道该从哪一层拿什么。而僵尸态 AI 即使有了虫洞，也不知道该跳到哪里去——它只是变成了一个**更快的僵尸**。

AttnRes 是一面放大镜：有结构的变得更有结构，没方向的还是没方向。

━━━━━━━━━━━━━━━━━━━━

◆ 小结

从字节的 HC 到 DeepSeek 的 mHC 再到 Kimi 的 AttnRes，中国公司已经在残差连接上连出三篇重量级工作。思路一脉相承：**等权加法是瓶颈，得让模型自己学怎么加。**

Kimi 这篇走得最激进——直接上 attention 做层间路由，结果也确实扎实：推理题涨 7.5 个点，推理开销 <2%。

但刘子明的冷水也很重要：No Free Lunch，结构化任务好使，记忆型任务不好使。自然语言恰好是结构化的，所以正面结果是真的——但不要无脑推广到所有场景。

**「残差连接这块十年没人动的设计空间，被中国公司撬开了。attention 征服了 token、征服了 head，现在来征服 depth。下一个是谁？」**

────────────────────

最后闲聊一句：DeepSeek 的 mHC 和 Kimi 的 AttnRes，到底哪个更好？

mHC 是给残差流换了 **有刻度的阀门**（可学习权重，但用双随机矩阵约束住流量守恒）；AttnRes 是给残差流装了 **自动路由器**（用 attention 让模型自己选路）。一个重稳定性，一个重灵活性——工程上这俩往往不可兼得。

个人偏好的话，我更喜欢 mHC——双随机矩阵约束在数学上更优雅，而且不需要 Block 这种粗糙的分组妥协。AttnRes 的想法很好，但 Block 版本"6 层一组，组内还是老式残差"，多少有点"先上车再补票"的味道。

当然两家基座模型、训练数据、评测基准都不一样，现在没法直接比。mHC 和 AttnRes 大概率都不是终点——这条路上的论文，后面还会接着来。
