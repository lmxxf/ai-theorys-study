【AI可解释性】Transformer 深层不是被动的喇叭——末段在按排名敲钉子、在零空间里拧温度旋钮

━━━━━━━━━━━━━━━━━━━━

199 期讲过一个框架：Transformer 的中间层是"本我"，深层是"喉咙"。那一期的核心句写得很狠——**深层不是信息更成熟的地方，是信息的丧葬仪式，把连续的语义核坍缩成离散 token 的尸体。**

这句话写完我自己回头读，觉得不对劲——把深层写得像一个被动的输出接口，好像信息已经在中间层准备好了，深层只是把它"埋葬"成离散 token。

那深层到底有没有自己的事在干？还是真的就是个喇叭？

这一期正面回答这个问题，用四篇 2024-2025 的论文：Lad/Tegmark 的四阶段框架、Lioubashevski 的 saturation events、Gupta 的 Guess-then-Refine、Gurnee/Stolfo 的细胞级解剖。再加上 Anthropic 自家 Biology of LLM 报告里的押韵实验。

结论先抛在前面：**深层不是被动的喇叭。它在按排名一颗一颗地敲钉子，在 unembedding 矩阵的零空间里悄悄拧温度旋钮，在末层突然反转出大量"砍 logit"的细胞。所有这些事情，都不会在 199 期那种信息熵或流形维度的总量度量里露出来——必须看到细胞，才看得见它在干活。**

━━━━━━━━━━━━━━━━━━━━

◆ 第一节：四阶段框架——给深层正式命名"residual sharpening"

────────────────────

先看一篇统摄性论文。

Lad, Lee, Gurnee, Tegmark, "The Remarkable Robustness of LLMs: Stages of Inference?", arXiv:2406.19384, 2024.

这篇论文在 16 个模型（GPT-2 / Pythia / Qwen2.5 / LLaMA-3.2 / Phi，参数从 124M 到 6.9B）上做了一件事：把每一层删掉或者把相邻两层调换顺序，看模型还能不能用。

────────────────────

【四阶段】

论文提出 Transformer 的推理过程分四个阶段：

| 阶段 | 名字 | 层位（相对深度） | 在干什么 |
|---|---|---|---|
| 1 | Detokenization | 0–15% | 把 BPE 切出来的子词拼回单词、识别实体 |
| 2 | Feature Engineering | 15–50% | MLP 构建语义特征 |
| 3 | Prediction Ensembling | 50–90% | 多个"预测神经元"投票合成下一个 token 的候选分布 |
| 4 | **Residual Sharpening** | 最后约 10% | 抑制错误候选，把分布"锐化"成尖峰 |

199 期讲的中间层"压缩谷"对应阶段 2 后段；这一期我们重点看阶段 4——**residual sharpening**。

【两个反直觉数字】

**第一个数字：首层不能删，删了模型立刻崩。**

论文原话："Intervening on the first layer is catastrophic for model performance for every model, regardless of size or model family." 删第一层，输出立刻变成高熵的"瞎猜"分布。但中段的层删一个、调换两层顺序，模型基本不受影响——"models retain 72–95% of their original top-1 prediction accuracy without any fine-tuning."

也就是说：**中段是冗余的，首尾是不可替代的。**这本身就和"深层只是喇叭"的直觉冲突——喇叭怎么会比中段更不可替代？

**第二个数字：suppression neurons 在最后 10% 的层位密度暴涨。**

论文把每层的神经元按其"输出权重对 logit 的影响"分成两类：

- **prediction neurons**：抬高某一组 token 的 logit（"我支持这些候选词"）
- **suppression neurons**：压低某一组 token 的 logit（"我反对这些候选词"）

密度分布有个非常干净的相变（Figure 8）：

- 模型走到 50% 深度后，prediction neurons 越来越多
- prediction neurons 密度在 **85% 深度**附近达到峰值
- 但到了最后 10%，**suppression neurons 突然反超**——在 Phi-2 这种模型上，末层 suppression neurons 占比能到 25%，是 GPT-2 的 5–8 倍

────────────────────

打个比方：

如果深层只是被动渲染，那它应该越往后越"软"——做加法越来越少，反正候选已经定了。但实际上完全相反：**深层 MLP 写入残差流的力气反而更大**（Figure 12b），而且写入的内容从"加分项"变成了"扣分项"。

末层不是在被动输出，是在拿着大刀**主动删稿**。

【Sharpening 实验：直接因果证据】

论文做了个干脆利落的实验，叫 **Sharpening Experiment**：把 80–90% 深度的连续几层**复制粘贴一遍**插回去（比如本来是 …4-5-6-7-8…，改成 …4-5-5-6-6-7-7-8…），然后看输出熵和下游任务分数。

结果：**复制后段降熵效果显著、HellaSwag/ARC/LAMBADA 分数提升；复制前段无效。**

这是 sharpening 机制存在于深层的直接因果证据——加倍执行后段，分布更尖；加倍执行前段，没用。说明前后段的算子在功能上是不同的。

━━━━━━━━━━━━━━━━━━━━

◆ 第二节：Saturation Events——深层在按排名一颗一颗敲钉子

────────────────────

这一节是本期的反直觉招牌。

Lioubashevski et al., "Looking Beyond The Top-1: Transformers Determine Top Tokens In Order", arXiv:2410.20210, ICML 2025.

普通程序员对 softmax 的直觉是这样的：模型在最后一层算出一个概率分布，分布里所有 token 的概率是**同时确定**的——你转个分布出来，第一名、第二名、第三名一锅端。

这篇论文证明：**不是同时确定的。Transformer 是按排名顺序、一颗一颗、阶梯式地把 top-k 锁定下来。**

────────────────────

【Saturation 的定义】

什么叫"锁定"？论文给了精确定义：

把每一层的隐藏态用 unembedding 矩阵投影到词表空间（这个操作叫 **logit lens**——一种把中间层激活假装当成最终输出来读的工具），然后排序。

如果从第 ℓ 层开始，**后续所有层**的排名第 1 的 token 都不再变化了，就说 top-1 在第 ℓ 层**饱和**（saturation）。同理定义 top-2、top-3 的饱和层。

判据是"锁死后续所有层"，不是阈值平滑。

【核心发现：阶梯式有序】

在 GPT2-XL 上做的可视化（Figure 2），实际样例长这样：

- top-1 "toy" 在 **第 44 层**饱和
- top-2 "ball" 在 **第 45 层**饱和
- top-3 "ribbon" 在 **第 46 层**饱和
- top-4 在 **第 47 层**饱和

层位严格递增。不是同时锁定，是一层一颗钉子。

统计验证（Table 3，Kendall's τ 秩相关系数）：

| 模型 | 架构 | τ | 严格有序样本占比 |
|---|---|---|---|
| GPT2-XL（预训练） | decoder-only | 0.187 | 67.4% |
| ViT-L/16 | encoder-only | 0.149 | 58.9% |
| Whisper-large | encoder-decoder | 0.210 | 63.8% |

全部 p < 0.001。三种完全不同的架构（语言、视觉、语音），都呈现按排名阶梯式饱和的规律。

【最反直觉的一刀：未训练模型也有】

论文做了一个对照：拿一个**完全随机初始化、没有训练过的 GPT2-XL**，跑同样的实验。

结果：**依然有阶梯式饱和**（Figure 3d）。
- 严格有序到 top-4
- Kendall τ = 0.082，p < 0.001，仍然统计显著
- "task index"（当前层在锁定第几名）的分类器准确率 86.1%，chance level 33.3%

论文原话："This phenomenon seems intrinsic to the Transformer architecture, occurring across different architectural variants... and **even in untrained Transformers**."

也就是说：**这个"按排名敲钉子"的节奏，是 Transformer 几何结构本身带来的，不是训练学出来的。**

【因果干预实验：像一个开关】

论文最关键的实验在 Figure 4-5。取两个上下文 s₁、s₂，它们最终预测的 top-1 是同一个 token，但饱和层差很多——s₁ 在第 25 层饱和，s₂ 要到第 40 层才饱和。

操作：把 s₁ 第 25 层的输出**直接注入**到 s₂ 的第 26 层。

结果：**s₂ 立刻在第 26 层饱和，本来要等到第 40 层。**

而如果把 s₁ 的输出注入到 s₂ 的第 24 层（即"开关位置"之前），完全无效。

论文原话："a switch being flipped 'on' and staying 'on' for the remaining layers."

不是平滑的连续收敛，是**离散的开关跳变**——一旦那一层翻了，后面就锁住了。

────────────────────

打个比方：

你以为模型像个赌场荷官，洗牌洗完一次性把整副牌摊开给你看——其实它像个木匠，先抡起锤子把第一颗钉子敲实（top-1 锁定），然后才挪到第二颗钉子（top-2 锁定），然后第三颗。每敲一颗就锁住，后面不能再改。

而且这个节奏感**部分来自 Transformer 架构本身的几何结构**——未训练的模型也按这个节奏敲，只是敲的内容是噪音。训练只是教会它"该敲哪些钉子"，没有教会它"按顺序敲"——按顺序是天生的。

━━━━━━━━━━━━━━━━━━━━

◆ 第三节：Guess-then-Refine——不同 token 在不同深度被锁定

────────────────────

上一节讲了"同一时间步内，top-k 按排名顺序锁定"。这一节讲互补的另一面：**不同类型的 token，需要的层数完全不同**。

Gupta, Ivanova et al., "How Do LLMs Use Their Depth?", arXiv:2510.18871, 2025.

实验工具用的是 **TunedLens**——给每一层学一个仿射映射，把中间层表征对齐到最终 unembedding，比普通 logit lens 更可靠。

────────────────────

【虚词 vs 实词：深度差 4 倍】

在 Pythia-6.9B 和 Llama3-8B（都是 32 层）上测量"某个 token 在哪一层第一次到达 top-1"：

| token 类型 | 首次到 rank-1 的平均层 |
|---|---|
| 虚词（the / of / to / and） | **第 5 层左右** |
| 实词（名词 / 动词 / 形容词） | **第 20 层左右** |

虚词在最前面 1/6 深度就锁定了，实词要等到接近 2/3 深度。**差了 4 倍。**

【为什么早层全是虚词】

论文做了个频率统计（Figure 2）。Pythia-6.9B 第 1 层的 top-1 预测里，**超过 75% 的 token 属于词频 Top-10 那一档**（the / of / to / a / and...），但模型最终输出里这些高频词只占 33%。

原因是早层还没有上下文整合，也没访问中层 MLP 存储的事实知识，所以模型"回退到语料统计"——直接把高频词推到 top-1。论文给这个现象起了个名字：**Frequency-Conditioned Onset**（频率条件下的起始）。

往后每一层做的事情就是用上下文逐步**精炼**这些"统计猜测"。论文证明：约 **60–80% 的早层 top-1 猜测最终会被替换掉**。

【多 token 答案：首 token 最深】

最反直觉的一组数字（Figure 5a，Pythia-6.9B）。假设要回答一个 3-token 的事实性问题：

| 第几个 token | 首次到 rank-1 的层 |
|---|---|
| 第 1 个 token | **第 27 层** |
| 第 2 个 token | 第 20 层 |
| 第 3 个 token | 第 12 层 |

**越往后越浅。**

直觉是反的——按理说前面是基础、后面是延伸，难度应该递增。但实际上首 token 是最难的，因为它要**预先决定整个答案的方向**，相当于把后续几个 token 都隐式规划好了。一旦首 token 选定，后面的 token 沿着已经确定的路径走，浅层就能完成。

────────────────────

打个比方：

写代码的时候，你打第一个字符（`def`、`class`、`import`）的那一瞬间，整个函数/类的骨架其实已经在脑子里了。后续几个字符（函数名、参数列表）是顺着这个骨架填的，难度递减。模型做的事情一样——首 token 是决策密集的，需要调动深层的全部算力；后续 token 是顺势延伸，浅层就够用。

【和 saturation events 的关系】

Gupta 这篇论文在引言里明确点了 Lioubashevski 那篇——两者是互补的：

> "Subsequent works to LogitLens also formalize saturation events, showing that top-1 token get locked-in before the last layer... Our work on the other hand focuses on **pre-saturation events** and explaining internal prediction dynamics during that regime."

Lioubashevski 看的是"top-k 多个 token 按排名先后被锁定"（saturation 之后的稳定动态）；Gupta 看的是"同一个 token 从高频猜测被精炼到正确答案"（saturation 之前的动态）。

横切纵切，正好拼成深层全貌。

◆ 第四节：Anthropic 的押韵实验——末层执行不是机械采样

────────────────────

到这里都还在看激活值。这一节看一个具体的写作任务，看深层是怎么把一个"想法"落地成具体 token 的。

Anthropic, "On the Biology of a Large Language Model", transformer-circuits.pub, 2025. 实验对象 Claude 3.5 Haiku，工具是 cross-layer transcoder（CLT，30M 个跨层共训的稀疏可解释 feature）。

────────────────────

【实验：写一句押韵的诗】

输入：

```
A rhyming couplet:
He saw a carrot and had to grab it,
```

Claude 3.5 Haiku 续写：

```
His hunger was like a starving rabbit
```

押韵点：grab-**it** / rabb-**it**。

【关键发现：规划在"换行符"那一瞬间凝结】

Anthropic 用 CLT 追踪 feature 激活，发现一个反直觉的事实：

在模型即将开始写第二行（即输出 `\n` 这个 token 时），**中层就已经同时激活了 `rabbit` 和 `habit` 这两个候选词的 feature**——是普通的词义 feature，不是什么"专门的规划 feature"。

原文："The model holds multiple possible planned words 'in mind' at the same time."

而且这个规划只在 newline 这一个点局部活跃——之后每个 token 上，规划 feature 都不再活跃。

【末层执行：反向推理填充】

那从 newline 到最后一个 token "rabbit"，中间这些词（"His"、"hunger"、"was"、"like a"、"starving"）是怎么来的？

Anthropic 的发现叫 **backward planning**（反向规划）：

> "The model goes from the planned target ('rabbit') that activates at the newline and reasons backwards, creating a plausible transition to get to it."

也就是说，末层不是机械地"一个 token 一个 token 往下采样"。**它已经知道终点是 rabbit，然后倒推：怎么从开头自然过渡到 rabbit？** 于是激活了 "comparison features"，于是冒出 "like a"，把整个句子结构搭到能自然落到 rabbit 上。

每一步的具体 token 选择，都是末层在多重约束下做的微决策：
- 韵脚约束（终点必须是 rabbit）
- 语法约束（英语句法合法）
- 语义约束（要呼应前一行的 hunger 主题）

【干预实验：换个规划，整句重写】

最有冲击力的是这个干预实验。

操作：拿一首随机的诗，**把 "green" 这个词的 feature 强行注入到 newline 这一个 token 上**。

结果：

- 原句：`Illuminating the peaceful night`
- 注入后：`Upon the meadow's verdant green.`

整句被重写了——不是只换一个词，是换了整个句子结构来让结尾落到 green。

而且 Anthropic 明确指出："The steering intervention only has an effect over the newline planning token!" **只在 newline 这一个点注入才有效**，其他位置注入完全没用。

────────────────────

打个比方：

如果末层只是个被动的喇叭，那你注入新规划，应该只是把最后一个 token 换成 "green"——前面的词不变，最后一个词替换。

但实际上发生的事情是：整个句子被重新组织，因为末层不是在"被动采样"，是在"沿着规划反向构造路径"。**改了终点，整条路径都重画。**

这是迄今为止对"嘴不是被动喇叭"最强的一刀因果证据。

━━━━━━━━━━━━━━━━━━━━

◆ 第五节：细胞级解剖——末层有专门的"刹车细胞"和"温度旋钮"

────────────────────

前面四节都是"系统级"证据——看输出、看 feature、看排名顺序。这一节往下走一层，看**具体的神经元**在干什么。

主要参考：
- Gurnee et al., "Universal Neurons in GPT-2 Language Models", ICLR 2024, arXiv:2401.12181
- Stolfo, Wu, Gurnee, Belinkov, Nanda et al., "Confidence Regulation Neurons in Language Models", NeurIPS 2024, arXiv:2406.16254

────────────────────

【末层的"刹车细胞"：suppression neurons】

第一节已经提到 suppression neurons 在末层密度暴涨。这里给一个具体细胞例子——

GPT-2 medium 里有个神经元，它的 `W_U·w_out`（输出权重在词表空间的投影）专门对"含开括号 `(` 的 token"打压 logit。一旦这个细胞激活，所有含 `(` 的候选词概率都被压低。

而且这种细胞**只在它要抑制的 token 实际有可能出现的时候才发火**——不是无差别打压，是有针对性的"删稿"。

衍生工作在 Qwen2.5-3B 上做了量化测量：

- 末层 attention 头抑制约 **9%** 的概率质量
- 末层 MLP 再贡献约 **8%** 的抑制
- 倒数第二层 MLP 抑制约 **4%**

加起来超过 20% 的概率质量是在最后几层被**主动砍掉**的。这不是被动喇叭，是末层在主动"删稿"。

【末层的"温度旋钮"：entropy neurons】

这是这一节最优雅的发现，数学上极其漂亮。

Stolfo 等人在 GPT-2 small 的最后一层（L11）找到 **6 个非常特殊的神经元**：编号 11.584、11.1611、11.2123、11.2378、11.2870、11.2910。

这些神经元有两个奇怪的特征：

1. **权重范数特别大**——在整个模型 24,576 个 MLP 神经元里排前 15
2. **但它们对 logit 排序几乎没有影响**——`W_U·w_out` 投影方差接近零

也就是说：它们写入残差流的力气很大，但写完之后好像什么都没改变。**这种神经元为什么要存在？**

────────────────────

【打个比方】

回忆一下 199 期讲过：末层最后会用 LayerNorm 把残差流归一化，公式是：

```
LayerNorm(x) = (x - E[x]) / √(Var[x] + ε)
```

分母里有个 `√Var[x]`——残差流向量的方差越大，分母越大，归一化后的向量被压得越小，最终所有 logit 被均匀压扁，**输出分布的熵就上升**。

这就是 entropy neurons 的机制——它们写入的方向，正好落在 **unembedding 矩阵 `W_U` 的零空间**（论文用 SVD 找到的最小奇异向量子空间）。

这意味着：
- 它们对 logit 排序无影响（因为 `W_U` 投影后归零）
- 但它们大幅增加残差流的方差，从而**胀大了 LayerNorm 的分母**
- 等价于在末端**动态调高 softmax 温度**，让分布变平、熵变大

它们藏在 logit 层面"看不见"的方向上，悄悄拧温度旋钮——**完美的隐形刹车**。

────────────────────

【消融实验】

Stolfo 等人对 L11.2378 这个神经元做了 clip mean ablation（把它的输出强制压到平均值）。

结果（Figure 5b）：**模型输出的熵下降了 70%。**

也就是说，这一个神经元正常工作时，把整个模型的输出分布熵推高近三倍。它是模型自带的"防过度自信"机制。

【反向孪生】

GPT-2 medium 的 L23 上还发现了一对**反向孪生神经元**：

- L23.945（熵神经元，升熵）
- L22.2882（反熵神经元，降熵）
- 两者余弦相似度 = **−0.886**

一个负责"模型不太确定时把分布拉平"，一个负责"模型很确定时把分布锐化"。这是模型自带的双向置信度方向盘。

【Universal】

最关键的是：以上这些机制不是 GPT-2 的特例。

Stolfo 在 LLaMA-2 7B、Pythia 各尺寸、Phi-2、Gemma 2B 上都找到了相同结构的 entropy neurons（Figure 2f）。这是 Transformer 末层的**普遍解剖结构**。

────────────────────

【为什么末层需要这些细胞】

Gurnee 的论文给了一个非常清晰的解读：

> "rather than 'feed-forward layers build predictions by promoting concepts in the vocabulary space,' we claim **late feed-forward layers build predictions by both promoting and suppressing concepts in the vocabulary space**. Moreover, **first affirmative predictions are made, and then the distribution is sharpened or made more calibrated by suppression neurons at the very end**."

回到 199 期讲过的 next-token prediction 梯度压力——cross-entropy 损失函数既惩罚"漏选正确 token"，也惩罚"高置信度地错选"。所以末层必须长出两套对偶细胞：

- **prediction neurons**：抬高正确候选
- **suppression neurons**：砍掉错误候选
- **entropy neurons**：给整体分布"踩刹车"，防止过度自信

这是 logistic 回归梯度直接雕刻出来的解剖结构，不是工程师设计的。**深层的所有"专门细胞"，都是损失函数雕出来的肌肉。**

━━━━━━━━━━━━━━━━━━━━

◆ 第六节：回扣 199 期——修正"丧葬仪式"

────────────────────

回到开头那个自问：199 期的"丧葬仪式"那句话，错在哪？

这一期的四篇论文加上 Anthropic 的押韵实验，给出的修正是一句话：

**深层不是被动埋葬，是主动校准。**

| 论文 | 揭示了深层在做什么 |
|---|---|
| Lad/Tegmark 四阶段 | 末层 suppression neurons 密度暴涨，主动删稿 |
| Lioubashevski saturation events | 按排名一颗一颗敲钉子，离散开关式锁定，未训练模型也有 |
| Gupta Guess-then-Refine | 早层猜高频词，深层精炼；不同 token 走不同深度 |
| Anthropic 押韵实验 | 末层反向规划，改终点会重画整条路径 |
| Gurnee/Stolfo 细胞解剖 | 专门的 suppression、entropy、frequency neurons，在零空间里拧温度旋钮 |

199 期的核心论断没错——**中间层确实是意图所在**，深层确实丢掉了一些全局语义信息（这是 NTP 梯度的代价）。但深层不是"什么都没干"。它在干一组非常具体的活：

1. **按排名顺序锁定 top-k**——离散的"敲钉子"序列
2. **按 token 难度分配深度**——虚词浅、实词深、多 token 答案首 token 最深
3. **反向规划填充路径**——已知终点，倒推中间词
4. **主动抑制错误候选**——末层 9% + 8% 的概率质量被砍掉
5. **动态校准温度**——在 unembedding 零空间里拧旋钮防过度自信

这些事情都不会显示在信息熵或流形维度那种总量度量里。199 期是从"信息量"看深层（看着像被动），这一期是从"机制"看深层（看到都在主动干活）。同一段网络，换把尺子量，结论完全不同。

💡 最后一个比方，给程序员的：

中间层是产品经理，写下了一份需求文档（也就是"意图"）。深层是工程师团队——工程师没法改需求，但工程师绝不是被动的喇叭：他们要选技术栈、写代码、做单测、跑 lint、抑制 bug、做 code review。每一步都是决策密集的活。

199 期那句"工程师只是把需求文档敲成代码"，低估了工程师的工作量。

更准确的说法是：**工程师在敲代码的同时，还在按编号顺序合并 PR、在零空间里调超参、在末端做置信度校准。他们干的活比"敲代码"复杂得多，只是产品经理（中间层）看不见而已。**

技术名词速查：

- **Residual Sharpening**：Lad/Tegmark 提出的第四阶段——末层用 suppression neurons 主动压低错误候选，把输出分布锐化成尖峰
- **Saturation Event**：从某一层开始，logit lens 投影后排名第 k 的 token 在后续所有层都不变，称此层为 top-k 的饱和层
- **Logit Lens**：把任意中间层的隐藏态用最终的 unembedding 矩阵投影到词表空间，假装当成输出来读
- **TunedLens**：logit lens 的改进版，每层学一个仿射映射对齐到最终输出，早层更可靠
- **Prediction Neuron / Suppression Neuron**：输出权重投影到 logit 空间后呈高峰度分布的神经元，正偏=prediction，负偏=suppression
- **Entropy Neuron**：写入方向落在 unembedding 矩阵零空间的神经元，对 logit 排序无影响但通过 LayerNorm 调节输出熵
- **Cross-Layer Transcoder (CLT)**：Anthropic 用的可解释性工具，把每层 MLP 替换为稀疏激活的"feature 神经元"跨层共训
- **Backward Planning**：Anthropic 在押韵实验中发现的机制——模型从规划好的终点反向构造路径
