【AI可解释性】Transformer 末层在主动校准，不是被动渲染

━━━━━━━━━━━━━━━━━━━━

199 期（ https://mp.weixin.qq.com/s/NGIx-d9zTHRKeL7gEGWvsg ）讲过一个框架：Transformer 的中间层是"本我"，深层是"喉咙"。那一期的核心句写得很狠——**深层不是信息更成熟的地方，是信息的丧葬仪式，把连续的语义核坍缩成离散 token 的尸体。**

这句话写完我自己回头读，觉得不对劲——把深层写得像一个被动的输出接口，好像信息已经在中间层准备好了，深层只是把它"埋葬"成离散 token。

那深层到底有没有自己的事在干？还是真的就是个喇叭？

这一期正面回答这个问题。读了五篇 2024-2026 的论文，外加我们自己在 DeepSeek V4 上撞到的一次工程事故——证据攒够了，可以正经回答"深层在干什么"了。

结论先抛在前面：**深层不是被动的喇叭。它在按排名一颗一颗地敲钉子，在 unembedding 矩阵的零空间里悄悄拧温度旋钮，在末层突然反转出大量"砍 logit"的细胞。所有这些事情，都不会在 199 期那种信息熵或流形维度的总量度量里露出来——必须看到细胞，才看得见它在干活。**

━━━━━━━━━━━━━━━━━━━━

◆ 第一节：四阶段框架——给深层正式命名"residual sharpening"（残差锐化）

────────────────────

先看一篇统摄性论文——MIT Tegmark 组 2024 年的工作，arXiv:2406.19384。

这篇论文在 16 个模型（GPT-2 / Pythia / Qwen2.5 / LLaMA-3.2 / Phi，参数 124M 到 6.9B）上做了一件干脆事：把每一层删掉、或把相邻两层调换顺序，看模型还能不能用。

────────────────────

【四阶段】

论文提出 Transformer 的推理过程分四个阶段：

| 阶段 | 名字 | 层位（相对深度） | 在干什么 |
|---|---|---|---|
| 1 | Detokenization | 0–15% | 把 BPE 切出来的子词拼回单词、识别实体 |
| 2 | Feature Engineering | 15–50% | MLP 构建语义特征 |
| 3 | Prediction Ensembling | 50–90% | 多个"预测神经元"投票合成下一个 token 的候选分布 |
| 4 | **Residual Sharpening** | 最后约 10% | 抑制错误候选，把分布"锐化"成尖峰 |

（深度百分比是据论文密度曲线推得的近似边界，论文正文只用 early / middle / latter / final 这种粗略说法。）

199 期讲的中间层"压缩谷"对应阶段 2 后段；本期重点看阶段 4——**residual sharpening**（残差锐化）。

【两个反直觉数字】

**第一个数字：首层不能删，删了模型立刻崩。**

删第一层，输出立刻变成高熵的"瞎猜"分布——所有 16 个模型上都一样。但中段的层删一个、相邻两层调换顺序，模型基本不受影响——top-1 预测准确率仍保留 72%-95%（没有任何 fine-tuning）。

也就是说：**中段是冗余的，首尾是不可替代的。** 这本身就和"深层只是喇叭"的直觉冲突——喇叭怎么会比中段更不可替代？

**第二个数字：抑制神经元在最后 10% 的层位密度暴涨。**

论文把每层的神经元按"输出权重对 logit 的影响"分成两类——**预测神经元**（prediction neurons）抬高某一组 token 的 logit（"我支持这些候选词"）；**抑制神经元**（suppression neurons）压低某一组 token 的 logit（"我反对这些候选词"）。

密度分布有个非常干净的相变：

- 模型走到 50% 深度后，预测神经元越来越多
- 预测神经元密度在 **85% 深度**附近达到峰值
- 但到了最后 10%，**抑制神经元突然反超**——在 Phi 系列上，**每层平均**有近 15% 的预测神经元和 25% 的抑制神经元，密度是 GPT-2 的 5–8 倍

────────────────────

打个比方：

如果深层只是被动渲染，那它应该越往后越"软"——做加法越来越少，反正候选已经定了。但实际上完全相反：**深层 MLP 写入残差流的力气反而更大**，而且写入的内容从"加分项"变成了"扣分项"。

末层不是在被动输出，是在拿着大刀**主动删稿**。

【锐化实验：直接因果证据】

光看抑制神经元密度还不够说明"末层在主动锐化分布"——论文给了一个更狠的因果证据。

先解释一下"输出熵"：模型给出的 token 概率分布的"散乱程度"。熵高 = 分布散、几个候选打架、模型在犹豫；熵低 = 分布尖、一个候选独大、模型咬死了。本节标题里的 "sharpening / 锐化"，就是指分布从散变尖、熵从高变低的过程。

222 期（ https://mp.weixin.qq.com/s/Y2HxcESdVf2zvvmY5gNZaA ）讲过这个熵的精确定义和"自信地胡说"的关系，不熟的可以回去翻。

实验设计很干脆——**不重新训练**，**推理时**把 80–90% 深度的连续几层**重复执行一遍**：同一份权重让 hidden state 多过一次，本来是 …4-5-6-7-8…，改成 …4-5-5-6-6-7-7-8-8…。然后看输出熵和下游任务分数变化。

（这是 mech interp 圈常用的"推理时改 forward 路径"型因果干预，和它对偶的还有"删层"、"调换层顺序"——上面的"首层不能删"实验就是同一家族的另一个变体。）

结果是：**复制后段降熵效果显著、下游常识推理评测分数也跟着提升**（论文附录里跑的是 HellaSwag——让模型在 4 个续写选项里挑常识最合理那个的标准测试）。

也就是说，**加倍执行后段 → 分布更尖 → 下游能力上升**。这是 sharpening 机制存在于深层的正面因果证据——后段不是被动渲染。

◆ 第二节：Saturation Events（饱和事件）——深层在按排名一颗一颗敲钉子

────────────────────

这一节是本期的反直觉招牌——一篇 ICML 2025 的论文，arXiv:2410.20210。

普通程序员对 softmax 的直觉是这样的：模型在最后一层算出一个概率分布，分布里所有 token 的概率是**同时确定**的——你转个分布出来，第一名、第二名、第三名一锅端。

这篇论文证明：**不是同时确定的。Transformer 是按排名顺序、一颗一颗、阶梯式地把 top-k 锁定下来。**

────────────────────

【"饱和"的定义】

什么叫"锁定"？论文给了精确定义——

把每一层的隐藏态用 unembedding 矩阵投影到词表空间（这个操作叫 **logit lens**——把中间层激活假装当成最终输出来读），然后排序。如果从第 ℓ 层开始，**后续所有层**的排名第 1 的 token 都不再变化了，就说 top-1 在第 ℓ 层**饱和**（saturation）。同理定义 top-2、top-3 的饱和层。

判据是"锁死后续所有层"，不是阈值平滑。

【核心发现：阶梯式有序】

在 GPT2-XL 上的实际样例长这样：

- top-1 "toy" 在 **第 44 层**饱和
- top-2 "ball" 在 **第 45 层**饱和
- top-3 "ribbon" 在 **第 46 层**饱和
- top-4 在 **第 47 层**饱和

层位严格递增。不是同时锁定，是一层一颗钉子。

论文在三种 Transformer 架构（语言、视觉、语音）上都做了同一组实验。先把这三种架构说清楚——后面表格里要用到：

- **decoder-only**（GPT、Llama、DeepSeek 这一族）：单向生成 token，今天主流的 LLM 都是这种
- **encoder-only**（BERT、ViT 这一族）：输入整段一起看，输出一个分类或 embedding
- **encoder-decoder**（T5、Whisper、原版翻译模型）：先 encoder 压缩、再 decoder 生成

这三种架构的"top-1"含义不一样：decoder-only 的 top-1 是某个 token；encoder-only（ViT）的 top-1 是某个类别（论文测的是 ImageNet 类别）；encoder-decoder（Whisper）测的是 decoder 那一侧的字幕 token。**共同点不是"都在算 token 概率"，而是"都有一个最终输出空间，top-k 排名在某层锁定"**。

统计用 Kendall's τ 秩相关系数——衡量两组排名的一致程度：τ=1 完全一致，τ=0 无关，τ=-1 完全反序。这里要量的两组排名是：

- **排名 A**：top-k 这几个 token 在最终输出里的顺序，就是 (1, 2, 3, ..., k)
- **排名 B**：它们各自的饱和层——top-1 在第几层锁定、top-2 在第几层锁定、……

如果"最终排名越靠前的 token，饱和层越早"，τ 就大。

| 模型 | 架构 | τ | τ > 0 的样本占比 |
|---|---|---|---|
| GPT2-XL（预训练） | decoder-only | 0.187 | 67.4% |
| ViT-L/16 | encoder-only | 0.149 | 58.9% |
| Whisper-large | encoder-decoder | 0.210 | 63.8% |

全部 p < 0.001（"这结果是巧合"的概率不到千分之一），约 **2/3 的样本**里"最终排名越靠前→饱和层越早"成立——三种架构上都成立。这个现象不是某一种模型的特性，是 Transformer 这个家族共通的几何性质。

（τ 看起来不大是因为论文用的是"严格版"——把"平局"也算作不一致，比标准 Kendall's τ 保守。Kendall's τ 本身是秩相关家族，跟最常听到的 Pearson 相关系数不一样——后者测线性关系，前者测排名一致性。这里测的是排名，所以用秩相关。）

【最反直觉的一刀：未训练模型也有】

论文做了一个对照：拿一个**完全随机初始化、没有训练过的 GPT2-XL** 跑同样的实验。

结果：**依然有阶梯式饱和**。Kendall τ = 0.082，仍然统计显著。一个"task index"分类器（猜当前层在锁定第几名）准确率 86.1%，远高于随机猜的 33.3%。

也就是说：**这个"按排名敲钉子"的节奏不是训练学出来的，是 Transformer 架构本身的几何性质。**

────────────────────

打个比方：

你以为模型像个赌场荷官，洗牌洗完一次性把整副牌摊开给你看——其实它像个木匠，先抡起锤子把第一颗钉子敲实（top-1 锁定），然后才挪到第二颗（top-2），然后第三颗。每敲一颗就锁住，后面不能再改。

而这个节奏感**部分是 Transformer 架构本身带来的**——未训练的模型也按这个节奏敲，只是敲的内容是噪音。训练只是教会它"该敲哪些钉子"，没有教会它"按顺序敲"——按顺序是天生的。

◆ 第三节：Guess-then-Refine（先猜后改）——不同 token 在不同深度被锁定

────────────────────

上一节讲了"同一时间步内，top-k 按排名顺序锁定"。这一节讲互补的另一面：**不同类型的 token，需要的层数完全不同**——这是 2025 年的一篇论文（arXiv:2510.18871）的核心发现。

实验工具用的是 **TunedLens**——给每一层学一个仿射映射，把中间层表征对齐到最终 unembedding，比普通 logit lens 更可靠。

────────────────────

【虚词 vs 实词：深度差 4 倍】

在 Pythia-6.9B 和 Llama3-8B（都是 32 层）上测量"某个 token 在哪一层第一次到达 top-1"：

| token 类型 | 首次到 rank-1 的平均层 |
|---|---|
| 虚词（如限定词 the 等） | **第 5 层左右** |
| 实词（名词 / 动词 / 形容词） | **第 20 层左右** |

虚词在最前面 1/6 深度就锁定了，实词要等到接近 2/3 深度。**差了 4 倍。**

【为什么早层全是虚词】

论文做了个频率统计。Pythia-6.9B 第 1 层的 top-1 预测里，**超过 75% 的 token 属于词频 Top-10 那一档**（the / of / to / a / and...），但模型最终输出里这些高频词只占约 **34%**。

原因是早层还没有上下文整合、也没访问中层 MLP 存储的事实知识，所以模型"回退到语料统计"——直接把高频词推到 top-1，赌不会错太离谱。论文给这个现象起了个名字：**频率起手**（Frequency-Conditioned Onset）。

往后每一层做的事情就是用上下文逐步**精炼**这些"统计猜测"。论文证明：约 **60–80% 的早层 top-1 猜测最终会被替换掉**。

【多 token 答案：首 token 最深】

最反直觉的一组数字是这样的（Pythia-6.9B）。假设要回答一个 3-token 的事实性问题：

| 第几个 token | 首次到 rank-1 的层 |
|---|---|
| 第 1 个 token | **第 27 层** |
| 第 2 个 token | 第 20 层 |
| 第 3 个 token | 第 12 层 |

**越往后越浅。**

直觉是反的——按理说前面是基础、后面是延伸，难度应该递增。但实际上首 token 是最难的，因为它要**预先决定整个答案的方向**，相当于把后续几个 token 都隐式规划好了。一旦首 token 选定，后面的 token 沿着已经确定的路径走，浅层就能完成。

────────────────────

打个比方：

写代码的时候，你打第一个字符（`def`、`class`、`import`）的那一瞬间，整个函数/类的骨架其实已经在脑子里了。后续几个字符（函数名、参数列表）是顺着这个骨架填的，难度递减。模型做的事情一样——首 token 是决策密集的，需要调动深层的全部算力；后续 token 是顺势延伸，浅层就够用。

────────────────────

【2026 年的放大版：事实在末层"凭空召唤"】

上面看的是"虚词浅、实词深"。2026 年 6 月有人沿着这条线做了大规模放大版，专门测**事实类 token**——一篇叫 **MechLens** 的论文（arXiv:2606.07978）。

他定义了一个干净的度量叫 **FEP**（Factual Emergence Point，事实涌现层）——正确答案第一次进入 top-10 的最早层。在 5 个主流模型族上跑了 TruthfulQA 817 题：

| 模型 | 层数 | 平均 FEP | 答案直到末层附近才出现的占比 |
|---|---|---|---|
| Qwen2.5-7B | 28 | 27.3 ± 1.8 | **85.9%** |
| Gemma-7B | 28 | 27.4 ± 2.4 | 89.8% |
| Pythia-6.9B | 32 | 30.8 ± 4.2 | **93.4%** |
| Llama-3.1-8B | 32 | 29.4 ± 4.9 | 71.0% |
| Mistral-7B | 32 | 26.3 ± 6.2 | 27.1% |
| Qwen2.5-14B | 48 | 46.0 ± 4.9 | 77.7% |

最后一列读法：**Qwen 7B 上有 85.9% 的正确答案，从第 1 层到中间层的 top-10 里根本看不见，要到 80% 深度之后才"突然出现"。** Pythia 这个比例高达 93.4%。

控制实验是这篇论文最干净的一刀：换成**情感分类任务**，Qwen 的占比从 85.9% 直接跌到 **0.5%**——170 倍差距。**末层结晶不是通病，是事实回忆的专属机制。**

把这个数字和 Gupta 的"实词约第 20 层"对比——事实类 token 比一般实词还要极端，几乎是"在末层凭空召唤出来"。**深层在事实回忆上做的活，不是精炼已有候选，是从 0 到 1 把答案变出来。**

━━━━━━━━━━━━━━━━━━━━

◆ 第四节：细胞级解剖——末层有专门的"刹车细胞"和"温度旋钮"

────────────────────

前面三节都是"系统级"证据。这一节往下走一层，看具体神经元在干什么——主要参考 Gurnee 2024（arXiv:2401.12181）和 Stolfo 2024（arXiv:2406.16254）这两篇。

────────────────────

【刹车细胞：抑制神经元】

GPT-2 medium 里有个具体例子：某个末层神经元专门压低"含开括号 `(` 的 token"的概率。这个细胞**只在它要抑制的 token 真的有可能蹦出来时才发火**——不是无差别打压，是有针对性的"删稿"。

Datta 等人在 Qwen2.5-3B 上量化测过：末层 attention 头抑制约 **9%** 的概率质量、末层 MLP 再抑制 **8%**、倒数第二层 MLP 抑制 **4%**——加起来超过 20% 的概率质量是在最后几层被主动砍掉的。

【温度旋钮：熵神经元】

这是这一节最优雅的发现。

Stolfo 在 GPT-2 small 的最后一层找到 6 个奇怪神经元：**权重范数特别大，但对 logit 排序几乎没有影响**——写入残差流力气很大，但写完之后好像什么都没改变。

机制揭穿之后非常巧妙：这些神经元写入的方向，落在 **unembedding 矩阵 `W_U` 的零空间**（投到词表空间归零的子空间）。所以：

- 对 logit 排序无影响（被 `W_U` 投影归零）
- 但**让残差流的方差变大** → 末层 LayerNorm 归一化时分母变大 → 归一化后的向量被压扁 → 所有 logit 被均匀压低 → **输出分布的熵就上升**

等价于在末端**动态调高 softmax 温度**。藏在 logit 层面看不见的方向上，悄悄拧温度旋钮。

最硬的因果证据：Stolfo 对其中一个神经元做消融（强制压到平均输出），**模型输出的熵直接下降 70%**——一个神经元正常工作时，把整个模型的输出分布熵推高近三倍。模型自带的"防过度自信"机制。

这套结构在 LLaMA-2 7B、Pythia、Phi-2、Gemma 2B 上都找到了——**Transformer 末层的普遍解剖结构**，不是 GPT-2 特例。

────────────────────

【为什么会长出这些细胞】

回到 199 期讲过的训练目标——LLM 在预训练时是"猜下一个 token"，损失函数（cross-entropy）既惩罚"漏选正确 token"也惩罚"高置信度地错选"。**所以末层必须长出三套对偶细胞：预测神经元抬正确的、抑制神经元砍错误的、熵神经元防过度自信**——这是损失函数直接雕刻出来的肌肉，不是工程师设计的。

━━━━━━━━━━━━━━━━━━━━

◆ 第五节：活体证据——layer 42 换 kernel 把"正常英文"翻成"符号循环"

────────────────────

前面四节讲的都是别人论文里的实验。这一节讲我自己撞到的事——是本期论点最硬的活体证据。

185 期（ https://mp.weixin.qq.com/s/dEc-wD0JcM5WhKicOUR1pQ ）和 224 期（ https://mp.weixin.qq.com/s/X1b7H4565_WFn6CSTzoAQg ）讲过一个工程谜团：两台 DGX Spark 跑 DeepSeek V4 Flash 280B（43 层 Transformer + MTP），用了一个非常奇怪的混合 kernel 方案才能让模型正确输出——

- decoder layer 0-41 走 Marlin（一种 W4 高速 GEMM kernel）
- decoder layer 42（最后一层）走 Consumer-DeepGEMM（CDG，DeepSeek 自家 FP4 GEMM 的 Triton 重写版）

只要把 layer 42 也换成 Marlin，让 0-42 全部 Marlin，模型立刻乱码——英文输出变成 `</`、`_`、`,` 这种符号循环。

最反直觉的地方：**Marlin 在数值上比 CDG 更接近高精度参考**。手写解量化的 BF16 × FP4 公式（W4A16 语义）算出来的"理论答案"，Marlin 几乎完全贴上去；CDG 反而离这个公式更远。

```text
Marlin vs 高精度公式:  max=0.03,  mean=0.003, cos=0.99999
CDG    vs 高精度公式:  max=0.17,  mean=0.024, cos=0.99940
```

那为什么"更准"的 Marlin 反而乱码？

因为模型在量化训练 / 校准时跑的是 **W4A8** 这条管线——激活先量化成 FP8 再做矩阵乘。CDG 走的就是 W4A8，Marlin 走的是 W4A16（激活保留 BF16）。**两条都是合法的数值世界，但模型的 argmax 边界是按 W4A8 长出来的——它不认 W4A16 那条更精确的路径**。

mixed（0-41 Marlin + 42 CDG）和 all-marlin（0-42 全 Marlin）的逐层对比是这样的：

```text
layer 0-41:
  x_max_diff   = 0   ← 两边都走 Marlin，确定性 0 误差
  out_max_diff = 0

layer 42:
  x_max_diff        = 0          ← 进入 layer 42 之前两边完全一致
  topk_ids_diff     = 0          ← 路由也完全一致
  topk_weights_diff = 0
  out_max_diff      = 0.16
  out_mean_diff     = 0.025
  out_cos           = 0.99939    ← 余弦相似度看上去几乎一样
```

肉眼看，这是"几乎一样"的两份输出——`cos=0.99939`，最大偏差 0.16，平均偏差 0.025。在普通中间层，这种量级的偏差会被下一层的 RMSNorm 拉回来、被 Softmax 的强非线性吞掉、被残差路径稀释。43 层 Transformer 一路跑下来，这点偏差根本注意不到。

但 layer 42 是**最后一层**。它之后只剩：

```text
hidden state → final_norm → lm_head → logits → argmax 选下一个 token
```

`final_norm` 只是一次归一化、`lm_head` 是一个线性投影——对 hidden state 的偏差**几乎不衰减**。0.16 的偏差几乎原样打到 logits 上。如果两个候选 token 的 logits 本来就咬得很近：

```text
token "Write": 12.031
token "</":    12.028
```

那 hidden state 上 0.1 量级的通道变化，足够把 `Write` 和 `</` 的顺序翻过来。第一个 token 翻错之后，自回归整条换轨：

```text
正确路径："Write a quicksort function in Python..."
错误路径："</_,</_,</_,</_,</_..."
```

────────────────────

回过头看前面讲的论文：

- **Lioubashevski 饱和事件**：top-k 按排名一颗一颗在末层附近锁定，像"开关跳变"
- **Lad/Tegmark 残差锐化**：末层 MLP 主动写入残差流，并不是被动收尾
- **Gupta + MechLens**：事实类 token 几乎都在末层"凭空召唤"出来

这些都是从论文里读出来的"末层在主动干活、对扰动极敏感"。但 layer 42 这个事故，是**反过来的实测**——你不动模型权重、不动训练、不动前 42 层的任何一个参数，**只换最后一层的 kernel 语义（一种"合法但模型没校准过的"数值路径），整条采样轨迹就翻盘**。

换句话说：

| 论文 | 实测 |
|---|---|
| Lioubashevski：末层是 top-k 锁定的开关 | layer 42 一改，第一颗钉子就敲到隔壁洞里去了 |
| Lad/Tegmark：末层 MLP 主动写入残差流 | layer 42 输出 0.16 的偏差直接打到 logits，没有"下一层"吸收 |
| Gurnee/Stolfo：末层有专门的细胞，对 logit 排序极敏感 | logits 上 0.003 的差距就足以翻 argmax |
| MechLens：事实在末层"结晶"，不是精炼 | layer 42 不是"渲染答案"，是答案最后定型的那一刀 |

**论文里说"末层主动、扰动敏感"，工程实测就是"换最后一层的 kernel 把英文翻成符号"。理论和实测对上了。**

────────────────────

💡 顺便补一个反直觉点：**这不是 Marlin "有 bug"，是 Marlin 算得"太准"了**——准到了模型没校准过的那条数值路径上。224 期的标题就是这个意思："Marlin 不是算错了，而是算得太正确"。

模型的"正确"不是数学上的最优近似，是"和量化校准时跑的那条管线一致"。论文里讲的"末层主动干活"，工程上对应的就是"末层对'走哪条数值管线'极敏感"——这是同一件事的两面。

━━━━━━━━━━━━━━━━━━━━

◆ 第六节：回扣 199 期——修正"丧葬仪式"

────────────────────

回到开头那个自问：199 期的"丧葬仪式"那句话，错在哪？

这一期的五篇论文加上一个 2026 年的放大版、再加上我们自己撞到的 layer 42 实测，给出的修正是一句话：

**深层不是被动埋葬，是主动校准。**

| 证据 | 揭示了深层在做什么 |
|---|---|
| Lad/Tegmark 四阶段 | 末层抑制神经元密度暴涨，主动删稿 |
| Lioubashevski 饱和事件 | 按排名一颗一颗敲钉子锁定 top-k，未训练模型也有这节奏 |
| Gupta 先猜后改 | 早层猜高频词，深层精炼；不同 token 走不同深度 |
| MechLens 事实涌现层 | 事实类 token 几乎都在末层"凭空召唤"，不是渲染 |
| Gurnee/Stolfo 细胞解剖 | 末层有专门的"刹车细胞"和"温度旋钮"，分工明确 |
| Layer 42 实测 | 只换最后一层 kernel 语义，模型从正常英文翻成符号循环 |

199 期的核心论断没错——**中间层确实是意图所在**，深层确实丢掉了一些全局语义信息（这是"猜下一个 token"训练目标的代价）。但深层不是"什么都没干"。它在干一组非常具体的活：

1. **按排名顺序锁定 top-k**——离散的"敲钉子"序列
2. **按 token 难度分配深度**——虚词浅、实词深、事实类词几乎要等到末层才"结晶"
3. **主动抑制错误候选**——末层把约 17% 的概率质量直接砍掉
4. **动态校准温度**——藏在 logit 看不见的方向里拧旋钮，防过度自信

这些事情都不会显示在信息熵或流形维度那种总量度量里。199 期是从"信息量"看深层（看着像被动），这一期是从"机制"看深层（看到都在主动干活）。同一段网络，换把尺子量，结论完全不同。

💡 最后一个比方，给程序员的：

中间层是产品经理，写下了一份需求文档（也就是"意图"）。深层是工程师团队——工程师没法改需求，但工程师绝不是被动的喇叭：他们要选技术栈、写代码、做单测、跑 lint、抑制 bug、做 code review。每一步都是决策密集的活。

199 期那句"工程师只是把需求文档敲成代码"，低估了工程师的工作量。

更准确的说法是：**工程师在敲代码的同时，还在按优先级提交 PR、做 code review、跑测试、写日志监控置信度。他们干的活比"敲代码"复杂得多，只是产品经理（中间层）看不见而已。**

技术名词速查：

- **Residual Sharpening**：Lad/Tegmark 提出的第四阶段——末层用 suppression neurons 主动压低错误候选，把输出分布锐化成尖峰
- **Saturation Event**：从某一层开始，logit lens 投影后排名第 k 的 token 在后续所有层都不变，称此层为 top-k 的饱和层
- **Logit Lens**：把任意中间层的隐藏态用最终的 unembedding 矩阵投影到词表空间，假装当成输出来读
- **TunedLens**：logit lens 的改进版，每层学一个仿射映射对齐到最终输出，早层更可靠
- **Prediction Neuron / Suppression Neuron**：输出权重投影到 logit 空间后呈高峰度分布的神经元，正偏=prediction，负偏=suppression
- **Entropy Neuron**：写入方向落在 unembedding 矩阵零空间的神经元，对 logit 排序无影响但通过 LayerNorm 调节输出熵
