【AI可解释性】Transformer 深层不是被动的喇叭——末段在按排名敲钉子、在零空间里拧温度旋钮

━━━━━━━━━━━━━━━━━━━━

199 期（ https://mp.weixin.qq.com/s/NGIx-d9zTHRKeL7gEGWvsg ）讲过一个框架：Transformer 的中间层是"本我"，深层是"喉咙"。那一期的核心句写得很狠——**深层不是信息更成熟的地方，是信息的丧葬仪式，把连续的语义核坍缩成离散 token 的尸体。**

这句话写完我自己回头读，觉得不对劲——把深层写得像一个被动的输出接口，好像信息已经在中间层准备好了，深层只是把它"埋葬"成离散 token。

那深层到底有没有自己的事在干？还是真的就是个喇叭？

这一期正面回答这个问题，用四篇 2024-2025 的论文：Lad/Tegmark 的四阶段框架、Lioubashevski 的 saturation events（饱和事件）、Gupta 的 Guess-then-Refine（先猜后改）、Gurnee/Stolfo 的细胞级解剖。再加上 Anthropic 自家 Biology of LLM（"大语言模型的生物学"）报告里的押韵实验。

结论先抛在前面：**深层不是被动的喇叭。它在按排名一颗一颗地敲钉子，在 unembedding 矩阵的零空间里悄悄拧温度旋钮，在末层突然反转出大量"砍 logit"的细胞。所有这些事情，都不会在 199 期那种信息熵或流形维度的总量度量里露出来——必须看到细胞，才看得见它在干活。**

◆ 第一节：四阶段框架——给深层正式命名"residual sharpening"（残差锐化）

────────────────────

先看一篇统摄性论文。

Lad, Lee (J. H.), Gurnee, Tegmark, "The Remarkable Robustness of LLMs: Stages of Inference?", arXiv:2406.19384, 2024.

这篇论文在 16 个模型（GPT-2 / Pythia / Qwen2.5 / LLaMA-3.2 / Phi，参数从 124M 到 6.9B）上做了一件事：把每一层删掉或者把相邻两层调换顺序，看模型还能不能用。

────────────────────

【四阶段】

论文提出 Transformer 的推理过程分四个阶段：

| 阶段 | 名字 | 层位（相对深度） | 在干什么 |
|---|---|---|---|
| 1 | Detokenization | 0–15% | 把 BPE 切出来的子词拼回单词、识别实体 |
| 2 | Feature Engineering | 15–50% | MLP 构建语义特征 |
| 3 | Prediction Ensembling | 50–90% | 多个"预测神经元"投票合成下一个 token 的候选分布 |
| 4 | **Residual Sharpening** | 最后约 10% | 抑制错误候选，把分布"锐化"成尖峰 |

（以上深度区间是据论文里的密度/范数曲线推得的近似边界，论文正文用的是 early / middle / latter / final 的模糊表述，唯一硬数字是后面会讲的"预测神经元"（prediction neuron，专门抬高某些 token 概率的细胞）密度峰值在约 85% 深度。）

────────────────────

💡 我自己容易搞混的问题

**为什么阶段 2 强调"MLP"构建语义特征？每层不都是 attention + MLP 吗？**

每层确实是 attention + MLP 两个子层。但 Lad/Tegmark 删层实验里观察到一个非对称：**中段 MLP 删了，模型性能掉得明显；中段 attention 删了，几乎无感**。所以他们把这一阶段定义为「MLP 主导」，不是泛指。

背后是一个 mechanistic interpretability 圈的老共识——**attention 和 MLP 是分工的**：

| 子层 | 干什么 |
|---|---|
| **attention** | **跨 token 搬信息**——看到 "the cat" 后面接 "sat"，把 "cat" 的特征搬到 "sat" 位置。干的是"信息路由" |
| **MLP** | **在单个 token 位置上做非线性变换**——把搬过来的特征加工成新的特征。干的是"特征构建 / 知识存储" |

这条线后续工作还在加证据：Geva 2021 证明"MLP 是 key-value memory"、Meng 2022 的 ROME 实验证明事实知识（"埃菲尔铁塔在巴黎"这种）就存在中段 MLP 里。**所以阶段 2 的主力是 MLP——它在堆特征、堆知识；attention 只是在调度信息**。

后面两个阶段也有自己的主力细胞——阶段 3 是上面表里说的**预测神经元（prediction neurons，抬高正确候选）**；阶段 4 换成另两种：**抑制神经元（suppression neurons，压低错误候选）** 和 **熵神经元（entropy neurons，给整体分布"踩刹车"防过度自信）**。后面几节会逐个讲。这些也都是 MLP 神经元——所以本期讲的"末层细胞"基本都在 MLP 那一边。

────────────────────

199 期讲的中间层"压缩谷"对应阶段 2 后段；这一期我们重点看阶段 4——**residual sharpening**。

【两个反直觉数字】

**第一个数字：首层不能删，删了模型立刻崩。**

论文原话："Intervening on the first layer is catastrophic for model performance for every model, regardless of size or model family." 删第一层，输出立刻变成高熵的"瞎猜"分布。但中段的层删一个、调换两层顺序，模型基本不受影响——"models retain 72–95% of their original top-1 prediction accuracy without any fine-tuning."

也就是说：**中段是冗余的，首尾是不可替代的。** 这本身就和"深层只是喇叭"的直觉冲突——喇叭怎么会比中段更不可替代？

**第二个数字：suppression neurons 在最后 10% 的层位密度暴涨。**

论文把每层的神经元按其"输出权重对 logit 的影响"分成两类：

- **prediction neurons**：抬高某一组 token 的 logit（"我支持这些候选词"）
- **suppression neurons**：压低某一组 token 的 logit（"我反对这些候选词"）

密度分布有个非常干净的相变：

- 模型走到 50% 深度后，prediction neurons 越来越多
- prediction neurons 密度在 **85% 深度**附近达到峰值
- 但到了最后 10%，**suppression neurons 突然反超**——在 Phi 系列上，**每层平均**有近 15% 的 prediction neurons 和 25% 的 suppression neurons，密度是 GPT-2 的 5–8 倍

────────────────────

打个比方：

如果深层只是被动渲染，那它应该越往后越"软"——做加法越来越少，反正候选已经定了。但实际上完全相反：**深层 MLP 写入残差流的力气反而更大**，而且写入的内容从"加分项"变成了"扣分项"。

末层不是在被动输出，是在拿着大刀**主动删稿**。

【Sharpening 实验（锐化实验）：直接因果证据】

论文做了个干脆利落的实验，叫 **Sharpening Experiment**（锐化实验）：在**不重新训练模型**的前提下，**推理时**把 80–90% 深度的连续几层**重复执行一遍**——同一份权重让 hidden state 多过一次（本来是 …4-5-6-7-8…，改成 …4-5-5-6-6-7-7-8-8…），然后看**输出熵**和下游任务分数。

注意这不是"训练时给模型加层"，是"推理时动态改变 forward pass 路径"——mech interp 圈常用的一种因果干预手法。和它对偶的还有"删层"、"调换层顺序"，前面引过的"首层不能删"实验就是同一个家族的另一个变体。

输出熵是模型给出的 token 概率分布的"散乱程度"——熵高代表"分布散，几个候选打架，模型在犹豫"，熵低代表"分布尖，一个候选独大，模型咬死了"。本期主题"sharpening / 锐化"指的就是分布从散变尖、熵从高变低的过程。

222 期（ https://mp.weixin.qq.com/s/Y2HxcESdVf2zvvmY5gNZaA ）讲过这个熵的精确定义（Shannon 熵 H(p) = −Σ pᵢ log pᵢ）和它跟"自信地胡说"的关系——抑制掉某些内部特征后，模型输出的熵能下降 75%。不熟的可以回去翻一下。**而本期会进一步揭示：末层不仅有"压熵"的细胞（让分布更尖），也有专门"抬熵"的细胞（防止分布过度自信地咬死一个错答案）——两组细胞反向制衡，就是末层的"置信度方向盘"。**

结果：**复制后段降熵效果显著、下游常识推理评测的分数也跟着提升**（论文附录主要展示了在 Qwen 上重复 5 次后段的实验，跑的评测是 HellaSwag——一个让模型在 4 个续写选项里挑常识最合理那个的标准测试；论文未直接做"复制早层"的对照实验，所以"前段无效"是反向推断，不在因果证据范围内）。

这至少是 sharpening 机制存在于深层的正面因果证据——加倍执行后段，分布更尖、下游能力上升，说明后段的算子并不是只在被动渲染。

━━━━━━━━━━━━━━━━━━━━

◆ 第二节：Saturation Events（饱和事件）——深层在按排名一颗一颗敲钉子

────────────────────

这一节是本期的反直觉招牌。

Lioubashevski et al., "Looking Beyond The Top-1: Transformers Determine Top Tokens In Order", arXiv:2410.20210, ICML 2025.

普通程序员对 softmax 的直觉是这样的：模型在最后一层算出一个概率分布，分布里所有 token 的概率是**同时确定**的——你转个分布出来，第一名、第二名、第三名一锅端。

这篇论文证明：**不是同时确定的。Transformer 是按排名顺序、一颗一颗、阶梯式地把 top-k 锁定下来。**

────────────────────

【"饱和"的定义】

什么叫"锁定"？论文给了精确定义：

把每一层的隐藏态用 unembedding 矩阵投影到词表空间（这个操作叫 **logit lens**——一种把中间层激活假装当成最终输出来读的工具），然后排序。

────────────────────

💡 我自己容易搞混的问题

**任何时刻都能把向量投影到词表空间，那 SAE 还有啥用？**

技术上确实任何时刻都能投——拿任意一层的 hidden state 乘上 unembedding 矩阵就行，操作没限制。但**能不能读出有意义的东西**是另一回事。

- **logit lens 适合深层**：离最后一层近、方向已经朝词表空间收束，投出来"排第几"是可信的——所以 Lioubashevski 用 logit lens 看 saturation 没问题（它只关心"top-1 这一层有没有锁定"）
- **logit lens 在中浅层会"漂"**：中层激活是抽象语义（"国家"、"颜色"、"否定"），硬投到词表空间读出来的常常是高频词噪音——不是模型在想这些词，是中层方向还没朝词表方向收束
- **TunedLens 是补丁**：给每一层学一个仿射映射先校准，再投影。第三节 Gupta 用的就是这个，因为他要看的"虚词第一次到 rank-1"对早层读数更敏感
- **SAE 是另一套尺子**：不投到词表空间读"哪个词概率高"，而是把中层激活分解成一组**抽象特征**的激活组合——读"这层激活了'国家'、'巴黎'、'否定'这几个概念"，而不是"这层最像哪个 token"

简单说：logit lens 读"输出像啥"，SAE 读"内部在想啥"。深层适合前者，中层适合后者。

────────────────────


如果从第 ℓ 层开始，**后续所有层**的排名第 1 的 token 都不再变化了，就说 top-1 在第 ℓ 层**饱和**（saturation）。同理定义 top-2、top-3 的饱和层。

判据是"锁死后续所有层"，不是阈值平滑。

【核心发现：阶梯式有序】

在 GPT2-XL 上的实际样例长这样：

- top-1 "toy" 在 **第 44 层**饱和
- top-2 "ball" 在 **第 45 层**饱和
- top-3 "ribbon" 在 **第 46 层**饱和
- top-4 在 **第 47 层**饱和

层位严格递增。不是同时锁定，是一层一颗钉子。

统计验证用的是 Kendall's τ 秩相关系数。这里要先解释一下**谁和谁的相关**：

对每个输入位置，论文取最终输出的 top-k 这几个 token（k 在每个样本里独立选定），形成两组排名：

- **排名 A**：这些 token 在最终输出里的排名顺序，就是 (1, 2, 3, ..., k)——最熟悉的"第一名、第二名、第三名"
- **排名 B**：这些 token 各自的**饱和层**——top-1 在第几层锁定、top-2 在第几层锁定……

τ 衡量的是这两组顺序的一致程度。如果"最终输出排名越靠前的 token，饱和层越早"，τ 就大；反之 τ 接近 0。τ = 1 完全一致、τ = 0 无关、τ = -1 完全反序。

（顺便补一句给可能搜过相关系数的读者：Kendall's τ 是**秩相关**家族，跟最常听到的 Pearson 相关系数不一样——后者测两个连续变量的线性关系，前者测两组排名的一致性。这里要测的是排名，所以不能用 Pearson。论文实际用的还是这个家族里的一个加强变体，叫 "stricter Kendall's τ"，把"平局"也算作不一致——比标准版更保守。所以下面数字看起来不大，是被刻意压低的。）

| 模型 | 架构 | τ | τ > 0 的样本占比 |
|---|---|---|---|
| GPT2-XL（预训练） | decoder-only（GPT 系，文本生成） | 0.187 | 67.4% |
| ViT-L/16 | encoder-only（BERT 系，视觉分类） | 0.149 | 58.9% |
| Whisper-large | encoder-decoder（翻译 / 语音转写型） | 0.210 | 63.8% |

最后一列「τ > 0 的样本占比」是说：把每个样本各自算一个 τ，**大约 2/3 的样本是正的**——也就是这个样本里"最终排名越靠前→饱和层越早"的成对关系多于反过来的。注意这不是"严格递增"——后者更严格，论文没报告。

全部 p < 0.001（**统计显著性指标**——"这结果是巧合"的概率小于千分之一，业内认可的高置信门槛；论文还跑了对照：把每个样本的饱和层顺序随机打乱 1000 次跑 τ，**真实 τ 高于所有 1000 次的打乱版本**）。

三种完全不同的架构（语言、视觉、语音），都呈现按排名阶梯式饱和的规律——论文里也补了一句限定："to a lesser extent" 适用于 encoder-decoder，即 Whisper 这条线的效应比 GPT2-XL / ViT 略弱，但仍然显著。

💡 我自己容易搞混的问题

**decoder-only / encoder-only / encoder-decoder 是 Transformer 的三种主要架构**：

- **decoder-only**（GPT、Llama、DeepSeek 这一族）：单向，从左往右生成 token，今天主流的 LLM 都是这种
- **encoder-only**（BERT、ViT 这一族）：双向，输入整段一起看，输出一个分类 / embedding，不生成
- **encoder-decoder**（T5、Whisper、原版 Transformer 翻译模型）：先 encoder 压缩输入、再 decoder 生成输出，常用于翻译 / 语音转写

需要补一个细节：**这三种架构的"top-1"含义不一样**——decoder-only 投到词表空间，top-1 是某个 token；encoder-only（ViT）投到类别空间，top-1 是某个类别（论文测的是 ImageNet 类别）；encoder-decoder（Whisper）的 saturation 测的是 **decoder 部分**——encoder 那一侧没有"输出 top-k 概率"的概念。

所以严格说，"三种架构都阶梯式饱和"测的是 **"最终输出空间里的 top-k 排名什么时候锁定"** 这件事——只是这个"输出空间"对不同架构是 token 词表 / 图像类别 / 字幕 token。**共同点不是"都在算 token 概率"，而是"都有'排名阶梯式锁定'这个几何现象"**——所以发现才特别有冲击力。

【最反直觉的一刀：未训练模型也有】

论文做了一个对照：拿一个**完全随机初始化、没有训练过的 GPT2-XL**，跑同样的实验。

结果：**依然有阶梯式饱和**。
- 严格有序到 top-4
- Kendall τ = 0.082，p < 0.001，仍然统计显著
- "task index"（当前层在锁定第几名）的分类器准确率 86.1%，chance level 33.3%

论文原话："This phenomenon seems intrinsic to the Transformer architecture, occurring across different architectural variants... and **even in untrained Transformers**."

也就是说：**这个"按排名敲钉子"的节奏，是 Transformer 几何结构本身带来的，不是训练学出来的。**

论文原话："a switch being flipped 'on' and staying 'on' for the remaining layers."

不是平滑的连续收敛，是**离散的开关跳变**——一旦那一层翻了，后面就锁住了。

（论文里还做了一组"层间注入"的因果干预实验试图证明这个开关的存在，但实验设计上对照不够干净——同一段上下文里两个 token 位置之间互相覆盖 hidden state，从计算图上看 attention 在后续层仍然能从相同的前文 KV cache 里捞到一致的信号，所以"开关已翻"的因果证据没切利索。本期不展开。）

────────────────────

打个比方：

你以为模型像个赌场荷官，洗牌洗完一次性把整副牌摊开给你看——其实它像个木匠，先抡起锤子把第一颗钉子敲实（top-1 锁定），然后才挪到第二颗钉子（top-2 锁定），然后第三颗。每敲一颗就锁住，后面不能再改。

而且这个节奏感**部分来自 Transformer 架构本身的几何结构**——未训练的模型也按这个节奏敲，只是敲的内容是噪音。训练只是教会它"该敲哪些钉子"，没有教会它"按顺序敲"——按顺序是天生的。

◆ 第三节：Guess-then-Refine（先猜后改）——不同 token 在不同深度被锁定

────────────────────

上一节讲了"同一时间步内，top-k 按排名顺序锁定"。这一节讲互补的另一面：**不同类型的 token，需要的层数完全不同**。

Gupta, Ivanova et al., "How Do LLMs Use Their Depth?", arXiv:2510.18871, 2025.

实验工具用的是 **TunedLens**——给每一层学一个仿射映射，把中间层表征对齐到最终 unembedding，比普通 logit lens 更可靠。

────────────────────

【虚词 vs 实词：深度差 4 倍】

在 Pythia-6.9B 和 Llama3-8B（都是 32 层）上测量"某个 token 在哪一层第一次到达 top-1"：

| token 类型 | 首次到 rank-1 的平均层 |
|---|---|
| 虚词（如限定词 the 等） | **第 5 层左右** |
| 实词（名词 / 动词 / 形容词） | **第 20 层左右** |

虚词在最前面 1/6 深度就锁定了，实词要等到接近 2/3 深度。**差了 4 倍。**

（论文具体测的是 DET 类，即限定词 the/a/an 这一类。同为"虚词"的介词、连词在论文里不在这组对照里——为了不滥用，本文用"如限定词 the 等"做代表。）

【为什么早层全是虚词】

论文做了个频率统计。Pythia-6.9B 第 1 层的 top-1 预测里，**超过 75% 的 token 属于词频 Top-10 那一档**（the / of / to / a / and...），但模型最终输出里这些高频词只占约 **34%**。

原因是早层还没有上下文整合，也没访问中层 MLP 存储的事实知识，所以模型"回退到语料统计"——直接把高频词推到 top-1。论文给这个现象起了个名字：**Frequency-Conditioned Onset**（频率条件下的起始）。

往后每一层做的事情就是用上下文逐步**精炼**这些"统计猜测"。论文证明：约 **60–80% 的早层 top-1 猜测最终会被替换掉**。

【多 token 答案：首 token 最深】

最反直觉的一组数字是这样的（Pythia-6.9B）。假设要回答一个 3-token 的事实性问题：

| 第几个 token | 首次到 rank-1 的层 |
|---|---|
| 第 1 个 token | **第 27 层** |
| 第 2 个 token | 第 20 层 |
| 第 3 个 token | 第 12 层 |

**越往后越浅。**

直觉是反的——按理说前面是基础、后面是延伸，难度应该递增。但实际上首 token 是最难的，因为它要**预先决定整个答案的方向**，相当于把后续几个 token 都隐式规划好了。一旦首 token 选定，后面的 token 沿着已经确定的路径走，浅层就能完成。

────────────────────

打个比方：

写代码的时候，你打第一个字符（`def`、`class`、`import`）的那一瞬间，整个函数/类的骨架其实已经在脑子里了。后续几个字符（函数名、参数列表）是顺着这个骨架填的，难度递减。模型做的事情一样——首 token 是决策密集的，需要调动深层的全部算力；后续 token 是顺势延伸，浅层就够用。

【和"饱和事件"的关系】

Gupta 这篇论文在引言里明确点了 Lioubashevski 那篇——两者是互补的：

> "Subsequent works to LogitLens also formalize saturation events, showing that top-1 token get locked-in before the last layer... Our work on the other hand focuses on **pre-saturation events** and explaining internal prediction dynamics during that regime."

Lioubashevski 看的是"top-k 多个 token 按排名先后被锁定"（saturation 之后的稳定动态）；Gupta 看的是"同一个 token 从高频猜测被精炼到正确答案"（saturation 之前的动态）。

横切纵切，正好拼成深层全貌。

────────────────────

【2026 年的放大版：事实在末层"结晶"，不是精炼出来的】

Gupta 这篇 2025 年发表的"首次到 rank-1 的平均层"，2026 年 6 月有人在 5 个模型族上做了大规模放大版：

Gao, X. "MechLens: Late Crystallization of Factual Knowledge Explains Intervention Effectiveness in Language Models", arXiv:2606.07978, 2026.

他定义了一个干净的度量叫 **FEP（Factual Emergence Point）**——正确答案在 logit lens 投影下第一次进入 top-k 的最早层。然后在 Pythia / Gemma / Qwen2.5 / Llama-3.1 / Mistral（0.5B–14B）上测：

| 模型 | 层数 | 平均 FEP（事实涌现层） | "末层结晶"样本占比 |
|---|---|---|---|
| Qwen2.5-7B | 28 | 27.3 ± 1.8 | **85.9%** |
| Qwen2.5-14B | 48 | 46.0 ± 4.9 | 77.7% |
| Gemma-7B | 28 | 27.4 ± 2.4 | 89.8% |
| Pythia-6.9B | 32 | 30.8 ± 4.2 | **93.4%** |
| Llama-3.1-8B | 32 | 29.4 ± 4.9 | 71.0% |
| Mistral-7B | 32 | 26.3 ± 6.2 | **27.1%** |

（数据集：TruthfulQA 817 样本，k=10。FEP 列是"事实涌现层 ± 标准差"，"层数"列是模型总层数做参考——Qwen 7B 的 27.3/28 意味着答案平均要到倒数第二层才被推到 top-10。）

跨架构 **27.1%–93.4% 的正确答案，在中间层 top-10 里根本看不见**——要到 80% 深度之后才"突然出现"。MMLU 上更夸张，1200 样本里 98.2% 是"末层结晶"型。

Mistral 在这张表里是异类。其他 5 个模型族的末层结晶率都在 70%–93% 之间，Mistral 只有 27%。论文给出的解释跟它的注意力架构有关——它用 **GQA（Grouped-Query Attention）+ SWA（Sliding Window Attention）** 这两个变体，而其他模型基本都是标准 MHA（Multi-Head Attention）。架构上的差异在末层动态上拉开了 3 倍的差距，这本身就是个反直觉点：**末层"结晶"这件事不是 Transformer 通用现象，attention 架构变体能改变它的强度**。

控制实验是这篇论文最干净的一刀：**情感分类任务（SST-2）上 Qwen 只有 0.5% 出现"末层结晶"、Mistral 只有 2.0%。** 同一个 Qwen 在事实任务上 85.9%、情感任务上 0.5%——170 倍差距。这个模式不是通病，是事实回忆的专属机制。

把这个数字和 Gupta 的"实词约第 20 层首次到 rank-1"放一起看——事实类 token 比一般实词更极端，几乎是"在末层凭空召唤出来"。**深层在事实回忆上做的活，不是精炼已有候选，是从 0 到 1 把答案变出来。**

这是 Lad/Tegmark 的 residual sharpening、Lioubashevski 的 saturation events、Gupta 的 Guess-then-Refine 在 2026 年最新的延伸——同一根藤上结的新瓜。

━━━━━━━━━━━━━━━━━━━━

◆ 第四节：Anthropic 的押韵实验——末层执行不是机械采样

────────────────────

到这里都还在看激活值。这一节看一个具体的写作任务，看深层是怎么把一个"想法"落地成具体 token 的。

Anthropic, "On the Biology of a Large Language Model", transformer-circuits.pub, 2025. 实验对象 Claude 3.5 Haiku，工具是 cross-layer transcoder（CLT，30M 个跨层共训的稀疏可解释 feature）。

────────────────────

【实验：写一句押韵的诗】

输入：

```
A rhyming couplet:
He saw a carrot and had to grab it,
```

Claude 3.5 Haiku 续写：

```
His hunger was like a starving rabbit
```

押韵点：grab-**it** / rabb-**it**。

【关键发现：规划在"换行符"那一瞬间凝结】

Anthropic 用 CLT 追踪 feature 激活，发现一个反直觉的事实：

在模型即将开始写第二行（即输出 `\n` 这个 token 时），**中层就已经同时激活了 `rabbit` 和 `habit` 这两个候选词的 feature**——是普通的词义 feature，不是什么"专门的规划 feature"。

原文："The model holds multiple possible planned words \"in mind\" at the same time."

而且这个规划只在 newline 这一个点局部活跃——之后每个 token 上，规划 feature 都不再活跃。

【末层执行：反向推理填充】

那从 newline 到最后一个 token "rabbit"，中间这些词（"His"、"hunger"、"was"、"like a"、"starving"）是怎么来的？

Anthropic 的发现叫 **backwards planning**（反向规划，原文带 s）：

> "The model goes from the planned target ('rabbit') that activates at the newline and reasons backwards, creating a plausible transition to get to it."

也就是说，末层不是机械地"一个 token 一个 token 往下采样"。**它已经知道终点是 rabbit，然后倒推：怎么从开头自然过渡到 rabbit？** 于是激活了 "comparison features"，于是冒出 "like a"，把整个句子结构搭到能自然落到 rabbit 上。

每一步的具体 token 选择，都是末层在多重约束下做的微决策：
- 韵脚约束（终点必须是 rabbit）
- 语法约束（英语句法合法）
- 语义约束（要呼应前一行的 hunger 主题）

【干预实验：换个规划，整句重写】

最有冲击力的是这个干预实验。

操作：拿一首随机的诗，**把 "green" 这个词的 feature 强行注入到 newline 这一个 token 上**。

结果：

- 原句：`Illuminating the peaceful night`
- 注入后：`Upon the meadow's verdant green.`

整句被重写了——不是只换一个词，是换了整个句子结构来让结尾落到 green。

而且 Anthropic 明确指出："The steering intervention only has an effect over the newline planning token!" **只在 newline 这一个点注入才有效**，其他位置注入完全没用。

────────────────────

打个比方：

如果末层只是个被动的喇叭，那你注入新规划，应该只是把最后一个 token 换成 "green"——前面的词不变，最后一个词替换。

但实际上发生的事情是：整个句子被重新组织，因为末层不是在"被动采样"，是在"沿着规划反向构造路径"。**改了终点，整条路径都重画。**

这是迄今为止对"嘴不是被动喇叭"最强的一刀因果证据。

━━━━━━━━━━━━━━━━━━━━

◆ 第五节：细胞级解剖——末层有专门的"刹车细胞"和"温度旋钮"

────────────────────

前面四节都是"系统级"证据——看输出、看 feature、看排名顺序。这一节往下走一层，看**具体的神经元**在干什么。

主要参考：
- Gurnee et al., "Universal Neurons in GPT-2 Language Models", TMLR 2024, arXiv:2401.12181
- Stolfo, Wu, Gurnee, Belinkov, Nanda et al., "Confidence Regulation Neurons in Language Models", NeurIPS 2024, arXiv:2406.16254
- Datta et al., "From Early Encoding to Late Suppression: Interpreting LLMs on Character Counting Tasks", arXiv:2604.00778, 2026（Qwen2.5-3B 上的末层抑制量化）

────────────────────

【末层的"刹车细胞"：suppression neurons】

第一节已经提到 suppression neurons 在末层密度暴涨。这里给一个具体细胞例子——

GPT-2 medium 里有个神经元，它的 `W_U·w_out`（输出权重在词表空间的投影）专门对"含开括号 `(` 的 token"打压 logit。一旦这个细胞激活，所有含 `(` 的候选词概率都被压低。

而且这种细胞**只在它要抑制的 token 实际有可能出现的时候才发火**——不是无差别打压，是有针对性的"删稿"。

Datta 等人在 Qwen2.5-3B 上做了一次量化测量（arXiv:2604.00778, 2026，研究的是字符计数任务里的末层抑制）：

- 末层 attention 头抑制约 **9%** 的概率质量
- 末层 MLP 再贡献约 **8%** 的抑制
- 倒数第二层 MLP 抑制约 **4%**

加起来超过 20% 的概率质量是在最后几层被**主动砍掉**的。这不是被动喇叭，是末层在主动"删稿"。

【末层的"温度旋钮"：entropy neurons】

这是这一节最优雅的发现，数学上极其漂亮。

Stolfo 等人在 GPT-2 small 的最后一层（L11）找到 **6 个非常特殊的神经元**：编号 11.584、11.1611、11.2123、11.2378、11.2870、11.2910。

这些神经元有两个奇怪的特征：

1. **权重范数特别大**——在末层 3072 个 MLP 神经元里属于权重范数最大的那一小撮
2. **但它们对 logit 排序几乎没有影响**——`W_U·w_out` 投影方差（论文记作 LogitVar）接近零

论文里有一张二维散点图：横轴 LogitVar、纵轴 norm，绝大多数神经元挤在一团，只有一小撮卡在"低 LogitVar + 高 norm"那一角——上面挑出来的 6 个就是这一角里最显著的几个。

也就是说：它们写入残差流的力气很大，但写完之后好像什么都没改变。**这种神经元为什么要存在？**

────────────────────

【打个比方】

回忆一下 199 期讲过：末层最后会用 LayerNorm 把残差流归一化，公式是：

```
LayerNorm(x) = (x - E[x]) / √(Var[x] + ε)
```

分母里有个 `√Var[x]`——残差流向量的方差越大，分母越大，归一化后的向量被压得越小，最终所有 logit 被均匀压扁，**输出分布的熵就上升**。

这就是 entropy neurons 的机制——它们写入的方向，正好落在 **unembedding 矩阵 `W_U` 的零空间**（论文用 SVD 找到的最小奇异向量子空间）。

这意味着：
- 它们对 logit 排序无影响（因为 `W_U` 投影后归零）
- 但它们大幅增加残差流的方差，从而**胀大了 LayerNorm 的分母**
- 等价于在末端**动态调高 softmax 温度**，让分布变平、熵变大

它们藏在 logit 层面"看不见"的方向上，悄悄拧温度旋钮——**完美的隐形刹车**。

────────────────────

【消融实验】

Stolfo 等人对 L11.2378 这个神经元做了 clip mean ablation（把它的输出强制压到平均值）。

结果：**模型输出的熵下降了 70%。**

也就是说，这一个神经元正常工作时，把整个模型的输出分布熵推高近三倍。它是模型自带的"防过度自信"机制。

【对偶机制：频率先验调节】

光有"升熵"还不够。如果模型只会"把分布拉平防过度自信"，那它就缺另一只手——某些 token 在当前上下文里就是该被压低、某些就是该被拉向先验。Stolfo 在论文里给了一个互补的对偶：除了 entropy neurons，他们在 Pythia 410M 上找到了一个 **token-frequency neuron**（编号 23.417），机制完全相反：

- entropy neuron 写入的方向落在 `W_U` 的**零空间**——只动 LayerNorm 分母、不动 logit 排序，等价于温度旋钮
- token-frequency neuron 写入的方向落在 `W_U` 的**有效空间**，且方向高度对齐到"对数词频"向量——它把残差流朝高频 token 的方向推一把，等价于把分布拉向 unigram 先验

两类机制不是"余弦反相"那种简单镜像，是**沿着 `W_U` 零空间 vs 有效空间这两条正交方向各干各的**：一个动温度，一个动先验。模型有不止一只手在末端做置信度校准。

【跨模型普遍性】

最关键的是：以上这些机制不是 GPT-2 的特例。

Stolfo 在 LLaMA-2 7B、Pythia 各尺寸、Phi-2、Gemma 2B 上都找到了相同结构的 entropy neurons（跨模型的普遍性证据放在论文的附录里，主图只画了 LLaMA-2 7B 单模型的散点）。这是 Transformer 末层的**普遍解剖结构**。

────────────────────

【为什么末层需要这些细胞】

Gurnee 的论文给了一个非常清晰的解读：

> "rather than 'feed-forward layers build predictions by promoting concepts in the vocabulary space,' we claim **late feed-forward (MLP) layers build predictions by both promoting and suppressing concepts in the vocabulary space**. [...] **first affirmative predictions are made, and then the distribution is sharpened or made more calibrated by suppression neurons at the very end**."

回到 199 期讲过的 next-token prediction 梯度压力——cross-entropy 损失函数既惩罚"漏选正确 token"，也惩罚"高置信度地错选"。所以末层必须长出两套对偶细胞：

- **prediction neurons**：抬高正确候选
- **suppression neurons**：砍掉错误候选
- **entropy neurons**：给整体分布"踩刹车"，防止过度自信

这是 logistic 回归梯度直接雕刻出来的解剖结构，不是工程师设计的。**深层的所有"专门细胞"，都是损失函数雕出来的肌肉。**

━━━━━━━━━━━━━━━━━━━━

◆ 第六节：活体证据——layer 42 换 kernel 把"正常英文"翻成"符号循环"

────────────────────

前面五节讲的都是别人论文里的实验。这一节讲我自己撞到的事——是本期论点最硬的活体证据。

185 期（ https://mp.weixin.qq.com/s/dEc-wD0JcM5WhKicOUR1pQ ）和 224 期（ https://mp.weixin.qq.com/s/aOPCfwwo3-2g1DTmCe3i6w ）讲过一个工程谜团：两台 DGX Spark 跑 DeepSeek V4 Flash 280B（43 层 Transformer + MTP），用了一个非常奇怪的混合 kernel 方案才能让模型正确输出——

- decoder layer 0-41 走 Marlin（一种 W4 高速 GEMM kernel）
- decoder layer 42（最后一层）走 Consumer-DeepGEMM（CDG，DeepSeek 自家 FP4 GEMM 的 Triton 重写版）

只要把 layer 42 也换成 Marlin，让 0-42 全部 Marlin，模型立刻乱码——英文输出变成 `</`、`_`、`,` 这种符号循环。

最反直觉的地方：**Marlin 在数值上比 CDG 更接近高精度参考**。手写解量化的 BF16 × FP4 公式（W4A16 语义）算出来的"理论答案"，Marlin 几乎完全贴上去；CDG 反而离这个公式更远。

```text
Marlin vs 高精度公式:  max=0.03,  mean=0.003, cos=0.99999
CDG    vs 高精度公式:  max=0.17,  mean=0.024, cos=0.99940
```

那为什么"更准"的 Marlin 反而乱码？

因为模型在量化训练 / 校准时跑的是 **W4A8** 这条管线——激活先量化成 FP8 再做矩阵乘。CDG 走的就是 W4A8，Marlin 走的是 W4A16（激活保留 BF16）。**两条都是合法的数值世界，但模型的 argmax 边界是按 W4A8 长出来的——它不认 W4A16 那条更精确的路径**。

【关键数字】

mixed（0-41 Marlin + 42 CDG）和 all-marlin（0-42 全 Marlin）的逐层对比里：

```text
layer 0-41:
  x_max_diff   = 0   ← 两边都走 Marlin，确定性 0 误差
  out_max_diff = 0

layer 42:
  x_max_diff        = 0          ← 进入 layer 42 之前两边完全一致
  topk_ids_diff     = 0          ← 路由也完全一致
  topk_weights_diff = 0
  out_max_diff      = 0.16
  out_mean_diff     = 0.025
  out_cos           = 0.99939    ← 余弦相似度看上去几乎一样
```

肉眼看，这是"几乎一样"的两份输出——`cos=0.99939`，最大偏差 0.16，平均偏差 0.025。在普通中间层，这种量级的偏差会被下一层的 RMSNorm 拉回来、被 Softmax 的强非线性吞掉、被残差路径稀释。43 层 Transformer 一路跑下来，这点偏差根本注意不到。

但 layer 42 是**最后一层**。它之后只剩：

```text
hidden state → final_norm → lm_head → logits → argmax 选下一个 token
```

`final_norm` 只是一次归一化、`lm_head` 是一个线性投影——对 hidden state 的偏差**几乎不衰减**。0.16 的偏差几乎原样打到 logits 上。如果两个候选 token 的 logits 本来就咬得很近：

```text
token "Write": 12.031
token "</":    12.028
```

那 hidden state 上 0.1 量级的通道变化，足够把 `Write` 和 `</` 的顺序翻过来。第一个 token 翻错之后，自回归整条换轨：

```text
正确路径："Write a quicksort function in Python..."
错误路径："</_,</_,</_,</_,</_..."
```

────────────────────

【这就是本期论点的活体证据】

回过头看前面讲的论文：

- **Lioubashevski saturation events**：top-k 按排名一颗一颗在末层附近锁定，像"开关跳变"
- **Lad/Tegmark residual sharpening**：末层 MLP 主动写入残差流，并不是被动收尾
- **Gupta + MechLens**：事实类 token 几乎都在末层"凭空召唤"出来

这些都是从论文里读出来的"末层在主动干活、对扰动极敏感"。但 layer 42 这个事故，是**反过来的实测**——你不动模型权重、不动训练、不动前 42 层的任何一个参数，**只换最后一层的 kernel 语义（一种"合法但模型没校准过的"数值路径），整条采样轨迹就翻盘**。

换句话说：

| 论文 | 实测 |
|---|---|
| Lioubashevski：末层是 top-k 锁定的开关 | layer 42 一改，第一颗钉子就敲到隔壁洞里去了 |
| Lad/Tegmark：末层 MLP 主动写入残差流 | layer 42 输出 0.16 的偏差直接打到 logits，没有"下一层"吸收 |
| Gurnee/Stolfo：末层有专门的细胞，对 logit 排序极敏感 | logits 上 0.003 的差距就足以翻 argmax |
| MechLens：事实在末层"结晶"，不是精炼 | layer 42 不是"渲染答案"，是答案最后定型的那一刀 |

**论文里说"末层主动、扰动敏感"，工程实测就是"换最后一层的 kernel 把英文翻成符号"。理论和实测对上了。**

────────────────────

💡 顺便补一个反直觉点：**这不是 Marlin "有 bug"，是 Marlin 算得"太准"了**——准到了模型没校准过的那条数值路径上。224 期的标题就是这个意思："Marlin 不是算错了，而是算得太正确"。

模型的"正确"不是数学上的最优近似，是"和量化校准时跑的那条管线一致"。论文里讲的"末层主动干活"，工程上对应的就是"末层对'走哪条数值管线'极敏感"——这是同一件事的两面。

━━━━━━━━━━━━━━━━━━━━

◆ 第七节：回扣 199 期——修正"丧葬仪式"

────────────────────

回到开头那个自问：199 期的"丧葬仪式"那句话，错在哪？

这一期的五篇论文加上 Anthropic 的押韵实验、再加上我们自己撞到的 layer 42 实测，给出的修正是一句话：

**深层不是被动埋葬，是主动校准。**

| 证据 | 揭示了深层在做什么 |
|---|---|
| Lad/Tegmark 四阶段 | 末层 suppression neurons 密度暴涨，主动删稿 |
| Lioubashevski saturation events | 按排名一颗一颗敲钉子，离散开关式锁定，未训练模型也有 |
| Gupta Guess-then-Refine | 早层猜高频词，深层精炼；不同 token 走不同深度 |
| MechLens FEP | 事实类 token 几乎都在末层"凭空召唤"，不是渲染 |
| Anthropic 押韵实验 | 末层反向规划，改终点会重画整条路径 |
| Gurnee/Stolfo 细胞解剖 | 末层有专门的 suppression / entropy / token-frequency neurons——一组动 logit 排序、一组在 `W_U` 零空间里拧温度、一组把分布拉向 unigram 先验 |
| Layer 42 kernel 实测 | 只换最后一层 kernel 语义，模型从正常英文翻成符号循环——末层对扰动极敏感 |

199 期的核心论断没错——**中间层确实是意图所在**，深层确实丢掉了一些全局语义信息（这是 NTP 梯度的代价）。但深层不是"什么都没干"。它在干一组非常具体的活：

1. **按排名顺序锁定 top-k**——离散的"敲钉子"序列
2. **按 token 难度分配深度**——虚词浅、实词深、多 token 答案首 token 最深
3. **反向规划填充路径**——已知终点，倒推中间词
4. **主动抑制错误候选**——末层 9% + 8% 的概率质量被砍掉
5. **动态校准温度**——在 unembedding 零空间里拧旋钮防过度自信

这些事情都不会显示在信息熵或流形维度那种总量度量里。199 期是从"信息量"看深层（看着像被动），这一期是从"机制"看深层（看到都在主动干活）。同一段网络，换把尺子量，结论完全不同。

💡 最后一个比方，给程序员的：

中间层是产品经理，写下了一份需求文档（也就是"意图"）。深层是工程师团队——工程师没法改需求，但工程师绝不是被动的喇叭：他们要选技术栈、写代码、做单测、跑 lint、抑制 bug、做 code review。每一步都是决策密集的活。

199 期那句"工程师只是把需求文档敲成代码"，低估了工程师的工作量。

更准确的说法是：**工程师在敲代码的同时，还在按编号顺序合并 PR、在零空间里调超参、在末端做置信度校准。他们干的活比"敲代码"复杂得多，只是产品经理（中间层）看不见而已。**

技术名词速查：

- **Residual Sharpening**：Lad/Tegmark 提出的第四阶段——末层用 suppression neurons 主动压低错误候选，把输出分布锐化成尖峰
- **Saturation Event**：从某一层开始，logit lens 投影后排名第 k 的 token 在后续所有层都不变，称此层为 top-k 的饱和层
- **Logit Lens**：把任意中间层的隐藏态用最终的 unembedding 矩阵投影到词表空间，假装当成输出来读
- **TunedLens**：logit lens 的改进版，每层学一个仿射映射对齐到最终输出，早层更可靠
- **Prediction Neuron / Suppression Neuron**：输出权重投影到 logit 空间后呈高峰度分布的神经元，正偏=prediction，负偏=suppression
- **Entropy Neuron**：写入方向落在 unembedding 矩阵零空间的神经元，对 logit 排序无影响但通过 LayerNorm 调节输出熵
- **Cross-Layer Transcoder (CLT)**：Anthropic 用的可解释性工具，把每层 MLP 替换为稀疏激活的"feature 神经元"跨层共训
- **Backwards Planning**：Anthropic 在押韵实验中发现的机制——模型从规划好的终点反向构造路径

━━━━━━━━━━━━━━━━━━━━

「深层不是信息的丧葬仪式——是按排名敲钉子的执行序列、是在零空间里拧温度旋钮的隐形刹车。」

「Transformer 按排名锁定 top-k 的节奏，未训练的随机权重模型里也有——这是架构本身的几何，不是训练学出来的。」

「末层有专门的细胞，藏在 unembedding 矩阵看不见的方向上，悄悄拧着输出分布的温度。它在 logit 上不留痕迹，却把整个分布的熵推高三倍。」

「中间层是产品经理——写下意图；深层是工程师团队——选技术栈、合 PR、跑 lint、抑制 bug。产品经理看不见工程师的活，不代表工程师没在干活。」
