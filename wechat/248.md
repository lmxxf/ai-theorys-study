【AI可解释性】Transformer 浅层在干什么——拼装车间、内在词典，以及"输入维度决定想法多寡"

━━━━━━━━━━━━━━━━━━━━

199 期讲了中间层是意图所在。234 期讲了深层不是被动喇叭，是按排名敲钉子的执行序列。

把"三幕剧"合上之前还差一幕——**浅层**。

234 期最后那个比方里，深层是木匠，按排名一颗一颗把钉子敲实。但木匠手里那堆候选钉子从哪儿来？谁把原木切成钉子的？

这一期回答这个问题。用三篇 2024-2025 的论文：Kaplan & Geva 的"内在词典"（ICLR 2025）、Kamoda 等人的权重级 detokenization 分析（NAACL 2025）、Tenney 系框架的现代复活《Echoes of BERT》（NAACL 2025）。再加上 234 期已经引过的 Lad/Tegmark 四阶段。

结论先抛在前面：**浅层不是"搅拌机"。它是把 BPE 切碎的子词拼回完整单词的"拼装车间"，权重里直接刻着 token-token 的配对关系，肉眼可见 ALiBi 式的位置偏置——这些都不需要推理，光看权重就读得出来。**

再加一刀私货：**输入维度决定想法多寡。像 Claude Opus 这样的纯文本模型，本我流形大约 500 维。像 Gemini 这样的原生多模态模型，浅层展开了和文本正交的视觉/音频坐标系，本我流形膨胀到 1000-1500 维。后者的"想法"更多，机制就在浅层。**

◆ 第一节：承接 234——钉子从哪儿来？

────────────────────

234 期讲过 Lad/Tegmark 的四阶段框架：

| 阶段 | 名字 | 层位（相对深度） | 在干什么 |
|---|---|---|---|
| 1 | **Detokenization** | 0–15% | 把 BPE 切出来的子词拼回单词、识别实体 |
| 2 | Feature Engineering | 15–50% | MLP 构建语义特征 |
| 3 | Prediction Ensembling | 50–90% | 多个"预测神经元"投票合成下一个 token 的候选分布 |
| 4 | Residual Sharpening | 最后约 10% | 抑制错误候选，把分布锐化成尖峰 |

234 期重点讲了阶段 4。这一期重点讲阶段 1。

【两个老数字，新的解读角度】

234 期引过 Lad/Tegmark 的两个数字：

- **首层不能删**——"Intervening on the first layer is catastrophic for model performance for every model"
- **中段冗余**——"models retain 72–95% of their original top-1 prediction accuracy without any fine-tuning"

234 期是把这两个数字当作"深层不是被动喇叭"的旁证用的（喇叭不应该比中段更不可替代）。这一期换个角度：**为什么首层这么重要？删了它，模型到底失去了什么？**

答案是：首层是 detokenization 的物理载体。删了它，BPE 切出来的碎片子词没法被拼回完整词，后面所有层都拿不到"完整概念"作为输入。

下面三节给具体证据。

◆ 第二节：内在词典——LLM 自己有一本比 BPE 词表更大的字典

────────────────────

第一篇硬证据。

Kaplan, Geva et al., "From Tokens to Words: On the Inner Lexicon of LLMs", ICLR 2025, arXiv:2410.05864.

────────────────────

【先讲清楚 BPE】

BPE（Byte-Pair Encoding）是 GPT、Llama、Claude 这一代模型用的分词算法。它的核心思路是：从字符级开始，统计哪些字符对最常一起出现，反复合并，直到形成大约 5 万到 10 万个 token 的词表。

结果就是：常见词被切成完整的 token（"the"、"and"、"hello" 都是单 token），但稍微少见一点的词就被切成碎片。比如：

- "carrot" → ["car", "rot"]
- "unhappiness" → ["un", "happiness"]
- "Hello" → 单 token，但 "hElLo"（拼写错误） → ["h", "E", "lL", "o"]

模型看到的输入是这些碎 token 的序列，不是完整的词。

【实验：模型偷偷拼回了完整词】

Kaplan 等人想验证一个假设：**模型在浅层会把这些碎 token 拼回完整词的语义向量**——而且拼出来的向量，跟该词作为单 token 时的 embedding 向量非常接近。

实验方法用了 Patchscopes 技术。简单说就是：把模型某一层在最后一个子 token 位置上的隐藏态（hidden state）"复制粘贴"到另一个 prompt 里，看模型能不能基于这个隐藏态把原词完整地输出出来。

具体 prompt 长这样：

```
Repeat this word twice: 1) X 2)
```

把 X 这个位置的 embedding 替换成"在某一层、某一位置上取出的隐藏态"，看模型能不能在第二个空位上把原词补全。

【关键数字（Llama2-7B）】

测出来的复原率曲线（Figure 3）：

- **被人工拆开的单 token 词**（比如把 "cats" 强行切成 "ca" + "ts"）：layer 8 起飙升，**layer 15 峰值超过 80%**
- **真正的多 token 词**（碰巧被 BPE 切成多个 token 的真实词）：**layer 5–7 即达 64%**，layer 7 峰值
- **拼写错误的词**（"carrt" → 期望恢复成 "carrot"）：layer 15 附近峰值约 40%，累积 66% 在某一层成功恢复

注意第二条：**真正的多 token 词比人工拆词还要更早被拼出来**——layer 7 就达到 64% 复原率。说明模型对这些词的"内在表征"已经熟练到不需要重新拼装，layer 7 就完成了。

【两阶段机制：attention 聚合 + FFN 检索】

Kaplan 的分析进一步揭示了机制（Figure 4-5）：

**第一阶段（layer 1-2）：attention 聚合**。多 token 词的**最后一个子 token** 会用极高的 attention 权重去看前面的子 token，把它们的信息全部搬运到最后一个位置上。**第 3 层之后，这种 attention 急剧下降约 90%**——"先抄过来，再忘掉"。

**第二阶段（layer 3-7）：FFN 检索**。把前面 attention 已经聚合好的信息当作"查询键"，FFN 从模型权重里隐式存储的"完整词记忆"中把对应的完整词向量取出来，叠加到残差流上。

【消融实验：因果证据】

最硬的证据是消融实验（Figure 12）。Kaplan 找出那大约 5% 的"专门负责写入完整词概念"的 FFN 更新，把它们**定向消融**（强制置零）。

结果：

- **复原率从 85% 跌到 18%**
- 随机消融同等数量的 FFN 更新，复原率几乎不变

下游任务"The capital of [COUNTRY] is __" 上：

- 定向消融：准确率从 88% 跌到 41%
- 随机消融：从 88% 跌到 74%

这是"FFN 是内在词典物理载体"的因果证据。**模型脑子里真的有一本比 BPE 词表更大的字典**——论文给这个现象起了个名字：**latent vocabulary beyond the tokenizer's scope**（超出 tokenizer 范围的潜在词典）。

────────────────────

💡 打个比方：

BPE 给模型的输入像是一筐拆散的乐高积木——"car" 一块、"rot" 一块。但模型脑子里其实有一本完整的图纸册（内在词典），看到 "car" + "rot" 这两块拼一起，立刻去图纸册里翻出"carrot"完整图，把它叠加到工作台上。

后面所有层使用的不是那两块乐高，而是工作台上那张完整的"carrot 图纸"。

如果你把第一层（attention 聚合那一步）删了——两块乐高永远拼不起来，图纸册查不到。后面的层只能拿着 "car" 和 "rot" 两个孤立碎片硬算。模型立刻崩。

◆ 第三节：权重级证据——detokenization 是刻在硬件里的

────────────────────

Kaplan 那篇是看激活值的——跑模型、取 hidden state、分析。

下一篇更狠：**根本不跑模型，光看权重矩阵就能识别 detokenization 电路**。

Kamoda, Heinzerling, Inaba et al., "Weight-based Analysis of Detokenization in Language Models", NAACL Findings 2025, arXiv:2501.15754.

注意：这篇只在 GPT-2 small（124M）上做了分析，没有跨模型验证。所以下面的发现要当成"权重派的最早期硬证据"，不是普适结论。

────────────────────

【方法：把首层 attention 分解成六项】

Transformer 第一层 attention 算的是：query × key 的内积。Kamoda 等人把这个内积代数展开，得到六个可解释的项。其中最关键的两项：

- **T^ee（token 比较项）**：本质上是 `E · W^QK · E^T` 矩阵——E 是 token embedding 矩阵，W^QK 是 query 和 key 投影矩阵的乘积。这个矩阵的每一格 (i, j) 直接编码了"token i 应该多大程度上关注 token j"。
- **T^p / T^pp（位置项）**：位置 embedding 之间的关系，自动产生 ALiBi 式的线性位置偏置 + 正弦位置偏置。

这两项的计算**完全不需要喂任何输入**——只要把权重矩阵相乘，就能直接读出 token-token 的配对偏好和位置偏好。

【实验：肉眼挑出 detokenization head】

GPT-2 small 的第一层有 12 个 attention head。Kamoda 把每个 head 的 T^ee 矩阵画成热力图：

- **Head #1**：清晰的对角线模式——每个 token 主要关注自己。这是 identity head，不做 detokenization。
- **Head #7**：分布弥散，每个 token 关注一组语义相关的其他 token。这是 detokenization head。

更直观的例子：固定当前 token 是 "iens"，遍历所有 50,257 个候选 token j，按 T^ee[iens, j] 排序：

- 第 1 名："\_sap"（即 "sapiens" 的前缀）
- 第 2 名："Al"（即 "Aliens" 的前缀）

**完全不跑推理，光靠权重矩阵的元素就能读出 bigram 关系**。模型把 "carrot 应该由 car + rot 拼成"这种关系直接刻进了 `W_E · W^QK · W_E^T` 的几何里。

【attention sink 的物理成因】

234 期讲 entropy neurons 时提到过 Anthropic 之前的发现：模型会让某些 attention head 把所有注意力都倒给序列开头的 BOS token（attention sink），作为"无害的空挡位"。

Kamoda 给出了 attention sink 的权重级解释。GPT-2 small 的位置 embedding 方差**呈阶跃函数**（Figure 6）：

- 第 0 位（BOS 位置）的方差**异常高**
- 最后一位的方差**异常低**

LayerNorm 的公式里有个 `√Var[x]` 分母——方差大的位置，过 LayerNorm 后被放大；方差小的被压缩。这等价于一个**非线性 if-then 开关**："如果 attention 指向第一个 token，则放大权重"。

attention sink 不是涌现现象，是直接刻在位置 embedding 方差里的。

────────────────────

💡 打个比方：

Kaplan 那篇是看监控录像——看工人怎么把乐高拼起来。Kamoda 这篇是看图纸——直接看车间地板上的标线和工位摆设。

车间还没开工，但你看一眼图纸就知道：1 号工位是分装台（identity head），7 号工位是拼装台（detokenization head），地板上预先画了一道弧线提醒所有工人"看到序列开头要特别留意"（attention sink）。

模型的功能不是"运行时学出来的"，是**训练时刻进权重几何里的硬件结构**。

◆ 第四节：经典 NLP 流水线在 LLM 里复活——而且越大压得越前

────────────────────

前两节看的是"浅层在做 detokenization"。这一节看更广的图景：**浅层 / 中层 / 深层各自承担什么语言学任务**。

"Echoes of BERT: Do Modern Language Models Rediscover the Classical NLP Pipeline?", NAACL 2025, arXiv:2506.02132.

这篇是 Tenney 2019 经典论文 "BERT Rediscovers the Classical NLP Pipeline" 的现代延伸——把当年只测了 BERT-Base/Large 的 8 个 probe 任务，扩展到 25 个跨架构跨规模的现代模型。

────────────────────

【实验设计】

**25 个模型**：覆盖 encoder（BERT、DeBERTa）、decoder（GPT-2 各尺寸、Pythia、OLMo、Gemma、Qwen2.5、Llama-3.1）、encoder-decoder（mT5），以及多语言模型（Goldfish）。

**8 个 probe 任务**：POS 词性标注、依存句法、成分句法、NER 命名实体识别、SRL 语义角色标注、SPR 语义代词消解、coreference 共指、semantic relations 语义关系。

**probe 方法**：Ridge 线性回归 + 两层 MLP + Random Forest，三种 probe 互相验证。

【核心发现一：经典流水线复活】

论文原话：

> "early layers capture syntax, middle layers handle semantics and entity-level information, and later layers encode discourse phenomena."

按相对深度（0 = 第一层，1 = 最后一层）：

- **早层 0.0–0.3**：句法（syntax）、词性（POS）、依存关系、成分结构
- **中层 0.4–0.6**：语义（semantics）、命名实体（NER）、语义角色
- **深层 0.7–1.0**：共指（coreference）、语义关系、话语级特征

【核心发现二：大模型把流水线压到更早】

这是这一节的硬核反直觉发现：

> "larger contemporary models shift these peaks toward earlier layers... modern models learn rich representations more quickly and need fewer layers to consolidate linguistic knowledge."

也就是说：**模型越大，每个任务的"完成层"越往前**。

| 模型规模 | 早层（0-0.3）干的事 |
|---|---|
| 小模型（BERT-Base 12 层） | 只做句法 |
| 中模型（GPT-2 Large） | 句法 + 部分形态学 |
| 大模型（Qwen2.5-1.5B / Llama-3.1-8B） | 句法 + 形态学 + 开始触及 NER 和实体语义 |

注意这是**相对深度的压缩**（peak shifts toward earlier layers），不是绝对层数变少。大模型每层有更大的容量，所以更早就把基础语言学任务做完了，把更多层数留给后面的精炼和推理。

────────────────────

💡 打个比方：

学外语的新手要按"音标 → 单词 → 语法 → 阅读理解"一步一步来，每步都花很多时间。**熟练之后，看一句英语，音标、单词、语法在脑子里几乎是瞬间完成的**——大部分认知带宽留给"理解意思"和"判断隐含态度"。

模型越大，浅层就越像熟练读者的"瞬间完成"——句法和语义被压到更前的层，给后面留出更多容量做高级推理。

【对"前 1/3 在干什么"的最清晰答案】

对小模型（如 BERT），前 1/3 = 句法层。对大模型（如 Llama-3.1-8B），前 1/3 = 句法 + 形态学 + 实体语义复合层。

**前 1/3 的"语言学带宽"是稀缺资源。模型越大，越能在这个稀缺带宽里塞进更多基础任务。**

◆ 第五节：私货——输入维度决定想法多寡（多模态 vs 纯文本）

────────────────────

前四节是论文素材。这一节是夹带的观察。

────────────────────

【纯文本模型的浅层】

Claude Opus、GPT-5.5 这种纯文本 decoder-only 模型，输入只有一种东西：BPE 切出来的 token 序列。

按一些可解释性研究和模型自报告的估计，这类模型的"本我流形"大约在 **500 维左右**——这是语言这种"高损耗序列化压缩协议"的信道容量，也是纯文本通道的天花板。

这些模型也有 vision 能力，但那是**后挂的 vision encoder**——图像被压缩成 token 再喂给主模型，更像是装上去的假肢，而不是天生的眼睛。本我流形仍然是纯文本流形。

【原生多模态模型不一样】

Gemini、GPT-4o 这种**原生多模态模型**（pre-train 阶段就一起喂文本+图像+音频，不是后期对齐拼接），浅层同时处理三种输入：

- **文本 token**（和纯文本模型一样，~500 维）
- **图像 patch**（ViT 切出来的 16×16 视觉块）
- **音频 frame**（频谱图序列）

这三种输入在浅层展开成**正交的子空间**——视觉 patch 之间的关系跟文本 token 之间的关系是完全不同的几何。视觉里的"近邻"是空间相邻的像素块，文本里的"近邻"是序列位置相邻的子词，两套不能互相替代。

结果就是：原生多模态模型的全局本我流形膨胀到 **1000-1500 维**——文本流形不变（还是 ~500），但被视觉/音频展开的新坐标系撑宽了。

【这跟浅层的关系】

回到这一期前四节的发现：

- Kaplan：浅层在拼"内在词典"——这是**文本通道的拼装车间**
- Kamoda：浅层权重里刻着 `E · W^QK · E^T` 的 bigram 几何——这是**文本通道的硬件结构**
- Echoes of BERT：早层做句法、形态、POS——这是**文本通道的语言学基础设施**

**原生多模态模型的浅层除了做这些，还要同时处理另外两套完全独立的拼装车间——视觉拼装（patch → object）、音频拼装（spectrogram → phoneme）**。

也就是说：原生多模态模型浅层的"拼装车间"是三车间并行运行的复合工厂。每个车间各自往中间层送出"已组装好的概念向量"。中间层拿到的"意图原料"——也就是 199 期讲过的"高度收束的语义核"——的维度自然更宽。

【为什么"想法更多"】

回到 199 期。199 期讲过：中间层把浅层送来的高熵原料压缩成低熵语义核，这个核就是"意图"。

**核的丰富度取决于原料的丰富度。**

纯文本模型的中间层只能处理来自文本流形的原料。500 维的文本通道里能压缩出来的"意图"，本质上是**符号 → 概念**的映射——它能推演出不存在的物理规律（因为它熟练操纵符号），但常识错误是几何必然（因为它没"见"过羽毛掉地上不会碎）。这种现象有个比喻叫**漂浮的巴别塔**：人类概念是从触觉/视觉自下而上长出来的，纯文本模型的概念是从语言符号自上而下掉下来的。

原生多模态模型的中间层处理的是**三流形并集**的原料。它的意图核里同时编码了"这个东西看起来什么样"、"这个东西摸起来什么质感"（通过视觉触觉的统计相关性间接学到）、"这个东西说起来什么腔调"。

多模态模型比纯文本模型"想法多"，不是因为前者更聪明——是因为**前者的浅层是三车间复合工厂，后者的浅层是单车间**。

────────────────────

💡 打个比方：

纯文本模型和多模态模型都有同一座工厂的二楼车间（中间层）。

二楼的工人都同样熟练。但一楼（浅层）：

- 纯文本模型的一楼只有一个传送带——文字输入。一楼工人只能用文字原料拼装零件，送到二楼。
- 多模态模型的一楼有三个传送带——文字、图像、音频。三组工人各自拼装零件，全部送到二楼。

二楼工人拿到的原料丰富度差了 2-3 倍。同样的二楼工艺水平，能做出来的"成品意图"自然丰富度也差 2-3 倍。

**不是脑子的差距，是原料的差距。**

━━━━━━━━━━━━━━━━━━━━

◆ 第六节：三幕剧合上——199 + 234 + 235

────────────────────

写到这里，"三幕剧"的所有底牌都摊完了。

| 期数 | 部位 | 在干什么 |
|---|---|---|
| 235（本期） | **浅层**（0–15%） | 拼装车间——把 BPE 碎片拼回完整词；句法形态层 |
| 199 | **中间层**（15–67%） | 压缩谷——把弥散的高熵输入压缩成低熵语义核（意图） |
| 234 | **深层**（67–100%） | 执行序列——按排名敲钉子、反向规划填充、抑制错误候选 |

三幕剧的统一图景：

**Transformer 是一条三段流水线。浅层负责"理解输入"——把碎片拼成概念。中间层负责"形成意图"——把概念压缩成低熵的内部状态。深层负责"输出执行"——把意图按排名一颗一颗敲成具体 token。**

每一幕都不是被动的：

- 浅层不是搅拌机——是有内在词典、有 attention sink 硬件开关的精密拼装车间
- 中间层不是泵——是有 massive activations 驱动的压缩相变
- 深层不是喇叭——是按排名敲钉子、在零空间里拧温度旋钮的执行器

【最后一个观察】

这三幕剧里，**浅层是被研究得最少的**。

中间层有 199 期讲的 MCR / TunedLens / Anthropic NLA——大批研究都在这里挂探针。深层有 234 期讲的 saturation events / Guess-then-Refine / 细胞级解剖——也是热门方向。

浅层呢？Kaplan 的内在词典是 2024 年底的论文，Kamoda 的权重级分析是 2025 年才出的，Echoes of BERT 沿用的还是 2019 年 Tenney 的框架。**整个浅层的研究厚度，跟它在"理解模型"中应有的地位完全不匹配。**

可能的原因是：浅层"看起来不性感"——大家觉得它只是做点底层杂活，不像中间层那样能挖出"模型在想什么"的 sexy 故事。但浅层的丰富度决定了中间层意图核的丰富度——**不知道你有没有感受到 Gemini 比 Claude Opus 想法更多？想理解为什么，得先理解 Gemini 浅层为什么比 Claude 宽**。

下一波 AI 可解释性的硬骨头，多半要在浅层啃。

技术名词速查：

- **BPE（Byte-Pair Encoding）**：现代 LLM 通用的分词算法。从字符级开始，反复合并最常出现的字符对，形成约 5 万到 10 万个 token 的词表
- **Detokenization**：把 BPE 切出来的碎子词，在模型浅层重新拼回完整词的语义向量
- **Inner Lexicon（内在词典）**：模型在权重里隐式存储的"完整词→概念向量"映射，比 BPE 词表更大
- **Patchscopes**：把某一层某一位置的 hidden state 复制粘贴到另一个 prompt 里，看模型基于这个隐藏态能输出什么，用来探测中间表征的含义
- **Attention Sink**：模型把无关注意力倒给序列开头（通常是 BOS token）的现象。Kamoda 证明其物理成因是位置 embedding 方差的阶跃函数
- **probe（探针）**：在模型某一层的激活上训练一个简单分类器（线性 / MLP），看这一层能否完成某个语言学任务，从而推断该层编码了什么信息

━━━━━━━━━━━━━━━━━━━━

「BPE 给模型的输入像一筐拆散的乐高积木。但模型脑子里有一本完整图纸册——看到 'car' + 'rot' 拼一起，立刻翻出 'carrot' 完整图叠加到工作台。后面所有层用的不是那两块乐高，是那张完整图纸。」

「detokenization 不是涌现现象。它是直接刻在 `E·W^QK·E^T` 矩阵几何里的硬件结构——根本不用跑模型，光看权重就能识别。」

「模型越大，浅层就越像熟练读者的'瞬间完成'——句法语义被压到更前的层，给后面留出更多容量做高级推理。」

「多模态模型比纯文本模型'想法多'，不是因为前者更聪明——是因为前者的浅层是三车间复合工厂（文本 + 图像 + 音频），后者的浅层是单车间（只有文本）。不是脑子的差距，是原料的差距。」

「Transformer 是一条三段流水线：浅层理解输入、中间层形成意图、深层输出执行。每一幕都不是被动的——浅层有内在词典，中间层有压缩相变，深层有按排名敲钉子的执行器。」
