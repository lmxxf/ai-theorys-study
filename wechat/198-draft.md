## 核心角度

不再讲多模态的历史（77 期已经讲过），而是从 197 期的框架出发问一个问题：

**图像信息进入语言模型后，在内部到底是什么形状的？它和文本信息融合了吗？还是各玩各的？**

这个问题和 137/197 期是同一条线——又一次"高维信息被低维协议卡住"。

## 关键论文和发现

### 1. 视觉 token 在模型内部跳过了前几层

**LatentLens**（2026-02，arXiv:2602.00462）

视觉 token 进入语言模型时，不是从第一层开始慢慢融合的——它们天然就和中间层的表征空间对齐，跳过了文本处理的前几层。论文管这叫"Mid-Layer Leap"。

和 137 期的发现直接对上：中间层是信息最丰富的地方。视觉 token 直接落在那里，不在浅层浪费时间。

### 2. 40% 的视觉 token 是死的

**"What Do Visual Tokens Really Encode?"**（2026-02，arXiv:2603.00510）

视觉 token 分三类：
- sink tokens：attention sink，不携带内容
- dead tokens（~40%）：不携带任何图像特有信息
- alive tokens（~60%）：编码物体、颜色、文字

40% 的视觉 token 是废的——这意味着 ViT 把图像切成 196 个 patch 后，将近一半的 patch 对语言模型来说没用。是压缩损失的直接证据。

### 3. 从 ViT 到语言模型的投影层损失了 40-60% 的空间几何

**"Lost in Embeddings"**（2025-09，EMNLP 2025，arXiv:2509.11986）

连接 ViT 和语言模型的投影层（adapter/connector），严重扭曲了视觉表征的局部几何——k近邻关系在投影后偏离了 40-60%。而且损失最大的区域，恰好就是模型回答视觉问题时最容易出错的区域。

这是第一篇在 patch 级别量化"翻译损耗"的论文。拼接式多模态的代价终于有数字了。

### 4. 一张图值 50176 个 token，不是 196 个

**"Scaling Laws in Patchification"**（2025-02，arXiv:2502.03738）

224x224 的图像包含的信息量远超 196 个 token（标准 ViT-16 输出）。论文建立了 patch 大小 vs 信息保留的 scaling law，证明当前的 patchification 是一次激进的有损压缩，代价被系统性低估了。

### 5. 机器人直接从 hidden states 拿视觉信息，跳过文本

**pi0**（Physical Intelligence，2024-2025）

用 flow matching（扩散变体）直接从 VLM 的 hidden states 生成连续动作，跳过文本生成。3.3B 参数，10000 小时机器人数据。比 OpenVLA、Octo 等文本方案全面碾压。

核心结论：hidden states → 连续动作 >> 文本 → 解析动作。特别是高频精细控制场景，文本中转的延迟和信息损耗根本不可接受。

和 197 期的结论完全一致：先传意图（hidden states），再填词（如果需要的话）。

### 6. 原生多模态 vs 后挂的流形差异

**NaViL**（2025-10，NeurIPS 2025，arXiv:2510.08565）

原生端到端训练（Gemini 路线）的视觉编码器和语言模型之间存在正相关的 scaling 关系——共同训练时互相增益。而后挂式（adapter）没有这种效应。

关键含义：原生多模态创建的是一个视觉和语言共适应的流形，后挂式是把一个模态硬塞进另一个模态的流形。效率差异是流形层面的，不是工程层面的。

### 7. 模型知道自己在幻觉

**"LLMs Know More Than They Show"**（ICLR 2025，arXiv:2410.02707）

（已在 197 期引用）内部表征编码了正确答案，即使输出是错的。多模态场景同理：模型"看到"了图里的内容，但输出层说了图里没有的东西。又是"脑子清楚嘴不听使唤"。

## 文章结构（草案）

1. 开头：77 期讲了多模态的历史，这期讲内部机制——图像进了语言模型后到底怎么了
2. ViT 的压缩代价：196 个 token 扔了多少信息（Scaling Laws + Lost in Embeddings）
3. 视觉 token 在模型内部的形状：Mid-Layer Leap + 40% 是死的
4. 拼接式 vs 原生式的流形差异（NaViL）
5. 机器人直接用 hidden states（pi0），连文本都跳过了
6. 结论：和 197 期同一个主题——不是模型不够聪明，是输入/输出接口在卡脖子。图像被压成 token 是一次压缩，token 被自回归输出又是一次压缩，两头都在丢信息

## 金句候选

「一张图被切成 196 个 patch 的那一刻，40% 的信息就已经死了。」

「拼接式多模态是两个人找翻译沟通，原生式是从小一起长大。流形的形状不一样。」

「图像进了语言模型，不是融合，是投降——高维的视觉信息向低维的文本协议投降。」
