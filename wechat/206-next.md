# 206 期后续实验备忘

## 实验：80 期 Encoder 模型的正交性验证

**背景**：206 期写作时发现一个未验证的推测——80 期用 Encoder（不是 ntp）训练的 LFSR-31 模型，表征坍缩到了 2-4 维。Shai et al. 用 ntp 的 GPT-2 证明了 Transformer 会自发正交分解。问题是：**非 ntp 的 Encoder 也会正交吗？**

**推测**：大概率会。正交压力来自 Transformer 的线性读写机制（QKV 矩阵乘法），不依赖于训练目标是 ntp 还是分类。

**实验方案**：
1. 加载 80 期训练好的 LFSR-31 模型（0.3M，ctx=16，test 50%）
2. 取中间层 activation（每个 token 一个 128 维向量）
3. LFSR-31 每步输出 8 bit，每个 bit 是一个天然因子
4. 按 bit 位分组：对每个 bit 位，找到该 bit=0 和 bit=1 的样本集，算两个集合在 activation 空间里的判别方向
5. 测量 8 个 bit 对应的 8 个判别方向之间的夹角（principal angles）
6. 如果趋近 90° = 正交实锤；如果不正交但低维 = 说明 Encoder 压缩但不正交

**代码位置**：https://github.com/lmxxf/grokking-train-learnability

**如果正交**：可以写一篇"80 期番外"——Encoder 做分类也会正交分解，正交压力来自架构不是训练目标。

**如果不正交**：更有意思——说明 ntp 训练目标对正交性有独立贡献，不只是架构的事。

## 实验：对权重矩阵做 TDA（拓扑数据分析）

**背景**：206 期第三篇论文（Gardinazzi et al.）对 activation（残差流的逐层快照）做 TDA，发现了四阶段拓扑模式（早层短命空洞爆发 → 中层长命空洞稳定 → 中后层专业化 → 末层再爆发）。但这测的是"水流过管道后的形状"。

**问题**：对权重矩阵本身做 TDA 会看到什么？权重是"管道的形状"——把权重矩阵的行或列当成高维点云，测拓扑空洞，能不能看到和 activation TDA 对应的结构？

**为什么有意思**：
- 我们 200 期对权重做的是奇异值分析（eRank/TwoNN），测的是"有效维度"——但维度不等于拓扑。低维流形可以有洞也可以没洞，拓扑空洞是独立于维度的几何信息
- 目前没看到有论文系统研究过权重矩阵的拓扑结构，可能是空白
- 如果权重 TDA 的逐层模式和 activation TDA 的四阶段模式能对应上，说明管道的形状决定了水流的形状（废话，但用拓扑语言验证就不是废话了）
- 如果对应不上，更有意思——说明 activation 的拓扑结构是数据和权重交互涌现的，不是权重单方面决定的

**实验方案**：
1. 取 Qwen3.6-27B 或 DeepSeek V4 Flash 的权重（我们已经有了）
2. 对每一层的 Q/K/V/O 权重矩阵，把行向量当成点云
3. 用 persistent homology 计算各层的拓扑空洞（Betti 数、persistence diagram）
4. 看逐层变化是否呈现类似四阶段的模式

**工具**：giotto-tda 或 ripser，之前 69/99 期 Grokking 实验用过
