【线性注意力】Mamba-3用复数和旋转干掉 Transformer 的 O(n²)——你的 API 账单有救了

━━━━━━━━━━━━━━━━━━━━

你有没有算过自己每个月在 OpenAI 上花多少钱？

我有个朋友做了个 AI 客服系统，每天处理几千条对话。上个月他发给我一张截图——API 账单 4700 美元。他说：代码没改过，用户也没暴涨，就是对话轮次多了一些。

为什么？因为 Transformer 的注意力机制天生是 **O(n²)** 的。虽然有 KV cache 优化（把历史的 Key/Value 缓存起来，这样 decode 阶段每生成一个新 token 只需要 O(n) 计算），但 prefill 阶段（一次性处理你的整个 prompt）还是 O(n²)，而且 KV cache 本身随上下文线性增长，上下文一长显存就爆。

这就像你用 MySQL 加了索引——单条查询快了，但索引本身占的磁盘空间跟数据量成正比，表一大索引也跟着爆。任何一个程序员都知道：O(n²) 的计算 + O(n) 的内存，这东西迟早要出事。

但 Transformer 的 O(n²) 已经统治了八年。原因很简单：**它效果太好了，好到你宁可花钱也不想换。**

直到有人证明：不换效果，也能把 O(n²) 干掉。

2026 年 3 月，Albert Gu 和 Tri Dao 团队发了 Mamba-3（arXiv: 2603.15569），被 ICLR 2026 收录。这俩人是 Mamba 系列的缔造者，都是斯坦福出来的亚裔——Gu 是华裔，Dao 是越南裔。有意思的是，Tri Dao 之前最出名的工作是 FlashAttention，也就是给 Transformer 的 attention 做加速。给 Transformer 续命的人，转头做了 Transformer 最大的竞争对手。核心结论：**1.5B 参数规模下，比当前最强的线性模型 Gated DeltaNet 高 1.8 个百分点，同时推理时计算量线性、内存恒定。**（Gated DeltaNet 是 NVIDIA 和杨松林团队在 ICLR 2025 发的架构，把 Mamba-2 的门控和 DeltaNet 的增量更新规则结合起来，阿里的 Qwen3.5 已经在用了——75% 的层用 Gated DeltaNet，25% 用传统 attention。所以 Mamba-3 赢的不是纸老虎，是已经上了生产线的对手。）

更狠的是：Mamba-3 用 Mamba-2 **一半的 state size**，就达到了同等 perplexity。

今天我们来拆这篇论文，看看他们到底用了什么数学魔法。

◆ 为什么 Transformer 这么贵——O(n²) 的罪与罚

先说清楚 Transformer 贵在哪。

Transformer 的核心是 self-attention：对于一句话里的每个词，它要去"看"这句话里的**所有其他词**，算出一个注意力权重。

如果你的输入有 n 个 token：
- 每个 token 都要跟 n 个 token 算注意力 → n × n = n² 次计算
- 还要存一个 n × n 的注意力矩阵 → 内存也是 O(n²)

n = 1000 的时候，100 万次计算，没问题。
n = 100,000 的时候，100 亿次计算，GPU 开始冒烟。
n = 1,000,000 的时候，1 万亿次计算，你的钱包开始冒烟。

💡 **翻译成人话：** Transformer 处理文本就像一个社恐被逼参加大型社交——每个人都要跟在场所有人握一次手。10 个人还行，1000 个人你试试？

而且这个问题在**推理阶段**（inference）更要命。训练是一次性的成本，推理是每天都在花的钱。你的 ChatGPT 每回答一个问题，背后都在做 O(n²) 的计算。上下文窗口越大，每一步推理越贵。

这就是为什么所有大模型厂商都在卷"如何让推理更便宜"——因为推理成本直接决定了 AI 能不能真正普及。

━━━━━━━━━━━━━━━━━━━━

◆ SSM 是什么——状态空间模型，程序员版解释

好，那有没有办法不做 O(n²) 的全员握手？

有。用**状态空间模型**（State Space Model，SSM）。

如果你入行够早，听到"维护一个隐藏状态，每来一个新输入就更新一次"，可能会说：这不就是 RNN 吗？

没错。SSM 本质上是 RNN 的高配复活。简单回顾一下这条线的历史：

- **RNN（1986）**：有状态递推，但梯度消失，记不住长距离依赖
- **LSTM（1997）**：加了门控机制（遗忘门、输入门、输出门），能记更长了，但串行训练慢
- **Transformer（2017）**：砍掉递推，全部并行算 attention——训练快了，但 O(n²)，推理贵
- **SSM/Mamba（2023）**：回到递推，但用连续时间微分方程的离散化替代 tanh，用选择性机制替代固定门控——**把 RNN 的 O(1) 推理优势拿回来了，同时解决了梯度消失和训练并行化**

兜了一大圈，又回到了"维护一个状态向量"的思路。只不过数学工具升了两代，从手动挡换成了自动挡。

SSM 的核心思路完全不同于 Transformer。Transformer 是"每个词看所有词"，SSM 是"每个词更新一个状态，下一个词只看这个状态"。

用程序员的话说：

**Transformer = 每次查询都全表扫描**
**SSM = 维护一个实时更新的缓存，每次查询只看缓存**

```python
# Transformer 的做法（伪代码）
for each new_token:
    attention = new_token.attend_to(ALL_previous_tokens)  # O(n)
    output = weighted_sum(ALL_previous_tokens, attention)

# SSM 的做法（伪代码）
state = initial_state  # 固定大小的缓存
for each new_token:
    state = A * state + B * new_token  # O(1) 更新
    output = C * state                  # O(1) 读取
```

看到区别了吗？SSM 的每一步都是 O(1)——不管你之前处理了 100 个词还是 100 万个词，每一步的计算量都一样大。总计算量 O(n)，线性。内存？state 的大小是固定的，不随序列长度增长。

💡 **翻译成人话：** Transformer 是图书馆管理员，每来一本新书就要把所有书重新翻一遍。SSM 是写日记的人——每天写一页总结，新事情发生了就更新今天的总结，不需要翻以前的日记。

拆开看这三个矩阵在干什么：

- **A（遗忘矩阵）**：决定旧记忆保留多少。A 的某个维度接近 1，那个维度的记忆就保留得久；接近 0，就快速遗忘。**A 就是遗忘曲线。**
- **B（写入矩阵）**：决定新信息怎么写进记忆。B 大，新信息写入力度强；B 小，轻轻记一笔。
- **C（读取矩阵）**：决定从压缩的状态里提取什么信息来输出。

A、B、C 都是训练学出来的参数。但这里有个关键区别——也是 Mamba 系列最重要的创新：

**早期的 SSM**（比如 S4，2021）：A、B、C 是固定的，对所有输入都一样。不管你输入什么，遗忘曲线和写入方式都不变。效果差强人意。

**Mamba-1 开始**：A、B、C **依赖于当前输入**，每个 token 进来都会动态生成不同的 A、B、C。重要的事多记点（B 大），不重要的事少记点（B 小），该忘的快点忘（A 小），该记住的慢点衰减（A 大）。

💡 **翻译成人话：** 早期 SSM = 一个**固定格式的日记本**，不管发生什么都用同一种方式记。Mamba = 一个**会根据内容调整记法的日记本**——听到八卦浓墨重彩记三页，听到废话一笔带过。这个"选择性"就是 Mamba 的核心卖点——全称 **Selective State Space Model**（选择性状态空间模型），Mamba 是它的品牌名（曼巴蛇，又快又毒 🐍）。也是它能逼近 Transformer 效果的核心原因。

熟悉 NLP 历史的同学会发现，这跟 Attention 的演进路径一模一样：Bahdanau Attention（2014）用两个固定的权重矩阵 W、U 算注意力分数（`score = V · tanh(W · s + U · h)`），到 Transformer（2017）变成 Q、K、V 全部由输入动态生成（`score = Q · K^T`，Q 和 K 都是输入乘以可学习矩阵得到的）——"Attention is All You Need"。SSM 这边走了完全相同的路：S4（2021）固定参数，Mamba（2023）输入依赖——"Selectivity is All You Need"。**两条技术路线独立走了一遍相同的进化：从固定到动态，从死记到选择性记。** 殊途同归，说明这不是巧合，是必经之路。

你马上会问：那 SSM 为什么之前没干掉 Transformer？

因为**压缩必然有损**。你把所有历史信息压缩进一个固定大小的 state，必然会丢东西。Transformer 的 O(n²) 虽然贵，但它**什么都不丢**——每个 token 的原始信息都保留着，随时可以回查。SSM 做不到这一点。

具体来说，早期的 SSM 在以下任务上被 Transformer 按在地上摩擦：

1. **精确检索（retrieval）**："第 3 段第 2 行说了什么？"——这种需要精确回查原文的任务，SSM 不行，因为原文信息已经被压缩进 state 了
2. **状态追踪（state tracking）**："Alice 把球给了 Bob，Bob 把球给了 Charlie，现在球在谁手里？"——这种需要精确维护多个实体状态的任务，SSM 也不行

Mamba-1（2023）和 Mamba-2（2024）已经大幅缩小了这个差距。但在 retrieval 和 state tracking 上，还是比 Transformer 弱。

**Mamba-3 的目标就是补上这两块短板。**

━━━━━━━━━━━━━━━━━━━━

◆ Mamba-3 的三个创新——复数、MIMO、更强递归

论文的核心贡献是三个互相配合的改进。我一个一个讲。

────────────────────

**创新一：复数值状态更新——最优雅的那个**

这是我个人觉得最漂亮的创新，值得多花点篇幅。

先说问题。Mamba-2 的状态更新是这样的：

```
state = A * state + B * input
```

这里 A 是一个实数矩阵。实数矩阵能做什么？**缩放。** 把 state 的每个维度放大或缩小。

但光有缩放是不够的。想象你要追踪一个绕圈跑的物体——它的位置在 x 轴和 y 轴上反复变化。用纯缩放，你只能"记住它在 x 轴上走了多远"，但没法表达"它正在转弯"。

Mamba-3 的解法：**把 state 从实数换成复数。**

💡 **翻译成人话：** 实数只有一个维度（数轴上的一个点），复数有两个维度（平面上的一个点）。实数乘法 = 拉伸或压缩。复数乘法 = 拉伸 + 旋转。**一个操作，信息量翻倍。**

如果你学过欧拉公式：e^(iθ) = cos(θ) + i·sin(θ)。一个复数可以表示为"长度 × 旋转角度"。复数相乘时，长度相乘，角度相加。

所以 Mamba-2 的 A 矩阵只能做"缩放"，Mamba-3 的 A 矩阵能做"缩放 + 旋转"。同样大小的 state，**表达能力翻倍**。

这就是为什么 Mamba-3 能用一半的 state size 达到 Mamba-2 同等的 perplexity——每个复数维度干了两个实数维度的活。注意这里"一半"省的是**维度数**，不是字节数：一个复数要存实部和虚部两个浮点数，内存占用跟两个实数差不多。但同样的内存预算下，复数版的**表达能力翻倍**（多了旋转自由度），所以用更少的维度就够了。

────────────────────

这里还有一个特别漂亮的数学巧合。

熟悉 Transformer 的同学都知道 **RoPE**（旋转位置编码）——LLaMA、Qwen、DeepSeek 全在用。RoPE 的核心就是：**把位置信息编码成复数平面上的旋转角度**。位置 1 转 θ 度，位置 2 转 2θ 度，位置 n 转 nθ 度。

Mamba-3 的复数状态更新，本质上做的是同一件事：**用旋转来编码序列中的位置关系**。

💡 **翻译成人话：** Transformer 是并行处理所有 token 的，不知道谁在前谁在后，所以需要额外加 RoPE 来告诉模型位置信息——方法是把位置编码成复数平面上的旋转角度。Mamba-3 是递推结构，天然有顺序，不需要额外加位置编码。但它的复数状态更新——用旋转来变换记忆——在数学上跟 RoPE 长得一模一样。一个是给"不知道顺序的模型"补位置信息，一个是给"记忆更新"加旋转自由度，**两个完全不同的动机，走到了同一个数学结构上**。

一个从 attention 方向走来，一个从 SSM 方向走来，殊途同归。数学真理不以人的意志为转移。

────────────────────

**创新二：更强的递归结构——从 SSM 离散化推导**

这个偏数学，我尽量讲直觉。

SSM 的原始形态是连续时间的微分方程——描述的是"状态在每一个瞬间怎么变化"：

```
dx/dt = Ax + Bu
```

但计算机没法处理"瞬间"，只能处理"一步一步"。所以你得把连续的微分方程**切成离散的时间步**，变成"每过一个 Δt，状态怎么更新"。这个"切"的过程就叫**离散化**。跟你写游戏物理引擎一样——物理世界是连续的，但你的 `for` 循环是离散的。你模拟一个球的轨迹，物理公式是微分方程，代码是每帧更新一次位置。**每帧更新的方式不同，精度和性质就不同。**

Mamba-2 用的离散化方法比较简单（假设每个时间步内输入不变），得到的递推是：

```
state_new = A * state_old + B * input
```

Mamba-3 用了更精细的离散化（考虑了步长 Δ 的影响），得到的递推多了一项：

```
state_new = A * state_old + B * input + D * (input - input_prev)
```

多出来的 `D * (input - input_prev)` 是什么？是**输入的变化量**。

💡 **翻译成人话：** Mamba-2 只看"当前输入是什么"。Mamba-3 还看"当前输入跟上一个输入差了多少"。就像一个股票分析师，Mamba-2 只看今天的股价，Mamba-3 还看今天比昨天涨了还是跌了。多了一个**微分信号**，对变化更敏感。

这个改进对 state tracking 特别有用——因为追踪状态变化，本质上就是在追踪"什么时候变了、变了多少"。

而且，这个新的离散化方式还带来了一个意外收获——它推出来的递推形式天然支持**多输入头**，直接为下面的 MIMO 铺好了路。

────────────────────

**创新三：MIMO——prefill 和 decode 解耦**

MIMO 是 Multi-Input Multi-Output 的缩写。这个创新之所以成为可能，正是因为上面讲的新离散化推导。

先解释一个背景。大模型的推理分两个阶段：

1. **Prefill（填充）**：把你的 prompt 一次性灌进去。比如你输入了 500 个词，模型一次性处理这 500 个词
2. **Decode（解码）**：一个词一个词地生成回答。每生成一个词都是一次前向计算

Prefill 是可以并行的（500 个词同时处理），Decode 是串行的（一个一个蹦）。

传统 SSM 的问题是：不管是 prefill 还是 decode，都用同一套 SISO（Single-Input Single-Output）递归。SISO 意味着每个时间步只有一个输入和一个输出。这在 decode 阶段没问题（反正也是一个词一个词来），但在 prefill 阶段就浪费了——你明明可以并行处理多个输入，却被 SISO 限制住了。

Mamba-3 的解法：**prefill 阶段用 MIMO（多个输入并行处理，多个输出同时产出），decode 阶段退化回 SISO（单输入单输出，跟以前一样）。**

你可能会问：Mamba-2 的 prefill 不也能并行吗？为什么之前做不了 MIMO？

关键在公式结构。Mamba-2 的递推是：

```
state = A * state + B * x_t      （x_t 是一个向量）
```

每个时间步的输入 x_t 是单个向量，状态更新做的是"矩阵 × 向量"运算。你可以用 parallel scan 把多个时间步展开并行，但**每一步内部还是向量运算**，算术强度上不去。

而 Mamba-3 的新离散化推出的递推形式，天然允许**多个输入头**打包成矩阵：

```
state = A * state + B * X_t      （X_t 是多个输入头拼成的矩阵）
```

表面上只是把小写 x 变成了大写 X，但背后是完全不同的离散化推导路径——Mamba-2 假设每个时间步内输入不变，推出来的递推天然只能接受单输入；Mamba-3 考虑了输入在步内的变化，推出来的递推天然支持多输入头。同一个微分方程，不同的离散化方法，解出来的递推公式性质完全不同。而这个新格式对硬件的意义巨大：

- **矩阵 × 向量**（SISO）：算完一次就要等内存搬下一批数据，Tensor Core 大部分时间在发呆
- **矩阵 × 矩阵**（MIMO）：一次搬进来的数据被反复复用（矩阵的行和列互相组合相乘），Tensor Core 终于被喂饱了

💡 **翻译成人话：** SISO 就像一个只有一个收银台的超市——就算同时来了 50 个人，也得排队一个个结账。MIMO 是开了 8 个收银台——50 个人分成几队，同时结账。但注意：**收银台多了不影响每个人的结账速度**。decode 阶段每次只来一个 token，没东西可以打包，自动退化回 SISO——延迟完全不变。

这意味着 prefill 更快了，但 decode（用户等待每个字蹦出来的延迟）**没有任何增加**。鱼和熊掌，它都要了。**不是新公式，是同一个公式在不同阶段用不同的并行度执行**——公式没变，喂数据的方式变了。

所以 MIMO 本质上是**新离散化的红利**：数学上推出了允许矩阵输入的递推形式，硬件上把"矩阵 × 向量"变成"矩阵 × 矩阵"，把 GPU 里闲置的 Tensor Core 压榨干净。同样的硬件，干了更多的活，延迟还不变。这不是算法优化，这是对 GPU 架构的逆向工程。

━━━━━━━━━━━━━━━━━━━━

◆ 实测数据：不是纸上谈兵

三个创新叠在一起，效果怎么样？

在 1.5B 参数规模下（这是 SSM 领域的标准测试规模）：

| 模型 | 平均下游准确率 |
|------|-------------|
| Mamba-2 | 基线 |
| Gated DeltaNet（此前最强线性模型） | 基线 + 0.6 |
| **Mamba-3** | **基线 + 0.6**（追平 DeltaNet） |
| **Mamba-3 MIMO** | **基线 + 1.8**（超过 DeltaNet 1.2 个点） |

你可能注意到 Mamba-3 和 Mamba-3 MIMO 是分开列的。因为 MIMO 是一个**可选配置**，不是默认开启。不开 MIMO 的 Mamba-3 已经追平了 DeltaNet（+0.6），开了 MIMO 再涨 1.2 个点。这说明多输入头不只是"让 prefill 更快"，**对模型质量本身也有提升**——多个输入头相当于让模型从多个角度同时观察输入，类似 Transformer 里 multi-head attention 的效果。所以 MIMO 的好处是双重的：速度更快，效果也更好。（论文只给了相对提升，没给绝对分数。1.8 个百分点听着不多，但在这个规模的 benchmark 上，各家模型通常挤在很窄的区间里，1.8 个点已经是碾压级别的差距。）

在检索和状态追踪任务上，改善更大。这正好击中了 SSM 之前最弱的地方。

另一个关键数据：**Mamba-3 用 Mamba-2 一半的 state size，达到同等 perplexity。** 这不是"差不多"，是数学上等价——用更少的内存做同样的事。

复数 state 的"每个维度干两个维度的活"，不是吹的。

━━━━━━━━━━━━━━━━━━━━

◆ 这跟你有什么关系——普通程序员的三个利好

你可能会说：我又不做 AI 研究，Mamba-3 跟我有啥关系？

关系很大。

────────────────────

**第一：API 账单要降了。**

O(n²) → O(n)，内存从 O(n) → O(1)。这意味着处理长文本的成本会大幅下降。现在 GPT-4 处理一个 128K 上下文的请求，大部分钱花在 attention 的 O(n²) 计算上。如果底层换成线性模型，同样的请求可能便宜一个数量级。

当然，替换不会一夜之间发生。但趋势已经很明确了。

────────────────────

**第二：本地部署变得现实。**

O(1) 的内存意味着什么？意味着**不管上下文多长，显存占用不变**。

现在你想在消费级 GPU（比如 RTX 4090，24GB 显存）上跑一个 7B 的 Transformer 模型，上下文长度基本被限制在几千到几万个 token——因为 KV cache 会随上下文线性增长，很快就把显存吃满。

但如果模型是 SSM 架构，state 大小是固定的。24GB 显存能跑多长上下文？**理论上无限长。** 当然实际上有其他瓶颈，但 KV cache 这个最大的内存杀手没了。

💡 **翻译成人话：** 以后你笔记本上的 GPU 可能就能跑一个"永远不会忘记你说过什么"的本地 AI 助手。不需要云端，不需要月费，不需要把你的聊天记录发到别人的服务器上。

────────────────────

**第三：端侧 AI 要起飞。**

手机、IoT 设备、嵌入式系统——这些地方算力和内存都极其有限。Transformer 的 O(n²) 在这些场景下根本跑不动。但 O(n) 计算 + O(1) 内存的 SSM？完全有可能。

想象你的手机上跑着一个本地的语音助手，不需要联网，不需要等服务器响应，响应延迟可以做到毫秒级。这在 Transformer 架构下很难实现，在 SSM 架构下是完全可行的。

━━━━━━━━━━━━━━━━━━━━

◆ "推理优先"——一种值得学习的设计哲学

最后聊点"虚"的，但我觉得这是这篇论文最有价值的东西。

Mamba-3 论文的摘要里有一个词组特别值得注意：**"inference-first perspective"（推理优先视角）**。

过去几年，AI 领域的主旋律是"scale up"——模型更大、数据更多、训练更久。GPT-3 到 GPT-4，参数量翻了不知道多少倍。这是一种暴力美学：**大力出奇迹**。

Mamba-3 代表的是另一种哲学：**不是问"怎么让模型更大"，而是问"怎么让模型更省"。**

不是用更多的资源做同样的事，而是用更少的资源做同样甚至更好的事。

这让我想起生物进化。恐龙是"大力出奇迹"的代表——体型巨大，统治地球 1.6 亿年。但最终活下来的不是恐龙，是鸟。鸟是恐龙的后代，但它们做了一件关键的事：**把身体缩小了，把骨头变空心了，把效率提到了极致。** 同样的飞行能力，用十分之一的体重实现。

人脑也是这个逻辑。人脑只有 1.4 公斤，功耗 20 瓦——跟一个灯泡差不多。但它能产生意识、能做数学、能写代码、能思考自身的存在。如果意识真的需要"规模"才能涌现，那整个大脑应该需要几兆瓦的功率才对。但它只用了 20 瓦。

**20 瓦就够了。这说明关键不是规模，是架构。**

这个思路在开源社区已经开始落地了。TII 的 Falcon-Mamba 是第一个大规模开源的纯 SSM 语言模型。DeepSeek 用极低的训练成本打出了震惊业界的效果。开源社区正在证明：你不需要百亿美元的 GPU 集群，也能做出好用的模型。

而且别忘了中国这边的玩家。Mamba-3 这种"同等智商，省一半显存"的架构，对于被推理成本压得喘不过气的大厂来说，就是救命稻草。阿里的 Qwen3.5 已经用上了 Gated DeltaNet（Mamba-3 刚刚超越的那个），说明国内大厂对线性注意力架构的嗅觉极其敏锐。DeepSeek 把训练成本打骨折了，阿里在架构上跟得最紧，字节在端侧部署上卷得最狠——当 Mamba-3 的论文和代码摆在桌上，你猜这帮被 996 驱动的工程师需要多久把 7B、34B 的权重炼出来？我赌不超过三个月。**资本的逐利本能，会用最暴躁的方式把好架构推向生产线。**

**AI 的未来不是"只有大厂才能玩"。AI 的未来是跑在你设备上的、属于你自己的智能。** 而中国的开源生态，很可能是让这件事最快发生的地方。

Mamba-3 的代码已经开源在 github.com/state-spaces/mamba。如果你想自己跑跑看，go ahead。

💡 **本文涉及的技术名词速查：**

- **Transformer**：2017 年 Google 提出的模型架构，核心是 self-attention 机制，计算量 O(n²)
- **SSM（State Space Model，状态空间模型）**：用固定大小的隐状态递推处理序列，计算量 O(n)，内存 O(1)
- **Mamba**：Albert Gu 团队提出的 SSM 系列模型，Mamba-1（2023）→ Mamba-2（2024）→ Mamba-3（2026）
- **O(n²) / O(n) / O(1)**：算法复杂度表示法。O(n²) 是二次方增长，O(n) 是线性增长，O(1) 是常数不增长
- **Perplexity（困惑度）**：衡量语言模型好坏的核心指标，越低越好。直觉上表示"模型对下一个词有多困惑"
- **State Size（状态大小）**：SSM 中隐状态的维度。越大理论上能记住越多信息，但也越占内存
- **复数（Complex Number）**：形如 a + bi 的数，其中 i² = -1。复数乘法 = 缩放 + 旋转
- **RoPE（Rotary Position Embedding，旋转位置编码）**：用复数旋转编码 token 位置的方法，LLaMA/Qwen/DeepSeek 都在用
- **SISO / MIMO**：Single-Input Single-Output / Multi-Input Multi-Output，信号处理术语
- **Prefill / Decode**：大模型推理的两个阶段。Prefill 是一次性处理输入，Decode 是逐词生成输出
- **KV Cache**：Transformer 推理时缓存的 Key/Value 矩阵，随上下文长度线性增长，是显存的大头
- **离散化（Discretization）**：把连续时间的微分方程转换成离散时间步的递推公式
- **线性注意力（Linear Attention）**：把 Transformer 的 O(n²) attention 降到 O(n) 或 O(1) 的方案的统称。SSM/Mamba、Gated DeltaNet、RWKV 都属于这个大类，目标一样，数学路径不同

────────────────────

💡 **延伸阅读：三种"干掉 O(n²)"的路线对比**

它们的状态更新公式长得很像，但各有侧重：

**Mamba（SSM 路线）**——来自控制论的微分方程离散化：
```
s_t = A · s_{t-1} + B · x_t        （A、B 由输入动态生成）
```
特点：A 是对角矩阵（每个维度独立衰减），Mamba-3 升级为复数对角矩阵（衰减 + 旋转）。来自连续动力系统的传统，数学最优雅。复数旋转意味着矛盾的信息可以转到另一个相位存起来，不需要擦除——旧记忆和新记忆可以共存在正交维度上。

**Gated DeltaNet（Delta Rule 路线）**——来自 Hebbian 学习的增量更新：
```
s_t = α · s_{t-1} + β · v_t · k_t^T - β · (k_t^T · s_{t-1}) · v_t · k_t^T
```
特点：状态是一个矩阵（不是向量），用外积 `v · k^T` 写入，用 delta rule 精确覆盖旧记忆。α 是门控（来自 Mamba-2），delta rule 负责"定点擦除再写入"。Qwen3.5 在用这个。精准擦除是它的优势也是它的性格——像个严谨的会计，每来一笔新账先把旧的错账平掉。适合需要逻辑严密的生产场景。

**RWKV（RNN 复兴路线）**——来自线性 attention 的 RNN 重写：
```
s_t = diag(w) · s_{t-1} + k_t · v_t^T
```
特点：w 是逐通道的衰减系数（类似 Mamba 的 A），用 `k · v^T` 外积写入。没有 delta rule 的精确擦除能力，新旧记忆直接叠加，像一张没有橡皮擦的草稿纸，只能等墨水慢慢褪色。实现最简单，纯国产开源（Bo Peng）。模糊但有人味儿。

**一句话总结**：Mamba 用微分方程，DeltaNet 用记忆覆写，RWKV 用线性 RNN。三条路都在解同一道题：怎么用 O(1) 的状态记住 O(n) 的历史。Mamba-3 目前跑分最高，但 Gated DeltaNet 已经进了 Qwen3.5 的生产线——工程落地和学术跑分是两回事。

**Gated DeltaNet 在 Qwen3.5 里的实战表现**：Qwen3.5 用 75% Gated DeltaNet + 25% 传统 attention 的混合架构，效果有多猛？Qwen3.5-9B 在 GPQA Diamond 上得分 81.7，打赢了 OpenAI 的 GPT-OSS-120B（71.5）——9B 打 120B，13 倍体量差距。推理速度是 Claude Sonnet 4.6 的 6 倍，Flash 版定价 $0.10/M tokens（Claude Sonnet 4.6 是 $1.30/M），便宜 13 倍。支持 100 万 token 上下文，计算近似线性增长。所以线性注意力不是实验室玩具，已经在生产线上赚钱了。现在 Mamba-3 说"我在 1.5B 上超了 Gated DeltaNet"——如果这个优势在大规模上成立，下一代 Qwen 换芯几乎是必然的
