【优化器是什么】从 SGD、AdamW 到 Muon

━━━━━━━━━━━━━━━━━━━━

◆ 先说人话：优化器到底干什么

━━━━━━━━━━━━━━━━━━━━

训练模型，说白了就是一件事：

**调参数。**

一个模型有几十亿、几千亿、甚至上万亿个参数。每个参数都是一个数字。训练时，模型先拿这些数字去算答案；答案错了，就算一个 **loss（损失）**；然后再根据 loss，把这些数字往更好的方向拧一点。

整个训练循环就是：

```text
输入一批数据
模型给出预测
算 loss
根据 loss 判断参数该怎么改
更新参数
重复几万亿次
```

**优化器**干的就是最后一步：

**根据梯度，决定这一步参数怎么更新。**

模型结构决定"它能表达什么"。  
训练数据决定"它见过什么"。  
优化器决定"它怎么从一堆随机数字，慢慢长成一个可用模型"。

所以优化器不是一个小细节。它是训练过程里的手。

DeepSeek V4 技术报告里说，1.6T 参数里 **99.9%** 都用了 Muon 优化器，只剩 embedding、输出头、RMSNorm 这几个模块继续用 AdamW。这个信息很重要：V4 不是随手换了个训练小配件，而是把大模型训练的"拧参数方式"换了。

今天就从零讲清楚：SGD 是什么，AdamW 为什么统治大模型，Muon 又到底改了哪里。

━━━━━━━━━━━━━━━━━━━━

◆ loss：模型错得有多离谱

━━━━━━━━━━━━━━━━━━━━

先别管优化器。先问一个更基础的问题：

**模型怎么知道自己错了？**

答案是 loss。

比如你让模型预测下一个字：

```text
输入：北京是中国的
正确答案：首都
模型预测：城市
```

模型答错了。那到底错多少？不能只说"错了"，训练需要一个数字。这个数字就是 **loss**。

loss 越大，说明错得越离谱。  
loss 越小，说明预测越接近正确答案。

训练的目标就是：

```text
让 loss 越来越小
```

如果把模型想象成一台有无数旋钮的机器，loss 就像屏幕上的错误分数。你每拧一下旋钮，分数可能下降，也可能上升。训练要做的事，就是找到一种拧法，让这个分数一路降下去。

优化器要解决的问题就是：

**这么多旋钮，每一步到底该往哪边拧、拧多少？**

━━━━━━━━━━━━━━━━━━━━

◆ 梯度：哪个方向最陡，反过来就是下降

━━━━━━━━━━━━━━━━━━━━

接下来是最容易吓人的词：**梯度**。

如果你学过微积分，梯度就是一堆偏导数。  
如果你偏导数算不明白，也没关系。这里完全可以用人话理解。

梯度回答一个问题：

**如果我稍微动一下这个参数，loss 会往哪边变？**

比如某个参数叫 `w`。现在 loss 是 10。

如果 `w` 增大一点，loss 变成 10.3，说明往这个方向拧不对。  
如果 `w` 减小一点，loss 变成 9.7，说明反方向更好。

梯度就是这个方向信号。严格说，梯度指向的是 **loss 上升最快的方向**；训练要让 loss 下降，所以更新参数时要反着走。

你可以把梯度理解成：

**loss 地形上的坡度。**

站在山坡上，梯度告诉你哪边最往上；反方向就是下山。模型训练就是在一个几百亿维的山地里找低处。最低处不是海拔，而是 loss。

所以最朴素的训练规则很直接：

```text
往 loss 下降最快的方向走一步
```

这就是 SGD。

━━━━━━━━━━━━━━━━━━━━

◆ SGD：最朴素的下山法

━━━━━━━━━━━━━━━━━━━━

SGD，全名 Stochastic Gradient Descent，随机梯度下降。

名字很长，公式很短：

```text
w = w - lr * grad
```

三个东西：

- `w`：当前参数
- `grad`：梯度，也就是 loss 上升最快的方向
- `lr`：learning rate，学习率，也就是这一步迈多大

为什么是减号？

因为梯度指向的是 loss 上升最快的方向。我们要让 loss 下降，所以要反着走。

这就是 SGD 的全部核心。

如果用下山比喻：

```text
grad 告诉你坡往哪边上
-grad 就是下山方向
lr 决定你一步迈多大
```

SGD 的优点是简单、直接、便宜。

但它有一个大问题：真实的 loss 地形不是一个平滑的小山坡，而是坑坑洼洼的高维山地。有些方向很陡，有些方向很平；有些方向梯度很吵，今天说往左，明天说往右。

SGD 每一步只看当前这一批数据给出的梯度，很容易抖。

像一个人闭着眼下山，每一步都只摸脚下这一小块地面。地面稍微有点石头，就开始左右晃。

于是人们给 SGD 加了一个东西：**动量**。

━━━━━━━━━━━━━━━━━━━━

◆ 动量：别被脚下一颗石头骗了

━━━━━━━━━━━━━━━━━━━━

动量的想法很简单：

**不要只看这一步的梯度，要看最近一段时间总体往哪走。**

如果最近 100 步大多数时候都在往东南方向下山，那这一步脚下有个小石头告诉你往西，也别立刻掉头。

公式可以这么理解：

```text
momentum = 0.9 * momentum + grad
w = w - lr * momentum
```

旧方向保留 90%，新梯度加进来。这样更新方向会更平滑，不容易被单次噪声带偏。

这就是 **SGD with Momentum**。

动量解决的是"方向太抖"的问题。

但它还没解决另一个问题：

**不同参数的尺度不一样。**

有些参数的梯度天生很大，有些参数的梯度天生很小。你用同一个学习率去拧所有参数，就像拿同一个扳手力度去拧所有螺丝。有的螺丝会拧爆，有的螺丝几乎没动。

这就是 Adam 要解决的问题。

━━━━━━━━━━━━━━━━━━━━

◆ Adam：给每个参数单独装油门和刹车

━━━━━━━━━━━━━━━━━━━━

Adam 的核心可以用一句话概括：

**每个参数都有自己的更新节奏。**

SGD 是所有参数共用一个学习率。  
Adam 会给每个参数估计两个东西：

```text
一阶动量：最近梯度的平均方向
二阶动量：最近梯度的波动大小
```

不用被"一阶""二阶"吓到。说人话：

**一阶动量**：最近总体往哪边走。  
**二阶动量**：这个参数最近抖得有多厉害。

如果某个参数梯度一直很稳定，那就可以大胆拧一点。  
如果某个参数梯度忽大忽小，那就少拧一点，别炸。

所以 Adam 像是给每个参数都装了独立的油门和刹车。

这就是它厉害的地方：**逐参数自适应**。

大模型训练数据噪声大、参数多、尺度复杂。Adam 这种每个参数自己调步长的机制，特别好用。所以过去几年，大模型训练几乎都离不开 Adam 或 AdamW。

那 AdamW 又是什么？

━━━━━━━━━━━━━━━━━━━━

◆ AdamW：Adam 加上更干净的减肥规则

━━━━━━━━━━━━━━━━━━━━

神经网络有一个老毛病：参数容易长得太大。

参数太大，模型可能会过拟合，也可能训练不稳定。于是训练时通常会加一个 **weight decay（权重衰减）**：

```text
每一步都让参数稍微变小一点
```

这有点像给模型减肥。别让某些参数无限膨胀。

Adam 早期把 weight decay 混在梯度里一起处理，效果不够干净。AdamW 的改动就是把 weight decay 从梯度更新里拆出来，单独做：

```text
先按 Adam 的规则更新参数
再单独把参数往 0 拉一点
```

这个改动看起来小，但对大模型很重要。它让"根据梯度学习"和"防止参数膨胀"这两件事分开处理，训练更稳定。

所以 AdamW 可以理解成：

**Adam 的自适应更新 + 更干净的权重衰减。**

这就是为什么 LLaMA、GPT、DeepSeek V3 这类模型长期依赖 AdamW。

但 AdamW 有一个根本局限：

**它把矩阵拆成一个个独立数字来看。**

━━━━━━━━━━━━━━━━━━━━

◆ 矩阵不是一袋散装数字

━━━━━━━━━━━━━━━━━━━━

Transformer 里最多的参数是什么？

不是一个个零散数字，而是一大堆二维矩阵。

比如最常见的线性层：

```text
y = W x
```

`x` 是输入向量。  
`W` 是权重矩阵。  
`y` 是输出向量。

这个 `W` 不是一张 Excel 表。它表示一次空间变换：把输入空间里的一个方向，映射到输出空间里的另一个方向。

注意力里的 Q/K/V 投影矩阵是 `W`。  
FFN 里的升维、降维矩阵是 `W`。  
MoE 专家里面的大部分参数也是 `W`。

也就是说，大模型的主体不是一袋散装参数，而是一堆矩阵。

AdamW 怎么看这些矩阵？

**逐元素看。**

矩阵里每个格子都是一个参数，每个参数自己算自己的梯度、动量、步长。这个方法很强，但它不关心矩阵整体的几何形状。

问题是：矩阵真正重要的，往往是整体形状。

一个矩阵会把某些方向放大，把某些方向压扁，把某些方向旋转过去。训练时更新矩阵，如果某几个方向更新太猛，其他方向几乎不动，矩阵就容易变得很偏。

这就是 Muon 的入口。

**AdamW 是把矩阵当 Excel 表格改。  
Muon 是把矩阵当几何对象改。**

━━━━━━━━━━━━━━━━━━━━

◆ Muon：矩阵级更新

━━━━━━━━━━━━━━━━━━━━

Muon 全名是：

**MomentUm Orthogonalized by Newton-Schulz**

名字先不用背。抓住核心操作就够了：

```text
1. 先像动量法一样，算出一个更新方向
2. 这个更新方向本身是一个矩阵
3. Muon 不直接用这个矩阵更新权重
4. 它先把这个更新矩阵做正交化
5. 再用正交化后的方向更新权重
```

关键是第 4 步：**正交化**。

正交化是什么意思？

在二维平面里，横轴和纵轴互相垂直，这叫正交。推广到高维空间，正交就是"方向之间尽量互不重叠、互不打架"。

Muon 对更新矩阵做正交化，就是想让更新方向更均匀地铺开：

```text
不要所有更新都挤在少数几个方向上
不要某些方向更新太猛，某些方向完全不动
让整个矩阵以更健康的方式一起变化
```

这就是 166 期里那句话的展开：

**AdamW 是逐元素更新。  
Muon 是矩阵级更新。**

AdamW 问的是：

```text
这个参数该怎么改？
```

Muon 问的是：

```text
这个矩阵整体该怎么改？
```

这个差别，对小模型可能没那么明显。  
但对 1.6T 参数的大模型，绝大多数参数都在大矩阵里，这个差别会被规模放大。

━━━━━━━━━━━━━━━━━━━━

◆ 正交化为什么有用

━━━━━━━━━━━━━━━━━━━━

还是用程序员能懂的方式讲。

假设一个矩阵有很多方向可以学。有的方向很常见，训练数据天天刺激它；有的方向很稀有，偶尔才出现。

普通逐元素更新很容易出现一个问题：

**常见方向越来越强，稀有方向越来越弱。**

就像一个团队里，嗓门大的人越来越有话语权，安静的人永远插不上嘴。最后整个团队只会沿着少数几个方向猛冲。

Muon 的正交化相当于给更新方向做一次整理：

**别挤在一起，分散一点。  
别只让几个方向说话，其他方向也要有空间。**

从几何上说，它在控制更新矩阵的形状，让更新不要过度集中在少数奇异方向上。

不用纠结"奇异方向"这个词。直觉就是：

**矩阵更新不要偏科。**

这会带来两个好处：

1. **收敛更快**：每一步更新更有效，不容易在少数方向上来回抖。
2. **训练更稳**：更新方向更均匀，少一些局部爆炸。

所以 Muon 不是神秘玄学。它做的是一件很朴素的事：

**既然大模型主体是矩阵，那更新时就别把矩阵拆成散装数字。**

━━━━━━━━━━━━━━━━━━━━

◆ Newton-Schulz：便宜地做近似正交化

━━━━━━━━━━━━━━━━━━━━

理论上，想把一个矩阵正交化，可以用 SVD。

SVD 会把矩阵拆成三块：

```text
M = U S Vᵀ
```

其中 `U` 和 `V` 代表方向，`S` 代表每个方向有多强。  
如果只保留方向，把强弱去掉，就得到一个正交化后的更新：

```text
orthogonal(M) = U Vᵀ
```

这很漂亮，但问题是：

**SVD 太贵。**

大模型训练每一步都有海量矩阵要更新。你不可能每一步都给每个矩阵做一次完整 SVD。那不是训练模型，是烧钱做线性代数祭祀。

Muon 用的是 **Newton-Schulz 迭代**。

它不是精确做 SVD，而是用几轮矩阵乘法，快速逼近"正交化"这个结果。

为什么这很重要？

因为 GPU 最擅长的就是矩阵乘法。只要把操作变成矩阵乘法，就能吃满硬件。

一句话：

**SVD 是精确但昂贵的手术刀。  
Newton-Schulz 是便宜很多、GPU 跑得飞快的近似刀。**

Muon 的现实价值就在这里：它不是只在数学上好看，而是能真的放进大模型训练流程里跑。

━━━━━━━━━━━━━━━━━━━━

◆ 为什么不是所有参数都用 Muon

━━━━━━━━━━━━━━━━━━━━

Muon 的核心优势是矩阵级更新。

所以它最适合二维权重矩阵：

- 注意力层的 Q/K/V/O 投影矩阵
- FFN 的升维、降维矩阵
- MoE 专家的 FFN 权重
- mHC 的混合矩阵

这些都是大矩阵，正好能吃到 Muon 的几何红利。

但有些参数不是这种形状：

- embedding
- 输出头（通常也是矩阵，但和词表/embedding 绑定，处理方式特殊）
- RMSNorm
- bias
- 标量参数
- 向量参数

这些东西要么不是二维矩阵，要么结构比较特殊。你硬给它用 Muon，没什么意义，甚至可能不稳定。

所以 DeepSeek V4 的做法很合理：

**大矩阵用 Muon，小零件继续用 AdamW。**

这就是 166 期里那组数字：

V4 的 1.6T 参数里，**99.9% 用 Muon**。  
只有 embedding、输出头、RMSNorm 这些模块还用 AdamW，合计不到 20 亿参数，占比 **0.1%**。

这不是因为 AdamW 过时了。  
而是因为不同形状的参数，适合不同的更新方式。

━━━━━━━━━━━━━━━━━━━━

◆ DeepSeek V4 为什么要换 Muon

━━━━━━━━━━━━━━━━━━━━

现在可以回到 DeepSeek V4。

V4 有 1.6T 总参数，384 个 MoE 专家，每次激活 6 个专家。真正的参数大头在哪里？

在专家矩阵里。

MoE 专家本质上就是一堆巨大的 FFN 矩阵。V4 的注意力投影、专家 FFN、mHC 混合矩阵，都是 Muon 最擅长处理的二维权重矩阵。

所以 Muon 对 V4 的意义不是"换个 optimizer 试试"。

它是在利用一个事实：

**Transformer 的主体是矩阵。**

AdamW 的成功来自逐参数自适应。  
Muon 的野心是再往前一步：

**从逐参数优化，走向矩阵几何优化。**

这就是为什么 V4 把 99.9% 参数交给 Muon。

在小模型上，Muon 多出来的正交化计算未必划算。  
但在 1.6T 这种规模上，如果它能让训练更快收敛、更稳定，省下来的训练 token、训练时间、训练事故，远远超过正交化本身的成本。

超大模型训练不是"每一步最便宜"就赢，而是：

**谁能更稳、更快地走到目标 loss，谁赢。**

━━━━━━━━━━━━━━━━━━━━

◆ 但 Muon 不是魔法

━━━━━━━━━━━━━━━━━━━━

Muon 很重要，但别把它神化。

它有代价：

1. **计算更复杂**：多了正交化步骤，哪怕 Newton-Schulz 已经很快，也不是免费。
2. **实现更麻烦**：要把参数分组，哪些用 Muon，哪些继续用 AdamW。
3. **不一定适合所有任务**：小模型、小数据、非矩阵参数，收益未必明显。
4. **调参仍然重要**：学习率、动量、权重衰减这些东西一样要调。

所以 Muon 不是 AdamW 的简单替代品。

更准确的说法是：

**在 Transformer 大矩阵占绝对主体的场景里，Muon 开始比 AdamW 更吃模型结构。**

AdamW 是通用型好扳手。  
Muon 是专门为大矩阵准备的几何扳手。

DeepSeek V4 这种模型，正好满身都是大矩阵。

━━━━━━━━━━━━━━━━━━━━

◆ 一张表总结

━━━━━━━━━━━━━━━━━━━━

| 优化器 | 核心想法 | 人话理解 | 问题 |
|---|---|---|---|
| SGD | `w = w - lr * grad` | 顺着坡往下走 | 容易抖，所有参数共用步长 |
| Momentum | 累积历史梯度方向 | 别被脚下一颗石头骗了 | 还是没有逐参数自适应 |
| Adam | 一阶动量 + 二阶动量 | 每个参数有自己的油门和刹车 | 把矩阵拆成散装数字 |
| AdamW | Adam + 解耦权重衰减 | 一边学习，一边干净减肥 | 仍然是逐元素更新 |
| Muon | 动量更新 + 矩阵正交化 | 把矩阵当几何对象整体更新 | 计算和实现更复杂 |

━━━━━━━━━━━━━━━━━━━━

◆ 最后

━━━━━━━━━━━━━━━━━━━━

SGD 像摸着坡往下走。  
Momentum 让你别被单次噪声带偏。  
AdamW 给每个参数单独装了油门和刹车。  
Muon 再往前走了一步：它不再把矩阵拆成散装数字，而是把矩阵当成一个整体来更新。

这就是优化器这条线的演化：

```text
从一个参数怎么改
到每个参数怎么自适应
再到整个矩阵怎么健康地一起改
```

Transformer 的主体就是一堆矩阵。  
当模型大到 1.6T 参数时，优化器就不再是训练细节。

**优化器，就是模型长成什么样的手。**

DeepSeek V4 把 99.9% 参数交给 Muon，本质上是在说：

**大模型训练，开始认真吃矩阵几何的红利了。**
