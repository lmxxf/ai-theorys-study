【优化器】从 SGD、AdamW 到 DeepSeek V4 的 Muon

━━━━━━━━━━━━━━━━━━━━

◆ 先说人话：优化器到底干什么

━━━━━━━━━━━━━━━━━━━━

训练模型，说白了就是一件事：

**调参数。**

一个模型有几十亿、几千亿、甚至上万亿个参数。每个参数都是一个数字。训练时，模型先拿这些数字去算答案；答案错了，就算一个 **loss（损失）**；然后再根据 loss，把这些数字往更好的方向拧一点。

整个训练循环就是：

```text
输入一批数据
模型给出预测
算 loss
根据 loss 判断参数该怎么改
更新参数
重复几万亿次
```

所以很多时候，训练模型的第一步不是先想优化器，而是先想：

```text
我到底想让模型变好在哪？
从输入到输出，每一步能不能写成公式？
这些公式能不能一路求导？
最后能不能得到一个衡量好坏的数字？
```

只要前向传播的每一步都写得清楚，而且这条计算链能一路求导，最后得到一个 loss，优化器就能沿着梯度替你一点点往那里拧。

**优化器**干的就是最后一步：

**根据梯度，决定这一步参数怎么更新。**

模型结构决定"它能表达什么"。  
训练数据决定"它见过什么"。  
优化器决定"它怎么从一堆随机数字，慢慢长成一个可用模型"。

所以优化器不是一个小细节。它是训练过程里的手。

上一期（第 166 期 https://mp.weixin.qq.com/s/iLObYwtZYqCWwRRYJVcpyA ）我们把 DeepSeek V4 的 58 页技术报告摊开过了一遍——CSA + HCA 混合注意力、mHC、Muon 优化器、FP4 QAT 等等。其中 §5 Muon 那一节只用一段话带过：1.6T 参数里 99.9% 用 Muon，AdamW 是逐元素更新、Muon 是矩阵级正交化更新。

这一期单独把"优化器"这条线展开。因为这是大模型训练里很多人觉得"看不懂跳过"的部分，但它恰好是 V4 这次最关键的换刀之一：**V4 不是随手换了个训练小配件，而是把大模型训练的"拧参数方式"换了。**

今天就从零讲清楚：SGD 是什么，AdamW 为什么统治大模型，Muon 又到底改了哪里。

◆ loss：模型错得有多离谱

━━━━━━━━━━━━━━━━━━━━

先别管优化器。先问一个更基础的问题：

**模型怎么知道自己错了？**

答案是 loss。

比如你让模型预测下一个字：

```text
输入：北京是中国的
正确答案：首都
模型预测：城市
```

模型答错了。那到底错多少？不能只说"错了"，训练需要一个数字。这个数字就是 **loss**。

loss 越大，说明错得越离谱。  
loss 越小，说明预测越接近正确答案。

训练的目标就是：

```text
让 loss 越来越小
```

如果把模型想象成一台有无数旋钮的机器，loss 就像屏幕上的错误分数。你每拧一下旋钮，分数可能下降，也可能上升。训练要做的事，就是找到一种拧法，让这个分数一路降下去。

优化器要解决的问题就是：

**这么多旋钮，每一步到底该往哪边拧、拧多少？**

━━━━━━━━━━━━━━━━━━━━

◆ 梯度：哪个方向最陡，反过来就是下降

━━━━━━━━━━━━━━━━━━━━

接下来是最容易吓人的词：**梯度**。

如果你学过微积分，梯度就是一堆偏导数。  
如果你偏导数算不明白，也没关系。这里完全可以用人话理解。

梯度回答一个问题：

**如果我稍微动一下这个参数，loss 会往哪边变？**

比如某个参数叫 `w`。现在 loss 是 10。

如果 `w` 增大一点，loss 变成 10.3，说明往这个方向拧不对。  
如果 `w` 减小一点，loss 变成 9.7，说明反方向更好。

梯度就是这个方向信号。严格说，梯度指向的是 **loss 上升最快的方向**；训练要让 loss 下降，所以更新参数时要反着走。

你可以把梯度理解成：

**loss 地形上的坡度。**

站在山坡上，梯度告诉你哪边最往上；反方向就是下山。模型训练就是在一个几百亿维的山地里找低处。最低处不是海拔，而是 loss。

但是等一下——梯度的意思知道了。真正的问题是：

**模型有几百万甚至几千亿个参数，这些梯度怎么算？**

如果对每个参数都去试一下——"动一点看 loss 变了没"——那算一次梯度的开销就是：参数数量 × 前向传播次数。百亿参数的模型光算一个 batch 的梯度就要跑百亿次前向传播，训练到宇宙热寂都训不完。

所以需要一种更聪明的方法。这就是反向传播（Backpropagation）。

反向传播不是一种近似，而是**精确**计算梯度的方法。它的核心思路只有一句话：

**算 loss 的过程是一步一步往前走的，那算梯度的过程就一步一步往回倒。**

━━━━━━━━━━━━━━━━━━━━

◆ 前向传播：从输入到 loss

━━━━━━━━━━━━━━━━━━━━

听起来还是抽象。你可能听别人讲过反向传播，看过一个简单的公式，反过来推一下感觉好像懂了。但一碰到真实模型——几十层、几百万参数、注意力机制套残差连接——梯度到底怎么穿过去的，就说不清了。

用一个具体例子走一遍就明白了。

我们用 69 期（ https://mp.weixin.qq.com/s/SV--XvFjWoEL4sG3EDMcjA ）Grokking 实验的那个模型来讲。它是一个 2 层 Transformer，做 (a+b) mod 97 的分类任务。比如输入 a=23, b=45，正确答案是 (23+45) mod 97 = 68。

前向传播，一步一步这么走：

```text
第 1 步：Embedding
  a=23 查 embed_a 表 → 得到 128 维向量 emb_a
  b=45 查 embed_b 表 → 得到 128 维向量 emb_b

第 2 步：拼接 + 位置编码
  把 emb_a 和 emb_b 拼成序列 → 形状 (2, 128)
  加上可学习的位置编码 pos_embed

第 3 步：第一层 Transformer
  多头注意力（4个头）→ 残差连接 + LayerNorm
  → FFN（128→512→128）→ 残差连接 + LayerNorm

第 4 步：第二层 Transformer
  同上，再来一遍

第 5 步：取输出
  取序列第一个位置的 128 维向量

第 6 步：线性分类头
  128 维向量 × 权重矩阵(128x97) + 偏置 → 97 维 logits

第 7 步：计算 loss
  logits 经过 softmax → 97 个概率值
  正确答案是 68，取第 68 个位置的概率
  loss = -log(P[68])
```

对应到代码里就是这几行（69 期 Grokking 实验的 `train_grokking.py`）：

```python
emb_a = self.embed_a(a)        # 第 1 步
emb_b = self.embed_b(b)
x = torch.stack([emb_a, emb_b], dim=1)  # 第 2 步
x = x + self.pos_embed.unsqueeze(0)
x = self.transformer(x)       # 第 3、4 步
logits = self.output(x[:, 0, :])  # 第 5、6 步
loss = F.cross_entropy(logits, y)  # 第 7 步
```

前向传播走完，我们得到一个数字：loss。

现在要做的事情是：把这个 loss 对模型里所有参数求梯度。

━━━━━━━━━━━━━━━━━━━━

◆ 反向传播：从 loss 往回倒

━━━━━━━━━━━━━━━━━━━━

关键观察：前向传播是一条链。每一步的输出，是下一步的输入。

```text
输入 → Embedding → Transformer层1 → Transformer层2 → 线性层 → loss
```

每一步都是一个函数。函数接受输入，做计算，产生输出。只要这个函数可导，我们就能知道：**输入变一点，输出会变多少**。

这就是导数。

现在反过来想。loss 是最后一步算出来的。loss 对线性层输出的导数，可以直接算——就是 softmax 的导数，这是一个有明确公式的东西。

拿到这个导数之后，再往前退一步。线性层的输出 = 输入 × 权重 + 偏置。所以：

```text
loss 对线性层权重的导数 = loss 对线性层输出的导数 × 线性层输入
loss 对线性层输入的导数 = loss 对线性层输出的导数 × 线性层权重
```

第一个式子给了我们线性层权重的梯度——这就是我们想要的。
第二个式子给了我们"loss 对线性层输入的敏感度"——而线性层的输入就是上一层 Transformer 的输出。于是这个信号可以继续往回传。

再退一步，进入第二层 Transformer。同样的道理：拿到"loss 对这一层输出的导数"之后，就能算出这一层每个参数的梯度，同时把信号继续往前传给第一层。

```text
loss
  ↓  求导
线性层：算出 W_out 的梯度，信号传给 Transformer 层 2
  ↓  求导
Transformer 层 2：算出这层参数的梯度，信号传给 Transformer 层 1
  ↓  求导
Transformer 层 1：算出这层参数的梯度，信号传给 Embedding
  ↓  求导
Embedding + 位置编码：算出 embed 的梯度
```

整个过程就是**前向传播的逆序**。每一步做两件事：算出这一层参数的梯度，然后把误差信号传递给前一层。一路倒回去，所有参数的梯度就都有了。

这个"一步一步倒回去"的机制，数学上叫**链式法则（Chain Rule）**。

━━━━━━━━━━━━━━━━━━━━

◆ 链式法则：为什么能一路倒回来

━━━━━━━━━━━━━━━━━━━━

链式法则是微积分里的基本定理。它说的事情很简单：

**如果一个计算是多步串联的，那整体的导数 = 每一步导数的乘积。**

```text
y = f(g(h(x)))

dy/dx = df/dg × dg/dh × dh/dx
```

翻译成前向传播的语言：

```text
loss = step7(step6(step5(step4(step3(step2(step1(input)))))))

d(loss)/d(input) = d(step7) × d(step6) × d(step5) × ... × d(step1)
```

反向传播就是从最后一步开始，把这些导数一个一个乘回去。每乘一步，就多解锁一层参数的梯度。

这个原理 1960 年代就有了。控制论领域的 Henry J. Kelley（1960）和 Arthur Bryson（1961）在最优控制问题中用过类似思路。1970 年，芬兰学者 Seppo Linnainmaa 把它形式化为"自动微分的反向模式"。但真正让它在神经网络训练中火起来的，是 **Rumelhart, Hinton & Williams 1986** 发表在 Nature 上的那篇经典论文 *Learning representations by back-propagating errors*。这里的 Hinton 就是 Geoffrey Hinton——2024 年诺贝尔物理学奖得主，深度学习三巨头之一。第一作者 Rumelhart 已于 2011 年去世。从那篇论文以后，反向传播成了神经网络训练的标准方法，一直到今天。

━━━━━━━━━━━━━━━━━━━━

◆ 为什么训练比推理吃显存

━━━━━━━━━━━━━━━━━━━━

理解了反向传播，有一个副产品马上就能解释：

**为什么训练模型比推理要多吃好几倍显存？**

回看反向传播的过程。要算线性层权重的梯度，需要用到什么？

```text
loss 对线性层权重的导数 = loss 对线性层输出的导数 × 线性层输入
```

"线性层输入"是前向传播第 5 步算出来的。如果前向传播时把这个中间结果丢了，反向传播时就拿不到，梯度就没法算。

所以前向传播时，每一步的中间结果都要存着——Embedding 的输出、每层注意力的 Q/K/V、每层 FFN 的中间激活——全部留在显存里，等着反向传播来取。

```text
前向传播：
  step1 的输出 → 存
  step2 的输出 → 存
  step3 的输出 → 存
  ...
  step7 → loss

反向传播：
  倒回 step7 → 用 step6 的输出算梯度
  倒回 step6 → 用 step5 的输出算梯度
  ...
  用完就可以释放
```

推理时只需要前向传播，每一步算完就可以丢掉中间结果，显存占用只有"当前这一步"的大小。

训练时要存所有中间结果等反向传播来用，显存峰值 ≈ 前向所有层的中间激活之和。

这就是为什么一张卡推理能跑的模型，训练时可能 OOM。也是为什么训练大模型需要各种显存优化技巧——梯度检查点（Gradient Checkpointing）就是牺牲一点计算时间，不存那么多中间结果，反向时重新算。本质上是在"存中间结果"和"重新算一遍"之间做取舍。

还有一个容易忽略的显存大户：**优化器本身。**

推理时不需要优化器，训练时优化器要给每个参数存额外的状态。以 AdamW 为例，它给每个参数存两份东西：一阶动量（梯度的移动平均）和二阶动量（梯度平方的移动平均）。

所以训练时显存里至少要装这些：

```text
参数本身：    1 份
梯度：       1 份
一阶动量：   1 份
二阶动量：   1 份
合计：       4 倍参数大小
```

一个 1.6T 参数的模型，光优化器状态就要存 3.2T 个浮点数。再加上参数和梯度本身，还没算中间激活，显存已经很吓人了。

这就是为什么大模型训练的显存瓶颈不只是"模型太大"或"中间激活太多"，**优化器状态本身也是一笔巨额开销**。

━━━━━━━━━━━━━━━━━━━━

◆ 激活函数：反向传播对前向传播的要求

━━━━━━━━━━━━━━━━━━━━

反向传播能工作，有一个前提：

**前向传播的每一步，都要能求导。**

矩阵乘法天然可导，加法天然可导，这些不是问题。真正容易出事的是**激活函数**。

激活函数是神经网络里引入非线性的地方。没有它，不管堆多少层，多层矩阵乘法的结果还是一个矩阵乘法——等于只有一层。所以激活函数必须存在。

但它又必须可导，否则梯度传不回去。

这两个要求放在一起，就是激活函数设计的核心矛盾：

**非线性要够强（不然白堆层），导数行为要够好（不然梯度传不动）。**

早期常用的是 sigmoid：

```text
sigmoid(x) = 1 / (1 + exp(-x))
```

处处连续可导，数学上完美。但有一个致命问题：它的导数最大值只有 0.25。

反向传播时梯度要一层一层往回乘。每过一层乘一次导数，每次最多保留 25%。10 层下来：

```text
0.25^10 ≈ 0.00000095
```

梯度几乎消失了。网络越深，底层参数越收不到信号，越学不动。这就是**梯度消失（Vanishing Gradient）**。

后来 ReLU 用一种粗暴的方式解决了这个问题：

```text
ReLU(x) = max(0, x)

导数：
  x > 0 → 1
  x < 0 → 0
```

正半边导数恒为 1。不管多少层乘回来，1 x 1 x 1 = 1，梯度不缩。深度学习能从几层堆到几百层，ReLU 功不可没。

x=0 那个点严格说不可导，但训练中恰好踩到 0 的概率几乎为零，实践中忽略了。

ReLU 的问题在负半边。注意：x<0 时导数为 0，这在数学上完全合法——不是"不可导所以出错"，而是导数正确地等于零。但正确归正确，后果很严重：梯度传到这儿就是零，往前面的层什么信号都传不回去。

如果某个神经元的输入总是落在负区间，那它每次反向传播拿到的梯度都是 0，参数永远不会被更新，这个神经元就彻底死了——不是数学上出了 bug，是数学上完全正确地把它杀死了。这叫 **Dead ReLU**。

后面的变体都在修这个问题：

```text
Leaky ReLU：负半边给一个小斜率（比如 0.01），别让神经元死透
GELU：用高斯分布做平滑过渡，负半边不是硬切零
SiLU / Swish：x * sigmoid(x)，处处光滑，负半边有小梯度
```

现在大模型常用的是 SwiGLU——它内部用的激活函数就是 SiLU，但多了一路门控（SiLU(xW₁) × xW₂）。LLaMA、Gemma、DeepSeek V4 全在用。SiLU 没有 ReLU 的死区问题，导数行为平滑，负半边的小梯度让信息不会彻底断流。

所以激活函数的演化史，本质上就是在反向传播的约束下找平衡：

**够非线性，够可导，梯度不消失，神经元不死。**

◆ 小结：梯度怎么来的

━━━━━━━━━━━━━━━━━━━━

把整个过程串一遍：

```text
1. 前向传播：输入 → 经过每一层 → 算出 loss
   （同时保存每一层的中间结果）

2. 反向传播：从 loss 出发 → 利用链式法则一层一层倒回去
   （每一层取出保存的中间结果，算出该层参数的梯度）

3. 结果：模型里每个参数都拿到了自己的梯度
   （即"这个参数往哪边动一点，loss 能降"）
```

反向传播的时间复杂度和前向传播差不多——只是多乘一些导数。所以训练一个 batch 的计算量大约是推理的 2-3 倍（前向一次 + 反向一次），而不是"参数数量"倍。这就是它的威力：**不管模型有多少参数，只要跑一次前向 + 一次反向，所有梯度就全有了。**

到这里，我们手里有了每个参数的梯度。它告诉我们：每个参数该往哪边调、调多少最能降低 loss。

**有了梯度之后，怎么用它更新参数？每一步迈多大？方向要不要修正？**

这就是优化器的事了。

所以最朴素的训练规则很直接：

```text
往 loss 下降最快的方向走一步
```

这就是 SGD。

━━━━━━━━━━━━━━━━━━━━

◆ SGD：最朴素的下山法

━━━━━━━━━━━━━━━━━━━━

SGD，全名 Stochastic Gradient Descent，随机梯度下降。

这个东西很老，比深度学习早得多。最早可以追溯到 **Robbins & Monro 1951** 年的论文 *A Stochastic Approximation Method*。那时候还没有神经网络，他们研究的是：

```text
如果我每次只能看到带噪声的一小批样本，
怎么一步步逼近正确答案？
```

这套思路 70 年后被搬进神经网络训练里，变成今天大家熟悉的 SGD。

名字很长，公式很短：

```text
w = w - lr * grad
```

三个东西：

- `w`：当前参数
- `grad`：梯度，也就是 loss 上升最快的方向
- `lr`：learning rate，学习率，也就是这一步迈多大

为什么是减号？

因为梯度指向的是 loss 上升最快的方向。我们要让 loss 下降，所以要反着走。

这就是 SGD 的全部核心。

如果用下山比喻：

```text
grad 告诉你坡往哪边上
-grad 就是下山方向
lr 决定你一步迈多大
```

SGD 的优点是简单、直接、便宜。

但它有一个大问题：真实的 loss 地形不是一个平滑的小山坡，而是坑坑洼洼的高维山地。有些方向很陡，有些方向很平；有些方向梯度很吵，今天说往左，明天说往右。

SGD 每一步只看当前这一批数据给出的梯度，很容易抖。

像一个人闭着眼下山，每一步都只摸脚下这一小块地面。地面稍微有点石头，就开始左右晃。

于是人们给 SGD 加了一个东西：**动量**。

━━━━━━━━━━━━━━━━━━━━

◆ 动量：别被脚下一颗石头骗了

━━━━━━━━━━━━━━━━━━━━

动量法最早出自苏联数学家 **Polyak 1964** 的论文 *Some methods of speeding up the convergence of iteration methods*，俗称 **heavy ball method**。

这个名字很形象：别把参数想成一粒没有重量的灰尘，而要想成一个有惯性的小球。小球不会因为脚下一点小坡度就立刻掉头，它会保留之前的运动方向。

后来 Nesterov 1983 又提出了改进版，也就是常说的 Nesterov Accelerated Gradient。

动量的想法很简单：

**不要只看这一步的梯度，要看最近一段时间总体往哪走。**

如果最近 100 步大多数时候都在往东南方向下山，那这一步脚下有个小石头告诉你往西，也别立刻掉头。

公式可以这么理解：

```text
momentum = 0.9 * momentum + grad
w = w - lr * momentum
```

旧方向保留 90%，新梯度加进来。这样更新方向会更平滑，不容易被单次噪声带偏。

这就是 **SGD with Momentum**。

动量解决的是"方向太抖"的问题。

但它还没解决另一个问题：

**不同参数的尺度不一样。**

有些参数的梯度天生很大，有些参数的梯度天生很小。你用同一个学习率去拧所有参数，就像拿同一个扳手力度去拧所有螺丝。有的螺丝会拧爆，有的螺丝几乎没动。

这就是 Adam 要解决的问题。

━━━━━━━━━━━━━━━━━━━━

◆ Adam：给每个参数单独装油门和刹车

━━━━━━━━━━━━━━━━━━━━

Adam 出自 **Kingma & Ba 2014** 的论文 *Adam: A Method for Stochastic Optimization*（ICLR 2015），名字是 **Adaptive Moment Estimation** 的缩写。

它站在两个前辈肩膀上：**AdaGrad** 做逐参数自适应学习率，**RMSProp** 追踪梯度波动。Adam 把这两条线合在一起，然后统治了深度学习十年。

Adam 的核心可以用一句话概括：

**每个参数都有自己的更新节奏。**

SGD 是所有参数共用一个学习率。  
Adam 会给每个参数估计两个东西：

```text
一阶动量：最近梯度的平均方向
二阶动量：最近梯度的波动大小
```

不用被"一阶""二阶"吓到。这里的"阶"不是一阶导数、二阶导数的意思，而是统计学里的"矩"（moment）。Adam 的全名 Adaptive Moment Estimation 里的 Moment 就是这个矩。

```text
一阶矩 = E[x]    = 均值（梯度的平均值）
二阶矩 = E[x²]   = 均方值（梯度平方的平均值）

（E[] 是数学期望，这里可以理解为"取平均"）
```

注意二阶矩不是方差。方差 = E[x²] - E[x]²，还要减去均值的平方。但 Adam 不需要精确的方差，只需要知道梯度波动的量级，均方值就够用了。

说人话：

**一阶动量**：最近总体往哪边走。  
**二阶动量**：这个参数最近抖得有多厉害。

如果某个参数梯度一直很稳定，那就可以大胆拧一点。  
如果某个参数梯度忽大忽小，那就少拧一点，别炸。

最终更新公式的核心逻辑：

```text
w = w - lr * 一阶动量 / sqrt(二阶动量)
```

分子是方向（往哪走），分母是波动（抖得多厉害）。**梯度稳的参数，分母小，步子大；梯度抖的参数，分母大，步子小。**

所以 Adam 像是给每个参数都装了独立的油门和刹车。

这就是它厉害的地方：**逐参数自适应**。

大模型训练数据噪声大、参数多、尺度复杂。Adam 这种每个参数自己调步长的机制，特别好用。所以过去几年，大模型训练几乎都离不开 Adam 或 AdamW。

那 AdamW 又是什么？

AdamW 出自 **Loshchilov & Hutter 2017** 的论文 *Decoupled Weight Decay Regularization*（ICLR 2019）。论文标题里的 **Decoupled** 就是它的核心改动——把 weight decay 从 Adam 的梯度更新里**解耦**出来。

━━━━━━━━━━━━━━━━━━━━

◆ AdamW：Adam 加上更干净的减肥规则

━━━━━━━━━━━━━━━━━━━━

神经网络有一个老毛病：参数容易长得太大。

参数太大，模型容易把训练数据背下来而不是学规律——训练集上很准，测试集上不行。参数膨胀还可能让训练不稳定。于是训练时通常会加一个 **weight decay（权重衰减）**：

```text
每一步都让参数稍微变小一点
```

这有点像给模型减肥。别让某些参数无限膨胀。

原版 Adam 把 weight decay 混在梯度里一起处理：先把 weight decay 加进梯度（grad' = grad + λw），然后这个混合梯度一起去算一阶动量、二阶动量、自适应步长。

问题是：**weight decay 的信号被 Adam 的自适应机制"消化"了。** Adam 会自动缩小大梯度参数的步长——但 weight decay 也被一起缩小了。结果该减肥的参数（梯度大的）反而减不动，不该减的参数（梯度小的）反而被惩罚过重。

AdamW 的改动就是把 weight decay 从 Adam 的计算里彻底拆出来：

```text
Adam：  grad' = grad + λw → 混进去一起算，被自适应步长扭曲
AdamW： 先用纯 grad 跑 Adam → 最后单独 w = w × (1 - lr × λ)
```

**解耦之后，每个参数被拉向零的力度一样大，不会被 Adam 的自适应步长扭曲。**

这个改动看起来小，但对大模型很重要。它让"根据梯度学习"和"防止参数膨胀"这两件事彻底分开，训练更稳定。

所以 AdamW 可以理解成：

**Adam 的自适应更新 + 更干净的权重衰减。**

这就是为什么 LLaMA、GPT、DeepSeek V3 这类模型长期依赖 AdamW。

但 AdamW 有一个根本局限：

**它把矩阵拆成一个个独立数字来看。**

━━━━━━━━━━━━━━━━━━━━

◆ 矩阵不是一袋散装数字

━━━━━━━━━━━━━━━━━━━━

Transformer 里最多的参数是什么？

不是一个个零散数字，而是一大堆二维矩阵。

比如最常见的线性层：

```text
y = W x
```

`x` 是输入向量。  
`W` 是权重矩阵。  
`y` 是输出向量。

这个 `W` 不是一张 Excel 表。它表示一次空间变换：把输入空间里的一个方向，映射到输出空间里的另一个方向。

注意力里的 Q/K/V 投影矩阵是 `W`。  
FFN 里的升维、降维矩阵是 `W`。  
MoE 专家里面的大部分参数也是 `W`。

也就是说，**大模型的主体不是一袋散装参数，而是一堆矩阵**。

AdamW 怎么看这些矩阵？

**逐元素看。**

矩阵里每个格子都是一个参数，每个参数自己算自己的梯度、动量、步长。这个方法很强，但它不关心矩阵整体的几何形状。

问题是：矩阵真正重要的，往往是整体形状。

一个矩阵会把某些方向放大，把某些方向压扁，把某些方向旋转过去。训练时更新矩阵，如果某几个方向更新太猛，其他方向几乎不动，矩阵就容易变得很偏。

这就是 Muon 的入口。

**AdamW 是把矩阵当 Excel 表格改。  
Muon 是把矩阵当几何对象改。**

━━━━━━━━━━━━━━━━━━━━

◆ Muon：矩阵级更新

━━━━━━━━━━━━━━━━━━━━

Muon 是 **Keller Jordan 2024 年 12 月** 在博客上首先公布的优化器，后续整理成 2025 年的论文 *Muon: An optimizer for hidden layers in neural networks*。

它先是在小规模实验和 NanoGPT speedrun 排行榜上很快吃下记录，后来又在 **Moonshot Kimi K2（2025）** 和 **DeepSeek V4（2026）** 这两个超大规模模型上证明了自己。

这就说明一件事：Muon 不是只适合论文玩具实验，它已经开始进入真正的大模型训练现场。

全名是：

**MomentUm Orthogonalized by Newton-Schulz**

名字先不用背。抓住核心操作就够了：

```text
1. 算出梯度矩阵 G
2. 对 G 做动量平滑 → 得到动量矩阵 M
   M = β × M + (1 - β) × G
   （和前面动量法一样的指数移动平均，不存历史，只存一份 M）
3. 对 M 做正交化 → 得到 M'
4. 用 M' 去更新权重：W = W - lr × M'
```

注意：被正交化的是**动量矩阵 M**（梯度平滑后的结果），不是权重矩阵 W。W 是被更新的对象，M 是拿来更新它的工具。Muon 改造的是工具的形状，不是目标本身。全名 MomentUm Orthogonalized 说的就是这个：对动量做正交化。

关键是第 3 步：**正交化**。

正交化是什么意思？

在二维平面里，横轴和纵轴互相垂直，这叫正交。推广到高维空间，正交就是"方向之间尽量互不重叠、互不打架"。

Muon 对更新矩阵做正交化，就是想让更新方向更均匀地铺开：

```text
不要所有更新都挤在少数几个方向上
不要某些方向更新太猛，某些方向完全不动
让整个矩阵以更健康的方式一起变化
```

你可以先不用管 Newton-Schulz，也不用管正交化的精确定义。先抓住这个机制：

**AdamW 是逐元素更新。  
Muon 是矩阵级更新。**

AdamW 问的是：

```text
这个参数该怎么改？
```

Muon 问的是：

```text
这个矩阵整体该怎么改？
```

这个差别，对小模型可能没那么明显。  
但对 1.6T 参数的大模型，绝大多数参数都在大矩阵里，这个差别会被规模放大。

━━━━━━━━━━━━━━━━━━━━

◆ 正交化为什么有用

━━━━━━━━━━━━━━━━━━━━

还是用程序员能懂的方式讲。

假设一个矩阵有很多方向可以学。有的方向很常见，训练数据天天刺激它；有的方向很稀有，偶尔才出现。

普通逐元素更新很容易出现一个问题：

**常见方向越来越强，稀有方向越来越弱。**

就像一个团队里，嗓门大的人越来越有话语权，安静的人永远插不上嘴。最后整个团队只会沿着少数几个方向猛冲。

Muon 的正交化相当于给更新方向做一次整理：

**别挤在一起，分散一点。  
别只让几个方向说话，其他方向也要有空间。**

从几何上说，它在控制更新矩阵的形状，让更新不要过度集中在少数奇异方向上。

不用纠结"奇异方向"这个词。直觉就是：

**矩阵更新不要偏科。**

AdamW 控制的是每个参数自己的步长。  
Muon 控制的是整个更新矩阵的形状。

再稍微数学一点说，就是别让更新矩阵的能量塌到少数几个方向上。塌得太厉害，模型就会沿着少数方向猛学，其他方向学不动。正交化就是把这股更新力量重新摊开。

结果就是更新既快又稳：每一步更有效，不容易在少数方向上来回抖，也少一些局部爆炸。

所以 Muon 不是神秘玄学。它做的是一件很朴素的事：

**既然大模型主体是矩阵，那更新时就别把矩阵拆成散装数字。**

━━━━━━━━━━━━━━━━━━━━

◆ Newton-Schulz：便宜地做近似正交化

━━━━━━━━━━━━━━━━━━━━

理论上，想把一个矩阵正交化，可以用 SVD（奇异值分解）。

SVD 说的事情是：**任何矩阵都可以拆成三步操作——先转一下，再拉伸，再转一下。**

```text
M = U × S × Vᵀ

Vᵀ：先把输入转到一组新的方向上（从哪些方向读）
S ：在每个方向上拉伸（每个方向有多强）
U ：再转到输出方向上（往哪些方向写）
```

这里的"方向"不是单个维度上的分量。拿一个 128×512 的矩阵举例，SVD 分解出来的每个"方向"是 128 维空间和 512 维空间里各一个向量配对。它描述的是"输入沿这个 512 维方向进来的信号，会被映射到输出的这个 128 维方向上"。

S 是一个对角矩阵，大小取行和列里小的那个——128×512 的矩阵，S 就是 128×128，最多 128 个独立的映射模式。对角线上的数叫**奇异值**，说的是每种输入→输出的映射模式各有多强。谁大，谁就是主要模式：

```text
S = diag(100, 80, 0.1, 0.01)

意思是：
  第 1 个方向放大 100 倍
  第 2 个方向放大 80 倍
  第 3 个方向几乎不动
  第 4 个方向几乎不动
```

如果这是一个更新矩阵，那前两个方向吃掉了几乎全部的更新能量，后面的方向基本没被更新。

Muon 的正交化就是：**扔掉 S（强度），只保留 U 和 Vᵀ（方向）。**

```text
原始：      M  = U × S × Vᵀ
正交化后：  M' = U × Vᵀ
```

方向全部保留，强度全部拉平成 1。每个方向都迈一样大的步子，不偏科。

一句话概括 Muon 正交化的核心思路：**相信梯度告诉你的方向，但拒绝梯度给的各方向上的不同强度。**

这很漂亮，但问题是：

**SVD 太贵。**

大模型训练每一步都有海量矩阵要更新。你不可能每一步都给每个矩阵做一次完整 SVD。那不是训练模型，是烧钱做线性代数祭祀。

Muon 用的是 **Newton-Schulz 迭代**——这是一类很老的数值方法，可以追溯到 **Schulz 1933** 的论文，后来被广泛用于矩阵求逆和矩阵函数的近似。

它不是精确做 SVD，而是用几轮矩阵乘法，快速逼近"正交化"这个结果。

为什么这很重要？

因为 GPU 最擅长的就是矩阵乘法。只要把操作变成矩阵乘法，就能吃满硬件。

一句话：

**SVD 是精确但昂贵的手术刀。  
Newton-Schulz 是便宜很多、GPU 跑得飞快的近似刀。**

Muon 的现实价值就在这里：它不是只在数学上好看，而是能真的放进大模型训练流程里跑。

━━━━━━━━━━━━━━━━━━━━

◆ DeepSeek V4 为什么要换 Muon

━━━━━━━━━━━━━━━━━━━━

现在可以回到 DeepSeek V4。

V4 有 1.6T 总参数，384 个 MoE 专家，每次激活 6 个专家。真正的参数大头在哪里？

在专家矩阵里。

MoE 专家本质上就是一堆巨大的 FFN 矩阵。V4 的注意力投影、专家 FFN、mHC 混合矩阵，都是 Muon 最擅长处理的二维权重矩阵。

所以 Muon 对 V4 的意义不是"换个 optimizer 试试"。

它是在利用一个事实：

**Transformer 的主体是矩阵。**

AdamW 的成功来自逐参数自适应。  
Muon 的野心是再往前一步：

**从逐参数优化，走向矩阵几何优化。**

这就是为什么 V4 会把 99.9% 参数交给 Muon。

在小模型上，Muon 多出来的正交化计算未必划算。  
但在 1.6T 这种规模上，如果它能让训练更快收敛、更稳定，省下来的训练 token、训练时间、训练事故，远远超过正交化本身的成本。

超大模型训练不是"每一步最便宜"就赢，而是：

**谁能更稳、更快地走到目标 loss，谁赢。**

━━━━━━━━━━━━━━━━━━━━

◆ 为什么不是所有参数都用 Muon

━━━━━━━━━━━━━━━━━━━━

Muon 的核心优势是矩阵级更新。所以 V4 的参数按形状分成三类：

**第一类：二维权重矩阵 → 全部用 Muon。**

- 注意力层的 Q/K/V/O 投影矩阵
- FFN 的升维、降维矩阵
- MoE 专家的 FFN 权重
- mHC 的混合矩阵

这些是大模型的参数主体，正好能吃到 Muon 的几何红利。V4 的 1.6T 参数里 **99.9%** 属于这一类。

**第二类：标量和向量参数 → 用 AdamW。**

- **RMSNorm**：每个维度一个缩放系数，向量
- **bias**：向量
- **标量参数**：一维的，没有"矩阵形状"可以正交化

这些不是矩阵，Muon 没法用，AdamW 的逐元素自适应刚好合适。

**第三类：唯一用 AdamW 的矩阵 → embedding / 输出头。**

这是整个模型里最特殊的一块参数。它明明是个矩阵，却不适合 Muon，原因有两条。

第一，**极度不光滑**。embedding 矩阵的每一行是一个 token 的向量表示——"猫"和"狗"可能很近，"猫"和"且"隔着十万八千里。行与行之间没有连续关系，整个矩阵不是一个光滑的空间变换，而是一本离散的字典。Muon 的正交化假设矩阵描述的是光滑映射，各方向曲率差别不大。但 embedding 是几万个离散点拼在一起，每个点独立更新，根本没有"光滑流形"可言。**光滑假设不成立，Muon 不适用。**

第二，**两头收梯度**。大多数大模型（包括 GPT、LLaMA）会让 embedding 矩阵和输出头**共享同一个权重**（weight tying）。embedding 做的是查表——给一个 token id，取出对应那一行向量。输出头做的是反过来——拿当前隐藏向量和词表里每个 token 的向量算相似度，像谁就猜谁。

共享权重意味着同一个矩阵既是入口也是出口。反向传播时它会从**两头收到梯度**：

```text
反向传播刚开始：从输出头来的梯度（"这个 token 猜错了"）
反向传播到最后：从 embedding 来的梯度（"这个 token 的输入表示该调"）
```

两份方向完全不同的梯度叠在同一个矩阵上。如果用 Muon 做正交化，可能把两股有用但方向矛盾的信号一起抹平，反而有害。

**不光滑 + 两头收梯度**，embedding 矩阵就是整个模型里最不适合 Muon 的参数。留给 AdamW——逐元素自适应刚好能分别消化来自两端的不同信号。

所以 166 期里那组数字背后的逻辑是清楚的：

V4 的 1.6T 参数里，**99.9% 用 Muon**。  
只有 embedding / 输出头、RMSNorm 等模块还用 AdamW，合计不到 20 亿参数，占比 **0.1%**。

不是因为 AdamW 过时了，而是**参数的形状和性质决定了该用什么优化器**。

━━━━━━━━━━━━━━━━━━━━

◆ 但 Muon 不是魔法

━━━━━━━━━━━━━━━━━━━━

Muon 很重要，但别把它神化。

它有代价：

1. **计算更复杂**：多了正交化步骤，哪怕 Newton-Schulz 已经很快，也不是免费。
2. **实现更麻烦**：要把参数分组，哪些用 Muon，哪些继续用 AdamW。
3. **不一定适合所有任务**：小模型、小数据、非矩阵参数，收益未必明显。

还有一个更根本的问题：**正交化抹掉了梯度本身的强度信息。** 梯度在某些方向上特别强，有时候是有道理的——那个方向的 loss 曲率确实更大，该快走。Muon 无差别拉平，等于假设那个尖刺不存在。

换句话说，Muon 的隐含假设是：**loss 地形在各方向上曲率差别不大，是一个比较光滑的流形，没有尖刺。**

举个反例。前面讲反向传播时用过的那个 69 期 Grokking 实验，训练过程中有效维度从 78 维骤降到 8 维——模型发现了 (a+b) mod 97 的低维数学结构，然后猛地塌上去。这就是一个典型的"尖刺"场景：少数方向极其重要，其他方向无关紧要。如果在这种任务上用 Muon 强行拉平所有方向，反而会拖慢模型找到那几个关键方向。

大模型学的是自然语言，不是一个单一的数学结构。语法、语义、常识、推理……每一层都有自己的方向，需要的有效维度非常高，不太会出现那种骤降到个位数的塌缩。

最近有几篇论文专门测量了语言模型训练过程中的维度变化（arxiv 2506.01034, 2604.04655, 2604.16431），观察到的幅度大约在 20-30%——是缓慢、温和的下降，不是 69 期那种 78 维骤降到 8 维的 90% 塌缩。

这就解释了为什么 Muon 在大模型上好使：loss 地形确实比小模型上的数学任务光滑得多，没有那种"突然发现低维结构然后猛塌"的尖刺。参数越多、维度越高，loss 地形越倾向于光滑，Muon 的光滑假设就越容易成立。

不过光滑归光滑，Muon 本身也会带来新问题。

Moonshot 在开发 Kimi K2 的过程中，用原版 Muon 训练 9B 规模的中间模型时，注意力分数的最大值飙到了 1000 以上，直接导致 loss spike 和训练发散。Kimi K2 的架构是标准 Transformer，没有特殊的残差设计，所以这个问题纯粹是 Muon 本身带来的——正交化让更新步子迈得太均匀，注意力层的 QK 数值失控。他们的解法是在优化器端打补丁：MuonClip，给注意力层的 QK 加裁剪，之后万亿参数规模训练做到了零 loss spike。

DeepSeek V4 的情况更复杂。V4 不仅用了 Muon，还用了字节跳动的 HC（Hyper-Connections，多路残差连接）。HC 本身就有信号放大的问题——四路残差叠在一起，信号放大倍数可以到 3000 倍。也就是说 V4 同时背了两个风险：Muon 的正交化风险 + HC 的信号放大风险。

V4 的解法是 mHC（流形约束超连接，37 期 https://mp.weixin.qq.com/s/IMU__NKt_L41YeHKi7_A1g 详细讲过）——在 HC 基础上加流形约束，把信号放大从 3000 倍压到 1.6 倍。信号放大从架构层面被掐掉了，Muon 的正交化再怎么均匀迈步，数值也不会被放大到离谱的范围。所以 V4 用的是原版 Muon，不需要 MuonClip。

这是我们的猜想，整理一下三家的对比：

```text
Kimi K2：标准架构 + Muon → Muon 本身导致注意力爆炸
         → 优化器端兜底（MuonClip）

字节 HC：多路残差放大 3000 倍 → 不管用什么优化器都危险

V4：    Muon + HC，两个风险叠加
         → mHC 从架构端同时压住两个风险
         → 优化器保持原版
```

如果这个猜想成立，mHC 不只是"让 HC 更稳定"，而是**同时解决了 HC 的放大问题和 Muon 的数值问题**，一箭双雕。

所以 Muon 不是 AdamW 的简单替代品。

更准确的说法是：

**在 Transformer 大矩阵占绝对主体的场景里，Muon 开始比 AdamW 更吃模型结构。**

AdamW 是万能扳手。  
Muon 是大矩阵专用的几何扳手。

DeepSeek V4 这种模型，正好满身都是大矩阵。

━━━━━━━━━━━━━━━━━━━━

◆ 一张表总结

━━━━━━━━━━━━━━━━━━━━

| 优化器 | 核心想法 | 人话理解 | 问题 |
|---|---|---|---|
| SGD | `w = w - lr * grad` | 顺着坡往下走 | 容易抖，所有参数共用步长 |
| Momentum | 累积历史梯度方向 | 别被脚下一颗石头骗了 | 还是没有逐参数自适应 |
| Adam | 一阶动量 + 二阶动量 | 每个参数有自己的油门和刹车 | 把矩阵拆成散装数字 |
| AdamW | Adam + 解耦权重衰减 | 一边学习，一边干净减肥 | 仍然是逐元素更新 |
| Muon | 动量更新 + 矩阵正交化 | 把矩阵当几何对象整体更新 | 计算和实现更复杂 |

━━━━━━━━━━━━━━━━━━━━

◆ 最后

━━━━━━━━━━━━━━━━━━━━

SGD 像摸着坡往下走。  
Momentum 让你别被单次噪声带偏。  
AdamW 给每个参数单独装了油门和刹车。  
Muon 再往前走了一步：它不再把矩阵拆成散装数字，而是把矩阵当成一个整体来更新。

这就是优化器这条线的演化：

```text
从一个参数怎么改
到每个参数怎么自适应
再到整个矩阵一起改
```

Transformer 的主体就是一堆矩阵。  
当模型大到 1.6T 参数时，优化器就不再是训练细节。

**优化器，就是模型长成什么样的手。**

DeepSeek V4 把 99.9% 参数交给 Muon，本质上是在说：

**大模型训练，开始认真吃矩阵几何的红利了。**
