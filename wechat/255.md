【Kimi K3】2.8T 全球最大开放权重模型预发布——这次 Moonshot 敢卖美国价了

━━━━━━━━━━━━━━━━━━━━

◆ 发生了什么：先说清楚这篇的边界

━━━━━━━━━━━━━━━━━━━━

2026 年 7 月 16 日，月之暗面（Moonshot AI）发布 Kimi K3：2.8T 总参数，第一个承诺开放权重的 3T 级模型。权重将在 7 月 27 日发布；按已公布规模计算，它落地后将从 DeepSeek V4-Pro（1.6T）手中接过"全球最大开放权重模型"的头衔。

但先把话说在前面：**正式技术报告要 7 月 27 日才随权重一起发布。** 这意味着层数、hidden size、激活参数量、词表大小、训练 token 量、训练 FLOPs、RL 算法细节、开源协议——这些我们读 V4 技术报告时最看重的数字，K3 全部**官方尚未披露**。所以这一期不是精读，是速报：基于官方博客 + 第三方独立评测，先把账对上。精读要等 7 月 27 日材料落地再说。

这期的主线不是架构，是定价。一句话：**K3 的 API 定价和 Claude Sonnet 5 一分不差**——输入 \$3.00、输出 \$15.00 每百万 token。这是中国前沿模型第一次定价完全对齐美国前沿模型。在我们看来，这个价格比任何 benchmark 都有信息量，下文展开。

◆ 架构速览：官方说了什么，没说什么

━━━━━━━━━━━━━━━━━━━━

先说官方博客给出的硬信息。

**MoE：896 个专家，每个 token 激活 16 个。** 名字叫 Stable LatentMoE。机制还是那个机制——每个 token 进来，路由器给它打分，从 896 个专家里挑 16 个做计算，其余 880 个待命。对比 DeepSeek V4-Pro 的 384 选 6，很容易脱口而出"稀疏度更激进"——但算一下就知道这话打折扣：384/6 = 64，896/16 = 56，激活比例其实接近，V4 反而更稀疏一点。K3 真正确定的变化是**专家粒度划分得更细，每个 token 的专家组合数更多**。至于单个专家是否更小，要等 hidden size 和专家 FFN 宽度公布后才能确认。

注意一个坑：**K3 的激活参数量没有披露**。V4 系列是明牌（比如 V4-Flash 284B 总 / 13B 激活），K3 只给了 2.8T 总数。不要拿"2.8T × 16/896"去估算——这种算法假设参数全在 MoE 里，实际还要扣掉注意力、embedding、共享层，估算出来的数字没意义。激活参数量是推理成本的核心指标，等 7 月 27 日技术报告。

**注意力：三件套的混搭。** 官方点名了三个组件：

1. **KDA（Kimi Delta Attention）**：一种改进 Gated DeltaNet 的线性注意力模块，用更细粒度的门控管理有限大小的循环状态。标准注意力每处理一个 token 要和所有历史 token 做点积，成本随长度平方增长；线性注意力把历史压缩成固定大小的状态，成本变线性，但会丢信息。KDA 出自 Kimi Linear 论文（arXiv 2510.26692）；Kimi Linear 才是把 KDA 与 MLA 按层混合的整体架构：大部分层走低成本的 KDA，少数层保留全局注意力，以弥补固定状态的信息损失。注意：**该论文的实验是 48B 总参数 / 3B 激活参数规模**，在 2.8T 上效果如何，技术报告出来前只能存疑。
2. **AttnRes（Attention Residuals，注意力残差）**：arXiv 2603.15031，同样是小模型实验，细节不展开，等技术报告。
3. **Gated MLA**：MLA 的低秩 KV 压缩思路（DeepSeek V2 首创）加一个门控。具体门控什么，官方没说细节。

**上下文 1M token**（1,048,576），和 V4-Pro 同级。

**量化：同族不同代。** V4 是 FP4（E2M1）专家权重 QAT + FP8 计算（166 期 https://mp.weixin.qq.com/s/iLObYwtZYqCWwRRYJVcpyA 精读过这套流程）；K3 是 MXFP4 权重 + MXFP8 激活——MX 系列是带微缩放（microscaling）的分块浮点格式，比传统 E2M1 多一层块级缩放因子。更关键的区别在 QAT 介入时机：V4 的量化感知训练在预训练/特定模块，K3 官方明确说**从 SFT 阶段就开始 QAT**。官方还建议 64+ 卡 supernode 部署，KDA 的 prefix caching 已贡献给 vLLM。

**几个新名词，先记住名字等报告**：分位数均衡（Quantile Balancing，按路由器分数的分位数而不是绝对分数来做专家负载均衡）、Per-Head Muon（Muon 优化器的逐头变体）、SiTU（Sigmoid Tanh Unit，新激活函数）。机制细节官方都没展开，现在展开就是脑补，不展开。

**另外两条**：相对 K2 的缩放效率约 2.5×（同样是官方口径）；原生多模态，支持文本/图像/视频输入。

**不知道清单**（诚实版）：层数、hidden size、激活参数量、词表、训练 tokens、训练 FLOPs、RL 算法、开源协议——以上全部官方尚未披露，等 7 月 27 日技术报告。

**和 V4 的硬对比**：

| | DeepSeek V4-Pro | Kimi K3 |
|---|---|---|
| 总参数 | 1.6T | 2.8T |
| MoE | 384 选 6 | 896 选 16 |
| 激活参数 | 明牌（如 Flash 284B 总 / 13B 激活） | 未披露 |
| 量化 | FP4 权重 QAT + FP8 计算 | MXFP4 权重 + MXFP8 激活，QAT 从 SFT 开始 |
| 上下文 | 1M | 1M |
| 技术报告 | 发布当日 58 页全公开 | 7 月 27 日随权重发布 |

◆ 独立评测对账：略低于前二，高于其余一切

━━━━━━━━━━━━━━━━━━━━

官方自报的 benchmark 我们只列不背书——官方脚注自认测试框架混用（K3 用 KimiCode、Claude 系用 Claude Code、GPT 系用 Codex），厂商自报口径，仅供参考：Terminal-Bench 2.1 88.3（Opus 4.8 84.6）、BrowseComp 91.2、DeepSWE 67.5、FrontierSWE 81.2（Fable 5 86.6）、Kimi Code Bench 2.0 72.9（Fable 5 76.9）、SWE Marathon 42.0、MMMU-Pro 81.6、MathVision 94.3、GPQA Diamond 93.5。

真正有分量的是第三方。

**Artificial Analysis（07-17）**——Intelligence Index 57 分，总榜第 4/189：Fable 5 60、GPT-5.6 Sol 59、K3 57、Opus 4.8 56、GLM-5.2 51。在已宣布开放权重的模型中，K3 是**当前评测得分最高的一个**，比前任开源第一 GLM-5.2 高 6 分。agentic 任务评测 GDPval-AA v2 上 K3 拿到 Elo 1668（Fable 5 1760、Opus 4.8 1600，而 K2.6 只有 1190——这个代际提升幅度相当夸张）；长程知识工作 AA-Briefcase Elo 1547，仅逊 Fable 5。

**LMArena（07-16）**——前端代码榜登顶，1679 Elo 超过 Claude Fable 5，7 个前端子域拿了 6 个第一，比 K2.6 的第 18 名跃升 17 位。这个榜有意思的细节是：K3 发布前用匿名代号 **"Kivine"** 参加盲测——分数是在没人知道它是谁的情况下打出来的，不存在"中国品牌被压分/抬分"的争议空间。这是盲测机制最有说服力的用法。

**社区实测**。HN 热帖 6 小时 836 分、500 评论，高赞原话："盲测里我不确定能把它和 Fable 区分开……\$100 的 Kimi Coding 套餐额度大致相当于我 \$200 的 Anthropic 套餐。"冷静派的声音同样有代表性："Fable 才出两周、GPT-5.6 Sol 才出一周。K3 略低于这两者、高于其他一切。"

Simon Willison（07-16）的实测补充了几个 API 层面的细节：目前只有 max 一档 reasoning effort，没有中低档；一个 SVG 鹈鹕 prompt 烧了 13241 个 reasoning token，花了 25 美分；还发现了一个约 85 token 的隐藏 system prompt（发一句 "hi" 计费 86 token）。视觉理解质量很好。

Reddit r/kimi 的口径相对克制："K3 不错，但比 K2.7 强得有限。"最具体的吐槽是配额：agent 持续负载下每小时烧约 6% 周配额。

把以上汇总成一句话：**独立评测的共识是"略低于 Fable 5 和 GPT-5.6 Sol，高于其余一切"；在已宣布开放权重的模型中，它目前得分最高。** 这个判断和官方自报的位次基本一致——这在厂商发布里是难得的。

◆ 主线：定价对齐 Sonnet 5，把自己摆上 Sonnet 价位

━━━━━━━━━━━━━━━━━━━━

这次最大的新闻不是分数，是价格表：

| 模型 | 输入（缓存命中/未命中）| 输出 |
|---|---|---|
| Kimi K3 | \$0.30 / \$3.00 | \$15.00 |
| Kimi K2.6 | \$0.16 / \$0.95 | \$4.00 |
| Claude Sonnet 5 | \$0.30 / \$3.00 | \$15.00 |
| Claude Fable 5 | \$1.00 / \$10.00 | \$50.00 |
| GPT-5.6 Sol | \$0.50 / \$5.00 | \$30.00 |

（表格来源：The Decoder，07-17）

K3 和 Claude Sonnet 5 的价格**一分不差**，连缓存命中价都一样。中国前沿模型第一次定价完全对齐美国前沿模型。

为什么这比 benchmark 有信息量？Benchmark 可以挑、可以刷、测试框架可以混用（官方自己就承认了），但定价是厂商对自己产品位置的商业判断。Moonshot 敢把 K3 挂到 Sonnet 5 同一价位，说明它判断自己已经进入同一个采购比较区间，押注这已经不是需要靠低价补偿的代差。至于市场认不认，要看后续调用量、续费率和收入——现在只是投票箱摆出来了，票还没开。

反过来，这个价格也意味着一个时代的转向。Simon Willison 直接说：定价是最大新闻，标志中国模型低价时代转向。The Decoder（07-17）的标题更直白："接近 GPT-5.6 Sol 与 Fable 5，同时宣告超便宜中国 AI 的终结。"摩根士丹利（07-17）的口径是"中国前沿大模型已实现全面追赶，规模、性能与定价同步突破"。Reuters 的表述相对平实：中国 Moonshot 发布全球最大开源模型，缩小与美国对手差距。

成本账也要算。Artificial Analysis 的每任务成本：K3 \$0.94，GPT-5.6 Sol \$1.04，Opus 4.8 \$1.80，GLM-5.2 \$0.32，DeepSeek V4-Pro **\$0.04**。K3 比 V4-Pro 贵 20 多倍——一个继续白菜，一个对标 Sonnet。中国开源模型内部已经分化出两条路线：DeepSeek 走极致成本，Kimi 走对标前沿。166 期我们聊 V4 时说的"白菜价"叙事，在 K3 这里正式翻篇了。

值得一提的是 AA 的一个细节：K3 的输出 token 用量比 K2.6 减少 21%——话变少了。这部分对冲了单价上涨，按每任务成本算，涨幅没有单价涨幅那么吓人。

◆ 两味冷静剂

━━━━━━━━━━━━━━━━━━━━

免得这篇写成通稿，泼两盆冷水。

**第一味：幻觉率上升了。** AA-Omniscience 评测显示，K3 的幻觉率从 K2.6 的 39% 升至 **51%**。公平起见要说全：准确率同时从 33% 升到 46%——模型知道得更多了，但"不知道的也说"的比例也上去了。这个 trade-off 怎么解读见仁见智：准确率 +13 个点是实打实的能力提升，幻觉率 +12 个点意味着把它用在需要事实可靠性的场景时，核验成本反而更高。技术报告出来前，我们倾向于把它当作"值得盯住的风险项"，而不是直接定性为退步。

**第二味：成本和使用门槛不再友好。** 配额吐槽不是孤例——agent 场景每小时烧约 6% 周配额，重度用户需要认真算账。API 侧目前只有 max 一档 reasoning effort，想省钱的用户没有中低档可选；那个 85 token 的隐藏 system prompt 虽然不大，但意味着每次调用都有一笔看不见的固定开销。这些都不是大问题，但和"便宜大碗"的老印象放在一起，落差是真实的。

◆ 结尾：等 7 月 27 日

━━━━━━━━━━━━━━━━━━━━

这一期的定位很清楚：材料没到齐之前的对账。官方博客 + 三家第三方评测（AA、LMArena、Simon Willison 实测）+ 社区反馈，交叉验证下来的图景是自洽的——2.8T 且已承诺开放权重、已宣布开放权重的模型中独立评测得分最高、定价对齐 Sonnet 5，三件事互相印证。

但架构层面我们还站在门外。896 选 16 的实际激活参数量、KDA/AttnRes 从 48B 小模型到 2.8T 的放大效果、Quantile Balancing 的具体机制、QAT 从 SFT 开始的实际收益、训练 FLOPs 和 RL 配方、开源协议——这些都要等 **7 月 27 日权重和技术报告落地**。166 期读 V4 那 58 页 PDF 的经验告诉我们，速报和精读的差别就在于"坐下来看代码"那一步。材料到齐，我们再对一次账。
