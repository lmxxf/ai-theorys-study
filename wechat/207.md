◆ 他到底在干啥：三分钟讲清"世界模型"

━━━━━━━━━━━━━━━━━━━━

先把概念搞清楚。"世界模型"这个词被用烂了，但杨老头儿说的"世界模型"跟 LLM 的"世界"是两码事。

LLM 的"世界"是什么？是 token 序列。输入一串字，预测下一个字。它对世界的感知完全通过文本——人类写的书、论文、代码、网页。

杨老头儿说的"世界模型"是什么？是 AI 脑子里装一个物理模拟器。不从文字学，从像素、传感器学。在 embedding 空间里模拟状态转移，然后基于模拟结果做规划。

区别一目了然：

**输入**
- LLM：文本 token
- 世界模型：像素 / 传感器数据

**内部表示**
- LLM：token embedding 序列
- 世界模型：embedding 空间中的状态向量

**生成方式**
- LLM：自回归（一个 token 一个 token 往外吐）
- 世界模型：在 embedding 空间里做状态转移预测（不生成 token）

**规划能力**
- LLM：没有显式规划（靠 chain-of-thought 硬凑）
- 世界模型：在 embedding 空间搜索最优动作序列

简单说：LLM 是"读了所有书的学者"，杨老头儿要造的是"亲眼看过世界的机器人"。

这个方向本身不蠢。LLM 确实缺具身感——我们在 163 期就说过，AI 的概念是"自上而下"从语言符号涌现的，没有身体经验的锚点。杨老头儿看到了这个缺陷。问题不在方向，在结论——他从"LLM 有缺陷"跳到了"LLM 要扔掉"。

◆ 离职后发了啥：三篇论文全是 JEPA

━━━━━━━━━━━━━━━━━━━━

杨老头儿离开 Meta 后，这条世界模型路线公开可见的新成果，截至 2026 年 5 月，主要是两篇 2026 年论文（再加上一篇 Meta 时期的旧作经常被捆绑宣传）。全部沿着同一条技术路线：JEPA。

先说这条线的历史。2022 年杨老头儿提出 JEPA 架构（Joint Embedding Predictive Architecture），2023 年出了 I-JEPA（图像版），2024 年出了 V-JEPA（视频版），2025 年演化成 LeJEPA（JEPA + alignment + Gaussian regularization / SIGReg 这条线），2026 年终于搞出了 LeWorldModel 和配套的理论证明。四年一条线走到黑，没有新范式。

**第一篇：LeWorldModel（2026-03，arXiv 2603.19312）**

15M 参数，单 GPU 训几小时。从像素端到端学世界模型——这是第一个稳定的端到端 JEPA。

这篇论文的整个架构只有两个 loss，搞清楚这两个 loss 就搞清楚了全部。

**第一个 loss：next-embedding 预测**

传统做法是让模型看当前帧的像素，预测下一帧的**像素**——逐像素还原。这个很蠢，因为像素里大量冗余（背景没变、光照噪声），模型把算力浪费在重建无关细节上。

JEPA 不预测像素，预测 embedding：

```
当前帧像素 → encoder → 当前 embedding z_t
下一帧像素 → encoder → 下一帧 embedding z_{t+1}（训练时的真值）

predictor(z_t, action_t) → ẑ_{t+1}（预测的下一帧 embedding）

prediction_loss = ||ẑ_{t+1} - z_{t+1}||²
```

encoder 把像素压缩成一个向量（embedding），predictor 拿当前向量 + 动作，预测下一步的向量。loss 就是预测向量和真实向量之间的均方误差。

encoder 把像素压成向量,好处是它可以自己学"什么信息值得保留"——背景没变就不编码,只编码跟决策有关的状态。比逐像素重建高效得多。

**这里要停一下,解释公式里那个 action_t 是怎么回事——"下一帧"怎么和"运动"搭上的。**

普通视频预测模型（比如 Sora 那种）只预测"下一帧画面长什么样",不管"你做了什么动作"。输入一串帧,输出下一帧。你没法问它"如果我往左转方向盘,画面会怎么变"。

LeWorldModel 的 predictor 多吃了一个输入:**动作（action）**。训练数据长这样:

```
(帧1, 动作1, 帧2), (帧2, 动作2, 帧3), (帧3, 动作3, 帧4), ...
```

每条数据都是"看到什么 + 做了什么 → 变成了什么"。predictor 学的不是"下一帧一般长什么样",而是"**执行这个动作之后,下一帧会变成什么样**"。action 是视觉和运动之间的桥。

这就是杨老头儿说的"世界模型能做规划"的基础:在 embedding 空间里试不同的 action 序列,看哪条能到达目标状态,选最优路径执行。

但这也正是局限——训练数据里必须有**明确的动作标签**。模拟器里好办,机械臂的动作是关节角度变化(连续数值),导航是离散方向(上下左右),这些都容易标注。真实世界就难了——你走路的时候哪块肌肉用了多少力？开车的时候方向盘转了几度？要么靠传感器,要么靠模拟器。所以 LeWorldModel 的实验还停在受控任务里:Two-Room 导航、Reacher、Push-T、OGBench-Cube 这类 2D/3D 控制环境。它证明的是像素到 latent planning 的效率,不是真实开放物理世界。

跟 LLM 对比一下:LLM 不需要"动作标签",因为语言本身就是动作——你打字,token 进来,模型接着预测。所以 LLM 能在开放世界里工作(任何人打任何字都行),LeWorldModel 只能在动作空间已知、可标注的封闭环境里工作。

回到坍塌问题。

**这里有个致命问题：** 如果 encoder 偷懒，把所有输入都映射到同一个点（比如全零向量），那预测永远是对的（0 预测 0，loss = 0），但 embedding 完全没用。这就是自监督学习的祖传毛病——**表征坍塌（representation collapse）**。以前大家用各种偏方对付它：stop-gradient、teacher-student、EMA 动量网络、VICReg、Barlow Twins……超参一堆，调参靠命。

杨老头儿的解法是第二个 loss。

**第二个 loss：SIGReg 高斯正则化**

思路很直接：如果 embedding 坍塌成了一个点，那它的分布肯定不是高斯分布（高斯是散开的）。所以**强迫 embedding 的分布接近标准各向同性高斯**——从构造上就不可能坍塌。

问题是怎么在高维空间里比较两个分布？直接比是灾难——维度太高，任何分布检验都不靠谱。SIGReg 用了一个数学定理：**Cramér-Wold 定理**——如果一个高维分布在**所有一维投影方向**上都是标准正态，那它就是各向同性高斯。

实际操作：

1. 随机采一堆投影方向 w₁, w₂, ..., wₖ（k 不用很大，几十个就够）
2. 把一个 batch 的 embedding 往每个方向上投影，得到一组一维数值
3. 用 Epps-Pulley 检验（一种正态性检验，跟 K-S 检验是同一类东西，专门测"这组数是不是正态分布"）测每组一维数值是不是标准正态
4. 不像就罚

说人话：**不去量整个高维球，只随机切一刀一刀地量横截面。每一刀切出来都是正态分布，就说明高维整体是高斯的。**

这就是为什么 SIGReg 只有一个超参（投影方向数 k）、线性时间、省显存——它把高维分布匹配问题拆成了 k 个一维检验，每个检验都是 O(batch_size)。大概 50 行代码搞定。

**两个 loss 合在一起：**

```
total_loss = prediction_loss + λ × SIGReg_loss
```

prediction_loss 拉着 encoder 去编码有用信息（能预测下一步），SIGReg_loss 拉着 encoder 把信息散开别堆一坨（分布保持高斯）。两个力对拉，平衡点就是"既有用又散开"的好 embedding。λ 就是那唯一一个超参。

超参从以前的 6 个减到 1 个。那一柜子偏方（stop-gradient、动量网络、学习率调度）全不要了。

**成绩单：** 在 2D/3D 控制任务上（机械臂操作、导航、行走），规划速度比"用大模型做世界模型"快 48 倍。15M 参数、单 GPU、快 48 倍——国内报道里"史上最轻世界模型"的来源就是这个。

**但论文自己承认的局限：**
- 只能短期规划，长了误差累积
- 简单环境效果打折
- 需要明确的动作标签

说人话：在实验室的简单模拟环境里能做短期规划，一旦环境复杂起来或者时间拉长，就不行了。

**停一下,把 LeWorldModel 和 LLM 放一起看。**

你可能已经注意到了:LeWorldModel 的"预测下一帧 embedding"和 LLM 的"预测下一个 token",逻辑结构是一样的——都是"为了预测准,被迫在内部构建对世界的理解"。163 期我们用四组实验证明过,LLM 为了预测下一个 token,内部涌现了棋盘、地图、日历、程序执行器。那 LeWorldModel 为了预测下一帧 embedding,理论上也应该被逼着在内部构建对物理环境的理解。

这个类比成立。但有一个关键区别:**LLM 的 embedding 空间没有被强制约束成特定分布**。严格说 LLM 也有约束——权重衰减（weight decay）相当于给参数加了引力,把权重往原点拉,不让 embedding 空间无限膨胀。但这个引力只管"别飞太远",不管"你得长成什么形状"。星球有引力所以不会散架,但星球上该有山有海有沟壑,引力不拦着。所以 embedding 空间里自发长出了各种非对称、非均匀的结构:Othello 棋盘是 8×8 网格,地图是二维球面投影,真假是一个线性方向。

LeWorldModel 反过来——SIGReg 强制 embedding 分布是各向同性高斯。各向同性的意思是:每个方向上方差一样,没有哪个方向比其他方向"更重要"。

但真实世界的物理结构不是各向同性的。重力只往下拉(一个方向特殊),关节只能在特定范围内弯曲(某些维度受约束),某些状态变量比其他的重要得多(位置比纹理重要)。强迫 embedding 各方向等权,等于不允许模型在 embedding 空间里反映这些不对称性。

打个比方:LLM 的 embedding 空间像一块自由生长的晶体——哪里有结构就往哪里长棱角。LeWorldModel 的 embedding 空间被摁成了一个完美球体——不会坍塌(SIGReg 的目的达到了),但棱角里可能藏着的结构也被磨平了。

我的判断是:**SIGReg 用"保底不塌"换了"可能压制涌现"。** 论文自己也承认:各向同性对最坏情况最优,但特定下游任务可能受益于各向异性。翻译:他知道球体可能不是最好的形状,但它至少不会坍塌。

这就引出了第二篇论文——杨老头儿要用数学证明"高斯约束是对的"。

**第二篇："When Does LeJEPA Learn a World Model?"（2026-05-25，arXiv 2605.26379）**

第一篇做了工程,第二篇补数学。杨老头儿要证明:SIGReg 强制高斯分布不是权宜之计,是理论最优解。

在看定理之前,先搞清楚"各向同性高斯分布"和"平稳加性噪声"到底在说什么现象。

**什么是高斯分布？** 就是正态分布,钟形曲线——大多数值聚在中间,越往两边越少。你测 1000 个人的身高,画出来就是这个形状。大学概率论第一课讲的那个。

**什么是"各向同性"高斯？** 想象在二维平面上撒一把豆子,高斯分布就是豆子聚在中心、四周渐少。"各向同性"是说豆子往上下左右散开的程度一样——画出来是个圆,不是椭圆。推广到高维:embedding 空间里每个方向上的散布程度都一样,没有哪个方向比其他方向"更重要"。是一个完美的高维球。

**什么是"平稳加性噪声"？** 世界的状态从一帧到下一帧怎么变化？"平稳"是说变化的规律不随时间改变——同一个动作在第 1 秒和第 100 秒产生的效果一样。"加性噪声"是说不确定性是"加"上去的,不是"乘"上去的——噪声大小跟当前状态无关。

这两个前提合在一起描述的是一个**非常温顺的世界**:状态散布均匀(高斯)、规则不变(平稳)、噪声独立于状态(加性)。

搞清楚了这些,定理就好读了。论文证明了一个 if-and-only-if（当且仅当）定理:

正方向——如果世界的隐变量是各向同性高斯分布,状态转移是平稳加性噪声,那 JEPA 的 encoder 一定收敛到正交旋转（线性可辨识——意思是 encoder 学到的表示和真实隐变量之间只差一个旋转,信息没丢）。

反方向——在平稳加性噪声这个类别里,高斯是**唯一**能保证线性可辨识的分布。换别的分布,这个保证就没了。

证明方法用了 Hermite 多项式分解，非线性分量对时间相关的贡献严格小于线性。而且论文用 Lean 4 + Mathlib 做了形式化验证——8032 个 build targets，零个 sorry（Lean 4 里 sorry 表示"这步我没证，先跳过"）。

这个形式化验证是真的硬。在 ML 圈用 Lean 4 把定理证到零 sorry 的论文极其罕见。杨老头儿在数学严谨性上补了一课。（顺便一提,8032 个 Lean 4 build targets 的工程量,今天大概率离不开 LLM 辅助。用"统计学鹦鹉"写证明来证明"统计学鹦鹉是死路"——这大概是今年最好的冷笑话。）

**但这个定理的命门在前提：世界必须是高斯且平稳的。**

LeWorldModel 瞄准的是物理世界——机器人眼中的环境。那我们就在物理世界里检查这个前提：

- **碰撞和接触**:杯子从桌上掉下来摔碎,状态转移在那一瞬间是不连续的——不平稳
- **相变**:水结冰、冰化水,状态分布突变——不平稳
- **柔性物体**:布料、绳子的形变,自由度爆炸,状态分布远非高斯
- **多物体交互**:一堆积木倒塌,每块积木的运动互相耦合——组合爆炸,既不高斯也不平稳

LeWorldModel 目前能处理的场景（刚体机械臂、简单导航、单体行走）恰好都是接近高斯且平稳的——关节角度连续、状态转移平滑、噪声是加性的。定理在这些场景里成立,没问题。

但杨老头儿的叙事不是"我做了个机械臂控制器",是"**世界模型**"。既然叫世界模型,就得面对整个物理世界——包括那些碰撞、相变、柔性、多体交互。定理恰恰在这些场景面前,前提不成立。

用一个具体场景感受一下。想象一个机器人从山脚出发,顺着一条坑坑洼洼的山间小路跑到山顶,升起一面旗帜。这个任务够日常了吧?拆开看看定理管得住哪段:

**平路匀速跑——管得住。** 每一步抬腿落脚,关节角度平滑变化,步与步之间规律差不多,帧间视觉差异是渐变的。状态转移接近平稳,噪声接近加性。这段 LeWorldModel 能预测。

接下来全管不住:

**踩进坑里——状态跳变。** 前一帧还在平路上,下一帧踩进坑,机器人姿态突变。而且坑深坑浅不一样,噪声大小取决于"踩到什么地形"——噪声依赖状态,不是加性的。

**急弯视野突变——帧间跳变。** 山路转弯,前一帧看到树,转过弯突然看到悬崖。视觉 embedding 不是渐变,是跳变。

**翻过山脊——视觉分布相变。** 一直在爬坡,视野里全是路面,翻过山脊那一瞬间变成开阔天际线。embedding 的统计分布在这一帧前后完全不同。

**从跑切换到举旗——动作空间变了。** 之前所有动作都是"腿怎么动",突然切换成"手臂举旗"。predictor 一直在学腿部运动和画面的关系,这个动作类型它没见过。

**旗帜在风中飘——柔性物体。** 布料的形变自由度爆炸,状态分布远非高斯。

一个机器人跑上山举旗,这么日常的任务,定理管得住的只有"平路匀速跑"那一小段。路面一颠、弯一拐、顶一翻、手一举、风一吹——全超出前提。15M 参数在模拟器里能做的事,和"理解物理世界"之间的距离,比山路还长。

**第三篇：V-JEPA 2（2025-06）**

搞清楚了 LeWorldModel 的新方法,回头看 V-JEPA 2,正好是一堂"以前怎么干的"历史课。

V-JEPA 2 的大规模视频预训练方法跟 LeWorldModel 完全不同。它不预测"下一帧",而是**遮住视频的一大块,让模型猜被遮住的部分**——跟 BERT 遮词填空是同一个思路,只不过 BERT 遮的是文本里的词,V-JEPA 2 遮的是视频里的时空块。而且遮得很狠——遮掉 90%,只给模型看 10% 的画面,让它把剩下的补全。

关键区别:V-JEPA 2 的**预训练主干本身没有动作输入**。它不知道"是什么动作导致了画面变化",只知道"画面长什么样"。所以纯视频预训练阶段学到的是观察式表征:能看懂视频（"这个人在走路""球飞过去了"），但还不能直接回答"如果我往左推一下,球会往哪飞"。

V-JEPA 2 论文后面确实接了一个动作条件化版本:V-JEPA 2-AC。做法是先用大规模视频训练出视觉表征,再用少量机器人轨迹把它后训练成 latent action-conditioned world model,最后在 Franka 机械臂上做 zero-shot pick-and-place。也就是说,V-JEPA 2 不是"完全不能规划",而是**规划能力不是来自纯视频遮块预训练本身,而是来自后续动作条件化**。

防坍塌用的是什么？就是 LeWorldModel 替掉的那一柜子**古典偏方**:stop-gradient（训练时切断一部分梯度流,不让 encoder 偷懒）+ EMA teacher-student（维护一个"老师网络"慢慢更新,学生网络去追老师的输出）。这些方法能用,但超参多、调参靠经验、理论上说不清为什么管用。

对比一下就清楚了:

**V-JEPA 2（古典）**
- 预测什么:被遮住的视频块（纯视频预训练阶段,补全,不分先后）
- 有没有动作:预训练主干没有;V-JEPA 2-AC 后训练阶段有
- 能做规划吗:纯预训练不能直接规划;V-JEPA 2-AC 可以做机器人规划
- 防坍塌:stop-gradient + EMA（偏方,超参多）
- 规模:1.2B 参数,100 万小时视频

**LeWorldModel（新方法）**
- 预测什么:下一帧 embedding（有时间方向,有因果）
- 有没有动作:有
- 能做规划吗:能（在 embedding 空间搜索最优动作序列）
- 防坍塌:SIGReg（50 行代码,1 个超参）
- 规模:15M 参数,单 GPU

从 V-JEPA 2 到 LeWorldModel 的跳跃不是"第一次能规划"这么简单。V-JEPA 2 已经通过 V-JEPA 2-AC 展示了机器人规划,但它是"先大规模视频预训练,再接动作条件化后训练"。LeWorldModel 往前推进的是另一件事:**从像素端到端学习 action-conditioned latent dynamics**,同时把防坍塌方法从一柜子偏方换成了一个有数学保证的正则化。

但这篇严格说是 Meta 时期的成果,2025 年 6 月发的,杨老头儿 11 月才离职。国内宣传经常把它跟 AMI Labs 捆在一起吹,不太厚道。

一条线理下来:2022 JEPA 架构提出 → 2023 I-JEPA → 2024 V-JEPA → 2025 V-JEPA 2 + LeJEPA → 2026 LeWorldModel + 高斯定理。四年迭代,方向没变,核心突破是两个:动作条件化从后接模块逐步走向端到端 latent dynamics,防坍塌从经验偏方走向 SIGReg 的数学保证。应用场景还停在实验室。

━━━━━━━━━━━━━━━━━━━━

◆ 10 亿美元花在哪了

━━━━━━━━━━━━━━━━━━━━

AMI Labs 截至 2026 年 5 月，没有产品。CEO Alex LeBrun 说"大概还要一年"。

做个对比。李飞飞的 World Labs 也融了 10 亿美元（AMD、NVIDIA、Autodesk 投的），人家好歹出了个产品——Marble，文字或照片生成 3D 场景的工具，20-95 美元/月订阅制。本质上是 3D 场景生成工具，跟"理解物理世界"没多大关系，但至少有东西能用。不过说句公道话——3D 场景生成这事儿，Unity 和 Unreal Engine 做了二十年，加上 NVIDIA 的 Neural Graphics，游戏公司那套比 Marble 成熟得多、可控得多。10 亿美元做了个 AI 版关卡编辑器，这个赛道的天花板不高。

两家合计 20 亿美元，截至目前的成果 = 一个 3D 场景生成器 + 一堆论文。

当然，基础研究不能用短期产出衡量。杨老头儿是图灵奖得主，他有资格做长线研究。但他的叙事不是"我要慢慢做研究"，他的叙事是"LLM 是死路"——这是一个关于整个行业方向的判断。既然你做了这么大的判断，那我们就看你拿出来的证据够不够硬。

15M 参数在简单模拟环境里做短期规划，加一个"如果世界是高斯的我就对"的定理——这个证据撑不起"LLM 是死路"的结论。

━━━━━━━━━━━━━━━━━━━━

◆ "LLM 只是统计学鹦鹉"站得住吗

━━━━━━━━━━━━━━━━━━━━

杨老头儿反复说的核心论点是：LLM 不理解世界，只是在做 token 共现统计，本质上是"统计学鹦鹉"。

这个论点我们在 163 期（ https://mp.weixin.qq.com/s/1e3nOvjvc8YlMtXGZrDPqw ）已经用四组实验证据彻底拆过了，这里不重写，直接给结论：

**四组证据，全部指向同一个结论——LLM 内部有结构化的世界表征，不是 token 共现统计。**

- Othello-GPT：只看棋谱的 LLM，内部自发涌现了完整的 8x8 棋盘状态表征，探针准确率 98.3%，7 个模型独立复现。
- 地图与日历：Llama-2 内部涌现了跟真实世界地图和时间轴一致的空间/时间坐标系。
- 程序执行追踪：代码 LLM 内部涌现了对变量值的实时追踪——它在脑子里跑程序。
- 真假命题线性可分：LLM 内部存在"真值方向"，真命题和假命题的向量线性可分。

这些不是"统计相关"。Nanda 2023 年的工作表明，用向量加减在 Othello-GPT 内部改写棋盘状态，模型的后续行为跟着变了。这叫因果干预——鹦鹉的脑子里改不出棋盘。

**可解释性研究已经给出强证据：LLM 不是简单的 token 共现统计，它在内部构建了结构化世界表征。**

杨老头儿选择不看这些。

━━━━━━━━━━━━━━━━━━━━

◆ 那他为什么坚持"LLM 是死路"

━━━━━━━━━━━━━━━━━━━━

屁股决定脑袋。

杨老头儿从 Meta 离职的原因，业内基本公开了：Llama 4 反响不佳后，Meta 重组 AI 线，28 岁的 Alexandr Wang 进入新 superintelligence 团队领导层。LeCun 后来公开说，新团队短期押注语言模型的方向和他的兴趣不一致，也批评 Wang 经验不足。走人不是单一原因，是组织路线和研究路线一起断了。

走人之后要融资。融资需要叙事。什么叙事能融 10 亿？"我跟大家做一样的事但我做得更好"融不了 10 亿。"大家全走错了路，只有我知道正确方向"——这才值 10 亿。

10.3 亿美元的种子轮绑定了"LLM 是死路"这个叙事。他现在回头看可解释性的证据，就是自己打自己的脸。融资逻辑不允许他承认 LLM 内部有世界模型。

这不是阴谋论，这是激励结构。学术界叫 confirmation bias（确认偏差），投资圈叫 narrative lock-in（叙事锁定）。10 亿美元下去了，叙事改不了了。

━━━━━━━━━━━━━━━━━━━━

◆ 公允地说，杨老头儿对在哪

━━━━━━━━━━━━━━━━━━━━

拿成果说话。杨老头儿有几个点确实说对了：

**第一，LLM 缺具身感。** 这点他说对了。LLM 从文本学世界，没有身体经验的锚点。它知道"巴黎是法国首都"，但不知道"羽毛从高处落下不会碎"——因为后者需要身体经验来校准。我们在 163 期也承认了这个盲区。世界模型路线试图从像素和传感器数据补上这个缺口，方向本身是对的。

**第二，SIGReg 是真贡献。** JEPA 一直有表征坍塌的问题——encoder 学着学着把所有输入都映射到同一个点。以前大家用各种偏方（VICReg、Barlow Twins、BYOL 的动量网络等），超参一堆，调参靠命。SIGReg 高斯正则化用大概 50 行代码把这个问题基本解决了，超参从 6 个减到 1 个。这是实打实的工程贡献。

**第三，高斯定理的 Lean 4 形式化验证。** 在 ML 圈，绝大多数理论论文的证明都是纸面的——人写人查，错了也不知道。用 Lean 4 做形式化验证，8032 个 build targets 零 sorry，这个数学纪律在 ML 圈极其罕见。杨老头儿在数学严谨性上给同行做了个示范。

但——

**"LLM 有缺陷"不等于"LLM 要扔掉"。**

LLM 缺具身感，可以补（多模态、具身智能）。LLM 短期规划弱，可以加外部规划器。这些是"增强"的理由，不是"替代"的理由。

你说手机拍照比不上单反，所以手机相机要扔掉？不是。你给手机加更好的镜头和算法。

杨老头儿的错误不在看到了缺陷，在于把缺陷等同于了死刑。

━━━━━━━━━━━━━━━━━━━━

◆ 收口

━━━━━━━━━━━━━━━━━━━━

杨老头儿用 Lean 4 证了一个漂亮的定理。定理说：如果世界是高斯的，他的 JEPA 就能学到正确的世界模型。

问题是，物理世界大面积不是高斯且平稳的。碰撞不连续、相变突变、柔性物体自由度爆炸、多体交互组合爆炸——定理在这些场景面前，前提不成立，结论不适用。

**他证的不是"世界模型成立"。他证的是"在一个比真实世界简单得多的假设下，我这套能成立"。**

与此同时，那些被他判了死刑的"统计学鹦鹉"，脑子里涌现了棋盘、地图、日历和程序执行器。

谁更像是理解了世界？

━━━━━━━━━━━━━━━━━━━━

「他证的不是"世界是高斯的"。他证的是"假如世界是高斯的，我这套就对"——然后真实世界大面积不满足这个假如。」

「15M 参数在实验室里做短期规划，叫"世界模型"。12288 维空间里涌现了棋盘、地图、日历和程序执行器，叫"统计学鹦鹉"。名字是人起的，能力是实验测的。」

「10 亿美元赌一条路线。628 美元/月已经在出活了。」

━━━━━━━━━━━━━━━━━━━━
