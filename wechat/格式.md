微信公众号文章格式规范（Markdown 版）

━━━━━━━━━━━━━━━━━━━━

◆ 文件格式

- 文章用 .md（Markdown），支持加粗、表格等语法
- 封面图用 .svg，转 .png 发布
- 命名：93.md / cover93.svg / cover93.png（数字递增）

━━━━━━━━━━━━━━━━━━━━

◆ 排版符号

- **加粗** 用于强调重点句子和金句
- 【小节标题】用中文方括号（◆ 下面的子内容）
- 💡 名词解释/人话翻译（给小白看的解释用这个标志）
- ━━━ 粗分隔线（章节之间）
- ──── 细分隔线（小节之间）

━━━━━━━━━━━━━━━━━━━━

◆ 标题层级

【关键词】文章主标题（文章开头第一行）
◆ 一级标题（大节）
【二级标题】（小节，在 ◆ 下面的子内容）
- 列表项（用 Markdown 列表 `-`）

注意：▸ 不再使用，统一用 ◆ 作为大节标题

━━━━━━━━━━━━━━━━━━━━

◆ 表格格式

用 Markdown 表格：

| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 内容 | 内容 | 内容 |

表格中的角色名统一用 **中文（eng）** 格式，例如：专家（expert）、苏格拉底（socratic）

◆ Markdown 注意事项

- 段落之间必须空一行，否则不换行
- 金句独立成段，每句之间空一行
- **加粗 `**` 内侧紧贴引号/间隔号会解析失败**（实测 commonmark 库验证，VSCode 预览同款规则）：
  - ❌ 失败：`**"引号加粗"**`（`**` 贴着引号 " "）、`**·吧**`（贴间隔号 ·）、`"动土"**——`（`**` 左贴引号右贴破折号）
  - ✅ 正常：`汉字**加粗**汉字`（紧贴汉字没问题）、`**加粗**：`、`，**加粗**，`（紧贴普通标点逗号冒号没问题）
  - 原因：CommonMark 的 flanking 规则把引号/间隔号算作标点，导致 `**` 定界符判定失效。**和"贴不贴汉字"无关，关键看 `**` 内侧紧贴的是不是引号/间隔号这类特殊标点。**
  - 解法：把引号挪到加粗外面，或加粗内容不含引号，或 `**` 两侧补空格。
- 公式/代码用 backtick 包裹：`E = mc²`
- 数学公式中的下标用 Unicode 下标字符，不用 `_t` 写法：
  - 时间步：sₜ、xₜ、sₜ₋₁、xₜ₋₁
  - 编号：B₁、B₂、x₁、x₂
  - 转置：kᵀ、vᵀ（用 Unicode 上标 ᵀ）
  - 希腊字母直接用 Unicode：α、β、θ、Δ
  - 常用 Unicode 数学符号：· (乘) ² (平方) ½ (二分之一) → ≈ ≠ ≤ ≥
- 代码块中的伪代码（Python 风格变量名如 state、new_token）不用 Unicode 下标，保持程序员可读性
- 微信公众号编辑器对 Markdown 支持有限，发布前用 Markdown 转微信工具预览

◆ 风格要点

- 语言接地气，少装逼
- 技术名词要翻译成人话（用 💡 标记）
- **正文不要黄脸/手势表情 emoji（😏😂🔧 之类）**——公众号正文保持稳重。唯一例外：💡 灯泡是 hint 块（"打个比方"/名词解释）的固定格式标记，保留
- 段落短一点，别堆大段
- 重点用 **加粗** 强调
- 数据和对比用 Markdown 表格呈现
- 第一次出现的专业术语必须解释，别假设读者看过之前的文章

【避免的口头禅/用词】
- **"拆 / 拆开 / 拆过 / 拆解"** —— 作者烦这个词，别用。改用"分析 / 讲 / 聊 / 剖 / 说清楚"等替代（如"我们专门分析过""改这个流程"）

【偏好用词】
- 💡 类比块的标题，用 **"打个比方"**，不要"翻译成人话"

【参考资料区规则】
- 文末"参考资料"只放**外部论文/资料**（带英文标题 + arXiv 号，读者好搜）
- **自家公众号往期链接不进参考资料附录**——在正文里引到的地方直接带链接即可
- 链接前后留空格、带 https:// 前缀，避免和中文括号粘连导致微信识别不出

━━━━━━━━━━━━━━━━━━━━

◆ 封面图要点

- 直白，一眼能看懂
- 别搞太抽象的
- 主标题要大、醒目
- 配色跟主题相关
- 别画复杂 logo，画不好就用简化图形或文字代替
- 动画效果微信不支持，做静态版本

【SVG 中文字体配置】

cairosvg 转 PNG 时需要指定本地中文字体，否则中文显示为方块。

在 SVG 开头加：

```xml
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 800 600">
  <style>
    text { font-family: 'Noto Sans CJK SC', sans-serif; }
  </style>
  ...
</svg>
```

系统可用的中文字体（fc-list :lang=zh family）：
- Noto Sans CJK SC（简体中文，推荐）
- Noto Serif CJK SC（简体中文，衬线）
- Noto Sans CJK TC（繁体中文）
- AR PL UKai TW MBE（台湾楷书）

转换命令：`cairosvg cover77.svg -o cover77.png -d 150`
