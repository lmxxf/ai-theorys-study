【AI可解释性】视觉 token 里到底装了什么？用 SAE 拆开多模态模型的黑箱

━━━━━━━━━━━━━━━━━━━━

◆ 承接上篇

━━━━━━━━━━━━━━━━━━━━

上篇说了多模态大模型其实是"拼出来的"——视觉编码器和语言模型各训各的，中间一个投影层搭桥。一张 448x448 的图片，经过视觉编码器，变成了 196 个 2048 维的向量。LLM 把这些向量当成 196 个"token"来处理。

那问题来了——**这 196 个视觉 token 里面到底装了什么？人类能看懂吗？**

这不是闲聊式的好奇。如果你不知道视觉 token 编码了什么，你就不知道模型"看到了什么"。它说"图里有一只猫"——是因为真的"看到"了猫，还是因为某个统计噪声碰巧触发了"猫"这个词？

这篇就来拆这个黑箱。

━━━━━━━━━━━━━━━━━━━━

◆ 视觉 token 不是像素，也不是文字

━━━━━━━━━━━━━━━━━━━━

先搞清楚视觉 token 到底是什么"物种"。

一张 448x448 的图片经过 ViT（patch size = 32），被切成 14x14 = 196 个 patch。每个 patch 过视觉编码器后变成一个 2048 维的向量。这 196 个向量就是视觉 token。

**它们保留了 14x14 的空间布局。** 左上角的 token 对应图片左上角那一块区域，右下角的 token 对应右下角。不是打乱的，是有空间对应关系的。

────────────────────

那每个向量里装的是什么？

**不是像素值。** 原始图片一个 patch 是 32x32x3 = 3072 个像素值（RGB 三通道）。经过编码器后变成 2048 维——维度还少了。信息被压缩了。

**也不是文字描述。** 不是说第 42 号 token 里存着"一只橘猫的左耳"这样的文字。它是一个 2048 维的浮点数向量，人类直接看就是一堆数字。

**它是语义特征向量。**

💡 翻译成人话：每个视觉 token 是对应图片区域的"语义压缩包"。比像素信息量小（丢掉了精确的颜色值、纹理细节），但比文字描述信息量大（保留了"这块区域是什么"、"和旁边有什么关系"等语义信息，而且是连续的、可微分的，不是离散的文字）。

打个比方：**像素是高清照片，文字描述是一句话摘要，视觉 token 是介于两者之间的"语义素描"——没有照片那么细，但比一句话丰富得多。**

────────────────────

更准确地说，这 196 个向量构成了一张"压缩过的语义地图"：

- **空间上**：14x14 的网格，每个格子对应原图的一块区域
- **语义上**：每个格子的 2048 维向量编码了该区域的语义——物体类别、纹理模式、空间关系等
- **格式上**：经过投影层后，这些向量被映射到语言模型的 embedding 空间，LLM 把它们当"文字"来读

**问题是：这 2048 维向量里的每一维代表什么？没人知道。** 这就是黑箱。

━━━━━━━━━━━━━━━━━━━━

◆ SAE：拆解纠缠特征的字典学习工具

━━━━━━━━━━━━━━━━━━━━

要理解视觉 token 里藏了什么，需要一个工具来"翻译"这些高维向量。

直觉上最简单的办法是看每一维——第 1 维大的时候图片里有猫，第 2 维大的时候图片里有车。但现实不是这样。

**神经网络的特征是"纠缠"的。** 一个维度可能同时编码了"猫"、"毛茸茸"、"暖色调"三个概念，具体表达哪个取决于其他维度的值。单看一维毫无意义。这叫 **polysemantic（多义性）**。

💡 翻译成人话：想象一个调音台，每个推子（维度）不是控制一种乐器的音量，而是同时影响鼓声、吉他和人声。你推第 3 号推子，三种声音同时变化。你没法通过看一个推子的位置来判断"现在吉他声有多大"。

要把纠缠的信号拆开，需要 **SAE（Sparse Autoencoder，稀疏自编码器）**。

────────────────────

【SAE 是什么】

SAE 的核心思路：**把一个低维的纠缠向量，映射到一个高维的稀疏空间，使得高维空间里每一维只对应一个概念。**

💡 翻译成人话：原来 2048 维里每一维都是多义的。SAE 把它"展开"成比如 65536 维，但同一时刻只有几十个维度是非零的（稀疏）。每个非零维度对应一个清晰的概念——"狗"、"红色"、"左上角有文字"。

结构很简单：

```
输入 x (2048 维) → 编码器 W_enc (2048 → 65536) + ReLU → 稀疏特征 z (65536 维，大部分为 0)
稀疏特征 z → 解码器 W_dec (65536 → 2048) → 重建 x̂ (2048 维)
```

训练目标：**让 x̂ 尽量接近 x，同时让 z 尽量稀疏（非零元素尽量少）。**

两个约束缺一不可：
- 只要求重建好 → z 不稀疏，每维还是多义的，等于没拆
- 只要求稀疏 → z 全是零，重建一塌糊涂，等于没学

**稀疏性迫使模型"用最少的特征来解释输入"——每个激活的特征必须承担独立的语义责任，不能偷懒共享。**

这就像逼一个人用最少的词来描述一张图片——你不会说"那个东西有点像猫又有点像狗"，你会说"猫"。少词约束迫使每个词精确。

────────────────────

SAE 在 NLP 领域已经被 Anthropic（对，就是做 Claude 的那家）广泛用于解释语言模型的内部表示。2024 年 Anthropic 在 Claude 3 Sonnet 上训了 SAE，发现了对应"金门大桥"、"代码错误"、"谄媚式赞美"等概念的单义特征——金门大桥特征放大后，模型甚至自称是金门大桥。

**那在视觉-语言模型上呢？视觉编码器的特征也能被 SAE 拆成单义的吗？**

━━━━━━━━━━━━━━━━━━━━

◆ 在 CLIP 上训 SAE：视觉特征确实是单义的

━━━━━━━━━━━━━━━━━━━━

2025 年 NeurIPS 论文 **"Sparse Autoencoders Learn Monosemantic Features in Vision-Language Models"**（arXiv: 2504.02821）回答了这个问题。

作者在 **CLIP 视觉编码器**（ViT-L/14）的多个层上训了 SAE（还顺带覆盖了 SigLIP、AIMv2 等编码器），然后系统性地验证：SAE 学出来的特征是不是单义的？人类能不能理解？

────────────────────

【发现一：视觉 SAE 特征确实是单义的】

训完 SAE 后，作者检查了大量特征的激活模式。论文里给出的真实例子：

- CLIP 第 22 层 SAE 的 **39 号神经元**：只在"铅笔"出现时激活
- 另一些高分特征分别只认**鹦鹉**、**考拉**、**冰球**——不管角度、姿势、背景怎么变

**一个 SAE 特征对应一个语义概念。** 这就是 **monosemantic（单义性）**——和原始网络里那种纠缠的 polysemantic 特征正好相反。

────────────────────

【发现二：量化评估——MonoSemanticity Score】

光看几个案例不够，需要系统化的量化指标。

作者提出了 **MS（MonoSemanticity Score）**：对每个特征，把激活它的图片全部收集起来，用另一个独立的视觉模型（DINOv2）给每张图算一个嵌入向量，然后计算这些图片**两两之间的余弦相似度**，按激活强度加权平均。

💡 翻译成人话：如果一个特征只被"长得像同一类东西"的图片激活——所有强激活图片互相都很相似——MS 分数就高；如果激活它的图片五花八门、互相不像，MS 分数就低。妙处在于全程不需要人工语义标签，用第三方模型的"眼睛"来当裁判。

实验结果：**SAE 特征的 MS 约 0.82~0.90，而原始 CLIP 神经元只有约 0.01。** 不是高一点，是两个数量级的差距——SAE 确实把纠缠的特征拆成了单义的。

────────────────────

【发现三：人类实验验证】

量化指标可能有偏差——万一 DINOv2 这个"裁判"自己有怪癖呢？所以作者还在 Amazon Mechanical Turk 上做了**人类实验**：71 名标注者、1000 次成对比较。每次给人看两组 16 张图的网格（分别对应两个不同 MS 分数的特征），问"哪一组更相似、更聚焦于同一个东西？"

结果：**人类的选择和 MS 指标的预测一致率 82.8%；当两个特征的 MS 差距很大时（≥0.8），一致率是 100%。**

**这意味着 SAE 学到的视觉特征不是统计幻觉——MS 说单义的特征，人类看了也觉得单义。**

━━━━━━━━━━━━━━━━━━━━

◆ 因果干预：改视觉 token 就能改 LLM 输出

━━━━━━━━━━━━━━━━━━━━

到这里你可能会说：好，SAE 特征是单义的，人类能看懂。但这只是"观察"——你只是看到某个特征和某个概念相关。**问题是：这个特征真的在因果上决定了 LLM 的输出吗？**

也许"狗"特征激活了，LLM 也提到了狗，但这可能只是相关性——也许 LLM 是通过别的路径"知道"图片里有狗的。

要回答因果问题，需要做**因果干预实验**——直接修改视觉特征，看 LLM 的输出会不会跟着变。

────────────────────

【实验设计】

作者用了 **LLaVA**（一个典型的拼接式多模态模型：CLIP 视觉编码器 + 投影层 + Vicuna-7B 语言模型）。

实验步骤：

1. 在 CLIP 视觉编码器的 SAE 里锁定一个单义特征（比如"铅笔"神经元）
2. 推理时**把这个神经元的激活值强制改写成指定的强度 α**——调高或压低
3. 改写后的视觉特征照常过投影层喂给 LLM（LLM 一个参数都不动）
4. 看输出怎么变

────────────────────

【结果】

**压低方向：** 给一张有小刀（或笔记本电脑）的图片，把对应神经元的激活逐步压低——LLM 的描述里，小刀逐渐消失了，只剩下草莓、木板这些其他物体。物体从模型的"眼睛"里被擦掉了。

**调高方向更好玩：** 把"铅笔"神经元放大，然后让模型**写一首情诗**——输出满篇都是铅笔：石墨的心、削尖的思念。模型像中了蛊，看什么都是铅笔。

LLM 全程没被改过一个参数——改的只是视觉编码器里一个 SAE 神经元的激活值。

────────────────────

**这证明了三件事：**

**1. 因果性成立。** 视觉 SAE 特征不只是和输出相关——它在因果上决定了 LLM 的输出。关掉它，输出就变。

**2. 投影层的对齐是有效的。** 视觉编码器里编码的语义，确实通过投影层传到了 LLM。不是"凑巧对上的"，是真的在传信息。

**3. 可以在视觉端控制语言输出。** 你不需要改 LLM 的任何参数——只需要在 CLIP 的视觉特征层面做干预，就能控制最终的文字输出。这为多模态模型的可控生成开了一扇门。

◆ 其他相关工作

━━━━━━━━━━━━━━━━━━━━

上面那篇 NeurIPS 论文不是孤例。2025 年前后有一批工作在探索视觉模型的可解释性，思路各有侧重。

────────────────────

【PatchSAE（ICLR 2025）/ saev】

PatchSAE（arXiv:2412.05276，ICLR 2025）在 CLIP 的 patch 级表征上训 SAE，提取出的概念跨越多个粒度——从形状、颜色这类低层视觉属性，到物体、场景这类高层语义都有，而且能定位到具体 patch。saev（arXiv:2502.06755）则是把 SAE 做成了分析视觉模型的开源工具库。

这和我们在上篇讲 DeepStack 时的说法呼应：ViT 的特征里既有纹理级信息也有语义级信息，都值得往语言模型里送——SAE 把"这些粒度确实同时存在"变成了可观察的实验事实。

────────────────────

【视觉 token 进入 LLM 后发生了什么】

另一个方向的研究关注的不是"视觉 token 编码了什么"，而是"视觉 token 进入 LLM 后经历了什么"。

Neo et al.（arXiv:2410.07149，ICLR 2025）在 LLaVA 上逐层追踪视觉 token 的表示，发现：**随着层数加深，视觉 token 在 LLM 的词表空间里越来越可解读，逐渐和对应概念的文字 token 对齐**——到后期，模型从视觉 token 里提取信息的方式，已经和从纯文本里回忆事实的机制很相似了。

💡 翻译成人话：视觉 token 刚进 LLM 时还是"外语"，每过一层就被消化一点，越往深走越像 LLM 自己的"母语"。注意这个翻译是**逐层渐进**的，不是前几层一次性完成的。

这也给投影层的定位补了一笔——**投影层做的是第一步粗对齐，进了 LLM 之后还有一个逐层消化的过程。** 如果投影层对齐得不好，这个消化过程就要占用更多本该用于理解的容量。

────────────────────

【CaFE：因果特征解释】

前面说的 SAE 分析有一个隐含假设：每个视觉 token 只编码它对应 patch 的语义。但 ViT 有 self-attention——经过多层注意力后，一个 token 的信息已经和其他 token 混合了。第 42 号 token 可能不只包含第 42 块 patch 的信息，还混入了周围 patch 甚至全图的信息。

**CaFE（Causal Feature Explanation，arXiv:2509.00749，首尔大学）** 用 **ERF（Effective Receptive Field，有效感受野）** 来解决这个归因问题——不只看一个 SAE 特征"在哪个 token 上激活了"，而是追溯"这个 token 的信息实际上来自原图的哪些区域"。

💡 翻译成人话：你检测到第 42 号 token 上的"狗"特征激活了。但第 42 号 patch 可能只是草地。实际上"狗"的信息是从邻近的 patch 通过 self-attention 流过来的。ERF 帮你追溯到真正的信息来源——第 35、36、43 号 patch 上的狗的身体部位。

◆ 还有什么不能解释

━━━━━━━━━━━━━━━━━━━━

说了这么多"能解释的"，也得说说"不能解释的"。

────────────────────

【2048 维里大部分维度是纠缠的】

SAE 能拆出单义特征，但 SAE 的重建不是无损的。一个 2048 维向量用 SAE 编码再解码回来，必然有信息损失。**SAE 捕捉的是最显著的、可以被稀疏表示的语义特征——剩下的部分，我们仍然不知道它编码了什么。**

这个"剩下的部分"在领域里有个正式名字：**SAE 暗物质**（Engels et al., arXiv:2410.14670《Decomposing the Dark Matter of Sparse Autoencoders》）。研究发现 SAE 的重建残差里有相当一部分是有结构的——不是噪声，是 SAE 还没学会表示的东西。黑箱拆开了一角，但没有拆完。

────────────────────

【投影层对齐的是 LLM 的语义，不是人类的语义】

投影层的训练目标是"让 LLM 能正确预测下一个 token"。所以视觉 token 被对齐到的是 **LLM 的内部语义空间**，不是人类的概念空间。

LLM 的内部语义和人类语义高度相关（因为 LLM 是用人类语言训练的），但不完全一样。可能存在 LLM "理解"但人类无法直接对应到词汇的概念——这些概念在视觉 token 里可能有编码，但 SAE 拆不出来，因为没有人类标签可以去验证。

────────────────────

【可解释性是 AI 的开放难题】

目前的技术能做到的是：**观察到"这个特征大概在表达什么"**——通过最大激活图片、因果干预、人类实验等方法。

做不到的是：**精确解读每一个维度的含义**，就像读一本书一样一字一字地理解。

这不是视觉 token 的特殊问题——语言模型的 token embedding 同样无法精确解读。**可解释性研究的目标不是"读懂每一个数字"，而是"理解到足以预测、控制、调试模型行为"。** SAE 是目前最接近这个目标的工具之一。

━━━━━━━━━━━━━━━━━━━━

◆ 总结：拼接的好处和代价

━━━━━━━━━━━━━━━━━━━━

回到架构层面。

**拼接架构让可解释性分析变得可能。** 因为视觉编码器是独立的模块，你可以单独拿出来分析——在 CLIP 上训 SAE、做因果干预、追溯信息流——不需要碰语言模型。这是拼接架构的一个被低估的好处。

**如果是原生多模态模型（比如 Gemini），情况就不同了。** Gemini 从一开始就把视觉和语言信号混合在同一个模型里联合训练。没有独立的"视觉编码器"可以单独分析——视觉信息和语言信息从第一层就纠缠在一起。你没法在不动语言模型的前提下单独分析视觉特征。

**拼接的代价是什么？** 上面说了，LLM 的浅层需要花几层来"翻译"视觉 token——把它们从视觉语义转换为语言语义。这几层的计算力被用于翻译而不是理解。原生多模态模型理论上不需要这个翻译步骤——视觉和语言从一开始就在同一个语义空间里。

| 维度 | 拼接式（LLaVA、Qwen3-VL） | 原生式（Gemini） |
|------|---------------------------|-----------------|
| 可解释性 | 可以单独分析视觉编码器 | 视觉-语言纠缠，难以单独分析 |
| 升级成本 | 换零件 + 重训投影层 | 基本从头训 |
| 浅层开销 | 需要"翻译层" | 理论上不需要 |
| 训练成本 | 低（复用现有模型） | 高（从头联合训练） |
| 深层融合度 | 有上限（取决于投影层质量） | 理论上更深 |

**目前开源多模态模型几乎全是拼接式，原生多模态是闭源模型的路线。** 不是因为原生方案不好——而是原生方案需要的训练资源，只有 Google、OpenAI 这个量级的公司才玩得起。

━━━━━━━━━━━━━━━━━━━━

**「196 个视觉 token，就是一张图片的语义骨架——比照片粗糙，比文字丰富，人类看不懂，但 LLM 读得懂。」**

**「SAE 不是万能翻译器，但它第一次让我们能指着一个神经元说：这个，代表狗。」**

**「拼接架构最大的隐藏收益不是便宜，是可拆——能拆开看的黑箱，就不算完全的黑箱。」**

技术名词速查：

- **SAE**（Sparse Autoencoder，稀疏自编码器）：一种字典学习工具，把高维纠缠的特征拆成高维稀疏的单义特征。每个激活的维度对应一个清晰的概念。
- **Monosemantic（单义性）**：一个特征只编码一个概念。与之相对的是 polysemantic（多义性）——一个特征同时编码多个概念。
- **MonoSemanticity Score（MS）**：量化评估一个特征是否单义的指标。用第三方视觉模型（DINOv2）给激活图片算嵌入，看它们两两相似度的激活加权平均——激活某特征的图片彼此越像，该特征越单义。
- **因果干预（Causal Intervention）**：直接修改模型内部的特征值，观察输出的变化。用来区分"相关性"和"因果性"。
- **ERF**（Effective Receptive Field，有效感受野）：一个 token 实际上能"看到"原图哪些区域的范围。经过 self-attention 后，感受野比原始 patch 大得多。
- **投影层**（Projector / Merger）：连接视觉编码器和语言模型的小型网络，做维度转换和特征对齐（上篇详细讲过）。
- **CLIP**（Contrastive Language-Image Pretraining）：OpenAI 的视觉-语言对比学习模型，SAE 可解释性研究的主要实验对象。
