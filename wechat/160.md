【AI的“海马体”战争】除了ZEP，你的Agent还能把记忆存哪？

━━━━━━━━━━━━━━━━━━━━

◆ 当上下文窗口不再是灵丹妙药

━━━━━━━━━━━━━━━━━━━━

我们在第 156 期测了那个火爆 GitHub 的多智能体推演项目 **MiroFish**。跑完之后，后台被问得最多的问题是：

**“那个必须填的 ZEP_API_KEY 到底是个啥？”**
**“如果我不想把数据传给 ZEP，能换成本地的平替吗？”**

这个问题非常有意思。它不只是关于 MiroFish，它涉及到了目前 AI Agent 领域最核心的基础设施之争：**AI 的“海马体”该长什么样？**

很多人的直觉是：随着大模型上下文窗口越来越长（比如 Gemini 的 200 万 token），我们不需要外挂记忆了，把历史记录全塞进 prompt 里不就行了？

答案是：**不行。**

这不只是成本问题。如果你把一个人从出生到现在的每一秒经历都塞进他的脑子里，他会疯掉的；大模型也一样，这叫“大海捞针”和“注意力稀释”。

Agent 真正需要的，是像人一样，能够**记住重点、遗忘细节、并且知道张三和李四到底有什么恩怨**。

这就催生了一门专门的生意：**记忆图谱服务**。ZEP 只是其中之一。

今天，我们来盘一盘 2026 年，如果你要给你的 Agent 外挂一个脑子，市面上都有哪些选择，它们之间的鄙视链又是怎样的。

━━━━━━━━━━━━━━━━━━━━

◆ 门派一：拎包入住的“傻瓜外挂流”

━━━━━━━━━━━━━━━━━━━━

代表选手：**Mem0**（前身是 EmbedChain）

这是目前生态最好、最火的 ZEP 替代品，没有之一。主打一个“一行代码接入”。

────────────────────

【核心机制：带冲突解决的更新】

如果你对传统的大模型说：“我喜欢吃苹果。”它记住了。
过了一个月你对它说：“我查出对苹果过敏了。”

如果你用的是传统的 RAG（比如用 Chroma 或者 Pinecone 存文档），它会把这两条记录都存进向量库。下次检索时，它可能把两条都捞出来，然后大模型就精分了。

**Mem0 解决的就是这个问题。** 它不仅是存文本，它会在存入的瞬间，调用底层大模型进行比对。发现冲突了？它会自动把旧记忆覆盖或打上废弃标签。

────────────────────

💡 人话翻译：

> 传统 RAG 是一个**不长脑子的记事本**，你写啥它留啥。
> Mem0 是一个**带自纠错功能的贴身秘书**，会帮你把前后矛盾的信息理顺。

**适合谁：**
做轻量级 Agent、个人助理，或者你的产品想用最快速度上线。它和 LangChain、CrewAI 绑得很深，闭着眼睛就能接入。

━━━━━━━━━━━━━━━━━━━━

◆ 门派二：自己管自己的“操作系统流”

━━━━━━━━━━━━━━━━━━━━

代表选手：**Letta**（前身是著名的 MemGPT）

如果说 Mem0 是外包秘书，那 Letta 就是给 Agent 动了个开颅手术，给它装了一套操作系统。

────────────────────

【核心机制：内存与硬盘】

Letta 的思路非常硬核。它把大模型的上下文窗口看作电脑的**内存（RAM）**，把外部数据库看作**硬盘（Disk）**。

最绝的是，它**把记忆的管理权交给了 Agent 自己**。

普通 Agent 是被动接受你塞进来的 prompt；而 Letta 驱动的 Agent，能在多轮对话中，自己决定：“这句话很重要，我要把它从上下文（内存）写入长期记忆（硬盘）”，或者“这个问题我答不上来，我得去搜索一下我的长期记忆”。

────────────────────

💡 人话翻译：

> ZEP/Mem0 是**你（开发者）**在帮大模型管记忆。
> Letta 是你赋予了大模型一套工具，**让它自己管自己的记忆**。

**适合谁：**
需要真正意义上“无尽上下文”，且要求 Agent 具备高度自主性的复杂系统。它非常适合用来做能陪你聊一辈子的赛博伴侣。

━━━━━━━━━━━━━━━━━━━━

◆ 门派三：防泄密的“本地平替流”

━━━━━━━━━━━━━━━━━━━━

代表选手：**Evermind.ai (EverOS)**

回到一开始那个问题：如果你在保密单位干活，打死也不能把数据传给 ZEP 或者其他 SaaS 服务，咋办？

这个时候你就需要 Evermind。

────────────────────

【核心机制：自组织的本地图谱】

ZEP 和 MiroFish 强绑定的原因，在于 ZEP 提供了带有时间戳的知识图谱功能（谁在什么时候说了什么，权重怎么算）。

Evermind 就是冲着这个痛点来的。它提供了一套开源的、高度针对本地部署优化过的记忆框架。它能在保持时间线一致性的前提下，让你把所有的记忆数据都存在本地的 SQLite 或者本地跑的图数据库里。

────────────────────

💡 人话翻译：

> 一套不联网也能跑出 ZEP 效果的记忆底座。

**适合谁：**
想在生产环境跑，但数据绝对不能出境的场景。

━━━━━━━━━━━━━━━━━━━━

◆ 门派四：学院派的“认知推演流”

━━━━━━━━━━━━━━━━━━━━

代表选手：**Cognee**

普通人的记忆是：“A 投资了 B”。
柯南的记忆是：“A 投资了 B，B 和 C 有仇，所以 A 遇到 C 时可能会警惕”。

────────────────────

【核心机制：多跳图谱检索】

Cognee 比其他家更学术，它不满足于仅仅把过去说的话原封不动地掏出来。它强依赖图数据库（比如 Neo4j）做底座，当你提问时，它会在知识图谱的节点上“多跳”（跳过好几个关联节点）去寻找线索。

────────────────────

💡 人话翻译：

> 别的记忆库是用来找证据的，Cognee 的记忆库是用来做**推理**的。

**适合谁：**
你在做金融分析 Agent、多方博弈推演（就像 MiroFish 想做的那样），或者任何需要处理错综复杂人物/利益关系的场景。

━━━━━━━━━━━━━━━━━━━━

◆ 结语：别再去手搓 Neo4j 了

━━━━━━━━━━━━━━━━━━━━

很多技术人员有个误区，一听到“知识图谱”，就想去搭个 Neo4j，自己写正则抽实体，自己写 Cypher 语句去查。

这就好比你要出门打车，非要去自己组装一辆汽车一样。

到了 2026 年的今天，底层的基础设施早就分工明确了。**Neo4j 是存砖头的水泥地，而 Mem0、ZEP、Letta 这些，是精装修好的智能家居。**

**一句话总结：**
- 图省事、想快速验证：上 **Mem0**。
- 想玩高阶的 Agent 自主意识：上 **Letta**。
- 想本地部署防泄密：上 **Evermind**。
- 做复杂逻辑推演：上 **Cognee**。

至于 MiroFish，它在代码里硬编码了对 ZEP 的依赖。如果你真想给它换个不花钱的本地脑子，不仅要改 `.env`，还得去它的源码里做一趟底层的器官移植手术。

如果有时间，我们下期把 MiroFish 的源码扒开，看看第一刀该从哪里下。
