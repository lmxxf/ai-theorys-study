【GPT5.6】新出的语音功能真不错，怎么做到的？

◆ 开篇：短视频里被玩坏的新语音

━━━━━━━━━━━━━━━━━━━━

短视频平台上这两天冒出一大堆调戏 GPT5.6 语音功能的视频。玩法五花八门：让它换口音、命令它 "说温柔一点"、故意半句话打断它看它怎么接、有人发现自己说话时它会 "嗯嗯""yeah" 地附和、还有人测出你停下来想事儿的时候它居然安静等着不抢话。评论区一片 "这也太像人了"。

这些事，旧版语音模式一件都做不到。旧版是你说一段、它答一段，像跟一台礼貌的对讲机聊天。

先纠正一个流传的小误会：这波能力其实不是 GPT-5.6 本体带来的。真正发布的东西叫 **GPT-Live**，7 月 8 日上线，比 GPT-5.6 正式开放还早一天。OpenAI 照例不说是怎么做到的——CloseAI 这个外号不是白叫的。

但有个好消息：**这条技术路线，开源界从 2021 到 2025 每一步都发过论文**。声音怎么变成 token、怎么让机器学会 "嗯嗯" 附和、怎么在陪聊的同时调用一个更聪明的大脑——全都有公开的原型。我们完全可以自己把 GPT-Live 大概的样子拼出来。

这就是本期要干的事：沿着四年的论文谱系走一遍，走到头你会发现，GPT-Live 虽然闭源，但它长什么样几乎是能推出来的。

◆ 第一节：旧三段管线为什么像不了人

━━━━━━━━━━━━━━━━━━━━

先看旧版语音是怎么工作的。它是三个模型串成的流水线：

1. **ASR**（Automatic Speech Recognition，语音转文字）：把你的语音听写成一行文本
2. **LLM**：拿这行文本当输入，生成一行回答文本
3. **TTS**（Text-to-Speech，文字转语音）：把回答文本朗读出来

每一段单独看都很成熟。但拼在一起，有两个**结构性死穴**——注意是结构性的，不是 "模型不够大" 能解决的。

**死穴一：每过一层丢一次信息。** 你说 "行吧……" 的时候，那个拖长的尾音、那声叹气、说到一半的迟疑——在 ASR 那一关全被扔掉了，LLM 拿到的只有 "行吧" 三个字。你带着哭腔说和带着笑意说，进到 LLM 是同一行文本。而 TTS 输出时再给回答 "配" 上的语气，是根据文本猜出来的表演——它根本没听过你的声音，谈不上回应你的情绪。

**死穴二：轮流说话是架构决定的。** 流水线必须等你说完，才能开始转文字；转完才能开始想；想完才能开始合成。"你说话时它插一句嗯嗯" 这件事在结构上就不存在——那个时刻整条流水线还卡在第一段。它不是不礼貌到不敢插话，是**架构里没有 "同时听和说" 这个概念**。

────────────────────

💡 打个比方

旧管线像找了个翻译开电话会：你说中文，翻译记成文字条递给外国专家，专家写好回复，翻译再念出来。专家永远听不到你的声音，你也永远等得到一板一眼的轮流发言。想让这场会像老友聊天，换更好的专家没用——**得把翻译撤了，让专家自己长耳朵和嘴**。

────────────────────

要撤掉翻译，第一个问题就来了：LLM 只认 token，声音是连续波形，怎么让模型直接 "吃" 声音？

◆ 第二节：声音变 token——先学会压缩，再学会接话

━━━━━━━━━━━━━━━━━━━━

【SoundStream：把波形量化成离散码字】

2021 年 Google 的 SoundStream（arXiv 2107.03312）本来是个音频压缩编解码器（codec），后来成了整条路线的地基。结构是全卷积的 encoder-decoder：encoder 把波形压成一串低帧率的向量，decoder 再还原回波形。

关键在中间的量化环节。要让向量变成离散 token，最直接的想法是 VQ（Vector Quantization，向量量化）：备一本 "码本"（codebook），每个向量找码本里最近的那个码字，用编号代替它。但算一笔账就知道单层 VQ 走不通：想在 6kbps 的码率下用一层码本表达，**码本得有 2 的 80 次方个条目**——这书没法印。

SoundStream 的解法是 **RVQ**（Residual Vector Quantization，残差向量量化）：多层码本级联。第一层粗略量化，算出误差（残差），交给第二层继续量化，再算残差交给第三层……8 层、每层只要 1024 个码字，就把不可能的问题变成了可能。效果也硬：3kbps 的 SoundStream 音质好于 12kbps 的 Opus、接近（注意，是接近，不是超过）9.6kbps 的 EVS。

2022 年 Meta 的 EnCodec（arXiv 2210.13438）走同一条路线，做了工程强化。到这一步，声音正式变成了一串串离散编号——**token 化完成**。

【AudioLM：把 token 变成一门生成语言】

token 有了，下一步是让模型学会 "接着说"。2022 年 Google 的 AudioLM（arXiv 2209.03143）干的就是这个：把音频 token 序列当成语言，训一个自回归模型做 "续写"。

它把 token 分成两种，各管一摊：

- **语义 token**：从 w2v-BERT 的中间层表示做 k-means 聚类得来，管长程内容——说的大致是什么、韵律怎么走
- **声学 token**：就是 SoundStream 的 RVQ 码字，管音质细节——音色、录音环境这些，主要由**粗量化层**（RVQ 前几层）捕获

生成分三阶段：先生成语义 token 定内容骨架，再生成粗声学 token 定音色，最后补细声学 token 出高保真波形。整个训练**纯音频、没有任何文本标注**。给它几秒钟没见过的说话人的录音，它能延续这个人的音色和韵律接着说下去。

────────────────────

💡 我问 Claude

你说了半天压缩和续写，短视频里那个 "说温柔一点" 它真的变温柔了，到底是怎么做到的？

*就藏在 AudioLM 这一步里。一旦音色、语速、韵律都变成了 token、进了自回归分布，"温柔" 就不再是某个 TTS 预设开关，而是声学 token 空间里的一片区域——语速慢、音高低、气声多的那一片。你说 "温柔一点"，这句指令改变了上下文条件，后续采样的概率质量就往那片区域漂。这和文本模型听到 "写正式一点" 就换文风，是同一个机制。所以严格说它不是 "切换到温柔模式"，而是 "朝着温柔的方向采样"——它在尝试温柔，像一个演员在找感觉，而不是按下了某个按钮。*

────────────────────

到这儿，机器能用自己的声音接话了。但它还是 "你说完我再说"——插话、附和这些事，需要把更根本的东西删掉。

◆ 第三节：全双工——把 "轮" 这个概念删掉

━━━━━━━━━━━━━━━━━━━━

【dGSLM：节奏像人了，内容还不知所云】

2022 年 Meta 的 dGSLM（Nguyen 等，arXiv 2203.16502，后发表于 TACL 2023）做了一个激进的实验：**不建模 "对话轮"，直接建模 "两条同时进行的声音"** 。

训练数据是 Fisher 语料：2000 小时双声道电话对话，16000 多通、每通约 10 分钟，两个人各占一个声道。语音用 HuBERT 特征加 k-means 离散成 500 类单元（50Hz 一帧），然后上一个**双塔 Transformer**：两座塔共享权重，各自建模一个说话人的 token 流，顶部 4 层用 cross-attention（交叉注意力）互相 "偷看" 对方——我说话的时候，实时知道你在干什么。

结果很妙。笑声、"嗯嗯" 这类 backchannel（附和声）、重叠说话、停顿——论文原话是这些行为 "从系统中自然涌现"（emerges naturally），没有任何一条规则写 "此处应附和"。它甚至复现了人类对话研究里一条著名规律：**同一个人话轮内的停顿，比换人说话的间隙更长**——你停下来想事儿，对方是等得出来的。

但论文自己也承认硬伤：这套系统无法生成语义连贯的语音。人类评分里，意义度只有 2.48 分（满分 5，真人 4.21），而自然度和真人只差 0.53。一句话总结：**节奏像人，内容不知所云**——像个特别会接话捧场、但完全不知道自己在说什么的家伙。

【Moshi：全双工的完全体】

节奏和内容怎么兼得？2024 年法国 Kyutai 实验室的 Moshi（一作 Défossez，arXiv 2410.00037）给出了完整答案，论文声称这是 "第一个实时全双工语音大模型"。理论延迟 160ms、实测约 200ms——而人类对话的平均反应间隙是 230ms。**它接话比人还快一点。**

Moshi 值得记住四个设计：

**一、底座是文本 LLM。** 7B 参数的 Helium，先用 2.1T 文本 token 预训练——先当个正经语言模型，再学说话。这直接解决 dGSLM "不知所云" 的问题：内容能力是从文本世界继承来的。

**二、Mimi codec：语义声学合一。** 24kHz 音频压到每秒 12.5 帧、1.1kbps，延迟只有 80ms。而且它不像 AudioLM 那样语义、声学两套 token 分开——Mimi 用 WavLM 当 teacher，把语义信息**蒸馏进 RVQ 的第一层**：第一层码字管 "说了什么"，后面几层管 "听起来怎么样"，一套 token 全包了。

**三、多流架构：删掉说话轮。** 自己的语音和用户的语音作为并行流同时建模，论文原文说这样 "去除显式说话轮"（removes explicit speaker turns）。它随时在听、随时可以出声——沉默也是一种输出。dGSLM 的双塔思想，在这里长成了正式架构。

**四、Inner Monologue（内心独白）。** 每帧音频 token 生成之前，先预测一段时间对齐的文本 token 当前缀——**先在心里想好要说什么字，再张嘴发出声音**。这个设计大幅提升了语音的语言质量，等于把文本 LLM 的智商实时注入到声音流里。

工程上用 RQ-Transformer 分层：32 层 Temporal Transformer 沿时间轴走，每个时间步内再由 6 层 Depth Transformer 逐层吐出 8 个 codebook 的码字。训练用了 700 万小时音频，而全双工对话能力，靠的还是那个 Fisher 语料——2000 小时微调就够了。

────────────────────

💡 常识科普

**全双工**（full-duplex）是通信术语：链路两端可以同时收发，像电话；对应的半双工是同一时刻只能一方发送，像对讲机——按下说话键就听不见对方。旧语音管线在交互形态上就是对讲机：说和听在架构上互斥。全双工语音模型的意思是，模型的 "听" 和 "说" 是两条并行流，每一帧都同时进行，插话、附和、被打断后立刻闭嘴，都只是这两条流的自然行为。

────────────────────

到 Moshi 这一步，"像人地聊天" 基本解决了。但还剩一个矛盾：陪聊要快，思考要深——一个模型很难两头占。

◆ 第四节：双脑分工——嘴和脑分家

━━━━━━━━━━━━━━━━━━━━

先算一笔物理账。人类对话里的 turn-taking gap（话轮转换间隙）在 200ms 量级，backchannel 附和也得在这个量级里响应，才不会显得迟钝。而一个前沿大模型光是 prefill 加吐出第一个 token，就不止这个数。结论很硬：**全双工的延迟预算把模型尺寸锁死了——负责出声的那个模型，必然是小模型。** 可小模型又扛不住复杂问题。怎么办？

【Qwen2.5-Omni：Thinker-Talker】

2025 年阿里的 Qwen2.5-Omni（arXiv 2503.20215）给这个矛盾起了个特别形象的架构名：**Thinker-Talker**。论文自己打的比方是：Thinker 是个 LLM，"像大脑"；Talker 是个双轨自回归模型，"像嘴"。

Talker 不自己从头理解问题——它直接拿 Thinker 的高层隐表示，外加已采样文本 token 的 embedding（用来消除同音歧义，比如光靠隐表示分不清要念哪个 "xíng/háng"），流式地生成语音 codec token。两者端到端联合训练，论文说这个设计让模型 "同时生成文本和语音而互不干扰"。

参数上说明一下：论文只给了 7B 主干的规模，Talker 的参数量论文没写；社区从开源权重实测约 1.35B，外加约 0.45B 的语音解码模块——这个数字来源是社区实测，不是官方口径。但量级信息够用了：**嘴比脑小一个数量级**，和上面那笔延迟账对得上。

【GPT-Live：把双脑做成两个独立模型】

GPT-Live 走的就是这个思路，而且分得更彻底：嘴和脑不再是一个模型的两个模块，而是**两个独立的模型**。官方博客的原话是，复杂任务会 "委托给我们最新的前沿模型"（媒体普遍解读为 GPT-5.5——发布当天 GPT-5.6 还没 GA）。你问个难题，嘴上那个小模型继续陪你聊着，后台大脑在算，结果好了自然带回对话，全程不冷场。

官方还有一句值得细品的硬话： **"模型每秒可做出多次交互决策——说话、继续听、停顿、打断，还是调用工具。"** 每秒多次——这意味着决策的粒度从 "轮" 降到了 "帧"。对照前面的谱系你会认出来：这正是 Moshi 那条 "每一帧都在选择说还是不说" 的多流架构，加上 Thinker-Talker 的分工，再加上把 "调用工具 / 委托大脑" 也塞进了每帧的决策空间。

────────────────────

💡 打个比方

这套架构像酒吧里的调酒师和后厨。调酒师（嘴）站在吧台前跟你聊天，反应快、会接梗、你抱怨工作他会 "嗯嗯" 附和；你要是点了一道费工夫的菜（复杂问题），他不会僵在原地等——单子递进后厨（前沿模型），转头继续跟你聊，菜好了端上来顺势接进话题。你全程的体验是 "这人真好聊"，但真正的手艺在后厨。

────────────────────

◆ 第五节：拼图完成——GPT-Live 大概长什么样

━━━━━━━━━━━━━━━━━━━━

回看官方给出的评测口径：让人跟 GPT-Live 和旧的 Advanced Voice Mode 各聊 5-10 分钟，然后打人类偏好分——总体偏好、话轮转换、打断处理、流畅度、自然度，**五个维度全胜**。注意这五个维度全是 "像不像人" 的维度，没有一个是 "答得对不对"——OpenAI 很清楚这次卖的是什么。

参数规模照例不披露。但走完这条谱系，你自己就能推个八九不离十：一张 Moshi 式的嘴——7B 量级、全双工多流、帧级决策、大概率也有某种内心独白机制；背后接一个前沿模型当脑。每一块拼图都有公开论文的原型，OpenAI 的本事在于把它们打磨到产品级——这不是贬义，工程打磨恰恰是最难抄的部分。

安全方面一笔带过：语音是新的攻击面，OpenAI 针对自残、情感依赖、暴力、精神病性内容、性内容做了音频原生的红队测试，高风险对话会直接挂断，并接入既有的家长控制体系（家长控制是原有功能，不是 GPT-Live 新做的）。语音的情感黏性比文字强得多，这块以后值得单独写一期。

最后说两个观察，不抒情，就摆事实。

第一个是祛魅的：**那些让人起鸡皮疙瘩的拟人感——喘气、附和、迟疑——是采样出来的表演，而且出自整个系统里最小、最没有内容的那个模型。** 嘴只负责流畅，不负责懂。发布演示里已经有露馅时刻：印地语实时翻译被媒体吐槽 "浓重美国口音、一股书面腔"——节奏是人的节奏，语言的里子还是训练分布的里子。这跟 dGSLM 那个 "自然度只差 0.53、意义度差一大截" 的老毛病，是同一条裂缝的两端。

第二个是提气的：dGSLM 到 GPT-Live，四年。从 "节奏像人但胡言乱语" 到 "短视频平台集体惊呼太像人了"，只用了四年。 **"像人" 的工程门槛，已经降到普通人的分辨率以下了** ——分辨不出来的东西，接下来就会大规模出现在你分辨不出来的地方。

━━━━━━━━━━━━━━━━━━━━

**拟人感是采样出来的表演——会喘气的，是系统里最小、最不懂内容的那个模型。**

**全双工的本质不是反应快，是把 "轮流说话" 这个概念从架构里删掉了——决策粒度从轮降到了帧。**

**dGSLM 到 GPT-Live 只用了四年："像人" 的工程门槛，已经低于普通人的分辨率了。**

| 语义 token / 声学 token | — | 前者管说了什么（内容、韵律），后者管听起来怎么样（音色、音质） |
| backchannel | 附和声 | 听者发出的 "嗯嗯""yeah"，表示 "我在听"，不抢话轮 |
| 全双工（full-duplex） | — | 同时收发，像电话；半双工像对讲机，同一时刻只能一方说 |
| Inner Monologue | 内心独白 | Moshi 的设计：每帧音频生成前先预测对齐的文本 token，先想好再开口 |
| Thinker-Talker | 思考者-说话者 | Qwen2.5-Omni 架构：大脑管理解和内容，嘴管流式出声，端到端联训 |
| turn-taking gap | 话轮转换间隙 | 人类对话中一方说完到另一方开口的间隙，平均约 200ms 量级 |
