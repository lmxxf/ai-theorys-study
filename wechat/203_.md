【AI架构原理】多模态大模型为什么是"拼"出来的？从 Qwen3-VL-2B 的两个文件说起

━━━━━━━━━━━━━━━━━━━━

◆ 起因

━━━━━━━━━━━━━━━━━━━━

最近在做一个项目——在 RK3588 的 NPU 上跑多模态大模型。下载 Qwen3-VL-2B 的时候发现要下两个文件：一个语言模型（2.3GB），一个视觉编码器（812MB）。两个模型分开下、分开加载、分开推理，最后串起来才能"看图说话"。

板端运行的时候，日志长这样：

```
LLM Model loaded in  2892.80 ms
ImgEnc Model loaded in  1008.81 ms
ImgEnc Model inference took  2749.71 ms
```

先加载语言模型，再加载视觉编码器，图片编码完了喂给语言模型——两步走。

第一个问题自然就来了：**为什么多模态模型要分成两个文件？不能训成一个整体吗？**

第二个问题更本质：**这种"拼接式"多模态是怎么训练出来的？凭什么把两个独立训练的模型拼在一起，就能"看图说话"？**

这篇文章从工程现象出发，把多模态模型的"拼接"架构拆清楚。

━━━━━━━━━━━━━━━━━━━━

◆ 多模态模型的"拼接"架构

━━━━━━━━━━━━━━━━━━━━

打开任何一个主流多模态大模型——Qwen3-VL、MiniCPM-V、LLaVA、InternVL——里面都是三个部件：

**1. 视觉编码器（Vision Encoder）**

💡 翻译成人话：把图片变成一组向量的模型。输入一张 448×448 的图片（几十万个像素），输出几百个向量（每个向量几百到一千多维）。这些向量包含了图片的语义信息——"这是一只猫"、"背景是草地"之类的。

Qwen3-VL 用的是 **SigLIP2**，一种基于 ViT（Vision Transformer）的图片编码器。MiniCPM-V 4.6 也是 SigLIP2。

**2. 投影层（Projector / Merger）**

💡 翻译成人话：把视觉编码器输出的向量"翻译"成语言模型能理解的格式。视觉编码器的输出维度可能是 1152 维，但语言模型期望的输入维度是 1536 维——投影层就是做这个维度转换的。

Qwen3-VL 的投影层叫 Merger，结构非常简单：一个 LayerNorm + 两个线性层（MLP）。

**3. 语言模型（LLM）**

💡 翻译成人话：就是我们熟悉的大语言模型——接收一串 token 向量，逐个预测下一个 token。Qwen3-VL-2B 用的是 Qwen3-2B，MiniCPM-V 4.6 用的是 Qwen3.5-0.8B。

────────────────────

三者的关系：

```
图片 → 视觉编码器 → 视觉向量 → 投影层 → "视觉 token" → 拼到文字 token 前面 → 语言模型 → 输出文字
```

打个比方：**视觉编码器是眼睛，投影层是翻译，语言模型是大脑。**

眼睛看到图片，用自己的"语言"描述了看到的东西。但大脑说的是另一种"语言"——它只认文字 token。翻译的工作就是把眼睛的描述转换成大脑能理解的格式。转换完了，大脑就像收到了一段特殊的"文字"，照常生成回答。

**这就是为什么可以分成两个文件**——视觉编码器和语言模型是两个独立训练出来的模型，中间只靠一个很小的投影层连接。在 RK3588 上部署时，视觉编码器跑 RKNN Runtime（NPU 加速矩阵运算），语言模型跑 RKLLM Runtime（NPU 加速 token 生成），各走各的推理流程。

━━━━━━━━━━━━━━━━━━━━

◆ 视觉编码器怎么训的——对比学习

━━━━━━━━━━━━━━━━━━━━

SigLIP2 不是凭空冒出来的，它的前身是 CLIP 和 SigLIP。训练方法的核心思路叫**对比学习（Contrastive Learning）**。

💡 翻译成人话：给模型看大量的"图片 + 文字描述"配对数据（几亿对），让模型学会——看到一张猫的图片，它的向量应该和"一只猫"这段文字的向量靠近；看到一张狗的图片，它的向量应该和"一只猫"的向量远离。

────────────────────

【CLIP 的做法：softmax 对比损失】

CLIP（OpenAI, 2021）同时训练一个图片编码器和一个文本编码器。一个 batch 里有 N 张图片和 N 段文字，算出 N×N 的相似度矩阵，对角线上的才是正确配对。

损失函数用 **softmax + 交叉熵**：对每一行（固定一张图片，和所有 N 段文字比较），做 softmax 归一化，然后算交叉熵——让正确配对的概率尽量高。

问题：softmax 是全局归一化——每个样本的损失依赖这个 batch 里所有其他样本。**batch 越大效果越好**，所以 CLIP 要用 32768 的巨大 batch size，训练成本很高。

────────────────────

【SigLIP 的改进：sigmoid 对比损失】

SigLIP（Google, 2023）改了一招：**把 softmax 换成 sigmoid。**

💡 翻译成人话：不再让 N 个样本互相竞争"谁的概率最高"。而是对每一对（图片ᵢ, 文字ⱼ）独立判断——"这对是不是正确配对？"输出一个 0 到 1 之间的概率。对角线上的配对标签是 1（正确），其他位置标签是 -1（错误）。

```
CLIP：  一行 N 个样本做 softmax → 概率和为 1 → 样本间竞争
SigLIP：每对独立做 sigmoid → 每对独立判断 → 样本间不耦合
```

**好处**：每对样本的梯度不再依赖其他样本，可以用更小的 batch size 达到相同效果，训练更高效。论文实测，SigLIP 在相同计算预算下，分类准确率和零样本性能都优于 CLIP。

**SigLIP2**（Google, 2025）在 SigLIP 基础上改进了训练策略（多分辨率、多任务、自蒸馏等），是目前最强的开源视觉编码器之一。Qwen3-VL 和 MiniCPM-V 4.6 都选了它。

────────────────────

**训练完得到什么？**

一个 ViT 模型，能把任意图片映射成一组固定格式的向量。语义相似的图片（比如两张不同角度的猫），向量在高维空间里距离很近。语义不同的图片（猫和汽车），向量距离很远。

**注意：这个阶段视觉编码器完全不知道什么是"语言模型"。** 它只学了"图片和文字的语义对齐"——知道什么图片对应什么文字描述，但不会生成文字。

━━━━━━━━━━━━━━━━━━━━

◆ 投影层怎么训的

━━━━━━━━━━━━━━━━━━━━

视觉编码器有了，语言模型也有了（比如 Qwen3-2B，用海量文本预训练好的）。两个模型各说各的"语言"——视觉编码器输出 1152 维向量，语言模型期望 1536 维 token embedding。怎么把它们接起来？

**训练策略：冻住两头，只训中间。**

💡 翻译成人话：视觉编码器的参数不动，语言模型的参数也不动。只训练投影层那几千万个参数——让投影层学会"怎么翻译"。

────────────────────

【训练数据】

用的是"看图说话"数据——每条数据是一张图片 + 一段对这张图片的文字描述。比如：

- 图片：一只橘猫趴在键盘上
- 文字：`A orange cat lying on a computer keyboard`

几百万到几千万条这样的配对。

────────────────────

【训练过程】

```
1. 图片过视觉编码器 → 得到视觉向量（参数冻结，不更新）
2. 视觉向量过投影层 → 得到"视觉 token"
3. "视觉 token" + 文字 token 拼在一起，喂给语言模型
4. 语言模型预测下一个 token（参数冻结，不更新）
5. 算损失：标准语言模型交叉熵（预测下一个 token 的概率 vs 真实下一个 token）
6. 梯度只回传到投影层，更新投影层参数
```

**损失函数就是最普通的语言模型交叉熵损失（next-token prediction）**——没有任何花活。投影层的目标就是：让翻译出来的"视觉 token"足够好，好到语言模型像读到了一段高质量的文字描述一样，能准确地预测后续文字。

────────────────────

【投影层有多小？】

以 MiniCPM-V 4.6 为例，我实际分析了模型权重：

```
model.vision_tower：5.22 亿参数（SigLIP2-400M）
model.language_model：7.52 亿参数（Qwen3.5-0.8B）
model.merger（投影层）：2597 万参数
```

投影层只占总参数的 **2%**。结构就是一个 LayerNorm + 两个线性层（MLP）——六个权重矩阵，49.5 MB。

**这意味着：把两个大模型（总计十几亿参数）连接起来的"翻译官"，只有 2600 万参数。** 整个多模态能力的"接口协议"就编码在这 2% 里面。

────────────────────

当然，实际训练多模态模型通常不止训投影层。比如 LLaVA 的两阶段方案：

- **第一阶段**：只训投影层（上面说的过程），用大规模图文配对数据
- **第二阶段**：解冻语言模型（甚至视觉编码器），用高质量的指令微调数据做全参数微调

但核心思路不变——**先用投影层把两个模型"接上"，再精调整体**。投影层是多模态能力的起点。

━━━━━━━━━━━━━━━━━━━━

◆ Qwen3-VL 的特色：DeepStack

━━━━━━━━━━━━━━━━━━━━

大多数多模态模型（LLaVA、MiniCPM-V 2.6）的投影层做的事很直接：**取视觉编码器最后一层的输出，投影一下，拼到语言模型的输入层。**

```
ViT 最后一层输出 → 投影层 → LLM 输入层
```

Qwen3-VL 做了一个更深度的融合，叫 **DeepStack**。

**不只是最后一层**——ViT 有很多层（比如 SigLIP2-Large 有 27 层），每一层捕捉的信息不同：

- **浅层**（1-9 层）：边缘、纹理、颜色——底层视觉特征
- **中层**（10-18 层）：物体部件、局部结构——中层语义
- **深层**（19-27 层）：物体类别、场景语义——高层语义

普通投影层只用深层输出，丢掉了浅层和中层的信息。**DeepStack 把多个层的 ViT 特征注入到 LLM 的不同深度。**

```
ViT 浅层特征 → 投影 → 注入 LLM 浅层
ViT 中层特征 → 投影 → 注入 LLM 中层
ViT 深层特征 → 投影 → 注入 LLM 深层
```

💡 翻译成人话：普通投影层像是翻译只听了对方最后一句总结。DeepStack 像是翻译在对方说话的全过程中，把不同阶段的意思分别翻译给大脑的不同处理层——底层视觉信息给大脑的底层处理，高层语义信息给大脑的高层处理。信息损失更少，融合更深。

Qwen3-VL 的技术报告指出，DeepStack 在视觉理解任务上比普通的单层投影有明显提升，尤其是在需要细粒度视觉理解的场景（OCR 识别、小物体检测等）。

━━━━━━━━━━━━━━━━━━━━

◆ 为什么"拼接"架构是主流

━━━━━━━━━━━━━━━━━━━━

回到最初的问题：为什么不从头训一个"天生就能看图"的大模型？

**答案是成本和效率。**

────────────────────

【原因一：复用已有的强模型】

训一个好的视觉编码器（SigLIP2）需要几亿图文配对数据 + 几千 GPU 小时。训一个好的语言模型（Qwen3-2B）需要几万亿 token 文本 + 几万 GPU 小时。

**这两个模型已经有人花了巨大代价训好了。** 你只需要训一个 2600 万参数的投影层——用几百万数据、几百 GPU 小时——就能把它们组装成一个多模态模型。

从头训一个端到端的多模态模型？你得重新学视觉特征、重新学语言能力、还得学两者的对齐——所有成本加在一起，是"拼接"方案的几十倍。

────────────────────

【原因二：升级方便】

视觉编码器从 CLIP 升级到 SigLIP 再到 SigLIP2——只需要重新训投影层，语言模型不用动。

语言模型从 Qwen2 升级到 Qwen3 再到 Qwen3.5——只需要重新训投影层，视觉编码器不用动。

**MiniCPM-V 系列就是这么迭代的：**

| 版本 | 视觉编码器 | 语言模型 | 时间 |
|------|------------|----------|------|
| MiniCPM-V 2.6 | SigLIP-400M | Qwen2-2B | 2024-08 |
| MiniCPM-V 4.5 | SigLIP2-400M | Qwen2.5-VL-3B | 2025-04 |
| MiniCPM-V 4.6 | SigLIP2-400M | Qwen3.5-0.8B | 2026-05 |

视觉编码器升级了一次（SigLIP → SigLIP2），语言模型升级了两次（Qwen2 → Qwen2.5 → Qwen3.5），每次都是"换零件 + 重新训投影层"。如果是端到端训练的模型，每次升级就意味着从头来。

────────────────────

【原因三：灵活的部署】

回到我们的 RK3588 项目。视觉编码器和语言模型分开之后：

- 视觉编码器用 RKNN 格式，跑在 NPU 上做矩阵运算
- 语言模型用 RKLLM 格式，跑在 NPU 上做 token 生成
- 两者用不同的量化策略——语言模型 W8A8，视觉编码器 FP16

**分开部署让每个部件都能用最适合它的推理优化。** 如果是一个大整体，你的量化、编译、内存管理都得统一处理，灵活性大打折扣。

◆ 用实际数据说明

━━━━━━━━━━━━━━━━━━━━

两个我实际接触过的多模态模型，参数分布对比：

| 部件 | Qwen3-VL-2B | MiniCPM-V 4.6 |
|------|-------------|---------------|
| 视觉编码器 | SigLIP2-Large（~300M） | SigLIP2-400M（522M） |
| 投影层 | Merger MLP（~26M） | Merger MLP（26M） |
| 语言模型 | Qwen3-2B（~1.7B） | Qwen3.5-0.8B（752M） |
| **总参数** | **~2.0B** | **~1.3B** |
| **投影层占比** | **~1.3%** | **~2.0%** |

两个模型的投影层参数占比都在 **1-2%** 之间——几千万参数就把十几亿参数的两个模型连起来了。

部署时的文件大小（Qwen3-VL-2B，W8A8 量化后）：

| 文件 | 大小 | 内容 |
|------|------|------|
| `qwen3-vl-2b-instruct_w8a8_rk3588.rkllm` | 2.3 GB | 语言模型（量化后） |
| `qwen3-vl-2b_vision_rk3588.rknn` | 812 MB | 视觉编码器 + 投影层 |

视觉编码器和投影层被打包在同一个 `.rknn` 文件里——因为推理时它们是串行的（图片先过编码器再过投影层），打包在一起减少一次数据搬运。

◆ 小结

━━━━━━━━━━━━━━━━━━━━

多模态大模型的"拼接"架构，本质上是工程和科学的双重最优解：

**科学上**：对比学习训出来的视觉编码器已经把图片的语义信息压缩进了向量空间，语言模型已经掌握了人类语言的生成能力。两者之间只差一个"维度对齐"——一个小小的投影层就够了。

**工程上**：复用已有的强模型，训练成本低几十倍；升级时换零件重训投影层就行；部署时各部件独立优化。

**在 RK3588 这种嵌入式设备上，这种拆分更是刚需**——16GB 内存装不下端到端训练的巨型模型，但分开量化、分开加载的"拼接"模型刚好能塞进去。

━━━━━━━━━━━━━━━━━━━━

**「把两个强模型连起来的不是一个同样强的模型，而是一个只有 2% 参数的翻译层。」**

**「好的架构不是把所有能力塞进一个模型，是让每个部件各自做到最好，然后用最小的代价连接它们。」**

技术名词速查：

- **ViT**（Vision Transformer）：把图片切成小块（patch），当作"token"输入 Transformer 的图片编码器。2020 年 Google 提出，现在是视觉编码器的主流架构。
- **SigLIP / SigLIP2**：Google 的视觉-语言对比学习模型，用 sigmoid 损失替代 CLIP 的 softmax 损失，训练效率更高。
- **对比学习**（Contrastive Learning）：让模型学会区分"什么和什么相似、什么和什么不同"的训练方法。核心是拉近正样本对、推开负样本对。
- **投影层**（Projector / Merger）：连接视觉编码器和语言模型的小型网络，做维度转换和特征对齐。
- **冻结**（Freeze）：训练时固定某个模块的参数不更新。"冻住视觉编码器"意味着只让梯度流过投影层，不改变视觉编码器的权重。
- **交叉熵损失**（Cross-Entropy Loss）：语言模型最常用的损失函数。模型预测下一个 token 的概率分布，和真实的下一个 token 比较，概率越接近损失越小。
- **DeepStack**：Qwen3-VL 提出的多层视觉特征注入策略，把 ViT 不同深度的特征注入 LLM 的对应层，实现更深度的视觉-语言融合。
