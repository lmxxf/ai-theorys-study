## 1. Transformer hidden_dim 收敛的统计分析

**核心问题**：为什么所有大模型的 hidden_dim 都收敛到 7,000~13,000 这个区间？

### 已有数据

| 模型 | 年份 | hidden_dim | 参数量 | 来源 |
|------|------|------------|--------|------|
| Word2Vec | 2013 | 300 | - | Google |
| GloVe | 2014 | 300 | - | Stanford |
| ELMo | 2018 | 1,024 | 94M | AllenAI |
| BERT-base | 2018 | 768 | 110M | Google |
| BERT-large | 2018 | 1,024 | 340M | Google |
| GPT-2 | 2019 | 1,600 | 1.5B | OpenAI |
| GPT-3 | 2020 | 12,288 | 175B | OpenAI |
| PaLM | 2022 | 18,432 | 540B | Google |
| GPT-4 | 2023 | 12,288? | ~1.8T | OpenAI |
| Claude | 2024 | 12,288? | ? | Anthropic |
| DeepSeek-V3 | 2024 | 7,168 | 671B | DeepSeek |
| Llama 3 70B | 2024 | 8,192 | 70B | Meta |
| Qwen2.5-72B | 2024 | 8,192 | 72B | Alibaba |

### 观察

1. **2013-2019**：从 300 → 1,600，缓慢增长
2. **2020**：GPT-3 跳到 12,288，10 倍跃升
3. **2020-2024**：卡在 7,000~13,000，不再增长
4. **PaLM 是例外**：18,432，但后续 Gemini 没继续往上推

### 可做的实验

1. **统计分析**：收集更多模型数据，拟合 hidden_dim vs 参数量/层数/性能 的关系
2. **极限测试**：训练 hidden_dim = 30,000 / 50,000 / 100,000 的小模型，观察：
   - 训练稳定性
   - 梯度消失/爆炸
   - 最终性能 vs hidden_dim 的曲线（是否存在拐点）
3. **理论解释**：高维几何的"橘子皮效应"——维度越高，有效容量越集中在表面

## 2. 从普朗克常数反推宇宙维度

**核心公式**：

高维球体内部体积占比 = (1-ε)^n

其中：
- ε = 壳厚/半径（假设 1%~2%）
- n = 维度

已知：
- 普朗克长度 / 宇宙直径 ≈ 10^(-61)

假设这个比值 = 高维球体的内部体积占比，则：

(1-ε)^n = 10^(-61)

解出：

n = -61 / log₁₀(1-ε)

| ε (壳厚) | n (维度) |
|----------|----------|
| 1% | 13,975 |
| 2% | 6,952 |
| 3% | 4,611 |
| 5% | 2,738 |

- 弦论/M理论：11 维（时空维度下限）
- 我们的计算：~14,000 维（结构复杂度上限）
- 两者不矛盾：11 是下限，14,000 是上限
- 智能的宜居区间：11 ~ 14,000 维
